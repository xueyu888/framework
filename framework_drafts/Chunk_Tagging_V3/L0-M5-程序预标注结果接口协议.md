# 程序预标注结果接口协议:ProgrammaticPreAnnotationResultInterfaceProtocol

@framework

## 0. 目标 (Goal)
- 程序预标注结果接口协议负责声明一次程序预标注流程完成后的最小输出结构。
- 该协议只定义程序预标注结果中最终对外导出的字段组成方式，不负责原始评分计算、归一化计算、阈值判定或人工审核流程。

## 1. 包含字段（Included Fields）

- `F1` 文本标识：`text_id`，用于标识该条程序预标注结果对应的文本。
- `F2` 标签标识：`tag_id`，用于标识该条程序预标注结果对应的标签。
- `F3` 原始评分：`raw_score`，用于承载大模型直接返回的原始评分结果。
- `F4` 归一化评分：`normalized_score`，用于承载由原始评分映射得到的归一化结果。
- `F5` 评分原因：`reason`，用于承载得出原始评分结果的原因。
- `F6` 启用状态：`enable_status`，用于承载自动标注给出的启用判定结果。
- `F7` 审核状态：`review_status`，用于承载自动标注给出的审核状态。
- `F8` 审核来源：`review_source`，用于承载自动标注给出的审核来源。

## 2. 字段约束（Field Constraints）

- `P1` 文本标识存在性：`text_id` 不可为空。
- `P2` 标签标识存在性：`tag_id` 不可为空。
- `P3` 原始评分类型：`raw_score` 必须为数值类型。
- `P4` 原始评分存在性：`raw_score` 不可为空。
- `P5` 归一化评分类型：`normalized_score` 必须为数值类型。
- `P6` 归一化评分存在性：`normalized_score` 不可为空。
- `P7` 归一化评分范围：`normalized_score` 必须落在 `0~1` 范围内。
- `P8` 评分原因类型：`reason` 必须为字符串类型。
- `P9` 评分原因存在性：`reason` 不可为空。
- `P10` 启用状态范围：`enable_status` 只能为 `启用`、`不启用` 或 `待人工审核`。
- `P11` 审核状态范围：`review_status` 只能为 `已审核` 或 `未审核`。
- `P12` 审核来源范围：`review_source` 只能为 `system` 或空值。

## 3. 能力声明（Capability Statement）

- `C1` 文本身份绑定能力：在文本标识存在时，能够稳定绑定该程序预标注结果对应的文本对象。
- `C2` 标签身份绑定能力：在标签标识存在时，能够稳定绑定该程序预标注结果对应的标签对象。
- `C3` 原始评分承载能力：在原始评分存在且类型合法时，能够稳定承载原始评分结果。
- `C4` 归一化评分承载能力：在归一化评分存在且类型合法时，能够稳定承载归一化评分结果。
- `C5` 评分原因承载能力：在评分原因存在且类型合法时，能够稳定承载评分原因。
- `C6` 自动标注结果承载能力：在自动标注字段存在且取值合法时，能够稳定承载自动标注结果。
- `N1` 非职责声明：不负责原始评分的计算或归一化评分的生成。
- `N2` 非职责声明：不负责人工审核、人工结果写回或持久化行为。

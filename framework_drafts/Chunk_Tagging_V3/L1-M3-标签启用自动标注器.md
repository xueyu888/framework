# 标签启用自动标注器:AutoEnableTagAnnotator

@framework

## 0. 目标 (Goal)
- 标签启用自动标注器负责接收一个归一化后的数值，并根据启用阈值和不启用阈值导出自动标注结果。
- 该模块只声明阈值分流规则与自动标注结果导出，不负责原始评分计算、人工审核或结果持久化。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 阈值分流规则：负责依据阈值判定规则执行自动标注计算，其中 `x` 为归一化结果，`a` 为启用判定阈值，`b` 为不启用判定阈值；
当 `a <= x <= 1` 时，输出 `enable_status = 启用`、`review_status = 已审核`、`review_source = system`；
当 `0 <= x <= b` 时，输出 `enable_status = 不启用`、`review_status = 已审核`、`review_source = system`；
当 `b < x < a` 时，输出 `enable_status = 待人工审核`、`review_status = 未审核`、`review_source = 空值`。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `自动标注结果组合规则`：由 `{B1}` 消费 `NORMALIZED_SCORE_IN` 形成 `AUTO_ANNOTATION_OUT`，导出 `C1 + C2`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `NORMALIZED_SCORE_IN`：运行时输入接口。载荷类型为数值；基数为 `1`；语义为待执行自动标注判定的归一化结果；结果范围为 `0~1`。
- `AUTO_ANNOTATION_OUT`：运行时输出接口。载荷类型为结构化对象；基数为 `1`；语义为自动标注结果；接口内容必须满足 `L0.M4`。

### 3.2 参数边界（Parameter Constraints）

- `P1` 启用判定阈值类型：启用判定阈值必须为数值类型。
- `P2` 不启用判定阈值类型：不启用判定阈值必须为数值类型。
- `P3` 判定阈值范围：启用判定阈值和不启用判定阈值都必须落在 `0~1` 范围内。
- `P4` 判定阈值大小关系：启用判定阈值必须大于不启用判定阈值。
- `P5` 启用状态范围：`enable_status` 只能为 `启用`、`不启用` 或 `待人工审核`。
- `P6` 审核状态范围：`review_status` 只能为 `已审核` 或 `未审核`。
- `P7` 审核来源范围：`review_source` 只能为 `system` 或空值。

## 4. 能力声明（Capability Statement）

- `C1` 自动标注判定能力：在归一化值和判定阈值参数合法时，能够按阈值分流规则完成自动标注判定。
- `C2` 自动标注结果导出能力：在自动标注判定完成时，能够稳定导出自动标注结果。
- `N1` 非职责声明：不负责生成或归一化原始评分。
- `N2` 非职责声明：不负责人工审核、人工结果写回或后续持久化。

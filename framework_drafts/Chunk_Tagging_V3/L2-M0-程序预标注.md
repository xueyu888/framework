# 程序预标注:ProgrammaticPreAnnotation

@framework

## 0. 目标 (Goal)
- 程序预标注模块负责接收一段文本和标签输入，依次完成大模型评分调用、原始响应解析、原始评分记录装配、归一化与自动标注判定，最终输出预标注结果，包含 `text_id`、`tag_id`、`raw_score`、`normalized_score`、`reason`、`enable_status`、`review_status` 和 `review_source` 字段。
- 该模块只负责标签预标注，不负责人工审核。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 大模型调用接口：`L1.M0`，负责对输入的文本和标签进行大模型评分，得到大模型原始响应。
- `B2` 原始响应解析器：`L1.M1`，负责将原始响应解析为 `raw_score` 和 `reason`。
- `B3` 线性映射归一化器：`L1.M2`，负责将`raw_score`映射为`normalized_score`。
- `B4` 标签启用自动标注器：`L1.M3`，负责根据`normalized_score`导出满足 `L0.M4` 的自动标注结果。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `程序预标注组合`：由 `<B1, B2, B3, B4>` 按`B1`->`B2`->`B3`->`B4`的顺序形成`PROGRAMMATIC_PREANNOTATION_OUT`，导出 `C1 + C2 + C3 + C4 + C5`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `TEXT_IN`：运行时输入接口。载荷类型为结构化对象；基数为 `1`；语义为待评分文本；接口内容必须满足 `L0.M0` 定义的字段约束。
- `TAG_IN`：运行时输入接口。载荷类型为结构化对象；基数为 `1`；语义为待评分标签；接口内容必须满足 `L0.M1` 定义的字段约束。
- `PROGRAMMATIC_PREANNOTATION_OUT`：运行时输出接口。载荷类型为结构化对象；基数为 `1`；语义为程序预标注结果；接口内容必须满足 `L0.M5`。

### 3.2 参数边界（Parameter Constraints）

- 本模块无新增独立参数。

## 4. 能力声明（Capability Statement）

- `C1` 评分调用编排能力：能够调用大模型接口生成原始响应。
- `C2` 原始响应解析能力：能够调用原始响应解析器恢复 `raw_score` 和 `reason`。
- `C3` 评分归一化编排能力：能够将一个数据进行归一化处理。
- `C4` 自动标注能力：能够自动根据归一化后的值输出自动标注结果。
- `C5` 程序预标注结果打包能力：在整条流程完成时，能够将一些中间产物和最终产物打包成为满足`L0.M5`协议的对象。
- `N1` 非职责声明：不负责人工审核或人工结果写回。
- `N2` 非职责声明：不负责将程序预标注结果写入数据库或其他存储介质。

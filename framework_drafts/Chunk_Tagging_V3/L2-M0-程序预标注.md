# 程序预标注:ProgrammaticPreAnnotation

@framework

## 0. 目标 (Goal)
- 程序预标注模块负责接收一段文本和标签输入，并最终输出预标注的结果，包含 `text_id`、`tag_id`、`raw_score`、`normalized_score`、`reason`、`enable_status`、`review_status` 和 `review_source`字段。
- 该模块只声明流程编排与结果导出，不负责人工审核、人工结果写回或持久化。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 大模型调用接口：`L1.M0`，负责导出单次匹配原始评分记录。
- `B2` 线性映射归一化器：`L1.M1`，负责将原始评分映射为归一化结果。
- `B3` 标签启用自动标注器：`L1.M2`，负责根据归一化结果导出满足 `L0.M4` 的自动标注结果。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `程序预标注编排规则`：由 `<B1, B2, B3>` 按“评分调用 -> 评分归一化 -> 自动标注判定”的顺序形成 `PROGRAMMATIC_PREANNOTATION_OUT`，导出 `C1 + C2 + C3 + C4`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `TEXT_IN`：运行时输入接口。载荷类型为结构化对象；基数为 `1`；语义为待评分文本；接口内容必须满足 `L0.M0` 定义的字段约束。
- `TAG_IN`：运行时输入接口。载荷类型为结构化对象；基数为 `1`；语义为待评分标签；接口内容必须满足 `L0.M1` 定义的字段约束。
- `PROGRAMMATIC_PREANNOTATION_OUT`：运行时输出接口。载荷类型为结构化对象；基数为 `1`；语义为程序预标注结果；接口内容必须满足 `L0.M5`。

### 3.2 参数边界（Parameter Constraints）

- 本模块无新增独立参数。

## 4. 能力声明（Capability Statement）

- `C1` 评分调用编排能力：在文本和标签输入存在时，能够调用大模型接口生成原始评分记录。
- `C2` 评分归一化编排能力：在原始评分存在时，能够调用线性映射归一化器得到归一化结果。
- `C3` 自动标注编排能力：在归一化结果存在时，能够调用标签启用自动标注器得到自动标注结果。
- `C4` 程序预标注结果导出能力：在整条流程完成时，能够在模块外部组合阶段稳定导出包含原始评分字段、归一化评分字段和自动标注字段的程序预标注结果。
- `N1` 非职责声明：不负责人工审核或人工结果写回。
- `N2` 非职责声明：不负责将程序预标注结果写入数据库或其他存储介质。

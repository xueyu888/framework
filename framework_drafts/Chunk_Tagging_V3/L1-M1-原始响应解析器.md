# 原始响应解析器:RawResponseParser

@framework

## 0. 目标 (Goal)
- 原始响应解析器负责接收大模型原始响应，并依据语义映射关系提取原始评分结果与评分原因。
- 该模块只声明原始响应解析所需的内部构件、输入输出接口和解析约束，不负责发起大模型调用、文本标签身份装配、归一化计算或阈值判定。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 原始响应到目标输出的映射关系：原始响应中的内容和目标输出接口协议要求的内容之间的映射关系。
  原始响应.raw_score -> 目标输出.raw_score
  原始响应.reason -> 目标输出.reason

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `原始响应解析规则`：由 `{B1}` 消费 `RAW_RESPONSE_IN` 形成 `PARSED_RESULT_OUT`，导出 `C1 + C2 + C3`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `RAW_RESPONSE_IN`：运行时输入接口。载荷类型为结构化对象或文本；基数为 `1`；语义为一次大模型调用返回的原始响应。
- `PARSED_RESULT_OUT`：运行时输出接口。载荷类型为结构化对象；基数为 `1`；语义为由原始响应解析得到的结果，包括 `raw_score` 和 `reason`。

### 3.2 参数边界（Parameter Constraints）

- `P1` 映射关系独立性：一个原始响应内容只能映射一个目标输出。

## 4. 能力声明（Capability Statement）

- `C1` 原始评分解析能力：能够在一段原始响应中解析出 `raw_score`。
- `C2` 评分原因解析能力：能够在一段原始响应中解析出 `reason`。
- `C3` 解析结果导出能力：在原始响应解析完成时，能够稳定导出包含 `raw_score` 和 `reason` 的结构化结果。
- `N1` 非职责声明：不负责定义原始响应的生成过程。
- `N2` 非职责声明：不负责将解析结果与 `text_id`、`tag_id` 装配为完整记录，也不负责目标结果之后的归一化、阈值判定或持久化行为。

# 大模型调用接口:LLMCallingInterface

@framework

## 0. 目标 (Goal)
- 大模型调用接口负责将一个输入文本和一个输入标签组织为一次评分调用，并通过大模型服务获得单次调用的原始响应。为了防止多标签污染大模型评分结果，每次只输入一个文本和一个标签定义。
- 该模块不负责后续原始响应解析。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 大模型服务：负责消费提示词、文本和标签，并返回大模型原始响应。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `评分调用组合`：由 `{B1}` 消费 `TEXT_IN`、`TAG_IN` 形成 `RAW_RESPONSE_OUT`，导出 `C1`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `TEXT_IN`：运行时输入接口。载荷类型为结构化对象；基数为 `1`；语义为单个待评分文本单元；接口内容必须满足 `L0.M0`。
- `TAG_IN`：运行时输入接口。载荷类型为结构化对象；基数为 `1`；语义为单个待评分标签定义；接口内容必须满足 `L0.M1`。
- `RAW_RESPONSE_OUT`：运行时输出接口。载荷类型为结构化对象或文本；基数为 `1`；语义为单次评分调用返回的原始响应。

### 3.2 参数边界（Parameter Constraints）

- `P1` 提示词存在性：大模型调用必须存在一份完整提示词参数，作为一次评分调用的统一提示词输入。
- `P2` 提示词合法性：该提示词参数必须能够同时表达角色定位、任务说明、输出格式、评分值域与输出约束。
- `P3` 提示词内容约束：提示词必须显式要求大模型输出 `raw_score` 和 `reason`，对原始评分范围的描述必须与 `L-1.M1[R1]` 一致，并且结果可被后续原始响应解析器稳定解析。
- `P4` 提示词配置来源约束：提示词通过 json 配置文件读取，不由外部模块运行时输入。

## 4. 能力声明（Capability Statement）

- `C1` 原始响应获取能力：在输入文本、输入标签和提示词同时存在且合法时，能够发起一次合法的大模型评分调用并获得原始响应。
- `N1` 非职责声明：不负责原始响应解析、评分结果归一化、阈值判定或审核流转。
- `N2` 非职责声明：不负责大模型服务底层部署、存储或资源调度实现。

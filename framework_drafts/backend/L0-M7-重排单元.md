# 重排单元:RerankUnit

@framework

## 0. 目标 (Goal)
- 重排单元负责把候选文本集合按提问语义重新排序并筛选为证据文本集合。
- 该单元聚焦“候选输入 -> 重排打分 -> 证据导出”的结构闭环，不承担候选召回与回答生成职责。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 提问文本：重排阶段消费的单个提问文本输入。
- `B2` 候选文本集合：重排阶段消费的 `1..N` 个候选文本。
- `B3` 重排策略：控制候选比较、排序与筛选的策略参数集合。
- `B4` 证据文本集合：重排阶段导出的 `0..N` 个证据文本。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `候选打分`：由 `{B1, B2, B3}` 形成候选打分结果，导出 `C1`。
- `R2` `证据筛选`：由 `{B2, B3}` 形成证据文本集合，导出 `C2`。
- `R3` `分值对齐导出`：由 `{B1, B2, B4}` 形成证据与分值对齐输出，导出 `C3`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `query_text_in`：运行时输入接口，接收单个提问文本。
- `candidate_text_list_in`：运行时输入接口，接收 `1..N` 个候选文本。
- `rerank_policy_in`：运行时输入接口，接收重排策略参数集合。
- `evidence_text_list_out`：运行时输出接口，导出 `0..N` 个证据文本。
- `evidence_score_list_out`：运行时输出接口，导出与证据文本一一对齐的分值列表。

### 3.2 参数边界（Parameter Constraints）

- `candidate_count` 候选数量：约束单次重排允许接收的候选数量范围。
- `top_k` 保留数量：约束 `evidence_text_list_out` 的最大保留数量。
- `score_range` 分值范围：约束 `evidence_score_list_out` 的合法分值区间。
- `alignment_rule` 对齐规则：约束证据文本列表与分值列表必须一一对齐。

## 4. 能力声明（Capability Statement）

- `C1` 候选打分能力：支持基于提问文本和重排策略对候选文本打分。
- `C2` 证据导出能力：支持按重排结果导出证据文本集合。
- `C3` 分值伴随导出能力：支持导出与证据文本一一对应的分值列表。
- `N1` 非职责声明：不负责提问嵌入、候选召回与对象索引读取。
- `N2` 非职责声明：不负责回答文本生成与最终业务结论输出。

# 嵌入单元:EmbeddingUnit

@framework

## 0. 目标 (Goal)
- 嵌入单元负责把输入文本编码为向量。
- 该单元支持单个文本或文本列表输入，并导出与输入一一对应的向量或向量列表。
- 该单元不负责候选召回、相似度比较、结果过滤与对象读取。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 嵌入映射：定义文本如何被映射到向量空间。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `文本向量化`：由 `{B1}` 作用于输入文本，导出 `C1`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `text_in`：运行时输入接口，接收单个文本或文本列表。
- `vector_out`：运行时输出接口，导出与输入顺序一致的向量或向量列表。

### 3.2 参数边界（Parameter Constraints）

- `vector_dimension` 向量维度：约束 `vector_out` 中每个向量的固定维度。
- `text_count` 文本数量：约束单次 `text_in` 可接收的文本数量范围。
- `text_length` 文本长度：约束 `text_in` 中每个文本的合法长度范围。

## 4. 能力声明（Capability Statement）

- `C1` 文本向量化能力：支持把输入文本稳定映射为与输入顺序一一对应的向量输出。
- `N1` 非职责声明：不负责候选召回、相似度比较、结果过滤与对象读取。

## 5. 验证（Verification）

- `V1` 规则闭环性：`R1` 必须只依赖 `B1` 与输入文本载荷，并且只能导出 `C1`。
- `V2` 输入输出对齐性：输入 `N` 个文本时，输出必须为 `N` 个向量，且顺序一致。
- `V3` 参数符合性：文本数量、文本长度与向量维度都必须满足声明约束。

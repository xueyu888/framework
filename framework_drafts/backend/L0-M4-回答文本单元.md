# 回答文本单元:AnswerTextUnit

@framework

## 0. 目标 (Goal)
- 回答文本单元定义“生成阶段最终导出的回答文本对象”。
- 该单元聚焦回答内容、会话归属与时间信息，不承担检索、重排与证据筛选职责。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：最终回答文本内容。
- `B2` 序号：回答文本在所属会话中的顺序位置。
- `B3` 用户溯源：回答文本所属用户标识。
- `B4` 会话溯源：回答文本所属会话标识。
- `B5` 时间戳：回答文本创建或修改时间。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `回答定位组合`：由 `{B2, B3, B4}` 形成回答定位键，导出 `C1`。
- `R2` `回答单元组合`：由 `{B1, B2, B3, B4, B5}` 形成完整回答文本单元，导出 `C2` 与 `C3`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `answer_text_in`：运行时输入接口，接收单个回答文本块。
- `answer_seq_in`：运行时输入接口，接收单个回答序号。
- `answer_user_id_in`：运行时输入接口，接收单个用户标识。
- `answer_session_id_in`：运行时输入接口，接收单个会话标识。
- `answer_timestamp_in`：运行时输入接口，接收单个时间戳。
- `answer_text_unit_out`：运行时输出接口，导出 `{answer_text, seq, user_id, session_id, timestamp}`。

### 3.2 参数边界（Parameter Constraints）

- `P1` 编号值域：约束 `answer_seq_in` 与 `B2` 的值域为无符号 64 位整型。
- `P2` 文本块长度：约束 `answer_text_in` 与 `B1` 的合法长度范围。
- `P3` 文本块语言：约束 `answer_text_in` 与 `B1` 的允许语言集合。
- `P4` 联合唯一性：约束 `(user_id, session_id, seq)` 在同一会话域内必须唯一。
- `P5` 时间戳格式：约束 `answer_timestamp_in` 与 `B5` 采用统一时间格式并可解析。

## 4. 能力声明（Capability Statement）

- `C1` 单元定位能力：支持通过 `(user_id, session_id, seq)` 唯一定位回答文本单元。
- `C2` 原文读取能力：支持读取回答文本单元的文本内容。
- `C3` 归属追溯能力：支持将回答文本单元追溯到所属用户与会话。
- `N1` 非职责声明：不负责具体实例值、业务文案与外部实现细节。
- `N2` 非职责声明：不负责候选召回、重排筛选与证据确认。

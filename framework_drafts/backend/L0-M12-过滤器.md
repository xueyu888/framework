# 过滤器:FilterUnit

@framework

## 0. 目标 (Goal)
- 过滤器负责对带分值候选列表执行阈值过滤、排序与数量裁剪。
- 该过滤器保留候选键、候选向量与分值，供后续对象读取或进一步处理。
- 该过滤器不负责候选召回、相似度打分、对象读取与回答生成。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 过滤规则：定义带分值候选列表如何按阈值、顺序与数量被过滤。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `候选过滤`：由 `{B1}` 作用于带分值候选列表，导出 `C1`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `scored_candidate_pair_list_in`：运行时输入接口，接收 `0..N` 个 `{candidate_key, candidate_vector, score}`。
- `filtered_candidate_pair_list_out`：运行时输出接口，导出 `0..N` 个 `{candidate_key, candidate_vector, score}`。

### 3.2 参数边界（Parameter Constraints）

- `min_score` 最低保留阈值：约束低于该阈值的候选必须被移除。
- `score_order` 分值优先方向：约束过滤后候选的排序方向。
- `top_k` 候选数量上限：约束过滤后最多保留的候选数量。
- `subset_alignment` 子集对齐性：约束输出只能是输入候选的有序子集。

## 4. 能力声明（Capability Statement）

- `C1` 带分值候选过滤能力：支持把带分值候选列表过滤为更小的带分值候选列表。
- `N1` 非职责声明：不负责候选召回、相似度打分、对象读取与回答生成。

## 5. 验证（Verification）

- `V1` 规则闭环性：`R1` 必须只依赖 `B1` 与输入候选载荷，并且只能导出 `C1`。
- `V2` 子集符合性：任一输出候选都必须来自输入候选，且字段保持一致。
- `V3` 参数符合性：最低保留阈值、排序方向、数量上限与子集对齐性都必须满足声明约束。

# 提问记录存取协议:QueryRecordAccessProtocol

@framework

## 0. 目标 (Goal)
- 提问记录存取协议定义提问记录在存取时必须携带的字段及其约束。
- 该协议关注字段协议本身，不关注数据库、KV、缓存、索引、事务等实现细节。
- 该协议不负责生成提问记录键，只要求该键满足稳定标识约束。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 提问文本：一次提问对应的文本内容。
- `B2` 提问记录键：用于标识并定位该提问记录的键。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `提问记录字段协议`：由 `{B1, B2}` 形成合法提问记录字段集合，导出 `C1`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `query_text_in`：运行时输入接口，接收单个提问文本。
- `record_key_in`：运行时输入接口，接收单个提问记录键。
- `query_record_out`：运行时输出接口，导出 `{record_key, query_text}` 提问记录字段结构。

### 3.2 参数边界（Parameter Constraints）

- `record_key_uniqueness` 提问记录键唯一性：约束任意两条不同提问记录不得共享同一 `record_key`。
- `record_key_format` 提问记录键格式：约束 `record_key` 的类型、格式与合法值域。
- `query_text_length` 提问文本长度：约束 `query_text` 的合法长度范围。
- `query_text_lang` 提问文本语言：约束 `query_text` 的允许语言集合。

## 4. 能力声明（Capability Statement）

- `C1` 字段协议定义能力：能够稳定定义 `{record_key, query_text}` 这组提问记录字段及其约束。
- `N1` 非职责声明：不负责数据库、KV、缓存、索引、事务或任何具体存储实现。
- `N2` 非职责声明：不负责生成提问记录键，只要求该键满足唯一性与格式约束。
- `N3` 非职责声明：不负责查询改写、纠错、语义规范化、向量化、检索、重排与回答生成。

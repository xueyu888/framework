# 候选文本单元:CandidateKnowledgeTextUnit

@framework

## 0. 目标 (Goal)
- 候选文本单元定义“可被检索命中的候选文本对象”，用于承载可读取的候选文本内容与可定位标识。
- 该单元只回答两件事：候选文本是什么、如何稳定定位与读取；不承担检索打分、重排筛选、证据确认或回答生成职责。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 候选文本块：候选文本单元的文本内容。
- `B2` 候选绑定键：唯一定位 `B1` 的绑定键。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `候选文本单元`：由 `{B1, B2}` 形成 `候选文本单元`，导出 `C1` 与 `C2`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `candidate_text_in`：运行时输入接口，接收单个候选文本。
- `candidate_binding_key_in`：运行时输入接口，接收单个候选绑定键。
- `candidate_text_out`：运行时输出接口，导出可读取的候选文本。
- `candidate_binding_key_out`：运行时输出接口，导出可定位的候选绑定键。

### 3.2 参数边界（Parameter Constraints）

- `candidate_text_length` 候选文本长度：约束 `candidate_text_in` 与 `B1` 的文本长度范围。
- `candidate_text_lang` 候选文本语言：约束 `candidate_text_in` 与 `B1` 的允许语言集合。
- `candidate_binding_key_uniqueness` 候选绑定键唯一性：约束 `candidate_binding_key_in` 与 `B2` 的唯一定位关系。

## 4. 能力声明（Capability Statement）

- `C1` 单元定位能力：支持通过候选绑定键唯一定位候选文本单元。
- `C2` 原文读取能力：支持读取候选文本单元的文本内容。
- `N1` 非职责声明：不负责检索命中计算、重排排序、最终证据确认与回答生成。
- `N2` 非职责声明：不负责规定下游必须以绑定键或文本原文消费本单元；下游消费协议由下游模块边界定义。

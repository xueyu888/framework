# 生成单元:GenerationUnit

@framework

## 0. 目标 (Goal)
- 生成单元负责在提问文本与证据文本输入存在时导出回答文本。
- 该单元聚焦“输入收敛 -> 生成执行 -> 回答导出”的结构闭环，不承担检索召回与重排职责。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 提问文本：生成阶段消费的单个提问文本输入。
- `B2` 证据文本集合：生成阶段消费的 `1..N` 个证据文本。
- `B3` 生成策略：控制回答生成行为的策略参数集合。
- `B4` 回答文本：生成阶段导出的回答文本结果。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `生成输入收敛`：由 `{B1, B2}` 形成生成上下文，导出 `C1`。
- `R2` `回答生成`：由 `{B1, B2, B3}` 形成回答文本输出，导出 `C2`。
- `R3` `证据关联导出`：由 `{B2, B4}` 形成回答到证据的可追溯关联，导出 `C3`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `query_text_in`：运行时输入接口，接收单个提问文本。
- `evidence_text_list_in`：运行时输入接口，接收 `1..N` 个证据文本。
- `generation_policy_in`：运行时输入接口，接收生成策略参数集合。
- `answer_text_out`：运行时输出接口，导出单个回答文本。
- `answer_evidence_trace_out`：运行时输出接口，导出回答文本与证据文本的关联信息。

### 3.2 参数边界（Parameter Constraints）

- `max_output_length` 最大输出长度：约束 `answer_text_out` 的最大长度范围。
- `output_lang` 输出语言：约束 `answer_text_out` 的允许语言集合。
- `evidence_count` 证据数量：约束 `evidence_text_list_in` 可接收的证据数量范围。
- `generation_temperature` 生成温度：约束生成策略中的采样温度值域。

## 4. 能力声明（Capability Statement）

- `C1` 生成输入收敛能力：支持把提问文本与证据文本收敛为可执行的生成上下文。
- `C2` 回答文本生成能力：支持在给定生成策略时稳定导出回答文本。
- `C3` 证据关联导出能力：支持导出回答文本与证据文本的关联关系。
- `N1` 非职责声明：不负责候选召回、向量比较、过滤裁剪与重排排序。
- `N2` 非职责声明：不负责回答文本的外部持久化、发布策略与权限控制。

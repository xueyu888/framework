# 证据文本单元:EvidenceTextUnit

@framework

## 0. 目标 (Goal)
- 证据文本单元定义“本次问答最终引用”的文本对象。
- 该单元承载证据文本内容、证据顺序与相关度分数，用于回答阶段稳定消费。
- 该单元不负责预处理、候选召回、重排计算与回答生成。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：最终确认并可被引用的证据文本内容。
- `B2` 序号：证据文本在当前候选集合中的顺序编号。
- `B3` 相关度分数：证据文本与提问之间的相关度分值。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `证据单元组合`：由 `{B1, B2, B3}` 形成证据文本单元，导出 `C1`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `evidence_text_in`：运行时输入接口，接收单个证据文本块。
- `evidence_rank_in`：运行时输入接口，接收单个证据序号。
- `evidence_score_in`：运行时输入接口，接收单个相关度分数。
- `evidence_text_unit_out`：运行时输出接口，导出 `{evidence_text, evidence_rank, evidence_score}`。

### 3.2 参数边界（Parameter Constraints）

- `P1` 编号值域：约束 `evidence_rank_in` 与 `B2` 的值域为无符号 64 位整型。
- `P2` 文本块长度：约束 `evidence_text_in` 与 `B1` 的合法长度范围。
- `P3` 文本块语言：约束 `evidence_text_in` 与 `B1` 的允许语言集合。
- `P4` 相关度分数值域：约束 `evidence_score_in` 与 `B3` 的取值范围在 `0..1`。

## 4. 能力声明（Capability Statement）

- `C1` 单元定位能力：支持通过证据序号与分值上下文稳定定位证据文本单元。
- `N1` 非职责声明：不负责预处理、切块、纠错、去噪等上游文本清洗实现。
- `N2` 非职责声明：不负责语义检索、内容匹配、向量召回与重排计算。
- `N3` 非职责声明：不负责回答生成与最终业务结论输出。

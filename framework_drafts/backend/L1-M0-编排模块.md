# 编排模块:OrchestrationModule

@framework

## 0. 目标 (Goal)
- 编排模块负责将检索、重排、生成节点按统一结构连接为从提问到回答的闭环链路。
- 该模块只声明节点连接关系、端口兼容关系与顺序约束，不重定义各节点内部算法。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 检索节点：`L0.M2[R1] + L1.M1[R1]`，负责消费提问并导出候选。
- `B2` 重排节点：`L0.M2[R1] + L0.M0[R1] + L0.M7[R1] + L0.M3[R1]`，负责消费提问与候选并导出证据。
- `B3` 生成节点：`L0.M2[R1] + L0.M3[R1] + L0.M6[R1] + L0.M4[R1]`，负责消费提问与证据并导出回答。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `检索节点规则`：`B1` 必须消费 `QUERY_IN` 并产出 `CANDIDATE_OUT`，导出 `C1`。
- `R2` `重排节点规则`：`B2` 必须消费 `QUERY_IN + CANDIDATE_IN` 并产出 `EVIDENCE_OUT`，导出 `C2`。
- `R3` `生成节点规则`：`B3` 必须消费 `QUERY_IN + EVIDENCE_IN` 并产出 `ANSWER_OUT`，导出 `C3`。
- `R4` `合法连接规则`：只允许 `CANDIDATE_OUT -> CANDIDATE_IN` 与 `EVIDENCE_OUT -> EVIDENCE_IN`，导出 `C4`。
- `R5` `阶段顺序规则`：检索节点必须在重排节点之前，重排节点必须在生成节点之前，生成节点为终端节点。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `QUERY_IN`：运行时输入接口，`L0.M2[R1]`，基数为 `1`。
- `CANDIDATE_OUT`：运行时输出接口，`L0.M0[R1]`，基数为 `1..N`。
- `CANDIDATE_IN`：运行时输入接口，`L0.M0[R1]`，基数为 `1..N`。
- `EVIDENCE_OUT`：运行时输出接口，`L0.M3[R1]`，基数为 `1..N`。
- `EVIDENCE_IN`：运行时输入接口，`L0.M3[R1]`，基数为 `1..N`。
- `ANSWER_OUT`：运行时输出接口，`L0.M4[R1]`，基数为 `1`。

### 3.2 参数边界（Parameter Constraints）

- `topology_acyclic` 拓扑无环性：约束节点连接必须为单向无环图。
- `stage_order` 阶段顺序：约束检索、重排、生成必须按固定顺序执行。
- `port_cardinality` 端口基数：约束各端口的输入输出基数必须满足定义范围。
- `channel_compatibility` 通道兼容性：约束 `CANDIDATE` 通道与 `EVIDENCE` 通道不可交叉混用。

## 4. 能力声明（Capability Statement）

- `C1` 候选导出能力：在提问输入存在时，能够导出候选文本单元集合。
- `C2` 证据导出能力：在提问输入与候选输入存在时，能够导出证据文本单元集合。
- `C3` 回答导出能力：在提问输入与证据输入存在时，能够导出回答文本单元。
- `C4` 闭环问答编排能力：在检索、重排、生成节点按合法顺序连接时，能够形成从提问到回答的完整编排链。
- `N1` 非职责声明：不负责检索、重排、生成节点内部算法实现。
- `N2` 非职责声明：不负责外部存储、部署拓扑与运行时资源调度策略。

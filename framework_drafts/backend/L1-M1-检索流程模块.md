# 检索流程模块:RetrievalFlowModule

@framework

## 0. 目标 (Goal)

- 检索流程模块负责把单个提问文本收敛为 `0..N` 个候选文本及其分值。
- 本模块不重新定义嵌入、对象索引、相似度比较与过滤的内部算法；只定义这些 `L0` 模块如何按固定顺序闭合成检索链。

## 1. 能力声明（Capability Statement）

- `C1` 候选文本导出能力：在提问输入存在时，能够导出一个或多个候选文本单元。
- `C2` 分值伴随导出能力：在候选文本导出时，能够同时导出与候选文本一一对齐的分值列表。
- `C3` 检索链路闭环能力：在查询编码、候选召回、精确比较、候选过滤与候选读取按合法顺序连接时，能够形成完整检索链。

## 2. 边界定义（Boundary / Parameter 参数）

- `QUERY_TEXT_IN` 提问输入：`text_in ∈ L0.M9[R1]` 基数 = `1`；检索链必须且只能接收一个提问文本作为查询输入。
- `CANDIDATE_PAIR_OUT` 召回候选：`candidate_pair_list_out ∈ L0.M10[R2]` 基数 = `1..*`；对象索引器召回一个或多个带键候选向量。
- `SCORED_CANDIDATE_OUT` 打分候选：`scored_candidate_pair_list_out ∈ L0.M11[R1]` 基数 = `1..*`；相似度比较器输出一个或多个带键带向量分值结果。
- `FILTERED_CANDIDATE_OUT` 过滤候选：`filtered_candidate_pair_list_out ∈ L0.M12[R1]` 基数 = `0..*`；过滤器输出按规则保留的带键带向量分值结果。
- `CANDIDATE_TEXT_OUT` 候选输出：`L0.M0[R1] ∈ B5` 基数 = `0..*`；检索链输出一个或多个候选文本单元。
- `SCORE_OUT` 分值输出：`score_out ∈ L0.M11[R1] + L0.M12[R1]` 基数 = `0..*`；检索链输出与候选文本一一对齐的分值结果。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 查询编码基：`L0.M9[R1]`。来源：`C3 + QUERY_TEXT_IN`。
- `B2` 候选召回基：`L0.M10[R2]`。来源：`C3 + CANDIDATE_PAIR_OUT`。
- `B3` 精确比较基：`L0.M11[R1]`。来源：`C2 + SCORED_CANDIDATE_OUT`。
- `B4` 候选过滤基：`L0.M12[R1]`。来源：`C2 + FILTERED_CANDIDATE_OUT`。
- `B5` 候选读取基：`L0.M10[R3] + L0.M0[R1]`。来源：`C1 + CANDIDATE_TEXT_OUT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 检索闭环：`QUERY_TEXT_IN -> L0.M9[R1] -> L0.M10[R2] -> L0.M11[R1] -> L0.M12[R1] -> L0.M10[R3] + L0.M0[R1] -> CANDIDATE_TEXT_OUT + SCORE_OUT`。
- `R2` 候选读取必须消费 `FILTERED_CANDIDATE_OUT` 中保留的候选键，并产出 `CANDIDATE_TEXT_OUT`。
- `R3` `SCORE_OUT` 只允许来自 `SCORED_CANDIDATE_OUT` 经 `FILTERED_CANDIDATE_OUT` 保留后的分值子集。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合顺序、输出能力或失效结论。
- `V2` 链路闭环性：`R1` 的输出能力并集必须覆盖 `C1 ~ C3`，且不得包含未声明能力。
- `V3` 边界符合性：任意有效检索链都必须同时满足输入基数、候选基数与分值对齐边界。
- `V4` 最小必要性：移除 `B1 ~ B5` 任一结构基后，`C1 ~ C3` 中至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。

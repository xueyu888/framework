# 检索单元:RetrievalUnit

@framework

## 0. 目标 (Goal)
- 检索单元负责接收本次提问文本，并在明确的外部输入输出契约与内部执行约束下，完成查询向量生成、索引向量比较、命中裁剪以及候选文本与分值导出。
- 该单元对外暴露的是“提问文本 -> N 个候选文本及其分值”的检索能力；候选向量到候选文本的映射属于单元内部转换，不作为外部边界暴露。

## 1. 能力声明（Capability Statement）

- `C1` 比较分值导出能力：在提问文本与索引向量集存在时，能够导出候选向量的比较分值集合。
- `C2` 候选文本及分值导出能力：在比较分值集合存在时，能够按外部输出约束导出有限个候选文本及其对应分值。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责候选文本原文载入、最终证据确认、重排排序与回答生成。
- `N2` 非职责声明：不负责在外部边界中暴露候选向量到候选文本的内部绑定细节；该映射只作为单元内部约束存在。
- `N3` 非职责声明：不负责在本单元边界中声明上游模块来源或跨模块连接拓扑；该类关系由编排模块统一表达。

## 2. 边界定义（Boundary / Parameter 参数）

### 2.1 外部边界（External Boundary）

- `query_text_length` 提问长度上限：对应 `B1`。必须给出正整数上限和计量单位（字符或 token）；有效提问长度必须落在闭区间 `[1, 上限]`。
- `query_text_lang` 提问语言集合：对应 `B1`。必须给出允许语言的有限枚举集合；输入语言不在集合内则请求非法。
- `top_k` 候选输出数量上限：对应 `B6 + B7`。必须给出正整数；最终导出的候选文本数量不得超过该值。
- `score_order` 分值优先方向：对应 `B5 + B7`。必须明确为 `desc` 或 `asc`；禁止使用“按默认顺序”这类不可测试表述。
- `min_score` 最低保留阈值：对应 `B5 + B7`。必须给出数值阈值，或显式声明为空；若非空，则未达到阈值的候选不得导出。

### 2.2 内部边界（Internal Boundary）
- `vector_dimension` 向量维度：对应 `B2 + B3 + B4`。必须给出正整数维度；`B2` 与 `B4` 中每个向量的维度都必须严格等于该值。
- `vector_norm` 向量归一化规则：对应 `B2 + B4`。取值必须明确为 `none` 或 `l2`；若为 `l2`，则 `B2` 与 `B4` 中每个向量的 L2 范数都必须在实现规定容差内等于 `1`。
- `retrieval_scope` 检索作用域：对应 `B4`。必须能解析为一个有限索引子集、命名索引集合或确定性过滤表达式；超出该作用域的向量不得进入本次比较。
- `comparator_kind` 比较器类型：对应 `B5`。必须明确为 `cosine_similarity`、`dot_product` 或 `l2_distance` 之一。
- `candidate_binding_uniqueness` 候选绑定唯一性：对应 `B6`。每个被保留的候选向量都必须映射到且只映射到一个候选文本单元。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 提问文本：用于发起本次检索的文本输入。
- `B2` 查询向量：由 `B1` 转换得到的查询向量，是本次比较的查询轴。
- `B3` 向量空间：`B2` 与 `B4` 中元素共同所属的可比较向量域。
- `B4` 索引向量集：被检索的一组候选向量，且每个候选向量都属于 `B3`。
- `B5` 比较分值：`B2` 与 `B4` 中各向量经合法比较后得到的数值结果。
- `B6` 候选文本绑定键：把 `B4` 中每个候选向量映射到其对应候选文本单元的内部绑定关系。
- `B7` 候选文本单元：由 `B6` 解析得到、可作为外部输出返回的一组候选文本单元。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法检索导出组合：
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6 + B7`
  - `R1.2` 组合方式：提问文本先被转换为查询向量；查询向量与检索作用域内的索引向量进行比较，形成比较分值集合；随后按分值方向排序、按阈值过滤、按 `top_k` 裁剪；每个被保留的候选向量都必须解析为且仅解析为一个候选文本单元，并与其原始分值一起导出。
  - `R1.3` 输出能力：`C1 + C2`
  - `R1.4` 参数绑定：`query_text_length + query_text_lang + vector_dimension + vector_norm + retrieval_scope + comparator_kind + top_k + score_order + min_score + candidate_binding_uniqueness`

- `R2` 非法组合：
  - `R2.1` 若任一外部边界或内部边界无法满足，或提问文本无法形成可比较的查询向量，则组合非法。
  - `R2.2` 若任一被保留的候选向量无法通过 `B6` 得到满足 `candidate_binding_uniqueness` 的唯一候选文本单元，则组合非法。
  - `R2.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1` 的输出能力必须覆盖 `C1 ~ C2`，且不得包含未声明的能力。
- `V3` 外部边界符合性：任意有效检索组合必须同时满足 `query_text_length + query_text_lang + top_k + score_order + min_score`。
- `V4` 内部边界符合性：任意有效检索组合必须同时满足 `vector_dimension + vector_norm + retrieval_scope + comparator_kind + candidate_binding_uniqueness`。
- `V5` 可度量性：每个边界项都必须能落到数值、有限枚举、精确标识或确定性过滤表达式；禁止使用“合适”“默认”“高质量”“常规”之类不可判定措辞。
- `V6` 可测试性：每个边界项都必须对应至少一个可执行的通过/失败断言，例如长度断言、维度断言、归一化断言、排序断言、阈值断言或唯一性断言。
- `V7` 输出完备性：`R1` 导出的每个候选文本都必须携带且只携带一个数值分值，且导出数量必须落在闭区间 `[0, top_k]`。
- `V8` 最小必要性：移除任一 `B*` 后，`C1 ~ C2` 中至少一项无法成立。
- `V9` 非职责隔离性：检索单元不得直接导出最终证据结论或回答文本。
- `V10` 边界职责隔离性：外部边界只声明提问输入与候选输出契约，内部边界只声明执行约束；两者不得混写。
- `V11` 结论表达：逐条输出 `R1 -> C* + 参数绑定` 或 `R2 -> 非法组合` 的通过或失败结论。

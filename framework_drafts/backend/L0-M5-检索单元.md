# 检索单元:RetrievalUnit

@framework

## 0. 目标 (Goal)
- 检索单元负责接收本次查询文本，并在明确的输入长度、表示模型、向量维度、比较器类型与召回窗口约束下，完成查询向量生成、索引向量比较、命中裁剪与来源定位导出。
- 该单元对外暴露的是“从查询文本到候选定位结果”的完整检索能力；查询文本与其他模块的连接关系由编排模块统一说明，本单元不在边界层重复声明跨模块连线。

## 1. 能力声明（Capability Statement）

- `C1` 合法查询表示形成能力：在查询文本满足长度与语言约束，且表示模型、向量维度、归一化规则与向量空间标识明确时，能够形成可进入检索比较阶段的查询向量。
- `C2` 比较结果导出能力：在查询向量、索引向量集与合法比较器存在时，能够导出候选向量的比较结果集合。
- `C3` 候选召回能力：在比较结果存在时，能够按明确的排序方向、数量上限与分值阈值导出有限命中集合。
- `C4` 来源定位导出能力：在命中集合存在时，能够导出对应来源文本单元的定位键集合。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责候选文本原文载入、最终证据确认、重排排序与回答生成。
- `N2` 非职责声明：不负责在本单元边界中声明上游模块来源或跨模块连接拓扑；该类关系由编排模块统一表达。

## 2. 边界定义（Boundary / Parameter 参数）

- `query_text_length` 查询长度上限：对应 `B1`。必须给出正整数上限和计量单位（字符或 token）；有效查询长度必须落在闭区间 `[1, 上限]`。
- `query_text_lang` 查询语言集合：对应 `B1`。必须给出允许语言的有限枚举集合；输入语言不在集合内则请求非法。
- `encoder_id` 表示模型标识：对应 `B1 + B2`。必须给出生成 `B2` 的精确模型或服务标识，禁止使用“任意 embedding 模型”这类不可判定表述。
- `vector_dimension` 向量维度：对应 `B2 + B3 + B4`。必须给出正整数维度；`B2` 与 `B4` 中每个向量的维度都必须严格等于该值。
- `vector_norm` 向量归一化规则：对应 `B2 + B4`。取值必须明确为 `none` 或 `l2`；若为 `l2`，则 `B2` 与 `B4` 中每个向量的 L2 范数都必须在实现规定容差内等于 `1`。
- `vector_space_id` 向量空间标识：对应 `B2 + B3 + B4 + B5`。必须给出一个可精确比较的空间标识；只有 `vector_space_id` 完全一致的查询向量、索引向量与比较器才允许组合。
- `retrieval_scope` 检索作用域：对应 `B4`。必须能解析为一个有限索引子集、命名索引集合或确定性过滤表达式；超出该作用域的向量不得进入本次比较。
- `comparator_kind` 比较器类型：对应 `B5`。必须明确为 `cosine_similarity`、`dot_product` 或 `l2_distance` 之一。
- `score_order` 分值优先方向：对应 `B5`。必须明确为 `desc` 或 `asc`；禁止使用“按默认顺序”这类不可测试表述。
- `top_k` 召回数量上限：对应 `B4 + B5`。必须给出正整数；最终命中数量不得超过该值。
- `min_score` 最低保留阈值：对应 `B4 + B5`。必须给出数值阈值，或显式声明为空；若非空，则未达到阈值的命中不得保留。
- `source_key_uniqueness` 来源定位唯一性：对应 `B6`。`R3` 保留的每个命中项都必须映射到且只映射到一个来源定位键。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 查询文本：用于发起本次检索的文本输入。
- `B2` 查询向量：由 `B1` 在 `encoder_id + vector_dimension + vector_norm + vector_space_id` 约束下生成的查询向量，是本次比较的查询轴。
- `B3` 向量空间：`B2` 与 `B4` 中元素共同所属的可比较向量域。
- `B4` 索引向量集：被检索的一组候选向量，且每个候选向量都属于 `B3`。
- `B5` 相似度比较器：定义在 `B3` 上的比较器，用于计算 `B2` 与 `B4` 中各向量的比较结果。
- `B6` 来源定位键：把 `B4` 中每个候选向量映射到其来源文本单元的定位键（必须唯一）。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法表示生成组合：
  - `R1.1` 参与基：`B1 + B2 + B3`
  - `R1.2` 组合方式：`B1` 的长度必须满足 `query_text_length`，语言必须属于 `query_text_lang`，并通过 `encoder_id` 被转换为 `B2`；生成后的 `B2` 维度必须等于 `vector_dimension`，归一化规则必须满足 `vector_norm`，且 `B2` 必须属于 `vector_space_id` 指定的 `B3`。
  - `R1.3` 输出能力：`C1`
  - `R1.4` 参数绑定：`query_text_length + query_text_lang + encoder_id + vector_dimension + vector_norm + vector_space_id`

- `R2` 合法比较组合：
  - `R2.1` 参与基：`B2 + B3 + B4 + B5`
  - `R2.2` 组合方式：`retrieval_scope` 解析出的 `B4` 中每个向量都必须属于 `vector_space_id` 指定的同一 `B3`，维度必须等于 `vector_dimension`，归一化规则必须满足 `vector_norm`；`B5` 必须与 `comparator_kind` 一致，并对 `B2` 与 `B4` 中各元素进行比较，形成比较结果集合。
  - `R2.3` 输出能力：`C2`
  - `R2.4` 参数绑定：`retrieval_scope + vector_dimension + vector_norm + vector_space_id + comparator_kind`

- `R3` 合法召回组合：
  - `R3.1` 参与基：`B4 + B5`
  - `R3.2` 组合方式：对 `R2` 导出的比较结果集合，依据 `score_order` 进行排序；若 `min_score` 非空，则先剔除未达到阈值的命中；最终结果按 `top_k` 裁剪为有限命中集合。
  - `R3.3` 输出能力：`C3`
  - `R3.4` 参数绑定：`score_order + top_k + min_score`

- `R4` 合法定位组合：
  - `R4.1` 参与基：`B6`
  - `R4.2` 组合方式：`R3` 导出的每个命中项都必须能够通过 `B6` 映射为且仅映射为一个来源定位键，形成来源定位结果集合。
  - `R4.3` 输出能力：`C4`
  - `R4.4` 参数绑定：`source_key_uniqueness`

- `R5` 非法组合：
  - `R5.1` 若 `B1` 长度不满足 `query_text_length`、语言不属于 `query_text_lang`，或 `encoder_id` 无法把 `B1` 形成满足 `vector_dimension + vector_norm + vector_space_id` 的合法 `B2`，则组合非法。
  - `R5.2` 若 `B4` 中存在向量维度不等于 `vector_dimension`、归一化不满足 `vector_norm`、`vector_space_id` 不一致，或 `B5` 与 `comparator_kind` 不匹配，则组合非法。
  - `R5.3` 若 `score_order`、`top_k` 或 `min_score` 任一无法被确定性执行，则召回组合非法。
  - `R5.4` 若 `R3` 导出的任一命中项无法通过 `B6` 得到满足 `source_key_uniqueness` 的来源定位键，则组合非法。
  - `R5.5` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1 ~ R4` 的输出能力并集必须覆盖 `C1 ~ C4`，且不得包含未声明的能力。
- `V3` 参数符合性：任意有效检索组合必须同时满足 `query_text_length + query_text_lang + encoder_id + vector_dimension + vector_norm + vector_space_id + retrieval_scope + comparator_kind + score_order + top_k + min_score + source_key_uniqueness`。
- `V4` 映射完备性：`R3` 导出的每个命中项都必须能通过 `B6` 导出唯一来源定位键。
- `V5` 可度量性：每个边界项都必须能落到数值、有限枚举、精确标识或确定性过滤表达式；禁止使用“合适”“默认”“高质量”“常规”之类不可判定措辞。
- `V6` 可测试性：每个边界项都必须对应至少一个可执行的通过/失败断言，例如长度断言、维度断言、归一化断言、排序断言、阈值断言或唯一性断言。
- `V7` 最小必要性：移除任一 `B*` 后，`C1 ~ C4` 中至少一项无法成立。
- `V8` 非职责隔离性：检索单元不得直接导出候选文本原文、最终证据结论或回答文本。
- `V9` 边界职责隔离性：本单元边界只声明本地契约，不在边界项中重复写死上游模块标识或跨模块连线。
- `V10` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R5 -> 非法组合` 的通过或失败结论。

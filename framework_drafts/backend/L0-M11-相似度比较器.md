# 相似度比较器:SimilarityComparator

@framework

## 0. 目标 (Goal)
- 相似度比较器负责在给定比较度量下，对查询向量与候选向量列表进行精确打分。
- 该比较器只负责比较本身，不负责候选召回、对象存取、文本解引用与向量生成。
- 该比较器采用带键带向量输入输出，使后续过滤与读取不必再次查询候选向量。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 比较度量：定义查询向量与候选向量如何计算相似度或距离分值。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `精确打分`：由 `{B1}` 作用于输入向量与候选键值对，导出 `C1`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `query_vector_in`：运行时输入接口，接收单个查询向量。
- `candidate_pair_list_in`：运行时输入接口，接收 `1..N` 个 `{candidate_key, candidate_vector}`。
- `scored_candidate_pair_list_out`：运行时输出接口，导出 `1..N` 个 `{candidate_key, candidate_vector, score}`。

### 3.2 参数边界（Parameter Constraints）

- `vector_dimension` 向量维度：约束 `query_vector_in` 与所有 `candidate_vector` 的维度必须一致。
- `metric_kind` 比较度量类型：约束比较器使用的相似度或距离度量方式。
- `candidate_count` 候选数量：约束单次比较允许接收的候选数量范围。
- `input_output_alignment` 输入输出对齐性：约束每个输入 `candidate_key` 在输出中都必须有且仅有一个对应 `{candidate_vector, score}`。

## 4. 能力声明（Capability Statement）

- `C1` 带键精确打分能力：支持把查询向量与候选键值对列表比较为带键带向量分值结果列表。
- `N1` 非职责声明：不负责候选召回、ANN 缩圈、对象存取、文本解引用与向量生成。
- `N2` 非职责声明：不负责阈值过滤、Top-K 裁剪与重排策略；这些属于更高层检索或重排流程。

## 5. 验证（Verification）

- `V1` 规则闭环性：`R1` 必须只依赖 `B1` 与输入向量载荷，并且只能导出 `C1`。
- `V2` 维度符合性：`query_vector_in` 与所有 `candidate_vector` 必须满足同一向量维度约束。
- `V3` 对齐符合性：每个输入 `candidate_key` 都必须在输出中且仅在输出中出现一次。
- `V4` 参数符合性：比较度量类型、候选数量与输入输出对齐性都必须满足声明约束。

# 批量程序预标注:BatchProgrammaticPreAnnotation

@framework

## 0. 模块身份（Module Identity）

    模块名 := 批量程序预标注:BatchProgrammaticPreAnnotation
    问题域 := 批量预标注编排域
    目标   := 批量程序预标注模块负责接收文本块集合和标签集合，并依次将单个文本块和单个标签传入程序预标注模块完成逐项标注，再将所有单次结果装配为批量输出结果。该模块只声明批量轮询执行与结果装配，不负责文本块集合或标签集合的来源生成。
    模块型 := 编排型

## 1. Base

    B1 程序预标注模块 := L1.M0，负责对单个文本块和单个标签执行一次程序预标注。

## 2. Combination Principles

只写原则，不写具体结果类。
记号：`x ≡ y` 表示同一结果类；`nf(x)` 表示 `x` 的规范形。

    R1 批量轮询执行原则:
      formation     := combinable(main_in, aux_in, B1) iff 文本块集合与标签集合可被逐项遍历，且每次遍历形成的单个文本块和单个标签都能作为 B1 的合法输入。
      satisfaction  := satisfied(main_in, aux_in, B1) iff 每个文本块与每个标签都已依次交给 B1 执行一次程序预标注。
      identity      := <main_in, aux_in, B1> ≡ <main_in', aux_in', B1'> iff 两者在轮询覆盖范围与单次预标注执行语义上等价。
      normalization := nf(<main_in, aux_in, B1>) = <批量轮询预标注> iff 该组合满足集合输入到逐项执行的批量预标注闭环。

    R2 批量结果装配原则:
      formation     := combinable(B1...) iff 每次单次程序预标注执行都导出可被纳入批量结果的单次结果。
      satisfaction  := satisfied(B1...) iff 所有单次结果都被装配到同一批量输出结果中。
      identity      := <B1...> ≡ <B1'...> iff 两者形成的批量输出在结果成员语义上等价。
      normalization := nf(<B1...>) = <批量程序预标注结果集合> iff 该组合满足单次结果到批量输出结果的装配闭环。

## 3. Result Space

    A1 批量程序预标注结果:
      form    := <B1...>
      by      := R1, R2
      meaning := 所有单次程序预标注结果被装配后的批量输出结果；其中每个结果成员都满足 L-1.M5。

## 4. Boundary

### 4.1 Input / Output

    main_in:
      payload     := 结构化对象集合；语义为待执行批量程序预标注的文本块集合；其中每个成员都满足 L-1.M1 定义的字段约束。
      cardinality := 1
      to          := B1

    aux_in:
      payload     := 结构化对象集合；语义为待参与批量程序预标注的标签集合；其中每个成员都满足 L-1.M2 定义的字段约束。
      cardinality := 1
      to          := B1

    main_out:
      payload     := 结构化对象集合；语义为批量程序预标注结果集合；其中每个成员都满足 L-1.M5。
      cardinality := 1
      from        := A1

### 4.2 Parameter

    P1 无新增独立参数:
      domain  := 本模块无新增独立参数。
      affects := 无

## 5. Capability Statement

    C1 批量轮询执行能力:
      requires   := main_in, aux_in, R1
      guarantees := 能够依次将单个文本块和单个标签传入程序预标注模块完成批量执行。

    C2 批量结果装配能力:
      requires   := A1, R2
      guarantees := 能够将所有单次程序预标注结果装配为同一批量输出结果。

    N1 非职责声明 := 不负责生成文本块集合或标签集合。
    N2 非职责声明 := 不负责领域筛选、数据库读取或结果持久化。

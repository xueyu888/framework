# 大模型服务:LLMService

@framework

## 0. 模块身份（Module Identity）

    模块名 := 大模型服务:LLMService
    问题域 := 评分调用处理域
    目标   := 大模型调用接口负责将一个输入文本和一个输入标签组织为一次评分调用，并通过大模型服务获得单次调用的原始响应。为了防止多标签污染大模型评分结果，每次只输入一个文本和一个标签定义。该模块不负责后续原始响应解析。
    模块型 := 处理型

## 1. Base

    B1 大模型服务 := 负责消费提示词、文本和标签，并返回大模型原始响应。

## 2. Combination Principles

只写原则，不写具体结果类。
记号：`x ≡ y` 表示同一结果类；`nf(x)` 表示 `x` 的规范形。

    R1 评分调用组合:
      formation     := combinable(B1) iff B1 同时接收满足 L-1.M1 的单个文本输入、满足 L-1.M2 的单个标签输入，且提示词参数满足 P1, P2, P3, P4。
      satisfaction  := satisfied(B1) iff B1 基于文本、标签和提示词完成一次单文本单标签评分调用，并导出原始响应。
      identity      := <B1> ≡ <B1'> iff 两者在评分调用的输入约束与原始响应输出语义上等价。
      normalization := nf(<B1>) = <单次评分调用> iff 该组合满足单文本单标签评分调用闭环。

## 3. Result Space

    A1 原始响应结果:
      form    := RAW_RESPONSE_OUT
      by      := R1
      meaning := 单次评分调用返回的原始响应。

## 4. Boundary

### 4.1 Input / Output

    main_in:
      payload     := 结构化对象；语义为单个待评分文本单元；接口内容必须满足 L-1.M1。
      cardinality := 1
      to          := B1

    aux_in:
      payload     := 结构化对象；语义为单个待评分标签定义；接口内容必须满足 L-1.M2。
      cardinality := 1
      to          := B1

    main_out:
      payload     := 结构化对象或文本；语义为单次评分调用返回的原始响应。
      cardinality := 1

### 4.2 Parameter

    P1 提示词存在性:
      domain  := 大模型调用必须存在一份完整提示词参数，作为一次评分调用的统一提示词输入。
      affects := R1, IO

    P2 提示词合法性:
      domain  := 该提示词参数必须能够同时表达角色定位、任务说明、输出格式、评分值域与输出约束。
      affects := R1

    P3 提示词内容约束:
      domain  := 提示词必须显式要求大模型输出 raw_score 和 reason，对原始评分范围的描述必须与 L-1.M0[R1] 一致，并且结果可被后续原始响应解析器稳定解析。
      affects := R1, A1

    P4 提示词配置来源约束:
      domain  := 提示词通过 json 配置文件读取，不由外部模块运行时输入。
      affects := IO

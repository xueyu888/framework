# 大模型调用接口:LLMInterface

@framework

0. 目标（Target）
声明一个大模型调用模块，用于接收一次评分请求内容和一次模型调用配置，通过大模型服务入口发起一次大模型评分调用，并导出未经上层解析与归一化处理的大模型原始响应。

## 1. 能力声明（Capability Statement）

- `C1` 评分请求内容消费能力：能接收一份可直接提交给大模型的评分请求内容。
- `C2` 模型调用配置消费能力：能接收一份用于约束本次大模型调用的模型调用配置。
- `C3` 大模型服务入口消费能力：能接收一个可用于发起大模型评分调用的大模型服务入口。
- `C4` 大模型评分调用能力：能根据评分请求内容、模型调用配置和大模型服务入口发起一次大模型评分调用。
- `C5` 大模型原始响应导出能力：能导出大模型返回的原始响应内容。

- `N1` 不提供原始响应解析能力：不负责从原始响应中稳定提取评分值或评分原因。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 评分请求内容存在性边界：评分请求内容不可为空。来源：`C1`。
- `P2` 评分请求内容可提交性边界：评分请求内容必须为单次调用可直接提交给大模型的一份完整内容。来源：`C1`。

- `P3` 模型调用配置存在性边界：模型调用配置不可为空。来源：`C2`。
- `P4` 模型调用配置最小完整性边界：模型调用配置必须至少包含单次调用所需的模型选择或等效调用参数。来源：`C2`。

- `P5` 大模型服务入口存在性边界：大模型服务入口不可为空。来源：`C3`。
- `P6` 大模型服务入口可调用性边界：大模型服务入口必须能够接收一次评分请求内容和模型调用配置并返回对应原始响应。来源：`C3`。

- `P7` 调用次数边界：单次有效组合只允许发起 `1` 次大模型评分调用。来源：`C4`。

- `P8` 原始响应存在性边界：大模型原始响应不可为空。来源：`C5`。
- `P9` 原始响应保持性边界：导出时必须保留大模型返回的原始响应，不在本层做字段级解析或归一化改写。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 评分请求内容：供一次大模型评分的完整请求内容。来源：`C1 + P1 + P2`。
- `B2` 模型调用配置：约束单次大模型评分调用的模型选择和等效调用参数。来源：`C2 + P3 + P4`。
- `B3` 大模型服务入口：供一次大模型评分调用提交请求并接收响应的大模型服务入口。来源：`C3 + P5 + P6`。
- `B4` 大模型原始响应：大模型返回的、未经上层解析和归一化处理的原始响应内容。来源：`C5 + P8 + P9`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单次评分调用组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：接收一份评分请求内容和一份模型调用配置后，通过大模型服务入口发起一次大模型评分调用，将请求内容与调用配置一并提交给大模型处理。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7`。

- `R2` 原始响应导出组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R2.2` 组合方式：在完成单次评分调用后，将大模型返回内容整体作为原始响应导出，不在本层对输出内容做字段级解释。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2` 推导得到，`B2` 必须可由 `C2 + P3 + P4` 推导得到，`B3` 必须可由 `C3 + P5 + P6` 推导得到，`B4` 必须可由 `C5 + P8 + P9` 推导得到。
- `V2` 规则一致性：`R1` 与 `R2` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1 + R2` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4 + C5`。
- `V4` 输入边界符合性：任意有效评分调用都必须满足 `P1 + P2 + P3 + P4 + P5 + P6 + P7`。
- `V5` 输出边界符合性：任意有效原始响应都必须满足 `P8 + P9`。
- `V6` 最小必要性：移除 `B1`、`B2`、`B3` 或 `B4` 任一结构基后，单次评分调用能力或原始响应导出能力至少一项无法成立。
- `V7` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

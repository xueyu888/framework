# 最小文本单元:SingleTextUnit

@framework

0. 目标（Target）
声明一个最小的文本存储单元，用于实现单个文本内容的存储，并且可以被定位索引。

## 1. 能力声明（Capability Statement）

- `C1` 身份标识能力：标识当前文本单元身份的能力。
- `C2` 内容承载能力：能存放一段字符串类型的文本。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 标识存在性边界：文本标识不可为空。来源：`C1`。
- `P2` 内容存在性边界：文本内容不可为空。来源：`C2`。
- `P3` 内容格式边界：文本内容支持字符串类型。来源：`C2`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本标识：文本单元的标识。来源：`C1 + P1`。
- `B2` 文本内容：该单元中存放的文本内容。来源：`C2 + P2 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单个文本单元组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：每个文本单元必须包含`B1`和`B2`。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 参数绑定：`P1 + P2 + P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1 + C2`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 标签启用自动标注器:AutoEnableTagAnnotator

@framework

0. 目标（Target）
声明一个标签启用自动标注器，支持接收一个归一化后的数值和一组启用判定阈值，
并基于阈值分流规则导出一条自动标注结果记录。
该记录至少包含 `enable_status`、`review_status` 和 `review_source`。

## 1. 能力声明（Capability Statement）

- `C1` 归一化值消费能力：能接收一个归一化后的待判定数值。
- `C2` 判定阈值消费能力：能接收启用判定阈值和不启用判定阈值。
- `C3` 启用状态导出能力：能根据归一化值与判定阈值导出 `enable_status`。
- `C4` 审核状态导出能力：能根据自动判定结果导出 `review_status`。
- `C5` 审核来源导出能力：能根据自动判定结果导出 `review_source`。
- `C6` 自动标注结果记录导出能力：能将 `enable_status`、`review_status` 和 `review_source` 组织为一条完整自动标注结果记录。

- `N1` 不提供原始评分计算能力：不负责生成或归一化原始评分。
- `N2` 不提供人工审核能力：不负责人工审核或人工结果写回。
- `N3` 不提供持久化能力：不负责将自动标注结果写入数据库或其他存储介质。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 归一化值类型边界：输入的归一化值为数值类型。来源：`C1`。
- `P2` 归一化值范围边界：输入的归一化值必须落在 `0~1` 范围内。来源：`C1`。

- `P3` 启用判定阈值类型边界：启用判定阈值为数值类型。来源：`C2`。
- `P4` 不启用判定阈值类型边界：不启用判定阈值为数值类型。来源：`C2`。
- `P5` 判定阈值范围边界：启用判定阈值和不启用判定阈值都必须落在 `0~1` 范围内。来源：`C2`。
- `P6` 判定阈值大小边界：启用判定阈值必须大于不启用判定阈值。来源：`C2`。

- `P7` 启用状态范围边界：`enable_status` 只能为 `启用`、`不启用` 或 `待人工审核`。来源：`C3`。
- `P8` 审核状态范围边界：`review_status` 只能为 `已审核` 或 `未审核`。来源：`C4`。
- `P9` 审核来源范围边界：`review_source` 只能为 `system` 或空值。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 归一化值：待执行自动标注判定的归一化数值。来源：`C1 + P1 + P2`。
- `B2` 启用判定阈值：高于或等于该阈值时直接判定为 `启用`。来源：`C2 + P3 + P5 + P6`。
- `B3` 不启用判定阈值：低于或等于该阈值时直接判定为 `不启用`。来源：`C2 + P4 + P5 + P6`。
- `B4` 启用状态：自动标注结果中的 `enable_status`。来源：`C3 + P7`。
- `B5` 审核状态：自动标注结果中的 `review_status`。来源：`C4 + P8`。
- `B6` 审核来源：自动标注结果中的 `review_source`。来源：`C5 + P9`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 自动启用组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R1.2` 组合方式：当归一化值大于或等于启用判定阈值时，导出 `enable_status = 启用`、`review_status = 已审核`、`review_source = system`。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9`。

- `R2` 自动不启用组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R2.2` 组合方式：当归一化值小于或等于不启用判定阈值时，导出 `enable_status = 不启用`、`review_status = 已审核`、`review_source = system`。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R2.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9`。

- `R3` 待人工审核组合
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R3.2` 组合方式：当归一化值大于不启用判定阈值且小于启用判定阈值时，导出 `enable_status = 待人工审核`、`review_status = 未审核`、`review_source = 空值`。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R3.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 + R2 + R3` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4 + C5 + C6`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 分流有效性：任意有效组合都必须且只能命中 `R1`、`R2`、`R3` 中的一条，不得同时命中多条规则。
- `V6` 最小必要性：移除任一 `B*` 后，自动启用、自动不启用或待人工审核至少一项无法成立。
- `V7` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

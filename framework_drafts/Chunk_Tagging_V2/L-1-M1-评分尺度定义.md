# 评分尺度定义:ScoreScaleDefinition

@framework

0. 目标（Target）
声明一份评分尺度定义，用于统一约束提示词中的原始评分区间与归一化器中的线性映射区间。
该模块只定义原始评分的下界和上界，供上层模块共同消费。

## 1. 能力声明（Capability Statement）

- `C1` 原始评分下界定义能力：能定义原始评分区间的下界。
- `C2` 原始评分上界定义能力：能定义原始评分区间的上界。
- `C3` 评分尺度导出能力：能将原始评分下界和原始评分上界组织为一份完整评分尺度定义。

- `N1` 不提供归一化计算能力：不负责执行实际归一化计算。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 原始评分下界类型边界：原始评分下界为数值类型。来源：`C1`。
- `P2` 原始评分上界类型边界：原始评分上界为数值类型。来源：`C2`。
- `P3` 原始评分区间有效性边界：原始评分下界必须小于原始评分上界。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 原始评分下界：原始评分区间的下界。来源：`C1 + P1`。
- `B2` 原始评分上界：原始评分区间的上界。来源：`C2 + P2 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整评分尺度定义组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：将原始评分下界和原始评分上界组合为一份完整评分尺度定义，供提示词模块和归一化器模块共同消费。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 参数绑定：`P1 + P2 + P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1 + C2 + C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 线性映射归一化器:LinearNormalizer

@framework

0. 目标（Target）
声明一个线性映射归一化器，支持接收一个输入值，并基于评分尺度定义中的原始评分下界和原始评分上界，通过线性映射公式将输入值映射为一个归一化结果。

## 1. 能力声明（Capability Statement）

- `C1` 输入值消费能力：能接收一个待归一化的输入值。
- `C2` 归一化区间消费能力：能接收一个归一化下界和一个归一化上界。
- `C3` 线性映射归一化能力：能依据线性映射公式完成归一化计算。
- `C4` 归一化结果导出能力：能导出一个归一化结果。

- `N1` 不提供输入来源追踪能力：不负责判断输入值来自哪个上游模块。
- `N2` 不提供业务语义判定能力：不负责解释输入值在业务上的含义。
- `N3` 不提供非线性归一化能力：不负责执行分段映射、对数映射或其他非线性归一化规则。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 输入值类型边界：输入值为数值类型。来源：`C1`。
- `P2` 下界类型边界：归一化下界为数值类型。来源：`C2`。
- `P3` 上界类型边界：归一化上界为数值类型。来源：`C2`。
- `P4` 下界来源一致性边界：归一化下界必须与 `L-1.M1[R1]` 定义的原始评分下界一致。来源：`C2`。
- `P5` 上界来源一致性边界：归一化上界必须与 `L-1.M1[R1]` 定义的原始评分上界一致。来源：`C2`。
- `P6` 区间有效性边界：归一化下界必须小于归一化上界。来源：`C2`。
- `P7` 输入值下界约束：输入值必须大于等于归一化下界。来源：`C3`。
- `P8` 输入值上界约束：输入值必须小于等于归一化上界。来源：`C3`。
- `P9` 结果类型边界：归一化结果为数值类型。来源：`C4`。
- `P10` 结果范围边界：归一化结果必须落在 `0~1` 范围内。来源：`C4`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 输入值：待归一化的原始输入值。来源：`C1 + P1`。
- `B2` 归一化下界：线性映射区间的下界。来源：`C2 + P2 + P4`。
- `B3` 归一化上界：线性映射区间的上界。来源：`C2 + P3 + P5 + P6`。
- `B4` 归一化结果：通过线性映射计算得到的结果。来源：`C4 + P9 + P10`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 线性映射归一化组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：接收输入值，并从 `L-1.M1[R1]` 对应的评分尺度定义中读取归一化下界和归一化上界，按照线性映射公式 `y = (x - a) / (b - a)` 计算归一化结果，并导出结果。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1 + C2 + C3 + C4`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

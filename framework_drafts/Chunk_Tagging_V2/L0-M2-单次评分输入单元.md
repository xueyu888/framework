# 单次评分输入单元:ScoreEnterUnit

@framework

0. 目标（Target）
声明一个提供给大模型进行处理的完整内容块。

## 1. 能力声明（Capability Statement）

- `C1` 文本内容承载能力：能承载一段文本内容。
- `C2` 标签评分语义承载能力：能承载一段用于评分的标签语义。
- `C3` 提示词声明能力：能声明提供给大模型的提示词。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文本内容存在性边界：文本内容不可为空。来源：`C1`。
- `P2` 文本内容类型边界：文本内容为字符串类型。来源：`C1`。

- `P3` 标签评分语义存在性边界：标签评分语义不可为空。来源：`C2`。
- `P4` 标签语义内容边界：标签语义必须包括标签名、标签说明、正向线索、反向线索。来源：`C2`。

- `P5` 提示词类型边界：提示词为字符串类型。来源：`C3`。
- `P6` 提示词来源边界：输入提示词必须满足 L-1.M0[R1] 的组合规则。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 单段文本内容：待匹配评分的一段文本。来源：`C1 + P1 + P2`。
- `B2` 单个标签评分语义：由标签名、标签说明、正向线索、反向线索组织得到的待匹配评分语义。来源：`C2 + P3 + P4`。
- `B3` 提示词：L-1.M0[R1]。来源：`C3 + P5 + P6`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 评分请求内容组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：将`B1`、`B2`、`B3`组合成为一个待发送给大模型的评分请求。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 提示词定义:PromptDefinition

@framework

0. 目标（Target）
声明一份用于文本-标签匹配评分的提示词定义。
该提示词定义按 `system / task / output_format / score_rubric / constraints` 五段稳定结构组织，供上层评分输入模块消费。

## 1. 能力声明（Capability Statement）

- `C1` system 段承载能力：能承载提示词中的 system 段。
- `C2` task 段承载能力：能承载提示词中的 task 段。
- `C3` output_format 段承载能力：能承载提示词中的 output_format 段。
- `C4` score_rubric 段承载能力：能承载提示词中的 score_rubric 段。
- `C5` constraints 段承载能力：能承载提示词中的 constraints 段。
- `C6` 完整提示词定义导出能力：能将五段结构组合为一份完整提示词定义。

- `N1` 不提供运行时实例值能力：不负责给出某次调用的具体提示词实例值。
- `N2` 不提供文本内容注入能力：不负责注入待评分文本内容。
- `N3` 不提供标签语义注入能力：不负责注入待评分标签语义。
- `N4` 不提供大模型调用能力：不负责执行实际大模型调用。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` system 类型边界：system 为字符串类型。来源：`C1`。
- `P2` system 存在性边界：system 不可为空。来源：`C1`。

- `P3` task 类型边界：task 为字符串类型。来源：`C2`。
- `P4` task 存在性边界：task 不可为空。来源：`C2`。

- `P5` output_format 类型边界：output_format 为字符串类型。来源：`C3`。
- `P6` output_format 存在性边界：output_format 不可为空。来源：`C3`。
- `P7` output_format 评分尺度一致性边界：output_format 中对原始评分区间的说明必须与 `L-1.M1[R1]` 定义的原始评分下界和上界一致。来源：`C3`。

- `P8` score_rubric 类型边界：score_rubric 为字符串类型。来源：`C4`。
- `P9` score_rubric 存在性边界：score_rubric 不可为空。来源：`C4`。
- `P10` score_rubric 评分尺度一致性边界：score_rubric 中对原始评分范围的描述必须与 `L-1.M1[R1]` 定义的原始评分下界和上界一致。来源：`C4`。

- `P11` constraints 类型边界：constraints 为列表类型。来源：`C5`。
- `P12` constraints 元素类型边界：constraints 中每一项都必须为字符串。来源：`C5`。
- `P13` constraints 存在性边界：constraints 不可为空。来源：`C5`。

- `P14` 提示词结构组成边界：完整提示词定义必须包含 `system`、`task`、`output_format`、`score_rubric`、`constraints` 五段。来源：`C6`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` system：提示词中的 system 段。来源：`C1 + P1 + P2`。
- `B2` task：提示词中的 task 段。来源：`C2 + P3 + P4`。
- `B3` output_format：提示词中的 output_format 段。来源：`C3 + P5 + P6 + P7`。
- `B4` score_rubric：提示词中的 score_rubric 段。来源：`C4 + P8 + P9 + P10`。
- `B5` constraints：提示词中的 constraints 段。来源：`C5 + P11 + P12 + P13`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整提示词定义组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R1.2` 组合方式：将 `system`、`task`、`output_format`、`score_rubric` 与 `constraints` 组合为一份完整提示词定义，其中 `output_format` 与 `score_rubric` 中涉及原始评分区间的部分必须共同满足 `L-1.M1[R1]` 的评分尺度定义。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11 + P12 + P13 + P14`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1 + C2 + C3 + C4 + C5 + C6`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

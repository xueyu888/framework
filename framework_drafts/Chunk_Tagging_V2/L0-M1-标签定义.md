# 标签定义:TagDefinition

@framework

0. 目标（Target）
声明一个标签单元，用于提供给大模型进行文本与标签的匹配程度的评分。需要包含标签标识，标签名，标签说明，正向线索，反向线索。

## 1. 能力声明（Capability Statement）

- `C1` 标签身份标识能力：标识该标签身份的能力。
- `C2` 标签名承载能力：标识该标签名字的能力。
- `C3` 标签说明承载能力：体现当前标签的内容含义的能力。
- `C4` 正向线索承载能力：能体现支持文本命中该标签的正向线索。
- `C5` 反向线索承载能力：能体现削弱或排除文本命中该标签的反向线索。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 标签标识类型：标签标识为字符串类型。来源：`C1`。
- `P2` 标签标识存在性边界：标签标识不可为空。来源：`C1`。

- `P3` 标签名类型：标签名为字符串类型。来源：`C2`。
- `P4` 标签名存在性边界：标签名不可为空。来源：`C2`。
- `P5` 标签名语言：标签名语言为中文。来源：`C2`。
- `P6` 标签名长度：标签名长度不超过10字符。来源：`C2`。

- `P7` 标签说明类型：标签说明为字符串类型。来源：`C3`。
- `P8` 标签说明存在性边界：标签说明不可为空。来源：`C3`。
- `P9` 标签说明长度：标签说明长度不超过100字符。来源：`C3`。

- `P10` 正向线索类型：正向线索为列表类型。来源：`C4`。
- `P11` 正向线索内容类型：正向线索中的每一项都必须是字符串。来源：`C4`。

- `P12` 反向线索类型：反向线索为列表类型。来源：`C5`。
- `P13` 反向线索内容类型：反向线索中的每一项都必须是字符串。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 标签标识：标识单个标签身份。来源：`C1 + P1 + P2`。
- `B2` 标签名：单个标签名称。来源：`C2 + P3 + P4 + P5 + P6`。
- `B3` 标签说明：单个标签的语义说明。来源：`C3 + P7 + P8 + P9`。
- `B4` 正向线索：贴近标签名和说明的线索。来源：`C4 + P10 + P11`。
- `B5` 反向线索：远离标签名和说明的线索。来源：`C5 + P12 + P13`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整定义组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R1.2` 组合方式：`B1`、`B2`、`B3`、`B4`和`B5`共同组成一个完整的标签定义。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11 + P12 + P13`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1~C5`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

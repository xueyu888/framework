# 中文模块名:EnglishName

@framework

0. 目标（Target）
声明一个评分器，输入带标识的文本单元和标签定义，返回评分记录，记录中包含文本单元标识、标签定义标识、评分、评分原因。

## 1. 能力声明（Capability Statement）

- `C1` 评分输入装配能力：支持从一个最小文本单元和一个标签定义中读取评分所需内容，并装配为一次评分输入。
- `C2` 单次评分调用能力：支持将一次评分输入通过大模型调用接口提交给大模型完成一次评分调用。
- `C3` 评分结果提取能力：支持从大模型原始输出中提取评分结果与评分原因。
- `C4` 评分记录装配能力：支持将评分结果、评分原因、文本标识与标签标识装配为一条单次匹配评分记录。

- `N1` 不提供评分归一化能力：不负责对提取出的评分结果做归一化或阈值判定。
- `N2` 不提供批量评分编排能力：不负责多文本、多标签或多请求的批量调度。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文本输入最小消费边界：输入对象必须至少包含可装配为最小文本单元的必要内容。来源：`C1`。
- `P2` 文本单元输入数量边界：单次有效组合只接收 `1` 个文本单元。来源：`C1`。

- `P3` 标签输入最小消费边界：输入对象必须至少包含可装配为标签定义的必要内容。来源：`C1`。
- `P4` 标签定义输入数量边界：单次有效组合只接收 `1` 个标签定义。来源：`C1`。

- `P5` 评分内容提取边界：从文本单元中只提取文本内容，从标签定义中只提取标签名、标签说明、正向线索、反向线索并组织为评分所需的标签语义。来源：`C1`。
- `P6` 评分输入合法性边界：装配后的评分输入必须满足本模块后续评分调用所需的最小完整性要求。来源：`C1 + C2`。

- `P7` 大模型调用接口边界：评分调用必须通过合法的大模型调用接口完成。来源：`C2`。
- `P8` 调用次数边界：单次有效组合只允许发起 `1` 次大模型评分调用。来源：`C2`。

- `P9` 原始输出存在性边界：大模型原始输出不可为空。来源：`C3`。
- `P10` 结果提取边界：从原始输出中必须能够提取出评分结果与评分原因。来源：`C3`。

- `P11` 评分记录输出边界：装配后的评分记录必须满足 `L0.M4[R1]` 的组合规则。来源：`C4`。
- `P12` 评分记录完整性边界：输出记录必须包含文本标识、标签标识、评分结果与评分原因。来源：`C4`。



## 3. 最小结构基（Minimal Structural Bases）

- `B1` 最小文本单元：`L0.M0[R1]`。来源：`C1 + P1 + P2 + P5`。
- `B2` 标签定义：`L0.M1[R1]`。来源：`C1 + P3 + P4 + P5`。
- `B3` 单次评分输入单元：`L0.M2[R1]`。来源：`C1 + P5 + P6`。
- `B4` 大模型调用接口：`L0.M3[R1,R2]`。来源：`C2 + P7 + P8 + P9 + P10`。
- `B5` 单次匹配评分记录：`L0.M4[R1]`。来源：`C4 + P11 + P12`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 最小结构装配组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：从外部输入对象中提取本模块所需内容，投影为最小文本单元和标签定义，再从最小文本单元中读取文本内容、从标签定义中读取标签名、标签说明、正向线索和反向线索并组织为评分所需的标签语义，装配为单次评分输入单元，作为后续评分流程的稳定输入。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`。

- `R2` 大模型评分调用组合
  - `R2.1` 参与基：`B3 + B4`。
  - `R2.2` 组合方式：将单次评分输入单元连同模型调用配置通过大模型调用接口提交给大模型完成一次评分调用，并从返回的原始输出中提取评分结果与评分原因。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 参数绑定：`P7 + P8 + P9 + P10`。

- `R3` 评分记录装配组合
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R3.2` 组合方式：在完成单次评分输入装配与大模型评分调用后，将提取出的评分结果与评分原因，连同最小文本单元中的文本标识、标签定义中的标签标识，一并装配到单次匹配评分记录中，并导出完整记录。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11 + P12`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1 + R2 + R3` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

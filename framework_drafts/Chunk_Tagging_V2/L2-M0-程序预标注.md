# 程序预标注:ProgrammaticPreAnnotation

@framework

0. 目标（Target）
声明一个程序预标注模块，支持先完成文本标签匹配评分，再完成评分归一化，最后完成标签启用自动标注，
并导出一条程序预标注结果。

## 1. 能力声明（Capability Statement）

- `C1` 原始评分消费能力：能消费文本标签匹配程度评分器导出的原始评分结果。
- `C2` 评分归一化能力：能将原始评分结果映射为 `0~1` 范围内的归一化值。
- `C3` 自动标注判定能力：能基于归一化值和判定阈值导出自动标注结果。
- `C4` 程序预标注编排能力：能将评分、归一化和自动标注串联为一条完整程序预标注流程。
- `C5` 程序预标注结果导出能力：能导出程序预标注结果，其中至少包含 `enable_status`、`review_status` 和 `review_source`。

- `N1` 不提供原始评分生成实现能力：不负责实现大模型评分本身。
- `N2` 不提供人工审核能力：不负责人工审核或人工确认结果写回。
- `N3` 不提供持久化能力：不负责将程序预标注结果写入数据库或其他存储介质。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 原始评分可提取边界：程序预标注流程中必须能够从原始评分记录中提取出可供归一化的原始评分值。来源：`C1 + C2`。
- `P2` 归一化结果边界：归一化结果必须为数值类型，且落在 `0~1` 范围内。来源：`C2`。
- `P3` 流程顺序边界：程序预标注必须按“原始评分 -> 评分归一化 -> 自动标注判定”的顺序展开。来源：`C4`。
- `P4` 程序预标注结果完整性边界：程序预标注结果必须至少包含 `enable_status`、`review_status` 和 `review_source`。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本标签匹配程度评分器：`L1.M0[R3]`。来源：`C1 + P1`。
- `B2` 线性映射归一化器：`L1.M1[R1]`。来源：`C2 + P2`。
- `B3` 标签启用自动标注器：`L1.M2[R1,R2,R3]`。来源：`C3 + P4`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 评分归一化组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先通过文本标签匹配程度评分器获得原始评分记录，并从中提取原始评分值，再通过线性映射归一化器将该评分值映射为归一化结果。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 参数绑定：`P1 + P2 + P3`。

- `R2` 程序预标注组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：先完成原始评分生成与评分归一化，再将归一化结果提交给标签启用自动标注器，导出包含 `enable_status`、`review_status` 和 `review_source` 的程序预标注结果。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 参数绑定：`P1 + P2 + P3 + P4`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 + R2` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4 + C5`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 流程顺序有效性：任意有效程序预标注流程都必须满足 `P3`，不得跳过评分归一化直接进入自动标注判定。
- `V6` 最小必要性：移除 `B1`、`B2` 或 `B3` 任一结构基后，评分归一化能力、自动标注判定能力或程序预标注结果导出能力至少一项无法成立。
- `V7` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

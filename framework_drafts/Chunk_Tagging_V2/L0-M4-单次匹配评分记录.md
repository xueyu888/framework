# 单次匹配评分记录:EnglishName

@framework

0. 目标（Target）
声明一个评分记录单元，用于记录某条文本和某个标签之前的匹配程度评分。

## 1. 能力声明（Capability Statement）

- `C1` 文本单元身份标识能力：能标识进行评分的文本单元对象身份。
- `C2` 标签定义身份标识能力：能标识进行评分的标签定义对象身份。
- `C3` 匹配评分结果承载能力：能承载对应文本单元与标签定义的匹配程度的评分结果。
- `C4` 匹配评分原因承载能力：能承载给出这个评分结果的原因。
- `N1` 不提供评分校验能力：不负责进行评分的值域校验、阈值判定。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文本标识存在性边界：文本标识不可为空。来源：`C1`。
- `P2` 标签标识存在性边界：标签标识不可为空。来源：`C2`。
- `P3` 匹配评分结果存在性边界：匹配评分结果不可为空。来源：`C3`。
- `P4` 匹配评分原因类型边界：匹配评分原因为字符串类型。来源：`C4`。
- `P5` 匹配评分原因存在性边界：匹配评分原因不可为空。来源：`C4`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本标识：标识该条记录对应的文本。来源：`C1 + P1`。
- `B2` 标签标识：标识该条记录对应的标签。来源：`C2 + P2`。
- `B3` 评分结果：该条文本和标签的匹配程度的评分。来源：`C3 + P3`。
- `B4` 评分原因：得出该评分结果的原因记录。来源：`C4 + P4 + P5`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完成评分记录组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：每个完整组合必定包含`B1`、`B2`、`B3`、`B4`四个基。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5`。


## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1~C4`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

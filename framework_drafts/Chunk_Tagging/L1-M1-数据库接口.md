# 数据库接口:DatabaseInterface

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本块查询能力：支持根据文档ID查询对应的文本块集合。
- `C2` 文档领域查询能力：支持根据文档ID查询对应的文档领域。
- `C3` 标签定义查询能力：支持根据文档领域查询对应的标签定义集合。
- `C4` 标注结果写入能力：支持将标注结果记录写入数据库。
- `C5` 标注状态批量更新能力：支持根据单个文档ID批量更新该文档下所有标注记录的启用状态、审核状态和审核来源。
- `N1` 不提供结构生成能力：不负责生成文档、文本块或标签定义，只负责读取已有结构。
- `N2` 不提供审核能力：不负责决定标注结果是否通过审核，只负责写入给定结果。
- `N3` 不提供筛选能力：不负责决定应查询哪些领域或哪些标签，只执行给定查询路径。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文档ID查询边界：输入查询条件必须为单个合法文档ID。来源：`C1 + C2`。
- `P2` 文本块查询结果边界：查询得到的文本块集合中的每个文本块都必须满足 `backend.L0.M0[R3]` 的组合规则。来源：`C1`。
- `P3` 领域查询结果边界：查询得到的文档领域必须为单个合法领域标识。来源：`C2`。
- `P4` 标签定义查询结果边界：根据领域查询得到的标签定义集合中的每个标签定义都必须满足 `L0.M0[R1]` 的组合规则。来源：`C3`。
- `P5` 写入结果字段边界：写入数据库的标注记录必须至少包含 `doc_id`、`chunk_id`、`tag_name`、`confidence`、`reason` 和 `review_status` 字段。来源：`C4`。
- `P6` 写入结果数量边界：单次写入的标注记录数量等于 `1`。来源：`C4`。
- `P7` 批量更新文档ID边界：批量更新条件必须为单个合法文档ID。来源：`C5`。
- `P8` 批量更新字段边界：批量更新必须作用于 `enable_status`、`review_status` 和 `review_source` 字段。来源：`C5`。
- `P9` 批量更新规则边界：批量更新必须依据给定阈值规则，对同一文档下的标注记录进行统一状态更新。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本块查询接口：根据文档ID查询文本块集合的接口。来源：`C1 + P1 + P2`。
- `B2` 领域查询接口：根据文档ID查询文档领域的接口。来源：`C2 + P1 + P3`。
- `B3` 标签定义查询接口：根据文档领域查询标签定义集合的接口。来源：`C3 + P4`。
- `B4` 标注结果写入接口：将标注结果记录写入数据库的接口。来源：`C4 + P5 + P6`。
- `B5` 标注状态批量更新接口：根据文档ID批量更新标注记录状态的接口。来源：`C5 + P7 + P8 + P9`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 查询接口组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：通过文本块查询接口、领域查询接口和标签定义查询接口，形成面向单个文档标注任务的完整查询能力，支持先按文档ID获取文本块集合与文档领域，再按领域获取标签定义集合。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4`。

- `R2` 文本块标签评分接口组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R2.2` 组合方式：在完整查询能力基础上补充标注结果写入接口，形成完整的标注数据库接口，支持文档标注模块完成“查询输入数据 + 写入标注结果”的完整数据库交互。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R2.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`。

- `R3` 标注状态批量更新接口组合
  - `R3.1` 参与基：`B4 + B5`。
  - `R3.2` 组合方式：在标注结果写入接口基础上补充按文档ID批量更新状态的接口，支持根据单个文档ID对该文档下全部标注记录统一更新 `enable_status`、`review_status` 和 `review_source`。
  - `R3.3` 输出能力：`C4 + C5`。
  - `R3.4` 参数绑定：`P5 + P6 + P7 + P8 + P9`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2` 推导得到，`B2` 必须可由 `C2 + P1 + P3` 推导得到，`B3` 必须可由 `C3 + P4` 推导得到，`B4` 必须可由 `C4 + P5 + P6` 推导得到。
- `V2` 规则完整性：`R1` 与 `R2` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1` 与 `R2` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4`。
- `V4` 查询结果边界符合性：任意有效数据库接口组合中，文本块查询结果必须满足 `P2`，领域查询结果必须满足 `P3`，标签定义查询结果必须满足 `P4`。
- `V5` 写入结果边界符合性：任意有效写入操作都必须满足 `P5 + P6`。
- `V6` 最小必要性：移除 `B1`、`B2`、`B3` 或 `B4` 任一结构基后，完整数据库接口组合不再成立。

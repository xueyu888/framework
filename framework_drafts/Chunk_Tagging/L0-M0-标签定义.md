# 标签定义:TagDefinition

@framework

## 1. 能力声明（Capability Statement）

- `C1` 标签名承载能力：能为单个标签提供稳定且唯一的标签名。
- `C2` 标签说明承载能力：能体现当前标签表示的内容含义。
- `C3` 正向线索承载能力：能体现支持文本命中该标签的正向线索。
- `C4` 反向线索承载能力：能体现削弱或排除文本命中该标签的反向线索。
- `C5` 示例承载能力：能体现该标签对应内容的示例文本。
- `C6` 标签定义导出能力：能将标签名、标签说明、正向线索、反向线索和示例组成一个完整标签定义，供上层评分模块消费。
- `N1` 不提供文本评分能力：不负责对输入文本执行匹配计算或输出最终置信度。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 标签名类型：字符串类型。来源：`C1`。
- `P2` 标签名语言：中文。来源：`C1`。
- `P3` 标签名长度：不超过10字符。来源：`C1`。
- `P4` 标签说明类型：字符串类型。来源：`C2`。
- `P5` 标签说明长度：不超过100字符。来源：`C2`。
- `P6` 正向线索类型：列表类型。来源：`C3`。
- `P7` 正向线索内容：列表中的每一项都必须是字符串。来源：`C3`。
- `P8` 反向线索类型：列表类型。来源：`C4`。
- `P9` 反向线索内容：列表中的每一项都必须是字符串。来源：`C4`。
- `P10` 示例类型：列表类型。来源：`C5`。
- `P11` 示例内容：列表中的每一项都必须是字符串。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 标签名：标签的唯一标识。来源：`C1 + P1 + P2 + P3`。
- `B2` 标签说明：当前标签表示的内容说明。来源：`C2 + P4 + P5`。
- `B3` 正向线索：支持文本命中该标签的线索。来源：`C3 + P6 + P7`。
- `B4` 反向线索：削弱或排除文本命中该标签的线索。来源：`C4 + P8 + P9`。
- `B5` 示例：该标签对应内容的示例文本。来源：`C5 + P10 + P11`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整定义组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R1.2` 组合方式：`B1`、`B2`、`B3`、`B4` 和 `B5` 共同组成一个完整的标签定义，作为数据库表中的一行数据，也作为上层AI评分模块的输入。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2 + P3` 推导得到，`B2` 必须可由 `C2 + P4 + P5` 推导得到，`B3` 必须可由 `C3 + P6 + P7` 推导得到，`B4` 必须可由 `C4 + P8 + P9` 推导得到，`B5` 必须可由 `C5 + P10 + P11` 推导得到。
- `V2` 规则完整性：`R1` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1` 的输出能力必须覆盖 `C1 + C2 + C3 + C4 + C5 + C6`。
- `V4` 标签名边界符合性：任意有效标签定义中的标签名都必须满足 `P1 + P2 + P3`。
- `V5` 标签说明边界符合性：任意有效标签定义中的标签说明都必须满足 `P4 + P5`。
- `V6` 正向线索边界符合性：任意有效标签定义中的正向线索都必须满足 `P6 + P7`。
- `V7` 反向线索边界符合性：任意有效标签定义中的反向线索都必须满足 `P8 + P9`。
- `V8` 示例边界符合性：任意有效标签定义中的示例都必须满足 `P10 + P11`。
- `V9` 最小必要性：移除 `B1`、`B2`、`B3`、`B4` 或 `B5` 任一结构基后，完整标签定义组合不再成立，且 `C6` 至少无法成立。

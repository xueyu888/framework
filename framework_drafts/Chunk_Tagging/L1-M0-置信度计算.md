# 置信度计算:ConfidenceCalculate

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本块输入能力：接收外部输入的文本块的能力。
- `C2` 标签定义输入能力：接收外部输入的标签定义的能力。
- `C3` 置信度计算能力：根据标签定义计算文本块与标签的匹配置信度的能力。
- `N1` 不提供标签创建能力：不创建新的标签。
- `N2` 不提供修改能力：不对输入内容进行修改。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文本块边界：待标注的输入文本块必须满足backend.L0.M0[R3]的组合规则。来源：`C1`。
- `P2` 文本块数量边界：单次输入文本块的数量为1。来源：`C1`。
- `P3` 标签定义边界：输入的标签和匹配规则必须满足L0.M1的组合规则。来源：`C2`。
- `P4` 标签定义数量边界：单次输入的标签定义数量为1。来源：`C2`。
- `P5` 置信度范围：置信度范围在0~1.0之间。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 待评分文本块：待标注的单个文本块 backend.L0.M0[R3]。来源：`C1 + P1 + P2`。
- `B2` 标签定义：单个标签，包括标签名以及对应的匹配规则 L0.M1[R1]。来源：`C2 + P3 + P4`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 匹配评分组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：以标签定义中的匹配规则作用于文本块内容，对该文本块与该标签定义的匹配程度进行计算，并输出单个置信度结果。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2` 推导得到，`B2` 必须可由 `C2 + P3 + P4` 推导得到。
- `V2` 规则完整性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V3` 结构闭环性：合法组合规则的输出能力并集必须覆盖 `C1 + C2 + C3`。
- `V4` 输入边界符合性：任意有效组合都必须同时满足 `P1 + P2 + P3 + P4`，即输入必须是单个合法文本块与单个合法标签定义。
- `V5` 评分结果合法性：凡由 `R2` 导出的置信度结果，都必须满足 `P5`，即结果值域落在 `0~1.0` 之间。
- `V6` 最小必要性：移除 `B1` 或 `B2` 任一结构基后，`C3` 至少无法成立，且合法评分组合不再成立。
- `V7` 非法组合可判定性：当缺少任一输入、输入数量不满足一对一约束、或任一输入越出对应边界时，必须能够明确判定该组合落入非法组合，而非继续导出评分结果。
- `V8` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论，并附对应证据。

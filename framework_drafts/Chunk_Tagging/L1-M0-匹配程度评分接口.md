# 匹配程度评分接口:SingleScoringInterface

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本块记录输入能力：支持输入带有 `doc_id`、`chunk_id` 与 `content` 字段的单条文本块。
- `C2` 单标签定义输入能力：支持输入单个完整标签定义。
- `C3` 提供大模型评分接口能力：提供连接到大模型进行文本块-标签关联程度评分的接口。
- `C4` 单次评分结果导出能力：能导出单条文本块与单个标签之间的匹配程度评分结果与评分理由。
- `N1` 不提供大模型实现能力：不负责实现底层大模型模型或外部接口本身。
- `N2` 不提供审核流转能力：不负责依据评分结果进行审核状态判定或后续人工处理。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文本块字段边界：输入文本块结构必须至少包含 `doc_id`、`chunk_id` 与 `content` 字段，并满足 `backend.L0.M0[R3]` 的组合规则。来源：`C1`。
- `P2` 文本块数量边界：单次输入的文本块数量为1。来源：`C1`。
- `P3` 标签定义边界：输入标签定义必须满足 `L0.M0[R1]` 的组合规则。来源：`C2`。
- `P4` 标签定义数量边界：单次输入的标签定义数量为 `1`。来源：`C2`。
- `P5` 大模型接口边界：输入大模型接口结构必须至少包含可执行评分调用所需的模型、提示词或调用入口字段。来源：`C3`。
- `P6` 评分范围边界：输出评分结果必须落在 `0~1.0` 范围内。来源：`C4`。
- `P7` 评分理由边界：输出结果必须包含与评分对应的理由说明，且理由类型为字符串。来源：`C4`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：承载 `doc_id + chunk_id + content` 的单条文本块输入 `backend.L0.M0[R3]`。来源：`C1 + P1 + P2`。
- `B2` 标签定义：承载单个标签名、标签说明、正向线索、反向线索和示例的完整标签定义 `L0.M0[R1]`。来源：`C2 + P3 + P4`。
- `B3` 大模型评分接口：将内容提供给大模型进行评分和获取评分结果与理由的接口。来源：`C3 + P5`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单次评分接口组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：将单条文本块记录、单个标签定义和当前大模型评分接口绑定，由大模型根据标签定义中的标签说明、正向线索、反向线索和示例，对该文本块与该标签之间的关联程度进行判断，并导出一条包含 `doc_id`、`chunk_id`、`tag_name`、`confidence` 和 `reason` 的评分结果。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2` 推导得到，`B2` 必须可由 `C2 + P3 + P4` 推导得到，`B3` 必须可由 `C3 + P5` 推导得到。
- `V2` 规则完整性：`R1` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1` 的输出能力必须覆盖 `C1 + C2 + C3 + C4`。
- `V4` 输入边界符合性：任意有效接口组合都必须同时满足 `P1 + P2 + P3 + P4 + P5`，即单次组合只处理一条文本块记录、一个标签定义和一个大模型评分接口。
- `V5` 评分结果合法性：任意有效评分结果都必须满足 `P6 + P7`，即评分值域合法，且结果包含对应理由。
- `V6` 最小必要性：移除 `B1`、`B2` 或 `B3` 任一结构基后，文本块记录输入能力、单标签定义输入能力、大模型评分接口输入能力或评分结果导出能力至少一项无法成立。

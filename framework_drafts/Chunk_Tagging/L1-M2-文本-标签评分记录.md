# 文本-标签评分记录:ChunkTagRecord

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本块定位能力：能通过原文档ID和文本块ID唯一定位到对应的文本块。
- `C2` 标签定位能力：能通过标签ID唯一定位到对应的标签。
- `C3` 匹配评分结果展示能力：能展示对应文本块和标签之间的匹配评分结果。
- `C4` 审核状态展示能力：能展示该条记录的审核状态。
- `C5` 审核来源展示能力：能回溯该条记录的审核来源。
- `C6` 启用状态展示能力：能展示该文本块对应的该标签是否启用。
- `C7` 记录导出能力：能将文本块定位信息、标签定位信息、评分结果、审核状态、审核来源和启用状态组成一条完整记录。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不提供评分能力：不负责计算文本块与标签之间的匹配程度。
- `N2` 不提供审核判定能力：不负责决定该条记录是否通过审核。
- `N3` 不提供启用判定能力：不负责决定该条记录是否启用。
- `N4` 不提供数据库写入能力：不负责将该条记录写入数据库。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 原文档ID类型：字符串类型。来源：`C1`。
- `P2` 文本块ID类型：字符串类型。来源：`C1`。
- `P3` 文本块联合定位性：原文档ID与文本块ID必须能联合定位到唯一文本块。来源：`C1`。
- `P4` 标签ID类型：字符串类型。来源：`C2`。
- `P5` 标签ID定位性：标签ID必须能唯一定位到对应标签。来源：`C2`。
- `P6` 匹配评分类型：数值类型。来源：`C3`。
- `P7` 匹配评分范围：匹配评分结果必须落在 `0~1.0` 之间。来源：`C3`。
- `P8` 审核状态类型：字符串类型。来源：`C4`。
- `P9` 审核状态范围：审核状态值范围为 `未审核` 或 `已审核`。来源：`C4`。
- `P10` 审核来源类型：字符串类型或空值。来源：`C5`。
- `P11` 审核来源范围：审核来源值范围为 `system`、对应用户名或空值。来源：`C5`。
- `P12` 启用状态类型：字符串类型。来源：`C6`。
- `P13` 启用状态范围：启用状态值范围为 `启用`、`不启用` 或 `待人工审核`。来源：`C6`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 原文档ID：用于定位文本块所属原文档。来源：`C1 + P1`。
- `B2` 文本块ID：用于定位原文档中的对应文本块。来源：`C1 + P2 + P3`。
- `B3` 标签ID：用于唯一定位标签。来源：`C2 + P4 + P5`。
- `B4` 匹配评分结果：文本块与标签之间的匹配评分结果。来源：`C3 + P6 + P7`。
- `B5` 审核状态：该条记录当前的审核状态。来源：`C4 + P8 + P9`。
- `B6` 审核来源：该条记录的审核来源。来源：`C5 + P10 + P11`。
- `B7` 启用状态：该条记录当前的启用状态。来源：`C6 + P12 + P13`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整记录组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6 + B7`。
  - `R1.2` 组合方式：`B1` 和 `B2` 联合定位到唯一文本块，`B3` 唯一定位到对应标签，在此基础上结合 `B4`、`B5`、`B6` 和 `B7`，共同组成一条完整的文本块-标签启用审核记录。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6 + C7`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11 + P12 + P13`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1` 推导得到，`B2` 必须可由 `C1 + P2 + P3` 推导得到，`B3` 必须可由 `C2 + P4 + P5` 推导得到，`B4` 必须可由 `C3 + P6 + P7` 推导得到，`B5` 必须可由 `C4 + P8 + P9` 推导得到，`B6` 必须可由 `C5 + P10 + P11` 推导得到，`B7` 必须可由 `C6 + P12 + P13` 推导得到。
- `V2` 规则完整性：`R1` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1` 的输出能力必须覆盖 `C1 + C2 + C3 + C4 + C5 + C6 + C7`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除 `B1`、`B2`、`B3`、`B4`、`B5`、`B6` 或 `B7` 任一结构基后，完整记录组合不再成立，且 `C7` 至少无法成立。

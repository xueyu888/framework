# 标签启用判定模块:TagEnableModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文档ID输入能力：支持输入单个需要进行标注的文档ID。
- `C2` 判定阈值输入能力：支持输入启用和不启用的判定阈值。
- `C3` 数据库数据获取能力：支持从数据库中读取相关数据。
- `C4` 数据库数据写入能力：支持将数据写入数据库中。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文档ID边界：输入文档ID必须为单个合法文档标识。来源：`C1`。
- `P2` 文档ID数量边界：单次输入的文档ID数量为 `1`。来源：`C1`。
- `P3` 阈值值域：阈值的范围在0.0到1.0。来源：`C2`。
- `P4` 阈值类型：float类型。来源：`C2`。
- `P5` 阈值大小边界：启动阈值必须大于不启动阈值。来源：`C2`。
- `P6` 数据库接口边界：输入数据库接口必须满足 `L1.M1[R3]` 的组合要求。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文档ID：文档的ID。来源：`C1 + P1`。
- `B2` 启用判定阈值：启用判定阈值，高于这个阈值的直接由系统判定为启用。来源：`C2 + P2`。
- `B3` 不启用判定阈值：不启用判定阈值，低于这个阈值的直接由系统判定为启用。来源：`C2 + C3 + P2 + P3`。
- `B4` 数据库接口：读取评分记录，将审核结果传入数据库的接口。L1.M1[R3]。来源：

## 4. 基组合原则（Base Combination Principles）

- `R1` 启用判定组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：先根据文档ID从数据库中读取相关的待判定记录，然后根据输入的阈值判定该条记录为`启用`/`不启用`/`待人工审核`，并将判定为`启用`/`不启用`的记录的审核来源修改为`system`。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1` 输出能力并集必须覆盖 `C1~C4`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

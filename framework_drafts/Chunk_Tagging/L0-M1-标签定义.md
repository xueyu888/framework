# 标签定义:TagDefinition

@framework

## 1. 能力声明（Capability Statement）

- `C1` 标签名承载能力：能为单个标签提供稳定且唯一的标签名。
- `C2` 匹配规则承载能力：能为该标签承载对应的匹配规则。
- `C3` 一对一映射能力：能将标签名和匹配规则进行一对一映射的能力。


## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 标签名类型：字符串类型。来源：`C1`。
- `P2` 标签名语言：中文。来源：`C1`。
- `P3` 标签名长度：不超过10字符。来源：`C1`。
- `P4` 匹配规则：匹配规则的格式必须满足L-1.M0[R1]。 来源：`C2`

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 标签名：标签的唯一标识。来源：`C1 + P1 + P2 + P3`。
- `B2` 标签匹配规则：该标签对应的匹配规则。L-1.M0[R1]。来源：`C2 + P4`

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整定义组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：`B1`和`B2`共同组成一个完整的标签定义，具有一对一的映射关系，作为数据库表中的一行数据。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2 + P3` 推导得到，`B2` 必须可由 `C2 + P4` 推导得到。
- `V2` 规则完整性：`R1` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1` 的输出能力必须覆盖 `C1 + C2 + C3`。
- `V4` 标签名边界符合性：任意有效标签定义中的标签名都必须满足 `P1 + P2 + P3`，即类型合法、语言合法且长度合法。
- `V5` 匹配规则边界符合性：任意有效标签定义中的匹配规则都必须满足 `P4`，即规则格式满足对应下层模块的合法组合要求。
- `V6` 一对一映射成立性：任意有效标签定义都必须且仅能包含一个标签名与一个匹配规则，且两者形成稳定绑定关系。
- `V7` 最小必要性：移除 `B1` 或 `B2` 任一结构基后，完整标签定义组合不再成立，且 `C3` 至少无法成立。
- `V8` 结论表达：逐条输出 `R1 -> C1 + C2 + C3 + 参数绑定` 的通过或失败结论，并附对应证据。

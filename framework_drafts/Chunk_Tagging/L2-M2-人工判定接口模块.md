# 人工判定接口模块:ManualReviewInterfaceModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文档ID输入能力：支持输入单个需要进行人工审核的文档ID。
- `C2` 人工审核数据输出能力：支持根据输入文档ID输出该文档对应的人工审核数据，所输出数据必须同时包含待人工审核记录与对应文本块内容。
- `C3` 人工确认结果输入能力：支持接收外部返回的人工确认结果。
- `C4` 审核结果写入能力：支持将人工确认后的审核结果写入数据库。
- `N1` 不提供前端交互能力：不负责登录、页面展示、页面跳转或人工审核界面的实现。
- `N2` 不提供自动评分能力：不负责重新计算文本块与标签之间的匹配评分。
- `N3` 不提供自动判定能力：不负责根据阈值自动决定记录应为 `启用`、`不启用` 或 `待人工审核`。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文档ID边界：输入文档ID必须为单个合法文档标识。来源：`C1`。
- `P2` 文档ID数量边界：单次输入的文档ID数量为 `1`。来源：`C1`。
- `P3` 待审核记录范围边界：输出记录必须限定为该文档下 `enable_status = 待人工审核` 且 `review_status = 未审核` 的记录。来源：`C2`。
- `P4` 待审核记录字段边界：输出记录必须至少包含 `doc_id`、`chunk_id`、`tag_name`、`confidence`、`reason`、`enable_status` 与 `review_status` 字段。来源：`C2`。
- `P5` 文本块内容边界：输出给外部的人工审核数据中，每条待人工审核记录都必须能关联到对应文本块内容，且文本块内容必须满足 `backend.L0.M0[R3]` 的组合规则。来源：`C2`。
- `P6` 人工确认结果字段边界：输入的人工确认结果必须至少包含记录定位字段、人工最终结论与审核来源字段。来源：`C3`。
- `P7` 人工确认结果范围边界：人工最终结论只能为 `启用` 或 `不启用`，不得继续写回 `待人工审核`。来源：`C3`。
- `P8` 审核来源边界：人工确认结果中的审核来源必须为单个合法外部身份标识。来源：`C3`。
- `P9` 写入结果边界：写回数据库后的记录必须更新 `enable_status`、`review_status` 和 `review_source` 字段，其中 `review_status = 已审核`。来源：`C4`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文档人工审核数据查询接口：接收单个文档ID，并导出该文档下全部待人工审核记录及其对应文本块内容的接口。待定义或数据库接口的对应展开。来源：`C1 + C2 + P1 + P2 + P3 + P4 + P5`。
- `B2` 人工审核结果回写接口：接收外部返回的人工确认结果，并将合法结果写回数据库的接口。待定义或 `L1.M1[R3]` 的人工审核展开。来源：`C3 + C4 + P6 + P7 + P8 + P9`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 人工审核数据查询组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：接收单个文档ID后，通过文档人工审核数据查询接口读取该文档下全部待人工审核记录，再根据记录中的 `doc_id + chunk_id` 补齐对应文本块内容，并按统一结构导出给外部模块。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5`。

- `R2` 人工确认结果回写组合
  - `R2.1` 参与基：`B2`。
  - `R2.2` 组合方式：接收外部返回的人工确认结果后，校验结果字段、人工结论范围与审核来源是否合法，再将对应记录写回数据库。
  - `R2.3` 输出能力：`C3 + C4`。
  - `R2.4` 参数绑定：`P6 + P7 + P8 + P9`。

- `R3` 人工审核闭环组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：先根据单个文档ID导出该文档下全部待人工审核记录及其对应文本块内容，再在收到外部返回的人工确认结果后完成合法性校验与数据库写回，形成完整的人工审核闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1 + R2 + R3` 的输出能力并集必须覆盖 `C1~C4`。
- `V4` 查询结果边界符合性：任意有效查询结果都必须满足 `P3 + P4`。
- `V5` 审核上下文完整性：任意导出的人工审核数据都必须满足 `P5`，即每条待人工审核记录都必须能关联到对应文本块内容。
- `V6` 回写结果边界符合性：任意有效回写结果都必须满足 `P6 + P7 + P8 + P9`。
- `V7` 最小必要性：移除 `B1` 或 `B2` 任一结构基后，人工审核闭环组合不再成立。
- `V8` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

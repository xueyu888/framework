# 可追溯性标准

## 1. 目标

确保任何结论都能沿唯一主链回溯：

`Framework -> Config -> Code -> Evidence`

## 2. 强制要求

- framework 来源必须可追到具体 `framework/*.md` 文件、模块 id、Base class 和 Rule class
- 配置来源必须可追到 `projects/*/project.toml` 的明确 section
- 代码绑定必须可追到对应 `CodeModule` export
- evidence 必须回落到 canonical JSON
- 禁止出现没有 framework、config、code 或 canonical 来源的孤儿结论

## 3. 机器追溯

- `projects/*/generated/canonical.json` 是唯一机器真相源
- canonical 必须记录：
  - framework module tree
  - config projection
  - code bindings
  - runtime projection
  - evidence
- 其他 manifest / tree / report 只能作为 canonical 派生视图存在

## 4. 代码追溯

- 关键结构语义必须能从 framework module 回到对应 code binding
- code 对外导出的 export 必须能回到对应的 compile result
- 运行时行为必须能回到 canonical 的 code/runtime projection
- 变更后必须运行严格验证，确认追溯链未断裂

## 5. 生成式实现追溯

- framework 文档是设计输入，不是被生成结果反向定义的附庸
- 生成出的代码、API、页面和报告都属于 evidence
- evidence 只能通过 canonical 回连到 framework、config 和 code
- 若实现倒逼标准变更，必须先改 framework 或 project config，再重新物化 canonical

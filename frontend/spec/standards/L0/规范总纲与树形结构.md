# 规范总纲与树形结构（L0）

## 1. 目的
定义前端规范的层级结构、映射关系与变更传导规则。

## 2. 标准树（强制）
```text
standards/
├── L0/
│   └── 规范总纲与树形结构.md
├── L1/
│   ├── 框架设计核心标准.md
│   ├── 可追溯性标准.md
│   └── 可删减性标准.md
├── L3/
│   └── mapping_registry.json
└── L4...Ln/
    └── 实现与证据扩展层（按需）
framework/
├── L0/
│   └── 前端框架标准.md
├── L1/
│   ├── 前端页面标准.md
│   └── instance/
│       └── 登录页面实例标准.md
├── L2/
│   └── 前端业务组件层标准.md
└── L3/
    └── 前端基础组件层标准.md
```

## 3. 树化映射规则（强制）
- 所有“标准 -> 实现”关系必须登记到 `standards/L3/mapping_registry.json`。
- `tree` 节点必须有稳定 `id`，并由 `children` 形成可遍历树。
- `kind=file` 节点必须唯一指向一个文件，禁止重复挂载。
- `standards/` 仅承载方法标准与映射注册；领域标准文件统一放在 `framework/`。
- `framework/L0-L3` 为领域分层命名，不等同于实现层 `L3...Ln`。

## 4. 变更传导规则（强制）
- 标准层固定为：`L0/L1 + framework(领域标准层)`。
- 实现层为可扩展后缀：`L3...Ln`。
- L0 变更：L1、framework、所有实现层必须同步评估并按需更新。
- L1 变更：framework、所有实现层必须同步评估并按需更新。
- framework 变更：所有实现层必须同步更新。
- 任一实现层变更：必须反向验证不违反 L0/L1/framework。

## 5. 验证机制
```bash
uv run python scripts/validate_strict_mapping.py
uv run python scripts/validate_strict_mapping.py --check-changes
```

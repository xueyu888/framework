# 框架设计标准（核心）

## 0. 定位
仅定义框架设计核心方法，不包含工程流程约束。

核心范围：
1. 目标
2. 边界定义
3. 模块（最小可行基）
4. 组合原则
5. 验证

## 1. 目标（Goal）
- 目标必须可量化
- 目标必须可验证
- 目标必须与边界形成约束关系

## 2. 边界定义（Boundary）
- 每个边界项都可测量
- 每个边界项都可校验
- 边界项需与目标直接相关

## 3. 模块（最小可行基，Module）
- 模块职责单一
- 模块可组合
- 模块可替换

## 4. 组合原则（Combination Principles）
- 规则必须可表达为显式条件
- 规则间不可冲突
- 组合结果必须可判定“有效/无效”

## 5. 验证（Verification）
- 必须有基线（baseline）
- 必须有目标值（target）
- 必须有通过条件（pass criteria）
- 必须有失败原因（failure reasons）

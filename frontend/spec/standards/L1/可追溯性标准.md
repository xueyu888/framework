# 可追溯性标准

## 1. 目标
确保任何前端结论都能追溯到来源目标、边界、模块与规则。

## 2. 强制要求
- 每个步骤必须有唯一 `step_id`
- 每个步骤必须声明 `depends_on`
- 每个结论步骤必须有 `evidence`
- 禁止无依赖来源的孤儿结论

## 3. 文档追溯
- L0/L1/L2/L3 关系必须在 L0 显式声明
- `mapping_registry.json` 必须包含标准锚点到实现符号的映射

## 4. 代码追溯
- Goal/Boundary/Module/Rule/Verification 必须在实现中可定位
- 变更后必须执行严格映射验证

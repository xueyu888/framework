# 前端业务组件层标准（L2）

## 0. 规范关系
本文件是前端业务组件层（业务语义层）标准，遵循：
- `standards/L0/规范总纲与树形结构.md`
- `standards/L1/框架设计核心标准.md`
- `standards/L1/可追溯性标准.md`
- `standards/L1/可删减性标准.md`
- `standards/L2/前端框架标准.md`

## 1. 目标（Goal）
业务组件层负责封装业务语义与业务交互规则，输出可复用、可测试、可组合的业务能力单元。

## 2. 边界定义（Boundary）
- 业务组件可以组合基础组件，但不得直接绑定具体页面容器
- 业务组件不得直接读取路由对象或页面实例
- 业务组件输入输出契约必须显式定义并稳定演进

## 3. 模块（最小可行基，Module）
- `M1 语义渲染模块`：业务语义 UI 表达
- `M2 基础组件编排模块（最小可行基）`：以基础组件作为业务组件的最小装配单元
- `M3 契约模块`：`Props/Events` 输入输出边界与状态约束
- `M4 质量模块`：业务组件测试与回归验证

说明：
- 业务组件层最小可行基定义到基础组件层面，业务组件应通过组合基础组件构建业务语义。

## 4. 组合原则（Combination Principles）
- `B-R1` 业务组件必须定义明确契约（props/events）
- `B-R2` 业务组件依赖通过页面层注入，不直接依赖路由容器
- `B-R3` 业务组件不得直接耦合业务 API SDK
- `B-R4` 业务组件必须有最小测试覆盖（渲染/交互/异常）
- `B-R5` 业务组件主结构应由基础组件构成，不得在业务组件中重复发明基础交互原语

## 5. 验证（Verification）
通过条件：  
- 契约变更可追溯并有影响说明
- 组件可脱离单页面上下文复用
- 关键交互规则通过回归测试
- 业务组件层规则违反项为 0
- 业务组件可追溯到基础组件清单（而非页面容器耦合）

## 1. 目标（Goal）
将前端框架标准实例化为可执行的数据清洗前端工程基线，使项目在统一框架约束下稳定交付“用户登录，用户注册，数据源、清洗任务编排、质量报告查看”页面能力。

可度量目标：
- 一致性：路由与页面入口规范违规项可统计并收敛
- 可用性：核心页面路由可达率可验证
- 稳定性：页面级异常具备兜底与恢复能力
- 交付效率：新增页面在既定框架下可快速接入

## 2. 边界定义（Boundary）
### 2.1 技术边界
- `Framework`：React
- `CSS`：Tailwind CSS
- `UI`：shadcn/ui
- `Icon`：lucide-react
- `Build Tool`：Vite
- `Runtime`：Browser SPA（如需 SSR，须在本实例中声明）

### 2.2 范围边界
- 本实例仅约束框架级能力：路由、页面入口、状态闭环、异常兜底、可追溯性
- 数据清洗业务语义（字段映射、清洗策略、质量规则）由业务标准定义，不在本实例展开实现细节

### 2.3 质量边界
- 页面入口必须可访问
- 页面状态必须可判定
- 关键失败路径必须可恢复

## 3. 模块（最小可行基，Module）
最小可行基仅定义到页面级（路由）：

- `M1 路由注册模块`
  - 定义路由路径、页面入口、访问策略：`/login`、`/register`、`/datasource`、`/task`、`/quality-report`
- `M2 页面入口模块`
  - 页面挂载点与页面级初始化逻辑
- `M3 页面状态闭环模块`
  - 页面最小状态集合：`loading / success / error / empty`
- `M4 路由兜底模块`
  - 未命中路由（404）、异常回退（500）、权限失败处理（401）
- `M5 消息中心模块（本实例强制启用）`
  - 统一页面级通知与系统事件广播（如任务提交结果、会话失效、全局公告）
  - 事件必须具备稳定命名、发布来源、消费边界
- `M6 数据中心模块（本实例强制启用）`
  - 统一共享会话数据与查询缓存（如当前用户、数据源清单、任务摘要、字典配置）
  - 数据必须声明域边界、缓存与一致性策略
- `M7 服务层模块（本实例强制启用）`
  - 统一封装 API 调用、重试/超时、错误映射与 DTO 转换
  - 服务必须声明稳定契约（input/output/error）并与页面层解耦

说明：
- 本实例在框架标准基础上，强制启用 `M5/M6/M7`，用于保障数据清洗场景的跨页面协同能力。
- 若项目需要业务组件层/基础组件层规则，应在后续层级标准中扩展，不回填本实例主模块定义。

## 4. 组合原则（Combination Principles）
先定义模块如何组合，再定义组合后的判定约束。

模块组合关系：
- `C0` 整体拓扑（总关系）
  - 主链路（正常访问）：`M1 -> M2 -> M3`
  - 回流链路（异常恢复）：`M1 -> M4 -> M2 -> M3`
  - 事件链路（强制启用）：`M2/M3 -> M5`
  - 数据链路（强制启用）：`M2/M3 <-> M6`
  - 服务链路（强制启用）：`M2/M3 <-> M7`
  - 服务数据链路（强制启用）：`M7 <-> M6`
  - 结构说明：`M1` 负责分发，`M2` 负责汇合，`M3` 负责状态终态承载
- `C1` `M1 路由注册模块 -> M2 页面入口模块`
  - 在 `C0` 中对应主链路第一跳
  - 通俗理解：先有路由，才有页面入口；每条路由都要“指向一个页面”
- `C2` `M2 页面入口模块 -> M3 页面状态闭环模块`
  - 在 `C0` 中对应主链路第二跳与回流链路终点
  - 通俗理解：进入页面后，页面必须能在 `loading/success/error/empty` 之间切换
- `C3` `M1 路由注册模块 -> M4 路由兜底模块`
  - 在 `C0` 中对应回流链路第一跳
  - 通俗理解：路由失败时不能崩掉，必须有 404/权限失败/异常回退入口
- `C4` `M4 路由兜底模块 -> M2 页面入口模块`
  - 在 `C0` 中对应回流链路第二跳
  - 通俗理解：兜底页不是终点，用户要能回到正常页面继续操作（回退/重试/重定向）
- `C5` `M2/M3 -> M5 消息中心模块`
  - 在 `C0` 中对应事件链路
  - 通俗理解：页面入口与状态变化可发布标准化消息事件，供跨页面消费
- `C6` `M2/M3 <-> M6 数据中心模块`
  - 在 `C0` 中对应数据链路
  - 通俗理解：页面读取共享数据并回写最新结果，保持跨页面状态一致
- `C7` `M6 数据中心模块 -> M3 页面状态闭环模块`
  - 在 `C0` 中对应数据回流
  - 通俗理解：缓存命中/失效/失败必须映射为页面状态变化
- `C8` `M5 消息中心模块 -> M4 路由兜底模块`
  - 在 `C0` 中对应异常协同
  - 通俗理解：系统级故障事件可触发统一兜底动作
- `C9` `M2/M3 <-> M7 服务层模块`
  - 在 `C0` 中对应服务链路
  - 通俗理解：页面层通过服务契约请求后端能力，禁止直接耦合 API SDK
- `C10` `M7 服务层模块 <-> M6 数据中心模块`
  - 在 `C0` 中对应服务数据链路
  - 通俗理解：服务层统一读写共享缓存并回流页面状态

组合约束（由组合关系导出）：
- `R1`：每个页面必须通过路由可达，禁止“不可达页面入口”
- `R2`：每条路由必须绑定唯一页面入口（对应 `C1`）
- `R3`：页面入口必须具备最小状态闭环（`loading/success/error/empty` 至少可触发其一，对应 `C2`）
- `R4`：必须存在路由兜底策略（404/权限失败/异常回退，对应 `C3`）
- `R5`：框架变更必须可追溯到实现文件与符号（映射登记，对应 `C1/C3`）
- `R6`：规则结果必须可判定为“有效/无效”，并输出失败原因
- `R7`：技术栈若偏离默认基线，必须在项目实例标准中声明替代方案与理由
- `R8`：消息中心事件必须具备事件名、来源、消费者、降级策略（对应 `C5/C8`）
- `R9`：数据中心必须声明数据域、TTL/失效策略、一致性级别（对应 `C6/C7`）
- `R10`：`M5/M6/M7` 必须启用并完成落地，禁止仅文档声明不实现（对应 `C0/C5/C6/C9`）
- `R11`：服务层必须声明稳定契约（input/output/error）与错误映射策略（对应 `C9`）
- `R12`：页面层与业务组件层禁止直接调用裸 `fetch/axios` 与业务 API SDK（对应 `C9`）

## 5. 验证（Verification）
通过条件：
- `framework_instance_valid == true`（满足 R1-R12）
- 模块组合关系 `C1-C10` 均可在工程文件中追溯
- `route_reachability_target > route_reachability_baseline`
- `page_error_recovery_target > page_error_recovery_baseline`
- `message_delivery_target > message_delivery_baseline`
- `data_consistency_target > data_consistency_baseline`
- `service_call_success_target > service_call_success_baseline`
- 验证输出必须包含失败原因列表（failure reasons）

验证口径（框架级）：
- 路由覆盖率（核心页面）
- 页面入口可达率
- 异常兜底触发成功率
- 消息链路投递成功率（M5）
- 数据链路一致性达标率（M6）
- 服务链路调用成功率（M7）
- 规则违规项数量

### 6.1 环境与依赖安装约定（新增）
目标：确保“实例化”不仅生成目录与代码骨架，还能在统一环境下自动完成依赖安装并可启动。

环境基线：
- `Node.js`：`>= 20.0.0`
- 包管理器：`pnpm >= 9`（默认）
- 锁文件：`pnpm-lock.yaml`（必须提交）

安装执行策略：
- `install_on_scaffold`：`true`（执行脚手架时默认自动安装依赖）
- `strict_pm`：`pnpm`（如改用 `npm/yarn`，必须在实例中声明原因）
- 若网络受限导致安装失败，需输出失败原因并允许重试

依赖基线（framework 层最小集）：
- 运行时依赖：
  - `react`

  - `lucide-react`
- 开发依赖：
  - `vite`

兜底回流约定：
- `404`：进入 `not-found`，支持“返回首页/返回上一页”
- `401`：进入 `unauthorized`，支持“跳转登录/重新鉴权”
- `500`：进入 `server-error`，支持“重试/回退”
- 所有兜底页必须提供至少一种回流动作，满足 `M4 -> M2` 回流链路

必备产物：
- 路由注册入口：`src/app/router/index.ts`
- 路由清单：`src/app/router/route-records.ts`
- 访问策略：`src/app/router/guards.ts`
- 兜底路由：`src/app/router/fallback.ts`
- 路由稳定标识：`src/app/router/route-names.ts`
- 页面入口映射：`src/app/entry/*.entry.tsx`
- 消息中心入口：`src/app/message-center/event-bus.ts`
- 消息事件清单：`src/app/message-center/event-names.ts`
- 数据中心入口：`src/app/data-center/store.ts`
- 数据域与缓存策略：`src/app/data-center/domains.ts`、`src/app/data-center/cache-policy.ts`
- 服务层契约与实现：`src/app/service-center/auth/contracts.ts`、`src/app/service-center/auth/auth-service.ts`

- 兜底页面（500）：`src/pages/server-error/index.tsx`

# 前端页面标准（L2）

## 0. 规范关系
本文件是前端页面层（功能模块编排层）标准，遵循：
- `standards/L0/规范总纲与树形结构.md`
- `standards/L1/框架设计核心标准.md`
- `standards/L1/可追溯性标准.md`
- `standards/L1/可删减性标准.md`
- `framework/L0/前端框架标准.md`

## 1. 目标（Goal）
页面层负责功能模块编排，确保页面具备清晰状态流、稳定异常恢复和可观测行为。

## 2. 边界定义（Boundary）
- 页面层只处理页面级流程与状态，不实现可复用业务语义细节
- 页面层可读取路由参数与全局上下文，但不直接承载基础交互细节
- 页面层负责页面级数据请求编排，不直接侵入基础组件实现

## 3. 模块（最小可行基，Module）
- `M1 页面入口模块`：路由挂载、页面生命周期、页面级权限与参数解析
- `M2 业务组件编排模块（最小可行基）`：以“业务组件”作为页面装配最小单元
- `M3 页面状态模块`：`loading/success/error/empty` 四态聚合与切换
- `M4 页面可观测模块`：埋点、异常恢复动作、关键流程追踪

说明：
- 页面层最小可行基定义到业务组件层面，页面不得以基础组件作为主要编排粒度。

## 4. 组合原则（Combination Principles）
先定义模块如何组合，再定义组合后的判定约束。

模块组合关系：
- `C0` 整体拓扑（总关系）
  - 主链路（正常访问）：`M1 -> M2 -> M3`
  - 回流链路（异常恢复）：`M1 -> M3 -> M2 -> M3`
  - 观测链路（行为追踪）：`M1/M2/M3 -> M4`
  - 结构说明：`M1` 负责页面启动，`M2` 负责业务编排，`M3` 负责状态闭环，`M4` 负责全链路观测
- `C1` `M1 页面入口模块 -> M2 业务组件编排模块`
  - 在 `C0` 中对应主链路第一跳
  - 通俗理解：页面入口负责挂载并组织业务组件，而不是直接拼基础组件
- `C2` `M2 业务组件编排模块 -> M3 页面状态模块`
  - 在 `C0` 中对应主链路第二跳与回流链路复入点
  - 通俗理解：业务编排必须显式驱动 `loading/success/error/empty` 状态切换
- `C3` `M1 页面入口模块 -> M3 页面状态模块`
  - 在 `C0` 中对应回流链路第一跳
  - 通俗理解：入口层必须定义异常进入状态闭环的策略（降级/回退/重试）
- `C4` `M1/M2/M3 -> M4 页面可观测模块`
  - 在 `C0` 中对应观测链路
  - 通俗理解：页面生命周期、关键行为、状态变化都要可观测

组合约束（由组合关系导出）：
- `R1`：页面主结构必须通过业务组件组织，不得以基础组件作为主要装配单元（对应 `C1`）
- `R2`：页面必须定义四态切换条件与降级策略（对应 `C2/C3`）
- `R3`：页面必须提供错误恢复路径（重试/回退/降级，对应 `C3`）
- `R4`：页面必须声明关键行为埋点（曝光/点击/成功/失败，对应 `C4`）
- `R5`：页面可直接使用基础组件仅限布局壳与通用容器，不得承载业务语义（对应 `C1`）

## 5. 验证（Verification）
通过条件：
- 模块组合关系 `C1-C4` 均可在页面实现中追溯
- 页面关键流程具备自动化用例
- 页面四态转换可触发并可观测
- 页面异常路径具备可恢复动作
- 页面编排规则违反项为 0
- 页面主结构可追溯到业务组件清单（而非基础组件清单）

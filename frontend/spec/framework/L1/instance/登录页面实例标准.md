# 登录页面实例标准（L2 Example）

## 0. 实例元数据
- `instance_id`：PAGE-LOGIN-001
- `category`：page
- `source_standard`：`framework/L1/instance/前端页面标准.md`
- `version`：0.1.0
- `status`：draft
- `owner`：frontend

## 1. 目标（Goal）
在登录页面中落实页面层规范，确保用户可完成登录、失败可恢复、行为可观测。

目标指标：
- 登录流程完成率提升（相对基线）
- 登录失败后恢复成功率可统计
- 页面关键状态与关键事件可观测

## 2. 边界定义（Boundary）
- 路由：`/login`
- 页面输入：账号、密码（可扩展验证码、二次验证）
- 页面输出：登录成功跳转、失败提示、恢复入口（重试/回退/找回）
- 页面状态：`loading/success/error/empty`（四态必须可判定）
- 异常场景：网络失败、凭证错误、服务端异常、权限限制
- 页面层职责：编排登录业务组件与页面状态，不承载基础组件级交互细节
- 非职责范围：账号体系策略、鉴权协议细节、基础组件实现

## 3. 模块（最小可行基，Module）
- `M1 页面入口模块`
  - 登录页路由挂载（`/login`）
  - 页面生命周期初始化与路由参数解析（如 `redirect`）
  - 未登录访问时可达，已登录访问时执行重定向策略
- `M2 业务组件编排模块（最小可行基）`
  - 业务组件：`LoginForm`
  - 业务组件：`LoginErrorNotice`
  - 页面主结构由业务组件编排
- `M3 页面状态模块`
  - 初始空态：`empty`（初次进入、待输入）
  - 提交中：`loading`（提交凭证到鉴权服务）
  - 登录成功：`success`（写入会话并跳转）
  - 登录失败：`error`（失败提示与恢复动作）
- `M4 页面可观测模块`
  - 埋点：页面曝光、提交点击、登录成功、登录失败
  - 埋点：状态切换事件（`empty->loading`、`loading->success/error`）
  - 恢复动作：重试登录、返回上一页、找回密码（可选）

## 4. 组合原则（Combination Principles）
先定义模块如何组合，再定义组合后的判定约束。

模块组合关系：
- `C0` 整体拓扑（总关系）
  - 主链路（正常访问）：`M1 -> M2 -> M3`
  - 回流链路（异常恢复）：`M1 -> M3 -> M2 -> M3`
  - 观测链路（行为追踪）：`M1/M2/M3 -> M4`
  - 结构说明：`M1` 负责入口启动，`M2` 负责业务组件编排，`M3` 负责状态闭环，`M4` 负责行为观测
- `C1` `M1 页面入口模块 -> M2 业务组件编排模块`
  - 在 `C0` 中对应主链路第一跳
  - 通俗理解：页面入口负责挂载并组织登录业务组件
- `C2` `M2 业务组件编排模块 -> M3 页面状态模块`
  - 在 `C0` 中对应主链路第二跳与回流链路复入点
  - 通俗理解：登录业务编排必须显式驱动四态切换
- `C3` `M1 页面入口模块 -> M3 页面状态模块`
  - 在 `C0` 中对应回流链路第一跳
  - 通俗理解：入口层必须定义异常进入 `error` 并恢复的策略
- `C4` `M1/M2/M3 -> M4 页面可观测模块`
  - 在 `C0` 中对应观测链路
  - 通俗理解：页面生命周期、提交行为、状态转换都要可观测

组合约束（由组合关系导出）：
- `R1`：页面主结构必须通过业务组件组织，不得以基础组件作为主要装配单元（对应 `C1`）
  - 判定：登录页主区以 `LoginForm/LoginErrorNotice` 作为业务语义主承载
- `R2`：页面必须定义四态切换条件与降级策略（对应 `C2/C3`）
  - 判定：`empty/loading/success/error` 触发条件明确且可测
- `R3`：页面必须提供错误恢复路径（对应 `C3`）
  - 判定：登录失败后至少存在“重试”路径，可选“回退/找回”
- `R4`：页面必须声明关键行为埋点（对应 `C4`）
  - 判定：曝光、提交、成功、失败及关键状态转换事件均已登记
- `R5`：页面可直接使用基础组件仅限布局壳与通用容器，不得承载业务语义（对应 `C1`）
  - 判定：基础组件不直接实现登录领域规则（如凭证校验流程）

## 5. 验证（Verification）
通过条件：
- 模块组合关系 `C1-C4` 均可在页面实现中追溯
- 登录成功路径自动化用例通过
- 凭证错误与网络异常路径自动化用例通过
- `empty/loading/success/error` 状态切换可触发且可观测
- 恢复路径（重试）可触发且可观测
- 页面编排规则违反项为 0
- 页面主结构可追溯到业务组件清单（而非基础组件清单）

失败原因（示例）：
- 未定义 `error -> retry` 恢复链路（违反 `R3`）
- 页面主结构直接由基础组件拼装并承载业务语义（违反 `R1/R5`）
- 关键埋点缺失（如登录失败事件未登记，违反 `R4`）
- 四态转换条件不完整（违反 `R2`）

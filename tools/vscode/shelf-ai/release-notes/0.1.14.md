# 0.1.14

## 中文说明

- 本次版本移除了扩展侧对校验命令中旧 `--json` 参数的静默剥离逻辑，插件会按用户当前配置原样执行校验命令。
- 强化了 Framework/Evidence Tree Webview 的 HTML 转义，补齐了文本节点和属性值中的 `"`、`'`、`<`、`>`、`&` 转义，降低属性注入风险。
- 发布工作流在打包 VSIX 前新增 Webview TypeScript 检查与扩展测试步骤，确保发布资产来自通过回归校验的代码状态。

对使用者可见的影响：

- 若你本地仍配置了过时的 `validate_canonical.py --json`，现在不会再被插件自动改写，请改回仓库支持的正式命令。
- 树视图展示与跳转语义不变，但渲染层安全边界更严格。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.14.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- This release removes extension-side normalization that previously stripped stale `--json` from validation commands; Shelf now runs the configured command as-is.
- Tree webview HTML escaping is hardened for both text nodes and attribute values (`"`, `'`, `<`, `>`, `&`) to reduce attribute-injection risk.
- The publish workflow now runs webview TypeScript checks and extension tests before VSIX packaging.

User-visible impact:

- If your local settings still use outdated `validate_canonical.py --json`, the extension no longer rewrites it automatically; update the command to the supported validator entrypoint.
- Tree navigation behavior is unchanged, while rendering safety is stricter.

How to install:

1. Download `shelf-ai-0.1.14.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

# 0.1.24

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.23..shelf-ai-v0.1.24`（包含该区间全部提交）。

- Shelf AI 移除了 governed-task / intent-gate 会话流：相关命令、设置项、测试和扩展内 grant 状态都已删除，插件不再维护一条独立于仓库校验链的“会话授权”分支。
- 对应地，作者侧工作流进一步收口到现有主链：保存仍不阻断，真正的 enforcement 继续落在仓库级 `validate_canonical.py`、git hooks 和正式发布校验上。
- framework 作者体验继续补强：标准二级标题顺序 lint 现在有直接 Quick Fix，可从 Problems 更快修正标题顺序与标准标题内容。
- framework 参数标识语法放宽：作者源 `boundary_id` 现在允许使用 `snake_case` 或混合大小写 token；parser、实时 lint、契约文档和测试已同步更新，而 config / code 锚点仍统一按小写归一化，不再依赖大小写携带语义。
- 同一提交区间里还同步调整了 backend framework 作者源基线，包括编排模块表述收口和当前 backend 模块结构整理，以保持 Shelf AI 直接消费的作者源与仓库现状一致。
- 发版前已基于当前版本重新完成插件测试、硬编码守卫、`validate_canonical.py`、`validate_canonical.py --check-changes` 以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 插件侧不再出现 governed-task 会话相关入口；如果需要门禁，仍应依赖仓库正式校验链，而不是扩展内临时 grant。
- framework 文档现在可以直接使用 `query_in`、`result_out` 这类参数标识，Problems / Quick Fix / canonical 映射会保持一致。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.24.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.23..shelf-ai-v0.1.24`.

- Shelf AI removed the governed-task / intent-gate session flow: the related commands, settings, tests, and extension-side grant state are gone, so the extension no longer maintains a parallel “session grant” path outside the repository validation chain.
- The author workflow is therefore further converged onto the existing repository mainline: saves remain non-blocking, while real enforcement stays in `validate_canonical.py`, git hooks, and the formal release pipeline.
- Framework authoring UX was tightened again: heading-order lint now has a direct Quick Fix path, so standard second-level heading mistakes can be repaired faster from Problems.
- Framework parameter-id syntax is now more permissive: author-source `boundary_id` tokens may use `snake_case` or mixed case; parser, realtime lint, contracts, and tests were updated together, while config/code anchors still normalize to lowercase and no longer depend on case for semantics.
- The same commit range also refreshes the backend framework author-source baseline, including orchestration-module wording cleanup and current backend module restructuring, so the author inputs consumed by Shelf AI stay aligned with the repository.
- Before release, the current version was revalidated with extension tests, the hardcode guard, `validate_canonical.py`, `validate_canonical.py --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- Governed-task session entry points are no longer present in the extension; gating should continue to rely on the repository validation chain instead of extension-local grants.
- Framework docs can now use parameter ids such as `query_in` or `result_out` directly, with Problems, Quick Fix behavior, and canonical mapping staying aligned.

How to install:

1. Download `shelf-ai-0.1.24.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

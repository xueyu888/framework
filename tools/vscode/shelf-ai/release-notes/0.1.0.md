# 0.1.0

## 中文说明

- 这次版本号提升到 `0.1.0`，因为 Shelf AI 对齐的是一次仓库级架构重写，而不是普通的 `0.0.x` 小修补。
- 插件继续围绕新的主链工作：
  `Framework -> Config -> Code -> Evidence`。
- 项目配置跳转、自动物化、生成物保护、预检和树视图都继续以 `projects/*/project.toml` 与 `generated/canonical.json` 为基础。
- 修复了已配置框架识别逻辑，使其直接读取统一 `[[framework.modules]]` 结构里的 `framework_file`。
- 仓库文档同步更新到当前主路径，补齐新的 pipeline 入口引用，并把 rewrite 执行账本收口到“全部完成”的实际状态。

## English Notes

- The version is now `0.1.0` because Shelf AI is aligned with a repository-wide architecture rewrite rather than another `0.0.x` patch.
- The extension continues to follow the rewritten mainline:
  `Framework -> Config -> Code -> Evidence`.
- Project-config navigation, auto-materialization, generated-artifact guarding, codegen preflight, and tree views remain grounded in `projects/*/project.toml` and `generated/canonical.json`.
- Fixed configured-framework detection so it reads `framework_file` from the unified `[[framework.modules]]` layout.
- Repository documentation now points at the current pipeline entrypoint and records the rewrite execution ledger as fully closed so the extension-facing guidance matches the shipped architecture.

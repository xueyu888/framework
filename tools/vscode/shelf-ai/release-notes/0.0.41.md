## 中文说明

- 这一版把“配置即功能”正式接入 Shelf 主链：`implementation_config.toml` 的实现层字段现在必须进入编译后的 `ui_spec`、`backend_spec`、运行时选择链或生成产物命名，不能再只是 bundle 里的死配置。
- `scripts/validate_strict_mapping.py` 新增了实现配置生效性检查。Shelf AI 继续沿用仓库验证链，因此在保存或手动校验时，会直接拦截“字段没有下游效果”“目标缺失”“配置值没有真正落到下游结构”等问题。
- 知识库工作台的运行时选择位已经补齐：前端 renderer/style/script、后端 renderer/transport/retrieval 以及产品真相接口地址都会进入编译结果，并由运行时代码显式消费。
- 本地安装脚本会顺手清理历史 `mapping` / `archsync` 扩展残留，避免同机保留多套旧版本插件。

## English Notes

- This release makes `config as functionality` an enforced contract in Shelf. Effective `implementation_config.toml` fields must now land in compiled `ui_spec`, `backend_spec`, runtime selection paths, or generated artifact names instead of remaining dead selectors inside bundles.
- `scripts/validate_strict_mapping.py` now checks implementation-config effectiveness. Shelf AI keeps using the repository validation chain, so save-time and manual validation now fail on dead config fields, missing downstream targets, or mismatched compiled values.
- The knowledge-base workbench now threads frontend renderer/style/script and backend renderer/transport/retrieval plus the product-spec endpoint through compiled output and explicit runtime consumption.
- The local install script also removes legacy `mapping` / `archsync` extension leftovers before installing the current Shelf AI build.

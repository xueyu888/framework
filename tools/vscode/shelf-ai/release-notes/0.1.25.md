# 0.1.25

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.24..shelf-ai-v0.1.25`（包含该区间全部提交）。

- Shelf AI 已同步到新版 framework 作者模板：parser、snippets、导航、补全、框架树相关测试与契约口径统一收敛到当前作者源结构，不再沿用旧模板假设。
- framework lint 契约检查进一步收紧；同一提交区间里，backend 检索流程相关 drafts 与正式 framework 结构也同步重排，插件直接消费的作者源基线已切到新的 retrieval / candidate 模块组合。
- 仓库与插件都已支持“零项目 bootstrap”状态：当仓库暂时没有 `projects/*/project.toml` 时，`validate_canonical.py`、`materialize_project.py` 和 Shelf AI preflight 会返回显式 no-op success；但 evidence tree、正式 canonical 导航与 `src/main.py serve` 仍要求真实项目配置。
- 旧的 `message_queue_basic` 样例项目与对应静态模块链路已从当前基线移除，仓库默认状态改为 framework authoring / bootstrap，而不是附带一个内置项目样板。
- 同一提交区间还新增了独立的 `tools/vscode/shelf-md-live` Markdown 实时渲染原型；它属于仓库范围内的新实验插件，但不打包进本次 `shelf-ai` 发布资产。
- 发版前已重新完成插件测试、webview 类型检查、硬编码守卫、`validate_canonical.py`、`validate_canonical.py --check-changes` 以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 使用 Shelf AI 编写 framework 时，新的作者模板结构现在可以直接得到一致的补全、导航、lint 与 canonical 入口体验。
- 如果你的仓库还处于“只有 framework、尚未创建项目配置”的阶段，Shelf AI 不会再把这种状态误报成失败；但要使用正式 evidence/canonical 导航，仍需先创建 `projects/<project_id>/project.toml`。
- 本次公开安装包仍只包含 `Shelf AI` 扩展本体，不包含 `shelf-md-live` 原型。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.25.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.24..shelf-ai-v0.1.25`.

- Shelf AI is now aligned with the current framework authoring template: parser behavior, snippets, navigation, completion, framework-tree-related tests, and contract wording all converge on the new author-source structure instead of the previous template assumptions.
- Framework lint contract checks were tightened again; in the same range, the backend retrieval-flow drafts and published framework modules were also reshaped, so the author-source baseline consumed by Shelf AI now reflects the newer retrieval / candidate module set.
- The repository and the extension now support explicit zero-project bootstrap mode: when no `projects/*/project.toml` exists yet, `validate_canonical.py`, `materialize_project.py`, and Shelf AI preflight report an intentional no-op success; evidence tree, formal canonical-backed navigation, and `src/main.py serve` still require a real project config.
- The old `message_queue_basic` sample project and its static-module chain were removed from the active baseline, so the repository now defaults to framework authoring / bootstrap state rather than shipping a built-in sample project.
- The same audited commit range also introduces a separate `tools/vscode/shelf-md-live` markdown live-rendering prototype. It is part of the repository history for this range, but it is not bundled into the `shelf-ai` VSIX released here.
- Before release, the current version was revalidated with extension tests, webview type checks, the hardcode guard, `validate_canonical.py`, `validate_canonical.py --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- If you author framework docs with the new template shape, Shelf AI should now provide matching completion, navigation, lint, and canonical entry behavior instead of lagging behind the authoring format.
- If your repository is still in a “framework only, no project config yet” phase, Shelf AI no longer misreports that state as a failure; formal evidence/canonical navigation still begins only after `projects/<project_id>/project.toml` exists.
- The public installable asset for this release still contains only the Shelf AI extension itself, not the `shelf-md-live` prototype.

How to install:

1. Download `shelf-ai-0.1.25.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

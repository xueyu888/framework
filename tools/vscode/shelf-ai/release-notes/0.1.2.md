# 0.1.2

## 中文说明

- 这个版本继续对齐仓库收口后的四层主链，runtime 由模块 export 自然装配。
- 扩展继续围绕统一主链工作：
  `Framework -> Config -> Code -> Evidence`。
- 文档与证据视图说明同步切到当前 canonical 派生命名。
- 本次发布同时重新打包并验证 `.vsix`，确保扩展资产与当前仓库状态一致。

安装或获取方式：

1. 下载发布资产 `shelf-ai-0.1.2.vsix`
2. 在 VSCode 中执行 `Extensions: Install from VSIX...`
3. 选择该文件完成安装

注意事项：

- 扩展依然要求仓库使用 `projects/*/project.toml` 作为唯一项目入口。
- 修改框架文档或项目配置后，仍需执行 `uv run python scripts/materialize_project.py` 刷新 canonical 与派生视图。

## English Notes

- This release aligns the extension with the repository's finalized four-layer runtime mainline.
- The extension continues to follow the unified chain:
  `Framework -> Config -> Code -> Evidence`.
- Release-facing docs and evidence wording now use the current canonical-derived artifact names consistently.
- The `.vsix` asset was rebuilt and revalidated against the current workspace so the published binary matches the validated repository state.

Install / access:

1. Download the release asset `shelf-ai-0.1.2.vsix`
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the file to complete installation

Notes:

- The extension still expects `projects/*/project.toml` to remain the only project entrypoint.
- After changing framework markdown or project config, rerun `uv run python scripts/materialize_project.py` to refresh canonical and derived views.

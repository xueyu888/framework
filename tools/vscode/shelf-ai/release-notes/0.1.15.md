# 0.1.15

## 中文说明

- 本次版本同步了框架规范：删除“基必须直接对应能力”的硬约束，能力导出继续由组合规则层负责。
- 新增机器校验：每个基必须至少声明一个参数边界，且该边界必须属于所属模块；不满足时 correspondence guard 将失败。
- Shelf AI 框架规则提示 `FW022` 已同步到“参数约束与边界归属”语义。

对使用者可见的影响：

- 框架文档与 runtime 校验对“基”的要求更聚焦在边界约束，不再要求基层直接维护能力归属映射。
- 若某个基没有绑定边界，`validate_canonical` 会直接报错并阻断后续流程。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.15.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- This release aligns framework standards by removing the mandatory base-to-capability direct mapping constraint; capability derivation remains at the rule-combination layer.
- Correspondence guard now enforces that every base declares at least one boundary and that each declared base boundary belongs to its owner module.
- Shelf AI framework lint hint `FW022` has been updated to match the new base-boundary semantics.

User-visible impact:

- Framework docs and runtime validation now focus base constraints on boundary bindings rather than direct capability ownership at the base layer.
- `validate_canonical` will fail when a base has no boundary binding.

How to install:

1. Download `shelf-ai-0.1.15.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

# 0.1.29

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.28..shelf-ai-v0.1.29`（包含该区间全部提交）。

- 本次发布把 `.sf` 实验语言的顶层块命名彻底收敛为单数：正式写法现在统一为 `Goal / Base / Principle / Space / Boundary`，旧的 `Principles / Spaces` 不再是正式拼写，lint、补全、TextMate 兜底高亮和语义高亮都已经跟到同一口径。
- 同一轮里，`map` 也被提升为共享 grammar 的正式语句种类，并且和 `op[...]` 明确区分：`map` 用来表达显式映射、索引和传播关系，可出现在 `Base` 与 `Principle`；`op[...]` 仍只表示带 shape 的操作子，留在 `Base`。
- `.sf` 的引用解析也一起补强了：当前引用路径已同步收敛为 `Base.xxx / Principle.xxx / Space.xxx / Boundary.xxx`，并接受中文顿号分隔的多引用写法，避免示例里类似 `Base.A、Base.B` 被误识别成单个长 token。
- README、`.sf` 协议文档、插件现状快照、契约第 13 章，以及 `合取模块.sf`、`父子任务状态记录模块.sf` 两个示例文件，均已同步到这套新口径。
- 发版前已重新完成 Shelf AI 扩展测试、webview 类型检查、仓库 `mypy`、硬编码守卫、`validate_canonical.py`、`validate_canonical.py --check-changes`，以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 现在写 `.sf` 时，顶层块命名终于一致了，不再需要记住“有的块写单数，有的块写复数”。
- `map` 和 `op[...]` 的作者语义已经分开，结构草稿里写映射关系或映射资源时，不需要再硬塞成操作子。
- 如果右值里包含 `Principle.xxx`、`Space.xxx` 或用中文顿号分隔的多个引用，lint 和高亮会更接近作者直觉。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.29.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.28..shelf-ai-v0.1.29`.

- This release fully normalizes the `.sf` experimental language around singular top-level block names. The formal block vocabulary is now `Goal / Base / Principle / Space / Boundary`, and the old `Principles / Spaces` spellings are no longer part of the accepted grammar for lint, completion, fallback TextMate highlighting, or semantic highlighting.
- In the same change set, `map` is promoted into a formal shared-grammar statement kind and clearly separated from `op[...]`: `map` is now used for explicit mappings, indexes, and propagation relations in `Base` and `Principle`, while `op[...]` remains the shaped operator form in `Base`.
- The `.sf` reference path handling was tightened as well. References now align with the singular namespace form `Base.xxx / Principle.xxx / Space.xxx / Boundary.xxx`, and the parser now accepts multiple references separated by the Chinese punctuation mark `、` instead of merging them into one token.
- The packaged README, the `.sf` protocol spec, the plugin evidence snapshot, chapter 13 of the plugin contract, and the shipped example files `合取模块.sf` and `父子任务状态记录模块.sf` have all been synced to the same model.
- Before release, the workspace was revalidated with Shelf AI extension tests, webview type checks, repository `mypy`, the hardcode guard, `validate_canonical.py`, `validate_canonical.py --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- Writing `.sf` drafts now uses a consistent singular top-level naming scheme instead of mixing singular and plural block names.
- `map` and `op[...]` now have clearly separated authoring roles, so explicit mapping resources no longer need to be forced into operator syntax.
- Lint and highlighting now behave more intuitively when a right-hand value references `Principle.xxx`, `Space.xxx`, or multiple references separated by the Chinese punctuation mark `、`.

How to install:

1. Download `shelf-ai-0.1.29.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

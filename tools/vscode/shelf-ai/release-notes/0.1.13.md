## 中文说明

- 本次发布把 framework 作者源术语从 `Boundary / 边界` 统一切到 `Parameter / 参数`，并同步更新了 Shelf AI 的框架模板、补全项、导航展示和当前作者文档。
- 兼容层仍保留现有 `boundary_id`、`boundary projection` 与相关 runtime / canonical 字段名，因此这次不是运行时模型重命名，而是作者侧表达与插件入口的统一。
- 已重新完成本地插件测试、仓库校验、VSIX 打包与安装验证，发布资产包含 `shelf-ai-0.1.13.vsix`。

对使用者可见的影响：

- 新建或编辑 framework 文档时，标准章节和规则子项现在默认显示为“参数定义 / 参数绑定”。
- 旧文档中的“边界定义 / 边界绑定”在当前迁移期仍可被解析和导航，不会因为这次升级立即失效。
- 插件命令、canonical 主链和运行时行为没有新增功能开口。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.13.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

注意事项：

- 这是一次术语统一与作者入口同步发布，不是 runtime schema 或 canonical 字段重命名发布。
- 当前仓库仍保留 `boundary_*` 兼容机器字段名，后续若进行真正的数据模型改名，将单独发布并说明迁移影响。

## English Notes

- This release renames the framework authoring term from `Boundary` to `Parameter` across current framework docs, Shelf AI snippets, completion entries, navigation copy, and author-facing plugin docs.
- The compatibility layer still keeps existing `boundary_id`, `boundary projection`, and related runtime/canonical field names, so this is not a runtime model rename; it is an author-surface and plugin-entry terminology alignment release.
- Local plugin tests, repository validation, VSIX packaging, and install verification were rerun. The release asset includes `shelf-ai-0.1.13.vsix`.

User-visible impact:

- When creating or editing framework documents, the default section and rule wording now uses `参数定义 / 参数绑定` (`Parameter definition / parameter binding`).
- Legacy framework files that still use `边界定义 / 边界绑定` remain parseable and navigable during the transition.
- No new plugin commands, canonical surfaces, or runtime features are introduced in this release.

How to install:

1. Download `shelf-ai-0.1.13.vsix` from the release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

Notes:

- This is a terminology-alignment and authoring-surface release, not a runtime schema or canonical field rename release.
- The repository still keeps `boundary_*` compatibility field names today; if a real data-model rename happens later, it should ship as a separate release with explicit migration notes.

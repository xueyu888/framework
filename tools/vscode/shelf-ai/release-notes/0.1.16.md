# 0.1.16

## 中文说明

- 本次版本移除了插件对 `projects/knowledge_base_basic/project.toml` 的硬依赖，不再把它当作唯一默认项目配置入口。
- framework 导航与 correspondence 运行时现在优先从工作区自动发现 `projects/*/project.toml`，使默认行为与当前仓库结构保持一致。
- 当问题定位缺少有效文件时，Shelf 会优先回退到已发现的项目配置文件；若不存在，则回退到仓库规范文档，避免打开不存在路径导致的失败。

对使用者可见的影响：

- 删除样例项目后，插件仍可正常工作，不会因旧默认路径失效而崩溃。
- 问题跳转在“无明确文件定位”场景下更稳定。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.16.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- This release removes the extension’s hard dependency on `projects/knowledge_base_basic/project.toml` as a fixed fallback project file.
- Framework navigation and correspondence runtime now prefer auto-discovered `projects/*/project.toml` files to align with current workspace layout.
- When issue locations do not provide a valid file, Shelf now falls back to an available project file first, then to the standards document, avoiding failures caused by missing paths.

User-visible impact:

- The extension remains usable after removing the sample project.
- Issue reveal becomes more stable when no direct file location is available.

How to install:

1. Download `shelf-ai-0.1.16.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

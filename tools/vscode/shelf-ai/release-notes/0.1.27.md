# 0.1.27

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.26..shelf-ai-v0.1.27`（包含该区间全部提交）。

- 本次发布继续收敛 `.sf` 实验语言的作者说明链路：README 已补上规范入口，`keyword-first-grammar-协议.md` 重新整理了当前写法，并把 `Boundary` 右值可直接绑定 `Base.xxx`、右值可多行续写、`{}` / `<>` 的约定，以及 `t(i)` 作为时间位点写法统一写清。
- Shelf AI 扩展侧同步补上了对应能力：`.sf` lint 现在接受“声明头单行 + 右值按更深缩进续写”的写法，`SFL009` 的提示语也改成了更精确的声明头 / 续行缩进说明；测试新增了多行有序集合样例回归。
- 同一提交区间里，插件现状证据与 `docs/back_zrx` 下的草案示例也一起对齐了“首次完成”语义：文档现在明确区分“历史上是否完成过”和“本次是否首次触发完成”，并用 `t(0) / t(1)` 表达时间位点。
- 这仍然是实验作者语言预览的文档与可用性收敛版本，不涉及 canonical / publish 主链语义扩展；`t(i)` 当前属于约定好的作者表达，不是新的 parser 强语法。
- 发版前已重新完成 Shelf AI 扩展测试、webview 类型检查、仓库 `mypy`、硬编码守卫、`validate_canonical.py`、`validate_canonical.py --check-changes` 以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 如果你现在用 `.sf` 写结构草案，可以把较长的 `seq` / 其他右值拆成多行，而不会因为续行本身被 lint 误判。
- `Boundary` 直接写 `Base.xxx` 引用、以及用 `t(0) / t(1)` 表达状态演进的写法，现在在规范文档里都有统一口径，编辑器诊断提示也更接近实际写作方式。
- 这版更偏向作者体验与文档一致性修复，而不是新增正式 framework 主链能力。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.27.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.26..shelf-ai-v0.1.27`.

- This release continues to tighten the `.sf` experimental authoring path: the README now links to the spec entry, `keyword-first-grammar-协议.md` has been reorganized, and the current rules for direct `Base.xxx` boundary bindings, multi-line right-hand values, `{}` / `<>` notation, and the `t(i)` time-point convention are now documented in one place.
- The extension-side `.sf` implementation has been aligned with that documentation: lint now accepts a single-line declaration head followed by deeper-indented continuation lines, `SFL009` now explains the declaration-head / continuation-indent rule more precisely, and tests now include a regression sample for multi-line ordered collections.
- The same commit range also syncs the plugin evidence snapshot and the draft `docs/back_zrx` examples so the “first completion” example clearly separates historical completion facts from the current trigger event, using `t(0) / t(1)` to express time points.
- This is still a documentation and authoring-usability release for the experimental preview language; it does not introduce a new canonical / publish-chain grammar feature. `t(i)` remains a documented authoring convention rather than a new parser-level construct.
- Before release, the workspace was revalidated with Shelf AI extension tests, webview type checks, repository `mypy`, the hardcode guard, `validate_canonical.py`, `validate_canonical.py --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- If you draft structures in `.sf`, longer `seq` values and similar right-hand content can now be split across multiple lines without being rejected just because of the continuation layout.
- Direct `Base.xxx` bindings in `Boundary` values, together with the `t(0) / t(1)` style for time-based state transitions, now have one documented writing convention and clearer editor diagnostics.
- This version is primarily about authoring experience and documentation consistency rather than new formal framework-chain capabilities.

How to install:

1. Download `shelf-ai-0.1.27.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

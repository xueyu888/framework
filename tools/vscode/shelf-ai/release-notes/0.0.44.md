## 中文说明

- 修复框架树画布拖拽的根问题：这次不是只补事件，而是改正了视口模型本身。生成页现在把中间区域当成真正可平移的工作区，左键拖动画布时会稳定移动整张图，而不是被内容宽度撑成“看起来能拖、实际上没空间拖”的伪视口。
- 框架树继续作为 Shelf AI 默认主视图，治理树保持独立次级入口。升级后，默认打开的仍是框架文档树，但这次画布会带着更大的工作区留白、初始化居中和适配后重新居中，整体更接近思维导图或 Visio 的操控方式。
- 仓库标准同步补了一条新的硬约束：本仓库 Git 提交标题默认使用中文自然语言书写。与之配套，`specs/code/` 里的代码治理标准也正式把 Git 提交信息和公开发布说明一起纳入标准树。
- 安装方式：下载本版本附带的 `shelf-ai-0.0.44.vsix`，在 VS Code 中执行 `Extensions: Install from VSIX...` 安装；若升级前已经打开树图页面，安装后建议重新打开该页面或执行一次 `Developer: Reload Window`，以拿到新的画布 DOM 与交互脚本。

## English Notes

- This release fixes the root cause behind framework-tree panning. The generated page now uses a real movable workspace surface instead of a content-stretched pseudo-viewport, so left-dragging the canvas reliably moves the whole graph rather than feeling stuck.
- The framework tree remains Shelf AI's default primary view, while the governance tree stays available as a separate secondary view. After upgrading, the default graph opens with larger workspace padding and proper recentering after fit/reset, making the interaction closer to a mind-map or Visio-style canvas.
- Repository governance is also tightened in this release: Git commit subjects in this repository are now required to use Chinese natural-language titles by default, and `specs/code/` now formally registers both the Git commit-message rule and the public release-note standard in the standards tree.
- Install by downloading the bundled `shelf-ai-0.0.44.vsix` and running `Extensions: Install from VSIX...` in VS Code. If you already had a tree page open before upgrading, reopen it or run `Developer: Reload Window` so the refreshed canvas DOM and interaction script take effect.

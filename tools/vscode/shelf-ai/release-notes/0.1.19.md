# 0.1.19

## 中文说明

- 本次版本聚焦仓库主链一致性与“默认项目回退”治理：运行时与脚本不再依赖固定 `projects/project.toml` 回退，改为 `projects/*/project.toml` 自动发现或显式参数传入。
- 补强了无项目场景下的行为：`main.py`、`materialize_project.py`、`validate_canonical.py` 在缺少项目配置时给出明确提示，并支持 bootstrap 检查流。
- 更新了硬编码守卫覆盖范围与文档一致性，避免“默认项目”措辞和固定路径语义继续回流。

对使用者可见的影响：

- 在非固定目录或多项目仓库中使用 Shelf AI 时，项目发现与验证流程更稳定，不再依赖单一路径约定。
- 发布产物仍为可直接安装的 VSIX，安装方式保持不变。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.19.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- This release focuses on mainline consistency and removing default-project fallbacks: runtime/scripts no longer rely on a fixed `projects/project.toml` fallback and now use `projects/*/project.toml` discovery or explicit input.
- Behavior for no-project workspaces is hardened: `main.py`, `materialize_project.py`, and `validate_canonical.py` now provide explicit guidance and support bootstrap-friendly checks.
- Hardcode-guard coverage and documentation consistency were synced to prevent default-project wording and fixed-path semantics from reappearing.

User-visible impact:

- Shelf AI project discovery and validation are more robust in non-fixed-path or multi-project repositories.
- The install path is unchanged and still ships a directly installable VSIX.

How to install:

1. Download `shelf-ai-0.1.19.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

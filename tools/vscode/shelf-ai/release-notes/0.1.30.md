# 0.1.30

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.29..shelf-ai-v0.1.30`（包含该区间全部提交）。

- 本次发布把 `.sf` 实验语言里“有限值域参数”的正式写法从 `enum` 收敛为 `set`。共享 grammar、上下文补全、模板 snippet、协议文档、现状快照、示例文件和测试基线现在统一使用 `param<set>`；`param<range>` 维持不变。
- 契约第 13 章也同步补了这条口径：`.sf` 里有限值域参数的正式写法是 `param<set>`，`param<enum>` 不再是正式拼写；`{}` 继续表示集合或枚举内容，`<>` 继续表示有序集合。
- 同一轮里，仓库内全部 `.sf` 文件也已经按当前最新 lint 口径修正完成。此前仍混用旧块名、`graph` / `enum` 边界 subtype 或旧声明形式的 back_zrx 草稿，现在都已经收敛到当前预览语法。
- 发版前已重新完成 Shelf AI 扩展测试、webview 类型检查、仓库 `mypy`、硬编码守卫、`validate_canonical.py`、`validate_canonical.py --check-changes`、仓库级 `.sf` lint，以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 现在写 `.sf` 时，有限值域参数统一写成 `param<set>`，不用再在 `set` 和 `enum` 之间来回切换。
- 打开的示例文件和仓库内现有 `.sf` 草稿，都会更接近当前插件真正执行的 lint 规则，不容易再出现“文档能看懂，但编辑器报错”的错位。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.30.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.29..shelf-ai-v0.1.30`.

- This release normalizes the `.sf` preview language around `set` for finite-domain parameters. The shared grammar, completion path, template snippet, protocol spec, evidence snapshot, shipped examples, and test baseline now all use `param<set>` as the formal spelling, while `param<range>` stays unchanged.
- Chapter 13 of the plugin contract is synced as well: finite-domain parameters in `.sf` now formally use `param<set>`, `param<enum>` is no longer part of the accepted spelling, `{}` still denotes set or enumeration content, and `<>` still denotes ordered collections.
- In the same release, every `.sf` file in the repository was realigned to the current lint baseline. The back_zrx drafts that still mixed older block names, `graph` / `enum` boundary subtypes, or outdated declaration shapes are now written in the same preview syntax the extension actually validates.
- Before release, the workspace was revalidated with Shelf AI extension tests, webview type checks, repository `mypy`, the hardcode guard, `validate_canonical.py`, `validate_canonical.py --check-changes`, repository-wide `.sf` lint, and local VSIX packaging/install verification.

User-visible impact:

- Finite-domain parameters in `.sf` now use one stable spelling: `param<set>`.
- The shipped examples and the repository drafts are now closer to the current editor behavior, reducing the mismatch between readable drafts and actual lint diagnostics.

How to install:

1. Download `shelf-ai-0.1.30.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

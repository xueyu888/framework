# 0.1.23

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.22..shelf-ai-v0.1.23`（包含该区间全部提交）。

- framework 作者语法进一步收口到平铺列表：`## 1. 能力声明` 中的 `N*` 非职责条目现在与 `C*` 一样直接写在章节列表里，不再要求额外的 `### 非职责声明` 小标题。
- framework completion / snippet 模板 / Quick Fix 已同步到这套写法：新模板直接给出 `C*` 后接 `N*`；能力章节格式修复也会按上下文选择生成标准 `C` 或 `N` 条目。
- 实时 framework lint 放松了对三级标题的处理：作者若保留或手写 `###`，插件不再把它当作结构错误；它只被视为中性分隔行，不构成正式语义，也不会阻断正常编辑。
- 同时，正式约束并没有放松到失真：`## 1~5` 标准章节顺序仍然严格检查，`C* / N* / P* / B* / R* / V*` 的合法条目格式和已定义符号引用仍会继续校验。
- 发版前已在 clean 基线完成插件测试、硬编码守卫、`validate_canonical --check-changes` 以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 旧 framework 文档即使还带着 `### 非职责声明`，也不会再在 Problems 里被当成语法错误。
- 新写 framework 时可以直接把 `N*` 放在 `C*` 后面，模板、补全和修复动作会与这种作者习惯保持一致。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.23.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.22..shelf-ai-v0.1.23`.

- Framework authoring was further flattened: `N*` non-responsibility items inside `## 1. Capability Statement` now live directly in the section list, without requiring a dedicated `### Non-Responsibility Statement` heading.
- Framework completion, snippet templates, and quick fixes were aligned with that style: new templates place `N*` directly after `C*`, and the capability-item quick fix now chooses a standard `C` or `N` entry based on context.
- Realtime framework lint now tolerates third-level headings as neutral separators. Authors can keep or type `###` lines without triggering structural errors; those lines no longer carry formal meaning in lint.
- The formal checks themselves remain strict where they matter: standard `## 1~5` section order is still validated, and entry syntax plus defined-symbol references for `C* / N* / P* / B* / R* / V*` are still enforced.
- Before release, the clean baseline passed extension tests, the hardcode guard, `validate_canonical --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- Existing framework docs that still contain `### Non-Responsibility Statement` will no longer show those lines as syntax problems in Problems.
- New framework authoring can keep `N*` immediately after `C*`, and templates/completion/fixes now match that workflow consistently.

How to install:

1. Download `shelf-ai-0.1.23.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

# 0.1.8

## 中文说明

- 本次发布是 Shelf AI 的发布链路修复版，不引入新的插件功能；重点是修复 GitHub `Publish Shelf AI` 工作流在 runner 上未安装扩展依赖、导致 `vsce package` 失败的问题。
- 发布工作流现在会在打包前执行 `npm ci`，并启用 `package-lock.json` 对应的 npm 缓存，使 GitHub 上的打包环境与本地 `install_local.sh` 的验证环境保持一致。
- 本地已重新完成 `0.1.8` 的打包与安装验证，确保新的公开 tag 可以重新产出可安装的 `.vsix` 资产。

对使用者可见的影响：

- 公开发布页会重新附带可直接安装的 `shelf-ai-0.1.8.vsix`
- 插件运行时行为与 `0.1.7` 保持一致，不需要额外迁移
- 这次变更主要修复发布可靠性，而不是树图、导航或 validation 功能

安装或获取方式：

1. 下载发布资产 `shelf-ai-0.1.8.vsix`
2. 在 VSCode 中执行 `Extensions: Install from VSIX...`
3. 选择该文件完成安装

注意事项：

- 若你本地已经安装 `0.1.7`，可直接覆盖安装 `0.1.8`
- 本次为流程修复版，canonical、freshness、generated guard 与现有 settings 行为均无语义变化

## English Notes

- This release is a Shelf AI publish-pipeline repair release rather than a feature release; it fixes the GitHub `Publish Shelf AI` workflow where the runner did not install extension dependencies before `vsce package`.
- The publish workflow now runs `npm ci` before packaging and uses the `package-lock.json` npm cache path so GitHub packaging matches the validated local `install_local.sh` environment.
- The `0.1.8` VSIX has already been repackaged and locally reinstalled to verify that the new public tag can produce an installable `.vsix` asset again.

User-visible impact:

- The public release page should again contain a directly installable `shelf-ai-0.1.8.vsix`
- Runtime behavior remains the same as `0.1.7`, so no migration is required
- This patch is about release reliability, not tree, navigation, or validation feature changes

Install / access:

1. Download the release asset `shelf-ai-0.1.8.vsix`
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the file to complete installation

Notes:

- If `0.1.7` is already installed locally, you can install `0.1.8` directly over it
- This is a process-fix release; canonical, freshness, generated-guard, and current settings behavior are unchanged

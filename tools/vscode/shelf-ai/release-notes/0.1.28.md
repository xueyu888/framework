# 0.1.28

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.27..shelf-ai-v0.1.28`（包含该区间全部提交）。

- Shelf AI 继续收敛 `.sf` 实验语言的编辑器链路：打开 `.sf` 时插件会直接激活，当前版本同时补上了由共享 grammar 派生的 TextMate 兜底高亮，并默认开启 `shelf-framework` 语言的 semantic highlighting，避免出现“provider 未就绪时整页无高亮”的首屏体验。
- `.sf` 的语义与语法边界也在这版继续收紧：声明名现在会独立高亮，`[shape]` 与 `<subtype>` 会按不同 token 语义区分；同时 `Boundary` 不再接受平行的 `op[...]` 写法，只保留 `in / out / param` 三类边界声明，`op[...]` 回到 `Base` 里的操作子定义。
- 与上述实现同步，README、`.sf` 作者规范文档、测试基线、插件现状快照，以及布尔逻辑 / XY 星型图等作者草稿都已对齐到当前 `set / elem / struct / seq / op` 与 `Boundary = in/out/param` 的口径。
- 同一仓库提交区间里还包含一批继续推进中的 `framework/**` 与 `docs/back_zrx/**` 作者草稿提交；这些文件属于本次 tag 的完整审计范围，但不作为 Shelf AI VSIX 的打包资产发布。
- 发版前已重新完成 `Shelf AI` 扩展测试、webview 类型检查、仓库 `mypy`、硬编码守卫、`materialize_project.py`、`validate_canonical.py`、`validate_canonical.py --check-changes` 以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 如果你现在用 `.sf` 写结构草案，首屏高亮会更稳定，不再只依赖语义 token provider 恰好先注册成功。
- `elem 点`、`param<enum> 状态边界` 这类声明名，以及 `op[4:1]` 的 shape、`in<enum>` 的 subtype，现在会被更明确地区分显示，作者更容易从颜色层级上看出“关键字 / 注记 / 名称”的结构差别。
- `Boundary` 的作者语义更单一：输入、输出和参数写在 `Boundary`，操作子仍写在 `Base`，不会再出现两套并行表达方式。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.28.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.27..shelf-ai-v0.1.28`.

- Shelf AI further stabilizes the `.sf` experimental editor path in this release: the extension now activates as soon as an `.sf` file is opened, ships a generated TextMate fallback grammar derived from the shared rules, and enables semantic highlighting by default for the `shelf-framework` language id so the first screen does not fall back to a completely unhighlighted buffer.
- The `.sf` semantic and grammar boundaries were also tightened: declaration names now get their own highlight token, `[shape]` and `<subtype>` are rendered as different semantic classes, and `Boundary` no longer accepts a parallel `op[...]` form. Only `in / out / param` declarations remain valid there, while `op[...]` stays in `Base` for operator definitions.
- The same implementation change set also syncs the packaged README, the `.sf` authoring spec, test baselines, the plugin evidence snapshot, and the boolean-logic / XY star-graph drafts to the current `set / elem / struct / seq / op` plus `Boundary = in/out/param` authoring model.
- The same repository-wide commit range also includes ongoing authoring work under `framework/**` and `docs/back_zrx/**`; those files are part of the audited tag interval, but they are not bundled into the Shelf AI VSIX payload.
- Before release, the workspace was revalidated with Shelf AI extension tests, webview type checks, repository `mypy`, the hardcode guard, `materialize_project.py`, `validate_canonical.py`, `validate_canonical.py --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- If you draft structures in `.sf`, the first-screen highlighting path is now more reliable instead of depending only on the semantic-token provider being ready in time.
- Declaration names such as `elem 点` or `param<enum> 状态边界`, together with `op[4:1]` shapes and `in<enum>` subtypes, now render with more distinct visual classes so the author can read keyword / annotation / name structure faster.
- `Boundary` authoring is now stricter and simpler: inputs, outputs, and parameters live in `Boundary`, while operators remain in `Base`, without two parallel ways to express the same layer.

How to install:

1. Download `shelf-ai-0.1.28.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

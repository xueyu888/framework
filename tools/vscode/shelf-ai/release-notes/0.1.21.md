# 0.1.21

## 中文说明

- 本次版本统一了 framework 作者侧标准章节命名：`## 2. 边界定义（Boundary / Parameter 参数）` 与 `## 3. 最小结构基（Minimal Structural Bases）`，并同步到解析器、lint、补全、导航、模板、测试与现有 framework 文档。
- 新增边界定义章节的输入体验：在该章节中输入 `-` 会自动展开为参数条目模板，并按当前文档上下文推断下一个 `P` 编号（例如 `P2`、`P3`）。
- 重新构建并验证插件发布产物，确保本次公开版本与当前仓库状态一致。

对使用者可见的影响：

- 章节标题的标准写法已切换为本版本新命名，旧命名不再作为当前标准入口。
- framework 编写效率提升：边界定义章节可以像代码自动补全一样，直接用 `-` 触发条目模板。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.21.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- This release standardizes framework authoring section names to `## 2. 边界定义（Boundary / Parameter 参数）` and `## 3. 最小结构基（Minimal Structural Bases）`, and syncs the change across parser, lint, completion, navigation, snippets, tests, and existing framework docs.
- Added an in-editor authoring improvement for the Boundary section: typing `-` now auto-expands into a parameter entry snippet with inferred next `P` numbering (for example, `P2`, `P3`).
- Rebuilt and revalidated the extension release artifact so the published package matches the current repository state.

User-visible impact:

- The standardized section naming now follows the new wording in this release.
- Framework writing is faster in the Boundary section via direct dash-triggered snippet expansion.

How to install:

1. Download `shelf-ai-0.1.21.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

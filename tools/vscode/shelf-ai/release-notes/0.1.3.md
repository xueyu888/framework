# 0.1.3

## 中文说明

- 这次发布只面向 `Shelf AI` VSCode 扩展，不再附带 `knowledge_base_basic` 的公开版本产物。
- 仓库内的 `knowledge_base_basic` 现在被明确视为本地验证与演示样例，用来测试 Shelf AI 在真实 `Framework -> Config -> Code -> Evidence` 流程下的效果，而不是独立发布对象。
- 发布规范已同步收紧：GitHub Release 只发布 Shelf AI 的双语说明和 `.vsix` 安装包，避免样例项目继续被误当成正式产品版本。
- 本次 `.vsix` 已基于当前已验证仓库重新打包并本地安装验证。

对使用者可见的影响：

- GitHub Release 页面将只提供 `shelf-ai-0.1.3.vsix`
- 扩展安装方式不变
- 样例项目仍保留在仓库中，继续用于本地验证、演示和回归测试

安装或获取方式：

1. 下载发布资产 `shelf-ai-0.1.3.vsix`
2. 在 VSCode 中执行 `Extensions: Install from VSIX...`
3. 选择该文件完成安装

注意事项：

- 扩展依然要求仓库使用 `projects/*/project.toml` 作为唯一项目入口
- `knowledge_base_basic` 仍会继续物化和参与验证，但不再单独发布版本

## English Notes

- This release is for the `Shelf AI` VSCode extension only and no longer ships public release payloads for `knowledge_base_basic`.
- The repository now treats `knowledge_base_basic` explicitly as a local validation and demo sample used to exercise Shelf AI against a real `Framework -> Config -> Code -> Evidence` workflow, not as a standalone release target.
- The release policy has been tightened accordingly: GitHub Releases now publish only the Shelf AI bilingual notes and the installable `.vsix` asset so the sample project is no longer mistaken for a product release.
- The `.vsix` asset was rebuilt and locally reinstalled against the current validated workspace.

User-visible impact:

- The GitHub Release page will provide only `shelf-ai-0.1.3.vsix`
- The extension install flow is unchanged
- The sample project remains in the repository for local validation, demos, and regression coverage

Install / access:

1. Download the release asset `shelf-ai-0.1.3.vsix`
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the file to complete installation

Notes:

- The extension still expects `projects/*/project.toml` to remain the only project entrypoint
- `knowledge_base_basic` will continue to materialize and validate locally, but it will no longer be published as its own versioned release

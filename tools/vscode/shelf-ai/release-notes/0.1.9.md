## English Notes

- This release does not add new Shelf AI UI, tree, or navigation features; it republishes the extension against the current validated repository state while cleaning redundant boundary mirror config in `knowledge_base_basic`.
- `projects/knowledge_base_basic/project.toml` no longer repeats `boundary_id` / `mapping_mode`; boundary identity continues to come only from framework and canonical instead of being manually mirrored in project config.
- Compiled `communication_export` / `exact_export` payloads also stop carrying those mirror keys, so dead projection metadata no longer leaks into the Config -> Code pipeline.
- Canonical validation, tests, and local VSIX packaging/install verification were rerun; the GitHub Release asset will provide `shelf-ai-0.1.9.vsix`.

How to install:

1. Download the release asset `shelf-ai-0.1.9.vsix`
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the file and install it

Notes:

- This is a repository-config cleanup and repackaging release, not a new plugin-UI feature release.
- `projects/knowledge_base_basic` remains a local validation, demo, and regression sample only; it is not a standalone public release target.

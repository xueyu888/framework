## 中文说明

- 修复 GitHub 发布工作流的 VSIX 打包路径问题：发布任务现在会先创建 `releases/` 目录，再执行 `vsce package`，避免云端打包时因目录缺失而失败。
- 保留并继续发布这一轮的主改动：`implementation_config.toml` 的实现层字段必须进入编译后的 `ui_spec`、`backend_spec`、运行时选择链或生成产物命名，Shelf AI 会通过 strict mapping 验证链直接检查这一点。
- 知识库工作台继续沿用新的配置生效链：前端 renderer/style/script、后端 renderer/transport/retrieval 以及产品真相接口地址都已进入编译结果并被运行时代码显式消费。
- 本地安装脚本仍会清理历史 `mapping` / `archsync` 残留扩展，方便直接升级到当前 Shelf AI。

## English Notes

- Fix the GitHub release workflow for VSIX packaging: the publish job now creates the `releases/` directory before running `vsce package`, preventing cloud packaging failures caused by a missing output directory.
- This release keeps the main feature from the previous build attempt: effective `implementation_config.toml` fields must land in compiled `ui_spec`, `backend_spec`, runtime selection paths, or generated artifact names, and Shelf AI now enforces that through the strict-mapping validation chain.
- The knowledge-base workbench continues to use the new configuration-effect path: frontend renderer/style/script and backend renderer/transport/retrieval plus the product-spec endpoint are all compiled and explicitly consumed at runtime.
- The local install script still removes legacy `mapping` / `archsync` extension leftovers before installing the current Shelf AI build.

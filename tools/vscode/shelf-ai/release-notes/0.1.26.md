# 0.1.26

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.25..shelf-ai-v0.1.26`（包含该区间全部提交）。

- Shelf AI 新增了关键字优先的 `.sf` 实验语言预览，并把 grammar、补全、lint、语义高亮收敛到同一套共享规则；当前预览语法覆盖 `MODULE` 模块头、`set / elem / relation[...]`、`sat / eq`、`comb / seq`、`param<enum>` 与 `param<range>` 等单行声明形态。
- `.sf` 相关的扩展实现已经和这套新语法同步：实时诊断、上下文补全、语义 token 图例、示例文件与测试基线都切到了最新 authoring 口径，不再沿用早期的 clause 风格原型。
- 同一发布区间里，也修复了仓库侧阻塞发版的 `Chunk_Tagging` 合同漂移：模块级 `P15` 配置路径恢复到 canonical 期望的 `l0_m0 / l1_m1` scope，静态模块注册表能够识别根级 contract bundle，多模块 `ModuleContract` 与 `L3.M0` base/rule 映射也已对齐，因而 `validate_canonical.py --check-changes` 重新恢复通过。
- 发版前已重新完成 `Shelf AI` 扩展测试、webview 类型检查、仓库 `mypy`、硬编码守卫、`validate_canonical.py`、`validate_canonical.py --check-changes` 以及本地 VSIX 打包安装验证。

对使用者可见的影响：

- 如果你现在用 `.sf` 写 Shelf 的结构草案，编辑器已经能直接理解新的关键字优先写法，并提供一致的高亮、补全和 lint 反馈。
- 这仍然是实验语言预览，不进入 canonical / publish 主链；它的价值在于作者侧预览和讨论，不替代正式 framework Markdown。
- 本次版本还修复了发版前的仓库校验链路，因此公开发布的 VSIX 来自一个 `validate_canonical.py --check-changes` 已通过的工作区，而不是跳过门禁的临时产物。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.26.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.25..shelf-ai-v0.1.26`.

- Shelf AI now ships a keyword-first `.sf` experimental language preview, with grammar, completion, lint, and semantic highlighting driven by one shared rule set. The current preview syntax covers the `MODULE` header, `set / elem / relation[...]`, `sat / eq`, `comb / seq`, and `param<enum>` / `param<range>` single-line declarations.
- The extension-side `.sf` implementation has been aligned with that syntax: realtime diagnostics, completion context detection, semantic token legend, example files, and test baselines now follow the latest authoring shape instead of the earlier clause-based prototype.
- The same release range also fixes the repository-side `Chunk_Tagging` drift that blocked release validation: module-scoped `P15` config paths were restored to the canonical `l0_m0 / l1_m1` scope, the static-module registry now discovers root-level contract bundles, and the multi-module `ModuleContract` / `L3.M0` base-rule mapping has been realigned. As a result, `validate_canonical.py --check-changes` passes again before packaging and push.
- Before release, the workspace was revalidated with Shelf AI extension tests, webview type checks, repository `mypy`, the hardcode guard, `validate_canonical.py`, `validate_canonical.py --check-changes`, and local VSIX packaging/install verification.

User-visible impact:

- If you draft Shelf structures in `.sf`, the editor now understands the new keyword-first form directly and provides matching highlight, completion, and lint behavior.
- The `.sf` path is still an experimental authoring preview; it does not replace canonical / publish inputs for formal framework Markdown.
- This version also restores the repository validation chain before release, so the public VSIX is packaged from a workspace that already passes `validate_canonical.py --check-changes`.

How to install:

1. Download `shelf-ai-0.1.26.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

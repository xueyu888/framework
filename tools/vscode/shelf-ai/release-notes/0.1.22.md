# 0.1.22

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.21..shelf-ai-v0.1.22`（包含该区间全部提交）。

- 发布流程标准进一步收紧：`Publish Shelf AI` 工作流现在会在发版时强制检查双语 release notes、完整 tag 区间字符串，以及与版本号匹配的 `.vsix` 资产，避免再出现“有 tag 但说明范围不完整”的发布。
- correspondence / Problems 诊断改为默认中文输出，常见审计漂移原因也会在插件侧本地化，不再把英文内部原因直接泄露给作者。
- framework 作者体验继续增强：新增未定义符号 lint 与导航兜底，规则里引用未定义 `C* / P* / B*` 会直接报错；当正式对应对象不存在时，导航仍可退回到作者源局部体验。
- framework tree 完整转向作者源实时视图：状态栏支持“直接打开框架树 / 查看问题”的分叉动作；树图关系来自 `framework/**` 作者源；保存 framework 时会先尝试刷新 canonical 的 framework 快照并后台刷新树图，不再自动抢焦点弹窗。
- framework tree 视觉语义继续收敛：在本次区间的最终状态里，画布只保留模块级作者关系，`B* / R*` 作为模块内部信息留在 hover / inspector 中；若上层基定义引用下层模块，会折叠成模块到模块的边。
- 与上述链路配套，仓库补齐了一批 backend / message_queue framework 模块与 materialize/compiler 支撑逻辑，使框架树作者源投影与 framework-only canonical 快照具备可验证的真实输入。
- 发版前已在该 clean 基线重新完成 webview 构建、插件测试、硬编码守卫、`validate_canonical --check-changes` 与本地 VSIX 打包安装验证。

对使用者可见的影响：

- 公开发布页会更严格，下载到的版本说明和 `.vsix` 资产会与 tag 区间一致。
- framework 文档编辑时能更早发现未定义符号问题，诊断文本也更贴近中文作者语境。
- framework tree 现在更明确地服务“作者看模块关系”这一目标，而不是把基层细节直接铺满画布。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.22.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.21..shelf-ai-v0.1.22`.

- The release pipeline is stricter now: the `Publish Shelf AI` workflow validates bilingual release notes, the full previous-tag-to-current-tag range string, and a version-matching `.vsix` asset before a tagged release is accepted.
- Correspondence / Problems diagnostics now default to Chinese, and common audit-drift reasons are localized in the extension instead of exposing raw English runtime text to authors.
- Framework authoring diagnostics were extended with unresolved-symbol lint and navigation fallback: undefined `C* / P* / B*` references are reported directly, and navigation can still fall back to local author-source behavior when no formal object target exists.
- The framework tree has been fully moved to an author-source real-time view: the status bar now supports an explicit “open framework tree / show issues” split action; graph relations come from `framework/**`; and framework saves refresh canonical framework snapshots plus the tree in the background without auto-revealing the panel.
- The framework-tree visual model was further converged in the final state of this range: the canvas now keeps a module-level author graph only, while `B* / R*` stay inside hover/inspector details; upstream references declared inside bases are collapsed into module-to-module edges.
- To support that chain, the repository also synced a batch of backend / message_queue framework modules plus materialize/compiler support so the author-source tree projection and framework-only canonical snapshots run on real validated inputs.
- Before release, the clean baseline was rebuilt and revalidated again: webview build, extension tests, hardcode guard, `validate_canonical --check-changes`, and local VSIX packaging/install verification all passed.

User-visible impact:

- Public releases are now stricter and more traceable: the published notes and `.vsix` asset should match the tagged commit range consistently.
- Framework Markdown authoring surfaces undefined-symbol problems earlier, with diagnostics that fit the Chinese author workflow better.
- The framework tree is now more clearly optimized for “module-level author relationships” instead of filling the canvas with base-level detail.

How to install:

1. Download `shelf-ai-0.1.22.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

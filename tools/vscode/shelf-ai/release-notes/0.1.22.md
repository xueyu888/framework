# 0.1.22

## 中文说明

本说明覆盖提交区间：`shelf-ai-v0.1.21..shelf-ai-v0.1.22`（包含该区间全部提交）。

- framework tree 改为优先基于作者源渲染，并避免在保存时默认自动弹出干扰性提示；即使 canonical 暂时不 fresh，作者侧结构阅读仍可继续。
- 新增 framework tree 的作者源实时模式与状态栏分叉动作，明确区分“作者侧可读视图”和“canonical-backed 正式结果”。
- 新增 unresolved framework ref 校验与 undefined-symbol 导航 fallback，framework 作者侧导航在结构未完全闭合时也能提供更稳的退路。
- correspondence 相关诊断进一步完全中文化，避免在插件 Problems 与提示信息中混入英文残留。
- runtime 同步支持框架保存时优先刷新 canonical framework snapshot，减少树视图和导航面对旧快照时的滞后。

对使用者可见的影响：

- framework tree 在 canonical stale 场景下更可用，作者仍可继续查看作者源结构而不是被强制打断。
- framework 导航和诊断在“引用未闭合 / 符号未定义”场景下更稳、更清楚。
- 本版本主要提升作者源优先体验与 stale 场景下的工作连续性。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.22.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

This note covers the full commit range: `shelf-ai-v0.1.21..shelf-ai-v0.1.22`.

- The framework tree now renders from author sources first and avoids disruptive save-time popups when author-side structure can still be shown even if canonical is not fresh.
- Added author-source live mode plus status-bar branch actions for the framework tree, making the split between author-side readability and canonical-backed formal results explicit.
- Added unresolved framework-reference validation and undefined-symbol navigation fallback so author-side navigation remains usable even when structure is not fully closed.
- Further localized correspondence diagnostics fully into Chinese to remove leftover English fragments from Problems and status messages.
- Synced runtime support so framework saves refresh canonical framework snapshots earlier for tree and navigation consumers.

User-visible impact:

- The framework tree is now more usable in stale-canonical scenarios because author-source structure remains browseable.
- Framework navigation and diagnostics are more resilient around unresolved references and missing symbols.
- This release mainly improves author-source-first workflow continuity rather than adding a new runtime feature surface.

How to install:

1. Download `shelf-ai-0.1.22.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

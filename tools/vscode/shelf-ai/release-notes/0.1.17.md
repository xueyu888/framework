# 0.1.17

## 中文说明

- 本次版本移除了 `project_runtime` 中对 root 角色关系的硬编码检查，改为基于 framework upstream 依赖自动推导一致性约束。
- 新增 `exact.evidence.root_role_dependencies` 配置入口，可在项目配置中显式补充 root 角色依赖，无需再改运行时代码。
- 允许仅选择 `message_queue` root 的项目正常物化与校验，不再强制要求 `frontend + knowledge_base + backend` 三根 root 同时存在。
- Code 层导出保持“按已选模块注入”：只注入当前实际存在的 module export，避免错误跨模块注入导致 correspondence 偏差。

对使用者可见的影响：

- 新框架场景（例如队列-only 项目）可以直接走 canonical 流程，不会被历史硬编码角色约束阻断。
- 需要额外 root 依赖的项目可以通过配置声明，不再依赖代码分支特判。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.17.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- This release removes hard-coded root-role coupling checks from `project_runtime` and derives consistency constraints from framework upstream dependencies.
- Added configurable root-role dependency overrides at `exact.evidence.root_role_dependencies`, so projects can declare extra root coupling without runtime code patching.
- Queue-only projects (for example, `message_queue` root only) can now materialize and validate without forcing the `frontend + knowledge_base + backend` triple.
- Code-layer exports remain strictly selected-root-scoped: each module only injects exports that actually exist in the current runtime assembly.

User-visible impact:

- New framework scenarios (such as queue-only projects) can run through canonical workflows without being blocked by legacy hard-coded role checks.
- Projects that need additional root coupling can now declare it in config instead of relying on special-case code paths.

How to install:

1. Download `shelf-ai-0.1.17.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

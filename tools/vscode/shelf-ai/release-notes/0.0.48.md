## 中文说明

- 这一版是 Shelf AI 的打包与说明同步发布。插件自带 README 现在明确把 `projects/*/generated/canonical_graph.json` 定义为当前项目主链唯一机器真相源，同时把 `governance_manifest.json`、`governance_tree.json`、严格校验报告等重新表述为由 canonical graph 派生出的视图，而不是平行真相源。
- 这版也把导航模型写清楚了。打包后的说明现在明确区分两种阅读方式：主路径必须沿 `Framework -> Product Spec -> Implementation Config -> Code -> Evidence` 逐层跳转；治理树中的 framework 直跳 code 只是一种诊断捷径，不代表系统允许跳层读取。
- 这一版没有新增插件命令，也没有改变运行时行为；它的主要价值是让公开发布的 Shelf AI 安装包、README、CHANGELOG 和仓库当前主架构保持一致，减少用户按旧叙事理解当前仓库的风险。
- 安装方式：下载本版本附带的 `shelf-ai-0.0.48.vsix`，在 VS Code 中执行 `Extensions: Install from VSIX...` 安装。若升级前工作区或树图页面已经打开，安装后建议执行一次 `Developer: Reload Window`，确保新的帮助说明、版本信息和侧边栏入口描述全部刷新。

## English Notes

- This is a packaging-and-guidance sync release for Shelf AI. The packaged README now explicitly treats `projects/*/generated/canonical_graph.json` as the sole machine truth for the current project mainline, while governance manifests / trees / strict reports are described as views derived from that canonical graph instead of parallel truth sources.
- The release also clarifies the navigation model shipped with the extension. The packaged docs now distinguish the primary layer-by-layer path (`Framework -> Product Spec -> Implementation Config -> Code -> Evidence`) from the secondary governance shortcuts that jump directly from framework-side structure to code-side governed symbols for diagnosis only.
- No new extension commands or runtime behaviors are introduced in this release. Its main value is consistency: the published VSIX, README, changelog, and repository-side architecture narrative now describe the same current structure, reducing the risk of users following outdated guidance.
- Install by downloading the bundled `shelf-ai-0.0.48.vsix` and running `Extensions: Install from VSIX...` in VS Code. If the workspace or tree pages were already open before upgrading, run `Developer: Reload Window` afterward so the refreshed guidance, version metadata, and sidebar entry descriptions are fully visible.

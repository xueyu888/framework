## 中文说明

- 本次发布修复了 Framework Tree 在 canonical `stale` 场景下的可读性回退问题：即使 correspondence 暂时不可用，模块拓扑边（箭头）仍会保留，不再退化成“只有节点、没有连线”的作者视图。
- Git hooks 现在支持分支范围控制：默认仅在 `framework` 与 `main` 分支执行拦截校验，其它分支默认跳过，不再阻塞日常 feature 分支提交与推送。
- 仓库已补充对 `projects/knowledge_base_basic/generated/canonical.json` 的显式忽略，保证新环境中由插件/脚本自动再生的 canonical 文件不会误入版本控制。
- 已完成本地插件测试、仓库校验、VSIX 打包与安装验证，发布资产包含 `shelf-ai-0.1.10.vsix`。

使用方式：

1. 下载发布资产 `shelf-ai-0.1.10.vsix`
2. 在 VSCode 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

注意事项：

- 这是一次稳定性与工程流程发布，不引入新的插件 UI 面板或交互入口。
- 若需要自定义受保护分支，可在仓库中设置：`git config --local shelf.hookProtectedBranches "framework main ..."`。

## English Notes

- This release fixes framework-tree readability under `stale` canonical: module topology edges/arrows remain visible even when correspondence freshness is temporarily unavailable.
- Git hooks now support branch scoping: blocking checks run by default only on `framework` and `main`, while other branches are skipped to avoid unnecessary feature-branch friction.
- The repository now explicitly ignores `projects/knowledge_base_basic/generated/canonical.json`, so canonical files regenerated by plugin/scripts in fresh environments do not get tracked accidentally.
- Local plugin tests, repository validations, VSIX packaging, and install verification were rerun. The release asset includes `shelf-ai-0.1.10.vsix`.

How to install:

1. Download `shelf-ai-0.1.10.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the VSIX file and install

Notes:

- This is a stability/workflow release and does not add new plugin UI panels or interaction entry points.
- To customize protected branches for hooks, set: `git config --local shelf.hookProtectedBranches "framework main ..."` in your repository.

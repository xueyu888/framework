# 0.1.18

## 中文说明

- 修复了 runtime/evidence 注入链中的“按模块名分支”硬编码：模块导出与 evidence 导出改为统一的映射注入流程，不再依赖 `if module_id == ...` 特判。
- 移除了 framework 层对 `frontend/knowledge_base/backend` 的固定分支 overlay 推导，改为按 framework 名动态生成 overlay 路径，并在 config 层按“路径存在才注入”处理。
- 新增仓库级硬编码守卫测试 `tests/test_no_hardcode_guard.py`，提交前可自动拦截固定 root 角色查找、固定项目回退路径、固定框架分支等回退模式。
- 将“禁止写死门禁/依赖/默认项目路径”和“必须运行硬编码守卫测试”写入 `AGENTS.md`，防止后续改动再次引入同类问题。

对使用者可见的影响：

- Shelf AI 所依赖的 canonical/runtime 装配链路更稳定，新增或裁剪 framework 组合时不需要再改代码里的固定分支。
- 发布前可通过新增守卫测试快速发现“写死回退”，降低回归风险。

安装方式：

1. 下载发布资产 `shelf-ai-0.1.18.vsix`
2. 在 VS Code 中执行 `Extensions: Install from VSIX...`
3. 选择该文件并安装

## English Notes

- Removed hard-coded per-module branching in runtime/evidence export injection. Exports are now assigned through a unified mapping flow instead of `if module_id == ...` special cases.
- Replaced fixed `frontend/knowledge_base/backend` overlay branching in framework projection with framework-name-driven overlay paths, and made config-layer overlay loading presence-aware.
- Added repository-wide hardcode regression guard `tests/test_no_hardcode_guard.py` to block fallback patterns such as fixed root-role lookups, fixed project fallback paths, and fixed framework branches.
- Added explicit anti-hardcode rules to `AGENTS.md`, including a mandatory hardcode-guard test step before submission.

User-visible impact:

- The canonical/runtime assembly path used by Shelf AI is more robust when framework combinations change, without requiring new hardcoded branches.
- Release-time regression checks now catch hardcoded fallback patterns earlier.

How to install:

1. Download `shelf-ai-0.1.18.vsix` from release assets
2. Run `Extensions: Install from VSIX...` in VS Code
3. Select the VSIX file and install

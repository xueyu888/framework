# 可测节点配置模块:TestNodeSpec

@framework

## 0.目的
定义单个可测节点规约的最小稳定结构，使其能够被稳定配置、读取与解释。
本模块采用如下测试前提：

完整测试结构 = 
在某个前置状态 W0 下，给定一个输入激励 M，
系统产生应当产生的输出 O，
并把环境推进到应当到达的新状态 W1，
同时产生的副作用 Σ。

记为：δ : W × M -> O × W × Σ

## 1. 能力声明（Capability Statement）

- `C1` 行为配置说明能力：把依赖输入、动作与接口出口写成稳定配置。
- `C2` 环境配置说明能力：把环境读取、环境写入与执行前提写成稳定。
- `C3` 判定配置说明能力：把观测点、断言条件与结果出口写成稳定配置。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责节点集合装配、输入文件转换、调度编排与实际执行。

## 2. 边界定义（Boundary / 参数）

- `IDENTITY` 标识：节点标识需可唯一判定并保持稳定。来源：C1。
- `PRECONDITION` 前提：前提谓词类型、守卫强度、依赖顺序与成立窗口需可判定。来源：C2。
- `ACTION` 动作：动作片段类型、目标引用方式、参数绑定完整性与动作出口集合需可判定。来源：C3。
- `READS` 读取：可读取对象类别、读取粒度、读取时机与可见域范围需可配置且可校验。来源：C2。
- `WRITES` 写入：目标状态变更域、允许副作用域、禁止副作用域与影响归属需可判定。来源：C5。
- `INPUT` 输入：输入项集合、来源类型、基数、必填性与默认绑定规则需可配置且可校验。来源：C3。
- `OUTPUT` 输出：
- `ACTION` 动作：动作片段类型、目标引用方式、参数绑定完整性与动作出口集合需可判定。来源：C3。
- `OBSERVATION` 观测：观测对象、采集字段、采集时机、观测窗口与结果提取规则需可判定。来源：C4。
- `ASSERTION` 断言：输出判定条件、后状态判定条件、失败条件与容差窗口需可判定。来源：C5。

## 4. 基组合原则（Base Combination Principles）

- `R1` 声明接口闭合组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定身份声明与端口槽位，再形成可被独立配置和引用的节点规约外壳。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`DECL + SLOT`。
- `R2` 挂接过程闭合组合
  - `R2.1` 参与基：`B2 + B3 + B4`。
  - `R2.2` 组合方式：将端口槽位、上下文挂接与过程体配置闭合，形成可被解释的节点规约主体。
  - `R2.3` 输出能力：`C2 + C3 + C4`。
  - `R2.4` 边界绑定：`SLOT + SCOPE + BODY`。
- `R3` 规约读取闭合组合
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R3.2` 组合方式：在完整节点规约上叠加读取投影，形成可被稳定读取与解释的规约结构。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`DECL + SLOT + SCOPE + BODY`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：缺失身份锚点、槽位方向不可判定、挂接作用域不明确、过程出口不可解释或读取投影与规约主体不一致的组合均无效。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`DECL + SLOT + SCOPE + BODY`。

## 5. 验证（Verification）

- `V1` 身份声明完整性：节点标识、类别与版本锚点必须满足 `DECL` 约束。
- `V2` 槽位完整性：输入槽位、输出槽位与观测槽位必须满足 `SLOT` 约束。
- `V3` 挂接清晰性：读取域、写入域与依赖可见域必须满足 `SCOPE` 约束。
- `V4` 过程可判定性：过程片段、参数绑定与终止出口必须满足 `BODY` 约束。
- `V5` 读取一致性：规约读取投影必须与身份、槽位、挂接与过程体配置保持一致。
- `V6` 最小必要性：移除任一 `B*` 后，`V1~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论。

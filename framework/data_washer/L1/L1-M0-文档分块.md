# 文档分块模块:DocChunk

@framework

## 1. 能力声明（Capability Statement）

- `C1` 资格识别能力：提供对已完成归一化文档执行清洗资格识别的能力。
- `C2` 分块结果构造能力：提供按输入分块标准执行分块，并按结果格式要求构造稳定块结果对象、填写追踪信息、原文映射与属性扩展接口的能力。
- `C3` 结果输出能力：提供按稳定输出格式生成新的结果文件并完成下游可消费交付的能力。
- `C4` 非能力项：不负责 `Tag` 标注、数据提取、结果对比与参数调整。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文档边界：`data_washer` 仅消费已完成归一化并已入库的文档。来源：`C1 + C2`。
- `QUALIFIED` 资格结论边界：每个归一文档在分块前必须被识别为具备清洗资格或不具备清洗资格。来源：`C1 + C2`。
- `CHUNK_SPEC` 分块标准边界：分块必须依据输入的分块标准执行。来源：`C2`。
- `FORMAT_SPEC` 结果格式边界：分块结果必须依据输入的结果格式要求构造。来源：`C2 + C3`。
- `B` 分块数量边界：输出内容块数量必须满足 `B > 1`。来源：`C2`。
- `ORDER` 顺序边界：内容块与结果对象顺序必须与原文档结构界标顺序一致。来源：`C1 + C2 + C3`。
- `COMPLETE` 完整性边界：每个内容块必须完整覆盖其所属结构区间内的正文内容，不得跨块丢失或重复。来源：`C2`。
- `TRACE` 追踪边界：每个结果对象都必须绑定稳定的 `block_id`。来源：`C2`。
- `SOURCE_MAP` 原文映射边界：每个结果对象都必须表达与原文标题、原文位置或原文范围的对应关系。来源：`C2`。
- `ATTR_EXT` 属性扩展边界：每个结果对象都必须预留可扩展属性接口或属性容器，但不在本层枚举具体属性字段。来源：`C2`。
- `RESULT_MODEL` 结果对象边界：分块结果在输出前必须被表达为稳定的结果对象集合。来源：`C2 + C3`。
- `OUTPUT_FORMAT` 输出格式边界：结果对象的输出格式、字段组织与状态表达必须明确。来源：`C3`。
- `ARTIFACT` 产物文件边界：分块流程必须生成独立于原文档的新文件用于存放结果，且该文件必须可定位、可读取、可替换。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 资格识别结构基：向下引用 `L2.M0[B1,B2,B3]` 作为展开。来源：`C1 + NORM_DOC + QUALIFIED`。
- `B2` 分块结果构造结构基：向下引用 `L2.M1[B1,B2,B3]` 作为展开。来源：`C2 + NORM_DOC + QUALIFIED + CHUNK_SPEC + FORMAT_SPEC + B + ORDER + COMPLETE + TRACE + SOURCE_MAP + ATTR_EXT + RESULT_MODEL`。
- `B3` 结果输出结构基：向下引用 `L2.M2[B1,B2,B3]` 作为展开。来源：`C3 + RESULT_MODEL + OUTPUT_FORMAT + ARTIFACT + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 输入执行组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由资格识别结构对已完成归一化的文档执行资格识别，再由分块结果构造结构基依据输入分块标准执行分块，并按结果格式要求同步补齐 `block_id`、原文映射与属性扩展接口；若分块标准采用标题区间规则，则当前标题 `A` 到下一标题 `B` 前形成一个块，末尾标题以文档结束位置闭合。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/FORMAT_SPEC/B/ORDER/COMPLETE/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL`。
- `R2` 构造输出组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：结果输出结构只能消费分块结果构造结构产出的稳定结果对象集合，并按统一输出格式写入新的结果文件；输出结构不得回头补写 `block_id`、原文映射或属性扩展接口。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`RESULT_MODEL/OUTPUT_FORMAT/ARTIFACT/ORDER`。
- `R3` 完整分块落地组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将资格识别、分块结果构造与结果输出收敛为完整分块落地管线，输出可供下游直接消费的结果文件。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/FORMAT_SPEC/B/ORDER/COMPLETE/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/OUTPUT_FORMAT/ARTIFACT`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：绕过 `B1` 直接让 `B2` 消费尚未完成资格识别的归一文档、绕过 `B2` 直接让 `B3` 消费归一文档或半成品结果、以及 `B3` 反向改写 `B2` 已确定的结果对象字段、追踪信息、原文映射或属性扩展接口的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/FORMAT_SPEC/B/ORDER/COMPLETE/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/OUTPUT_FORMAT/ARTIFACT`。

## 5. 验证（Verification）

- `V1` 资格识别有效性：资格识别结构必须满足 `NORM_DOC/QUALIFIED` 约束。
- `V2` 结果构造合法性：分块结果构造结构必须满足 `CHUNK_SPEC/FORMAT_SPEC/B/ORDER/COMPLETE/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL` 约束。
- `V3` 结果输出一致性：结果输出结构必须满足 `OUTPUT_FORMAT/ARTIFACT/ORDER` 约束。
- `V4` 管线完整性：完整分块落地管线必须满足从资格识别到结果输出的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

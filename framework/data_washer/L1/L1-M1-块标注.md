# 块标注模块:BlockTag

@framework

## 1. 能力声明（Capability Statement）

- `C1` 块记录接入能力：提供对上游分块结果中的稳定块记录集合执行接入与就绪校验的能力。
- `C2` Tag判定能力：提供依据标签集合与标注规则为块记录判定适用 `Tag` 的能力。
- `C3` 标注关系组织能力：提供将块标识与 `Tag` 绑定为稳定标注结果集合的能力。
- `C4` 非能力项：不负责文档分块、数据提取、结果对比与参数调整。

## 2. 边界定义（Boundary / 参数）

- `BLOCK_SET` 块记录集合边界：仅消费来自上游分块结果的稳定块记录集合。来源：`C1 + C2`。
- `TRACE` 追踪边界：每个块记录都必须携带稳定的 `block_id`。来源：`C1 + C3`。
- `TAG_SET` 标签集合边界：可用标签集合、标签标识与标签语义必须明确。来源：`C2 + C3`。
- `TAG_RULE` 标注规则边界：块内容到标签集合的匹配条件、优先级与互斥关系必须明确。来源：`C2`。
- `BT_SET` 标注关系集合边界：标注结果必须被表达为稳定的块标识与标签标识绑定集合。来源：`C3`。
- `TN` 标签数量边界：参与当前标注流程的标签数量必须满足 `TN >= 1`。来源：`C2 + C3`。
- `TT` 标签种类边界：参与当前标注流程的标签种类必须满足 `TT >= 1`。来源：`C2 + C3`。
- `ORDER` 顺序边界：块记录顺序、标签判定顺序与标注输出顺序必须保持一致。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 块记录接入结构基：`input_access.L0.M0[B1,B2,B3]`。来源：`C1 + BLOCK_SET + TRACE + ORDER`。
- `B2` Tag判定结构基。来源：`C2 + BLOCK_SET + TAG_SET + TAG_RULE + TN + TT + ORDER`。
- `B3` 标注关系结构基。来源：`C3 + TRACE + TAG_SET + BT_SET + TN + TT + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 输入判定组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由块记录接入结构基确认上游块记录集合与 `block_id` 已稳定可消费，再由 `Tag` 判定结构基依据标签集合与标注规则为每个块记录判定适用标签。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`BLOCK_SET/TRACE/TAG_SET/TAG_RULE/TN/TT/ORDER`。
- `R2` 判定绑定组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：`Tag` 判定结构基产出的标签结果必须经过标注关系结构基处理后，才能形成稳定的块标识与标签标识绑定集合；未绑定 `block_id` 的标签结果不得输出。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`TRACE/TAG_SET/BT_SET/TN/TT/ORDER`。
- `R3` 完整块标注组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将块记录接入、`Tag` 判定与标注关系组织收敛为完整块标注闭环，输出稳定标注结果集合供后续提取消费。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`BLOCK_SET/TRACE/TAG_SET/TAG_RULE/BT_SET/TN/TT/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造块记录、`B3` 绕过 `B2` 直接自造标签绑定结果、以及 `B3` 反向改写 `B1/B2` 已确定的块标识、标签语义或顺序关系的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`BLOCK_SET/TRACE/TAG_SET/TAG_RULE/BT_SET/TN/TT/ORDER`。

## 5. 验证（Verification）

- `V1` 块记录接入有效性：块记录接入结构必须满足 `BLOCK_SET/TRACE/ORDER` 约束。
- `V2` Tag判定一致性：`Tag` 判定结构必须满足 `BLOCK_SET/TAG_SET/TAG_RULE/TN/TT/ORDER` 约束。
- `V3` 标注绑定一致性：标注关系结构必须满足 `TRACE/TAG_SET/BT_SET/TN/TT/ORDER` 约束。
- `V4` 标注闭环完整性：完整块标注流程必须满足从块记录接入到标注关系输出的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

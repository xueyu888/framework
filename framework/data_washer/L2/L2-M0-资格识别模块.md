# 资格识别模块:Qualify

@framework

## 1. 能力声明（Capability Statement）

- `C1` 属性读取能力：提供对归一文档变量按指定读取目标执行属性读取的能力。
- `C2` 简单对比能力：提供将读取结果与对比目标执行简单对比并得出资格结论的能力。
- `C3` 资格结果封装能力：提供对资格识别结果执行稳定封装的能力。
- `C4` 非能力项：不负责分块规则执行与分块结果文件产出。

## 2. 边界定义（Boundary / 参数）

- `DOC_VAR` 文档变量边界：输入变量必须稳定指向已完成归一化并已入库的文档对象。来源：`C1`。
- `READ_TARGET` 读取目标边界：读取时指定的属性名、字段名、路径或内容目标必须明确。来源：`C1 + C2`。
- `READ_RESULT` 读取结果边界：属性读取后的结果必须稳定、可比较。来源：`C1 + C2`。
- `COMPARE_TARGET` 对比目标边界：参与简单对比的目标值、目标身份或目标规则必须明确。来源：`C2 + C3`。
- `QUALIFIED` 资格结论边界：每个归一文档必须被判定为具备清洗资格或不具备清洗资格。来源：`C2 + C3`。
- `QUAL_RESULT` 资格结果边界：资格识别结果必须被封装为稳定结果对象，且其状态表达与下游消费语义必须明确。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 属性读取结构基。来源：`C1 + DOC_VAR + READ_TARGET + READ_RESULT`。
- `B2` 简单对比结构基。来源：`C2 + READ_RESULT + COMPARE_TARGET + QUALIFIED`。
- `B3` 资格结果封装结构基。来源：`C3 + QUALIFIED + QUAL_RESULT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 读取对比组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由属性读取结构基基于文档变量与读取目标读取属性结果，再由简单对比结构基将该读取结果与对比目标进行简单对比并形成资格结论。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`DOC_VAR/READ_TARGET/READ_RESULT/COMPARE_TARGET/QUALIFIED`。
- `R2` 对比封装组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：简单对比结构基产出的资格识别结果必须经过资格结果封装结构基处理后，才能形成稳定资格结果对象并输出给下游分块结果构造模块消费。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`QUALIFIED/QUAL_RESULT`。
- `R3` 完整资格识别组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将属性读取、简单对比与资格结果封装收敛为完整资格识别闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`DOC_VAR/READ_TARGET/READ_RESULT/COMPARE_TARGET/QUALIFIED/QUAL_RESULT`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过 `B1` 直接让 `B2` 自造读取结果、跳过 `B2` 直接让 `B3` 自造资格识别结果、以及 `B3` 反向改写 `B1/B2` 已确定的读取结果或资格结论的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`DOC_VAR/READ_TARGET/READ_RESULT/COMPARE_TARGET/QUALIFIED/QUAL_RESULT`。

## 5. 验证（Verification）

- `V1` 属性读取有效性：属性读取结构必须满足 `DOC_VAR/READ_TARGET/READ_RESULT` 约束。
- `V2` 简单对比一致性：简单对比结构必须满足 `READ_RESULT/COMPARE_TARGET/QUALIFIED` 约束。
- `V3` 资格结果封装一致性：资格结果封装结构必须满足 `QUALIFIED/QUAL_RESULT` 约束。
- `V4` 识别闭环完整性：完整资格识别流程必须满足从属性读取到资格结果封装的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

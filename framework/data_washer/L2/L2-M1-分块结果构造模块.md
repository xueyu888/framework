# 分块结果构造模块:ChunkBuild

@framework

## 1. 能力声明（Capability Statement）

- `C1` 分块标准组织能力：提供对输入分块标准建立稳定结构表达的能力。
- `C2` 分块执行能力：提供按分块标准对归一文档执行分块的能力。
- `C3` 结果格式化能力：提供按结果格式要求构造稳定分块结果模型的能力。
- `C5` 非能力项：不负责资格识别与结果文件输出。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文档边界：仅消费来自上游并已完成归一化的文档。来源：`C2`。
- `QUALIFIED` 资格结论边界：仅消费已通过资格识别的归一文档。来源：`C2`。
- `CHUNK_SPEC` 分块标准边界：分块标准中的切分规则、边界条件与顺序语义必须明确。来源：`C1 + C2`。
- `CHUNK_SET` 分块集合边界：分块执行后的块集合必须稳定、可枚举。来源：`C2 + C3`。
- `FORMAT_SPEC` 结果格式边界：分块结果的字段组织、必填项与格式要求必须明确。来源：`C3`。
- `ORDER` 顺序边界：块顺序必须与原文顺序及分块标准定义保持一致。来源：`C1 + C2 + C3`。
- `B` 分块数量边界：输出内容块数量必须满足 `B > 1`。来源：`C2 + C3`。
- `COMPLETE` 完整性边界：每个内容块必须完整覆盖其区间范围内的正文内容。来源：`C3`。
- `TRACE` 追踪边界：每个结果对象都必须具备稳定的 `block_id`。来源：`C3`。
- `SOURCE_MAP` 原文映射边界：每个结果对象都必须表达与原文标题、原文位置或原文范围的对应关系。来源：`C3`。
- `ATTR_EXT` 属性扩展边界：每个结果对象都必须预留可扩展属性接口或属性容器。来源：`C3`。
- `RESULT_MODEL` 结果对象边界：分块构造后的结果必须被表达为稳定结果对象集合。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 分块规范结构基：向下引用 `L3.M0[B1,B2,B3]` 作为展开。来源：`C1 + CHUNK_SPEC + ORDER`。
- `B2` 分块执行器结构基：向下引用 `L3.M1[B1,B2,B3]` 作为展开。来源：`C2 + NORM_DOC + QUALIFIED + CHUNK_SPEC + CHUNK_SET + ORDER + B + COMPLETE`。
- `B3` 结果格式器结构基：向下引用 `L3.M2[B1,B2,B3]` 作为展开。来源：`C3 + CHUNK_SET + FORMAT_SPEC + TRACE + SOURCE_MAP + ATTR_EXT + RESULT_MODEL + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 规范执行组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由分块规范结构基建立稳定分块标准，再由分块执行器结构基依据该标准对归一文档执行分块；若标准采用标题区间规则，则当前标题 `A` 到下一标题 `B` 前形成一个块，末尾标题以文档结束位置闭合。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/CHUNK_SET/ORDER/B/COMPLETE`。
- `R2` 执行格式化组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：分块执行器结构基产出的分块集合必须经过结果格式器结构基处理后，才能形成稳定结果模型；格式化阶段必须补齐 `block_id`、原文映射与属性扩展接口，并满足结果格式要求。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`CHUNK_SET/FORMAT_SPEC/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/ORDER`。
- `R3` 完整结果构造组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将分块规范、分块执行器与结果格式器收敛为完整分块结果构造闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/CHUNK_SET/FORMAT_SPEC/ORDER/B/COMPLETE/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造分块标准、`B3` 绕过 `B2` 直接消费归一文档或半成品结果、以及 `B3` 反向改写 `B1/B2` 已确定的分块标准、块顺序或块主内容的组合均无效。
  - `R4.3` 输出能力：`C5`。
  - `R4.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/CHUNK_SET/FORMAT_SPEC/ORDER/B/COMPLETE/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL`。

## 5. 验证（Verification）

- `V1` 分块规范一致性：分块规范结构必须满足 `CHUNK_SPEC/ORDER` 约束。
- `V2` 分块执行一致性：分块执行器结构必须满足 `NORM_DOC/QUALIFIED/CHUNK_SPEC/CHUNK_SET/B/COMPLETE` 约束。
- `V3` 结果格式化一致性：结果格式器结构必须满足 `CHUNK_SET/FORMAT_SPEC/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL` 约束。
- `V4` 规则执行闭环完整性：完整结果构造流程必须满足从分块规范到结果格式器的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

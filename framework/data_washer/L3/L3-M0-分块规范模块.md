# 分块规范模块:ChunkSpec

@framework

## 1. 能力声明（Capability Statement）

- `C1` 规则集组织能力：提供对分块规则集合建立稳定结构表达的能力。
- `C2` 边界策略组织能力：提供对分块边界策略建立稳定结构表达的能力。
- `C3` 规范模型组织能力：提供将规则集与边界策略组装为稳定分块规范模型的能力。
- `C4` 非能力项：不负责文档扫描、规则匹配与结果格式化。

## 2. 边界定义（Boundary / 参数）

- `RULE_SET` 规则集边界：分块规则项、规则顺序与规则作用范围必须明确。来源：`C1 + C2`。
- `BOUNDARY_POLICY` 边界策略边界：块起止、块闭合与跨块处理策略必须明确。来源：`C2 + C3`。
- `CHUNK_SPEC` 分块规范边界：规则集与边界策略形成的分块规范模型必须稳定、可消费。来源：`C3`。
- `ORDER` 顺序边界：规则顺序与策略绑定顺序必须保持一致。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 规则集结构基。来源：`C1 + RULE_SET + ORDER`。
- `B2` 边界策略结构基。来源：`C2 + RULE_SET + BOUNDARY_POLICY + ORDER`。
- `B3` 规范模型结构基。来源：`C3 + BOUNDARY_POLICY + CHUNK_SPEC + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 规则策略组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由规则集结构基建立稳定规则集合，再由边界策略结构基将块起止、闭合与跨块处理策略绑定到规则集合。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`RULE_SET/BOUNDARY_POLICY/ORDER`。
- `R2` 策略建模组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：边界策略结构基产出的规则策略组合必须经过规范模型结构基处理后，才能形成稳定分块规范模型。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`BOUNDARY_POLICY/CHUNK_SPEC/ORDER`。
- `R3` 完整分块规范组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将规则集、边界策略与规范模型收敛为完整分块规范闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`RULE_SET/BOUNDARY_POLICY/CHUNK_SPEC/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造规则集、`B3` 绕过 `B2` 直接自造分块规范、以及 `B3` 反向改写 `B1/B2` 已确定的规则项、策略语义或顺序绑定的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`RULE_SET/BOUNDARY_POLICY/CHUNK_SPEC/ORDER`。

## 5. 验证（Verification）

- `V1` 规则集一致性：规则集结构必须满足 `RULE_SET/ORDER` 约束。
- `V2` 边界策略一致性：边界策略结构必须满足 `RULE_SET/BOUNDARY_POLICY/ORDER` 约束。
- `V3` 规范模型一致性：规范模型结构必须满足 `BOUNDARY_POLICY/CHUNK_SPEC/ORDER` 约束。
- `V4` 规范闭环完整性：完整分块规范流程必须满足从规则集到规范模型的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 结果格式器模块:ChunkFormat

@framework

## 1. 能力声明（Capability Statement）

- `C1` 追踪绑定能力：提供对分块集合绑定稳定追踪标识的能力。
- `C2` 映射绑定能力：提供对分块集合绑定原文映射与属性扩展信息的能力。
- `C3` 结果模型组织能力：提供按结果格式要求组织稳定结果模型的能力。
- `C4` 非能力项：不负责分块规范建模与分块执行。

## 2. 边界定义（Boundary / 参数）

- `CHUNK_SET` 分块集合边界：仅消费稳定、可枚举的分块集合。来源：`C1 + C2`。
- `FORMAT_SPEC` 结果格式边界：结果模型的字段组织、必填项与格式要求必须明确。来源：`C2 + C3`。
- `TRACE` 追踪边界：每个结果对象都必须具备稳定的 `block_id`。来源：`C1 + C3`。
- `SOURCE_MAP` 原文映射边界：每个结果对象都必须表达与原文标题、原文位置或原文范围的对应关系。来源：`C2 + C3`。
- `ATTR_EXT` 属性扩展边界：每个结果对象都必须预留可扩展属性接口或属性容器。来源：`C2 + C3`。
- `RESULT_MODEL` 结果对象边界：格式化后的结果必须被表达为稳定结果对象集合。来源：`C3`。
- `ORDER` 顺序边界：结果对象顺序必须与分块集合顺序一致。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 追踪绑定器结构基。来源：`C1 + CHUNK_SET + TRACE + ORDER`。
- `B2` 映射绑定器结构基。来源：`C2 + CHUNK_SET + TRACE + SOURCE_MAP + ATTR_EXT + ORDER`。
- `B3` 结果模型结构基。来源：`C3 + CHUNK_SET + FORMAT_SPEC + TRACE + SOURCE_MAP + ATTR_EXT + RESULT_MODEL + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 追踪映射组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由追踪绑定器结构基为分块集合绑定稳定追踪标识，再由映射绑定器结构基补齐原文映射与属性扩展信息。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`CHUNK_SET/TRACE/SOURCE_MAP/ATTR_EXT/ORDER`。
- `R2` 映射建模组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：映射绑定器结构基产出的结果必须经过结果模型结构基处理后，才能形成满足格式要求的稳定结果模型。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`CHUNK_SET/FORMAT_SPEC/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/ORDER`。
- `R3` 完整结果格式化组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将追踪绑定、映射绑定与结果模型组织收敛为完整结果格式化闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CHUNK_SET/FORMAT_SPEC/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造追踪标识、`B3` 绕过 `B2` 直接自造原文映射或属性扩展信息、以及 `B3` 反向改写 `B1/B2` 已确定的追踪标识、原文映射或属性容器的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`CHUNK_SET/FORMAT_SPEC/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/ORDER`。

## 5. 验证（Verification）

- `V1` 追踪绑定一致性：追踪绑定器结构必须满足 `CHUNK_SET/TRACE/ORDER` 约束。
- `V2` 映射绑定一致性：映射绑定器结构必须满足 `CHUNK_SET/TRACE/SOURCE_MAP/ATTR_EXT/ORDER` 约束。
- `V3` 结果模型一致性：结果模型结构必须满足 `CHUNK_SET/FORMAT_SPEC/TRACE/SOURCE_MAP/ATTR_EXT/RESULT_MODEL/ORDER` 约束。
- `V4` 格式化闭环完整性：完整结果格式化流程必须满足从追踪绑定到结果模型的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 分块执行器模块:ChunkExec

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文档扫描能力：提供对归一文档执行顺序扫描的能力。
- `C2` 规则匹配能力：提供依据分块规范将扫描结果执行规则匹配的能力。
- `C3` 分块组装能力：提供依据匹配结果组装稳定分块集合的能力。
- `C4` 非能力项：不负责分块规范建模与结果格式化。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文档边界：仅消费已完成归一化的文档。来源：`C1`。
- `QUALIFIED` 资格结论边界：仅消费已通过资格识别的归一文档。来源：`C1`。
- `CHUNK_SPEC` 分块规范边界：分块执行必须依据稳定分块规范进行。来源：`C2`。
- `SCAN_SET` 扫描单元边界：扫描得到的单元集合必须稳定、可枚举。来源：`C1 + C2`。
- `MATCH_SET` 匹配结果边界：规则匹配结果必须稳定、可消费。来源：`C2 + C3`。
- `CHUNK_SET` 分块集合边界：组装后的分块集合必须稳定、可枚举。来源：`C3`。
- `ORDER` 顺序边界：扫描顺序、匹配顺序与块顺序必须保持一致。来源：`C1 + C2 + C3`。
- `B` 分块数量边界：输出内容块数量必须满足 `B > 1`。来源：`C3`。
- `COMPLETE` 完整性边界：每个内容块必须完整覆盖其区间范围内的正文内容。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文档扫描器结构基：向下引用 `L4.M0[B1,B2,B3]` 作为展开。来源：`C1 + NORM_DOC + QUALIFIED + SCAN_SET + ORDER`。
- `B2` 规则匹配器结构基：向下引用 `L4.M1[B1,B2,B3]` 作为展开。来源：`C2 + SCAN_SET + CHUNK_SPEC + MATCH_SET + ORDER`。
- `B3` 分块器结构基：向下引用 `L4.M2[B1,B2,B3]` 作为展开。来源：`C3 + MATCH_SET + CHUNK_SET + B + COMPLETE + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 扫描匹配组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由文档扫描器结构基按原文顺序产出扫描单元，再由规则匹配器结构基依据分块规范对扫描单元执行规则匹配。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`NORM_DOC/QUALIFIED/SCAN_SET/CHUNK_SPEC/MATCH_SET/ORDER`。
- `R2` 匹配分块组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：规则匹配器结构基产出的匹配结果必须经过分块器结构基处理后，才能形成稳定分块集合。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`MATCH_SET/CHUNK_SET/B/COMPLETE/ORDER`。
- `R3` 完整分块执行组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将文档扫描、规则匹配与分块组装收敛为完整分块执行闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/SCAN_SET/MATCH_SET/CHUNK_SET/ORDER/B/COMPLETE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造扫描单元、`B3` 绕过 `B2` 直接自造匹配结果、以及 `B3` 反向改写 `B1/B2` 已确定的扫描顺序、匹配结果或块边界的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/SCAN_SET/MATCH_SET/CHUNK_SET/ORDER/B/COMPLETE`。

## 5. 验证（Verification）

- `V1` 文档扫描一致性：文档扫描器结构必须满足 `NORM_DOC/QUALIFIED/SCAN_SET/ORDER` 约束。
- `V2` 规则匹配一致性：规则匹配器结构必须满足 `SCAN_SET/CHUNK_SPEC/MATCH_SET/ORDER` 约束。
- `V3` 分块组装一致性：分块器结构必须满足 `MATCH_SET/CHUNK_SET/B/COMPLETE/ORDER` 约束。
- `V4` 执行闭环完整性：完整分块执行流程必须满足从文档扫描到分块组装的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

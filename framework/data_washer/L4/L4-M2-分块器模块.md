# 分块器模块:ChunkAssemble

@framework

## 1. 能力声明（Capability Statement）

- `C1` 起块控制能力：提供依据匹配结果确定块起始位置的能力。
- `C2` 闭块控制能力：提供依据匹配结果与起块状态确定块结束位置的能力。
- `C3` 块集合组织能力：提供将起止边界组织为稳定分块集合的能力。
- `C4` 非能力项：不负责文档扫描、规则匹配与结果格式化。

## 2. 边界定义（Boundary / 参数）

- `MATCH_SET` 匹配结果边界：仅消费稳定、可枚举的匹配结果集合。来源：`C1 + C2`。
- `BLOCK_OPEN` 起块边界：每个块的起始位置、起始状态与顺序必须明确。来源：`C1 + C2`。
- `BLOCK_CLOSE` 闭块边界：每个块的结束位置、闭合状态与顺序必须明确。来源：`C2 + C3`。
- `CHUNK_SET` 分块集合边界：块集合必须稳定、可枚举。来源：`C3`。
- `ORDER` 顺序边界：起块顺序、闭块顺序与块集合顺序必须保持一致。来源：`C1 + C2 + C3`。
- `B` 分块数量边界：输出内容块数量必须满足 `B > 1`。来源：`C3`。
- `COMPLETE` 完整性边界：每个内容块必须完整覆盖其区间范围内的正文内容。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 起块结构基。来源：`C1 + MATCH_SET + BLOCK_OPEN + ORDER`。
- `B2` 闭块结构基。来源：`C2 + BLOCK_OPEN + BLOCK_CLOSE + ORDER`。
- `B3` 块集合结构基。来源：`C3 + BLOCK_CLOSE + CHUNK_SET + B + COMPLETE + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 起块闭块组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由起块结构基依据匹配结果确定每个块的起始位置，再由闭块结构基依据起块状态确定每个块的结束位置。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`MATCH_SET/BLOCK_OPEN/BLOCK_CLOSE/ORDER`。
- `R2` 闭块建集组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：闭块结构基产出的块边界必须经过块集合结构基处理后，才能形成稳定分块集合。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`BLOCK_CLOSE/CHUNK_SET/B/COMPLETE/ORDER`。
- `R3` 完整分块组装组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将起块控制、闭块控制与块集合组织收敛为完整分块组装闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`MATCH_SET/BLOCK_OPEN/BLOCK_CLOSE/CHUNK_SET/B/COMPLETE/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造起块位置、`B3` 绕过 `B2` 直接自造块结束位置、以及 `B3` 反向改写 `B1/B2` 已确定的起止边界或块顺序的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`MATCH_SET/BLOCK_OPEN/BLOCK_CLOSE/CHUNK_SET/B/COMPLETE/ORDER`。

## 5. 验证（Verification）

- `V1` 起块一致性：起块结构必须满足 `MATCH_SET/BLOCK_OPEN/ORDER` 约束。
- `V2` 闭块一致性：闭块结构必须满足 `BLOCK_OPEN/BLOCK_CLOSE/ORDER` 约束。
- `V3` 块集合一致性：块集合结构必须满足 `BLOCK_CLOSE/CHUNK_SET/B/COMPLETE/ORDER` 约束。
- `V4` 组装闭环完整性：完整分块组装流程必须满足从起块控制到块集合组织的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

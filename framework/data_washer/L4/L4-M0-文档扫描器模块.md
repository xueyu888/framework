# 文档扫描器模块:DocScan

@framework

## 1. 能力声明（Capability Statement）

- `C1` 扫描游标控制能力：提供对归一文档建立稳定扫描位置与前进顺序的能力。
- `C2` 扫描单元抽取能力：提供基于扫描位置抽取稳定扫描单元的能力。
- `C3` 扫描集组织能力：提供将扫描单元组织为稳定扫描集合的能力。
- `C4` 非能力项：不负责规则匹配、分块组装与结果格式化。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文档边界：仅消费已完成归一化的文档。来源：`C1`。
- `QUALIFIED` 资格结论边界：仅消费已通过资格识别的归一文档。来源：`C1`。
- `CURSOR` 扫描游标边界：扫描位置、前进方向与终止条件必须明确。来源：`C1 + C2`。
- `SCAN_UNIT` 扫描单元边界：每个扫描单元的内容范围与结构表达必须明确。来源：`C2 + C3`。
- `SCAN_SET` 扫描集合边界：扫描单元集合必须稳定、可枚举。来源：`C3`。
- `ORDER` 顺序边界：游标顺序、单元顺序与集合顺序必须保持一致。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 扫描游标结构基。来源：`C1 + NORM_DOC + QUALIFIED + CURSOR + ORDER`。
- `B2` 单元抽取结构基。来源：`C2 + CURSOR + SCAN_UNIT + ORDER`。
- `B3` 扫描集结构基。来源：`C3 + SCAN_UNIT + SCAN_SET + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 游标抽取组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由扫描游标结构基确定稳定扫描位置与前进顺序，再由单元抽取结构基依据游标位置抽取扫描单元。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`NORM_DOC/QUALIFIED/CURSOR/SCAN_UNIT/ORDER`。
- `R2` 抽取建集组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：单元抽取结构基产出的扫描单元必须经过扫描集结构基处理后，才能形成稳定扫描集合。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`SCAN_UNIT/SCAN_SET/ORDER`。
- `R3` 完整文档扫描组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将扫描游标、单元抽取与扫描集组织收敛为完整文档扫描闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`NORM_DOC/QUALIFIED/CURSOR/SCAN_UNIT/SCAN_SET/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造扫描位置、`B3` 绕过 `B2` 直接自造扫描单元、以及 `B3` 反向改写 `B1/B2` 已确定的扫描位置、单元边界或顺序的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`NORM_DOC/QUALIFIED/CURSOR/SCAN_UNIT/SCAN_SET/ORDER`。

## 5. 验证（Verification）

- `V1` 扫描游标一致性：扫描游标结构必须满足 `NORM_DOC/QUALIFIED/CURSOR/ORDER` 约束。
- `V2` 单元抽取一致性：单元抽取结构必须满足 `CURSOR/SCAN_UNIT/ORDER` 约束。
- `V3` 扫描集一致性：扫描集结构必须满足 `SCAN_UNIT/SCAN_SET/ORDER` 约束。
- `V4` 扫描闭环完整性：完整文档扫描流程必须满足从扫描游标到扫描集的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

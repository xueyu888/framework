# 规则匹配器模块:RuleMatch

@framework

## 1. 能力声明（Capability Statement）

- `C1` 规则选择能力：提供从分块规范中选择当前适用规则的能力。
- `C2` 条件判定能力：提供基于扫描单元与当前规则执行条件判定的能力。
- `C3` 匹配结果组织能力：提供将判定结果组织为稳定匹配集合的能力。
- `C4` 非能力项：不负责文档扫描、分块组装与结果格式化。

## 2. 边界定义（Boundary / 参数）

- `SCAN_SET` 扫描集合边界：仅消费稳定、可枚举的扫描集合。来源：`C1 + C2`。
- `CHUNK_SPEC` 分块规范边界：规则匹配必须依据稳定分块规范进行。来源：`C1 + C2`。
- `RULE_REF` 规则引用边界：当前选中的规则引用、作用范围与顺序必须明确。来源：`C1 + C2`。
- `JUDGE_SET` 判定结果边界：条件判定结果必须稳定、可消费。来源：`C2 + C3`。
- `MATCH_SET` 匹配结果边界：匹配结果集合必须稳定、可枚举。来源：`C3`。
- `ORDER` 顺序边界：规则顺序、判定顺序与匹配结果顺序必须保持一致。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 规则选择结构基。来源：`C1 + CHUNK_SPEC + RULE_REF + ORDER`。
- `B2` 条件判定结构基。来源：`C2 + SCAN_SET + RULE_REF + JUDGE_SET + ORDER`。
- `B3` 匹配结果结构基。来源：`C3 + JUDGE_SET + MATCH_SET + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 规则判定组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由规则选择结构基选出当前适用规则，再由条件判定结构基依据扫描集合与当前规则执行条件判定。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SCAN_SET/CHUNK_SPEC/RULE_REF/JUDGE_SET/ORDER`。
- `R2` 判定建匹配组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：条件判定结构基产出的判定结果必须经过匹配结果结构基处理后，才能形成稳定匹配集合。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`JUDGE_SET/MATCH_SET/ORDER`。
- `R3` 完整规则匹配组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将规则选择、条件判定与匹配结果组织收敛为完整规则匹配闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SCAN_SET/CHUNK_SPEC/RULE_REF/JUDGE_SET/MATCH_SET/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造规则引用、`B3` 绕过 `B2` 直接自造判定结果、以及 `B3` 反向改写 `B1/B2` 已确定的规则顺序、判定结论或匹配关系的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SCAN_SET/CHUNK_SPEC/RULE_REF/JUDGE_SET/MATCH_SET/ORDER`。

## 5. 验证（Verification）

- `V1` 规则选择一致性：规则选择结构必须满足 `CHUNK_SPEC/RULE_REF/ORDER` 约束。
- `V2` 条件判定一致性：条件判定结构必须满足 `SCAN_SET/RULE_REF/JUDGE_SET/ORDER` 约束。
- `V3` 匹配结果一致性：匹配结果结构必须满足 `JUDGE_SET/MATCH_SET/ORDER` 约束。
- `V4` 匹配闭环完整性：完整规则匹配流程必须满足从规则选择到匹配结果的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 数据清洗模块:DataWasher

@framework

本模块定义数据清洗的主线结构：归一文档接入、清洗资格判断、稳定分块产物、Tag 标注、数据提取、结果对比、调参回环与人工审核。它只负责清洗链路结构，不直接定义具体行业字段词典或下游存储模型。

## 1. 能力声明（Capability Statement）

- `C1` 清洗入口能力：把归一文档、资格判断与稳定分块产物收敛为可进入清洗链路的起点结构。
- `C2` 标注能力：把块记录与 Tag 绑定关系收敛为稳定标注结构。
- `C3` 提取能力：把标注结果、提取目标与提取版本收敛为稳定提取结构。
- `C4` 对比能力：把多次提取结果的可比范围、对比结论与差异信息收敛为稳定对比结构。
- `C5` 调参闭环能力：把提取结果与调参动作组织为可回放的调参闭环结构。
- `C6` 人工审核能力：把自动结果与人工审核结果对齐为稳定审核收口结构。
- `C7` 非能力项：不负责具体行业字段词典、下游持久化模型与跨项目调度。

## 2. 边界定义（Boundary / 参数）

- `NORMDOC` 归一文档边界：仅处理已完成归一化并已入库的单文档输入。来源：`C1`。
- `QUALIFY` 资格边界：每个归一文档在进入清洗主线前都必须产出明确资格结论。来源：`C1`。
- `CHUNKARTIFACT` 分块产物边界：合法清洗输入必须产出稳定块记录、原文映射与可扩展属性位。来源：`C1 + C2`。
- `TAGLINK` 标注边界：每个 Tag 绑定都必须显式关联块标识、Tag 标识与标注来源。来源：`C2 + C3`。
- `EXTRACTTARGET` 提取边界：提取目标、提取依据与结果版本必须明确。来源：`C3 + C4 + C5`。
- `COMPARE` 对比边界：对比范围、差异表达与对比结论必须统一。来源：`C4 + C5 + C6`。
- `TUNE` 调参边界：调参动作、参数版本与回放路径必须可追踪。来源：`C5 + C6`。
- `REVIEW` 审核边界：人工审核意见、审核结果与收口状态必须稳定。来源：`C6`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 清洗入口结构基：由归一文档、资格结论与稳定分块产物构成的清洗入口结构。来源：`C1 + NORMDOC + QUALIFY + CHUNKARTIFACT`。
- `B2` 块标注结构基：由块记录、Tag 绑定与标注来源构成的标注结构。来源：`C1 + C2 + CHUNKARTIFACT + TAGLINK`。
- `B3` 数据提取结构基：由提取目标、提取依据与提取版本构成的提取结构。来源：`C2 + C3 + TAGLINK + EXTRACTTARGET`。
- `B4` 对比判断结构基：由可比结果集合、差异表达与对比结论构成的对比结构。来源：`C3 + C4 + EXTRACTTARGET + COMPARE`。
- `B5` 调参闭环结构基：由调参动作、参数版本与结果回放路径构成的闭环结构。来源：`C3 + C4 + C5 + EXTRACTTARGET + COMPARE + TUNE`。
- `B6` 人工审核结构基：由审核样本、审核结论与收口状态构成的人工审核结构。来源：`C4 + C5 + C6 + COMPARE + TUNE + REVIEW`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 清洗入口收敛
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：只有具备清洗资格并已生成稳定分块产物的文档，才允许进入标注链路；Tag 绑定不得脱离块记录集合单独存在。
  - `R1.3` 输出结构：`TaggedChunk_set`（元素：`TC`，每个 `TC` 必须包含 `chunk_id`、`tag_id`、`qualification_status`）。
  - `R1.4` 输出能力：`C1 + C2`。
  - `R1.5` 边界绑定：`NORMDOC/QUALIFY/CHUNKARTIFACT/TAGLINK`。
- `R2` 提取结构闭合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：每条提取结果都必须绑定来源块、触发 Tag、提取目标与提取版本；缺失任一绑定的提取结果均不构成合法组合。
  - `R2.3` 输出结构：`ExtractResult_set`（元素：`ER`，每个 `ER` 必须包含 `chunk_id`、`tag_id`、`extract_target`、`extract_version`）。
  - `R2.4` 输出能力：`C2 + C3`。
  - `R2.5` 边界绑定：`CHUNKARTIFACT/TAGLINK/EXTRACTTARGET`。
- `R3` 对比判断组合
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：只有提取目标一致、结果结构可比且差异范围明确的结果集才允许进入对比；不可比结果不得强行给出对比结论。
  - `R3.3` 输出结构：`CompareResult_set`（元素：`CR`，每个 `CR` 必须包含 `extract_target`、`compare_status`、`difference_summary`）。
  - `R3.4` 输出能力：`C3 + C4`。
  - `R3.5` 边界绑定：`EXTRACTTARGET/COMPARE`。
- `R4` 调参回环组合
  - `R4.1` 参与基：`B3 + B4 + B5`。
  - `R4.2` 组合方式：只有已进入对比阶段的结果才允许进入调参闭环；每次调参都必须生成新的提取版本并保留可回放路径。
  - `R4.3` 输出结构：`TuneLoop_set`（元素：`TL`，每个 `TL` 必须包含 `extract_target`、`parameter_version`、`replay_path`）。
  - `R4.4` 输出能力：`C3 + C4 + C5`。
  - `R4.5` 边界绑定：`EXTRACTTARGET/COMPARE/TUNE`。
- `R5` 人工审核收口
  - `R5.1` 参与基：`B4 + B5 + B6`。
  - `R5.2` 组合方式：进入人工审核的结果必须已经经过可比判断并保留调参路径；审核结论必须显式给出收口状态，不得只保留口头结论。
  - `R5.3` 输出结构：`ReviewResult_set`（元素：`RR`，每个 `RR` 必须包含 `compare_result_id`、`review_status`、`review_decision`）。
  - `R5.4` 输出能力：`C4 + C5 + C6`。
  - `R5.5` 边界绑定：`COMPARE/TUNE/REVIEW`。
- `R6` 全链路合法组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R6.2` 组合方式：合法链路顺序固定为 `B1 -> B2 -> B3 -> B4 -> B5 -> B6`；下游结构不得逆向改写上游块标识、Tag 绑定或资格结论。
  - `R6.3` 输出结构：`CleaningChain`（必须包含 `qualified_doc_id`、`chunk_artifact_id`、`extract_version_set`、`review_status`）。
  - `R6.4` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R6.5` 边界绑定：`NORMDOC/QUALIFY/CHUNKARTIFACT/TAGLINK/EXTRACTTARGET/COMPARE/TUNE/REVIEW`。
- `R7` 禁止组合
  - `R7.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R7.2` 组合方式：跳过资格判断直接标注、跳过标注直接提取、跳过提取直接对比、调参覆盖旧版本、审核无收口状态或下游逆向改写上游结构的方案均无效。
  - `R7.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效基、失效关系与失效原因）。
  - `R7.4` 输出能力：`C7`。
  - `R7.5` 边界绑定：`NORMDOC/QUALIFY/CHUNKARTIFACT/TAGLINK/EXTRACTTARGET/COMPARE/TUNE/REVIEW`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个边界项推导得到。
- `V2` 规则完整性：每个 `R*` 必须明确参与基、组合方式、输出能力与边界绑定。
- `V3` 链路顺序合法性：所有有效组合都必须满足 `B1 -> B2 -> B3 -> B4 -> B5 -> B6` 的固定顺序。
- `V4` 绑定一致性：归一文档、资格结论、分块产物、Tag 绑定、提取结果、对比结果、调参版本与审核结论之间必须存在可校验绑定。
- `V5` 回放有效性：进入调参与审核阶段的结果都必须能回放到具体版本与具体对比结论。
- `V6` 最小必要性：移除任一参与合法组合的 `B*` 后，该组合至少一项合法性条件失败。
- `V7` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 身份校验模块:IdentityVerificationModule

@framework

组件目标：对标识输入执行凭证记录定位，并对凭证输入执行凭证校验，输出身份校验结论。

## 1. 能力声明（Capability Statement）

- `C1` 标识凭证定位能力：根据标识输入定位对应的已存凭证记录。
- `C2` 凭证校验能力：根据凭证表示方式和凭证校验方式，对输入凭证与已存凭证执行校验。
- `C3` 身份结论输出能力：输出通过或失败结论。

## 2. 非职责声明（Non-Capability Statement）

- `N1` 不负责输入格式合法性校验。
- `N2` 不负责账户创建、资料维护和登录后信息修改。
- `N3` 不负责 token、会话与权限控制。

## 3. 边界定义（Boundary）

- `d1` 标识定位规则：同一标识必须唯一定位到一条有效凭证记录。来源：`C1`
- `d2` 凭证表示方式：明文、MD5 等。来源：`C2`
- `d3` 凭证校验方式：完全匹配、哈希校验、归一化后校验等。来源：`C2`
- `d4` 结论范围：通过、失败。来源：`C3`

## 4. 最小可行基（Minimum Viable Bases）

- `B1` 身份校验输入结构：由标识输入和凭证输入构成。来源：`C1 + C2`。
- `B2` 标识凭证映射表(map)：由标识值、凭证值和凭证表示方式构成。来源：`C1 + C2 + d1 + d2`。
- `B3` 身份校验结果映射表：由标识定位结果、凭证校验结果和身份结论值构成。来源：`C3 + d4`。
- `B4` 字符串比较：用于对输入凭证与已存凭证执行校验。来源：`C2 + d3`。

## 5. 基组合原则（Base Combination Principles）

- `R1` 标识凭证定位组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：标识输入必须通过标识凭证映射表定位对应的已存凭证记录，并将定位结果写入身份校验结论结构中的标识定位结果位。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`d1/d4`。

- `R2` 凭证校验组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：仅当标识定位结果为通过时，输入凭证才允许与已存凭证执行校验；校验必须依据凭证表示方式和凭证校验方式完成，并将校验结果写入身份校验结论结构中的凭证校验结果位。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`d2/d3/d4`。

- `R3` 结论成立组合
  - `R3.1` 参与基：`B3`。
  - `R3.2` 组合方式：标识定位结果和凭证校验结果全部为通过时，身份校验模块输出通过；任一结果为失败时，身份校验模块输出失败。
  - `R3.3` 输出能力：`C3`。
  - `R3.4` 边界绑定：`d4`。


## 6. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个边界项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力和边界绑定。
- `V3` 闭环性：`R4` 必须形成“输入承载 -> 格式校验 -> 凭证定位 -> 结果输出”的完整登录校验链。
- `V4` 边界符合性：所有有效校验组合都必须满足 `D1/D2/D3/D4`。
- `V5` 最小必要性：移除任一 `B*` 后，至少一项能力无法成立。

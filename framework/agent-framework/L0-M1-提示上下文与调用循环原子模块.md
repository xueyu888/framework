# 提示上下文与调用循环原子模块:PromptContextInvocationLoopAtoms

@framework

本模块定义 agent bot 最底层的提示上下文和调用循环部件。它只回答 5 件事：引导上下文怎样被装成 bootstrap block、调用消息怎样接成序列、provider 入口怎样被登记、工具调用怎样回填、迭代状态怎样持续推进；不负责渠道收发、记忆归档和后台任务触发。

## 1. 能力声明（Capability Statement）

- `C1` 引导上下文定义能力：定义身份说明、引导文件和运行时元数据怎样被稳定表达。
- `C2` 调用消息定义能力：定义 system、history、user、assistant 和 tool 消息怎样被稳定接成一次调用消息列表。
- `C3` Provider 入口定义能力：定义 provider 注册元数据和模型入口怎样被稳定表达。
- `C4` 工具调用回填定义能力：定义 tool call、工具结果和回填位置怎样被稳定表达。
- `C5` 迭代状态定义能力：定义当前轮次、最大轮次和停止原因怎样被稳定表达。
- `C6` 非能力项：不负责渠道收发、历史归档和后台任务触发。

## 2. 边界定义（Boundary / 参数）

- `WORKROOT` 工作区边界：引导文件、技能目录和运行时上下文文件只能来自声明过的 workspace 根目录。来源：`C1 + C2`。
- `CTXSET` 上下文装配边界：调用消息只能由引导上下文、运行时元数据、会话历史和用户输入组成。来源：`C1 + C2 + C5`。
- `PROVIDERSET` 模型入口边界：provider 入口只能来自注册表或显式配置过的 provider 集合。来源：`C3 + C5`。
- `TOOLSET` 工具集合边界：tool call 只能来自已注册工具集合和声明过的 schema。来源：`C4 + C5`。
- `ITERMAX` 迭代上限边界：一次调用循环允许的最大轮次必须显式声明。来源：`C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 引导上下文块：汇集身份说明、引导文件引用和运行时元数据，能表达一个稳定 bootstrap block。来源：`C1 + WORKROOT + CTXSET`。
- `B2` 调用消息序列：排列 system、history、user、assistant 和 tool 消息顺序，能表达一次完整调用消息列表。来源：`C2 + CTXSET`。
- `B3` Provider 入口登记项：登记 provider 名、模型前缀、API 入口和默认模型，能表达一个唯一模型入口。来源：`C3 + PROVIDERSET`。
- `B4` 工具调用记录：记录 `tool_call_id`、工具名、参数和工具结果，能表达一次完整工具调用事实。来源：`C4 + TOOLSET`。
- `B5` 迭代状态记录：记录当前轮次、最大轮次、停止原因和当前消息序列位置，能表达一次主循环推进状态。来源：`C4 + C5 + ITERMAX`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 提示上下文成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定引导上下文块，再把当前历史和用户输入挂到调用消息序列上，形成一次完整调用上下文。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`WORKROOT/CTXSET`。
- `R2` Provider 入口成立
  - `R2.1` 参与基：`B3`。
  - `R2.2` 组合方式：先登记 provider 注册元数据、模型前缀和 API 入口，形成一个唯一模型入口。
  - `R2.3` 输出能力：`C3`。
  - `R2.4` 边界绑定：`PROVIDERSET`。
- `R3` 工具调用成立
  - `R3.1` 参与基：`B4 + B5`。
  - `R3.2` 组合方式：先记录工具名、参数和工具结果，再由迭代状态记录固定当前轮次和消息位置，形成一次完整工具调用回填事实。
  - `R3.3` 输出能力：`C4 + C5`。
  - `R3.4` 边界绑定：`TOOLSET/ITERMAX`。
- `R4` 主循环事实成立
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：把引导上下文、调用消息、provider 入口、工具调用和迭代状态接到同一轮调用上，形成完整主循环事实。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R4.4` 边界绑定：`WORKROOT/CTXSET/PROVIDERSET/TOOLSET/ITERMAX`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：上下文越出工作区、provider 入口未声明、工具不在工具集合或迭代状态无上限的组合无效。
  - `R5.3` 输出能力：`C6`。
  - `R5.4` 边界绑定：`WORKROOT/CTXSET/PROVIDERSET/TOOLSET/ITERMAX`。

## 5. 验证（Verification）

- `V1` 提示上下文有效性：引导上下文块和调用消息序列必须满足 `WORKROOT/CTXSET` 约束。
- `V2` Provider 入口有效性：Provider 入口登记项必须满足 `PROVIDERSET` 约束。
- `V3` 工具调用有效性：工具调用记录和迭代状态记录必须满足 `TOOLSET/ITERMAX` 约束。
- `V4` 主循环事实有效性：同一轮调用必须能同时回看上下文、provider 入口、工具调用和停止状态。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

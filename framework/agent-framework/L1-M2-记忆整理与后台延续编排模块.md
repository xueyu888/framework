# 记忆整理与后台延续编排模块:MemoryConsolidationBackgroundContinuationOrchestration

@framework

本模块把长期记忆写回、历史归档、后台任务触发和结果回投接成一条后台延续线。它只负责 4 件事：长期记忆怎么更新、历史记录怎么归档、后台任务怎么触发、后台结果怎么回投；不负责渠道入站和主循环推理。

## 1. 能力声明（Capability Statement）

- `C1` 长期记忆写回能力：根据整理结果稳定更新 `MEMORY.md`。
- `C2` 历史归档能力：根据归档范围和游标推进稳定写入 `HISTORY.md`。
- `C3` 后台任务触发能力：根据任务类型、调度条件和任务载荷触发后台任务。
- `C4` 结果回投能力：根据来源渠道和目标聊天把后台结果稳定回投。
- `C5` 非能力项：不负责渠道入站、提示上下文装配和主循环推理。

## 2. 边界定义（Boundary / 参数）

- `WORKROOT` 工作区边界：`MEMORY.md`、`HISTORY.md` 和后台任务文件只能来自声明过的 workspace 根目录。来源：`C1 + C2 + C3`。
- `MEMWINDOW` 记忆窗口边界：归档范围、保留窗口和 `last_consolidated` 推进规则必须显式声明。来源：`C1 + C2`。
- `TASKSET` 后台任务边界：后台延续类型只能来自子代理、cron 和 heartbeat 三类任务集合。来源：`C3 + C4`。
- `TASKSCHED` 任务调度边界：cron 表达式、一次性触发和心跳间隔只能来自声明过的调度规则。来源：`C3`。
- `TASKTARGET` 结果回投边界：后台结果必须绑定一个明确来源渠道和一个明确聊天目标。来源：`C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 长期记忆写回器：L0.M2[R1]：把整理后的完整记忆文本写回 `MEMORY.md`，能输出当前长期记忆版本。来源：`C1 + WORKROOT + MEMWINDOW`。
- `B2` 历史归档器：L0.M2[R1]：把历史条目和游标推进写入 `HISTORY.md` 与会话归档范围，能输出当前归档结果。来源：`C2 + WORKROOT + MEMWINDOW`。
- `B3` 后台任务调度器：L0.M2[R2]：维护子代理、cron 和 heartbeat 的任务载荷与调度条件，能触发一条可执行后台任务。来源：`C3 + TASKSET + TASKSCHED`。
- `B4` 结果回投器：L0.M2[R3]：绑定来源渠道、聊天目标和通知正文，能把后台结果回投到一个明确目标。来源：`C4 + TASKSET + TASKTARGET`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 记忆整理成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先写回完整长期记忆，再补齐历史条目和游标推进，形成一条稳定记忆整理结果。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`WORKROOT/MEMWINDOW`。
- `R2` 后台任务成立
  - `R2.1` 参与基：`B3 + B4`。
  - `R2.2` 组合方式：先固定后台任务类型、调度条件和任务载荷，再补齐来源渠道和回投目标，形成一条可执行后台任务链。
  - `R2.3` 输出能力：`C3 + C4`。
  - `R2.4` 边界绑定：`TASKSET/TASKSCHED/TASKTARGET`。
- `R3` 后台延续闭环成立
  - `R3.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R3.2` 组合方式：把记忆整理、后台触发和结果回投接成一条完整后台延续闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 边界绑定：`WORKROOT/MEMWINDOW/TASKSET/TASKSCHED/TASKTARGET`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：记忆写回不推进游标、后台任务类型未声明、调度条件无效或结果没有明确回投目标的组合无效。
  - `R4.3` 输出能力：`C5`。
  - `R4.4` 边界绑定：`WORKROOT/MEMWINDOW/TASKSET/TASKSCHED/TASKTARGET`。

## 5. 验证（Verification）

- `V1` 长期记忆写回有效性：长期记忆写回器必须满足 `WORKROOT/MEMWINDOW` 约束。
- `V2` 历史归档有效性：历史归档器必须满足 `WORKROOT/MEMWINDOW` 约束，且必须推进归档游标。
- `V3` 后台任务触发有效性：后台任务调度器必须只能触发 `TASKSET` 内且满足 `TASKSCHED` 的任务。
- `V4` 结果回投有效性：结果回投器必须把后台结果投到满足 `TASKTARGET` 的明确目标。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

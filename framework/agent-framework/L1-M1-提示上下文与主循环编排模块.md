# 提示上下文与主循环编排模块:PromptContextMainLoopOrchestration

@framework

本模块把提示上下文、provider 入口、工具调用和主循环推进接成一条 agent 推理线。它只负责 4 件事：提示上下文怎么生成、模型入口怎么定位、工具调用怎么执行和回填、一次 agent 轮次怎么稳定结束；不负责渠道收发、记忆整理和后台任务触发。

## 1. 能力声明（Capability Statement）

- `C1` 提示上下文生成能力：根据引导文件、运行时元数据、长期记忆和当前历史生成稳定调用上下文。
- `C2` Provider 入口定位能力：根据 provider 注册元数据和模型配置定位唯一模型入口。
- `C3` 工具调用回填能力：根据 tool call 名称、参数和结果完成稳定工具执行与结果回填。
- `C4` 主循环结束能力：根据当前轮次、工具结果和停止条件稳定结束一次 agent 轮次。
- `C5` 非能力项：不负责渠道入站、会话续写、记忆整理和后台任务触发。

## 2. 边界定义（Boundary / 参数）

- `WORKROOT` 工作区边界：引导文件、技能目录和长期记忆文件只能来自声明过的 workspace 根目录。来源：`C1`。
- `CTXSET` 上下文装配边界：system prompt 与调用消息只能由引导文件、运行时元数据、长期记忆和当前历史组成。来源：`C1 + C4`。
- `PROVIDERSET` 模型提供商边界：模型入口只能来自注册表或显式配置过的 provider 集合。来源：`C2 + C4`。
- `TOOLSET` 工具集合边界：可执行工具只能来自已注册工具集合和声明过的 schema。来源：`C3 + C4`。
- `ITERMAX` 迭代上限边界：一次主循环允许的迭代轮数上限必须显式声明。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 提示上下文装配器：L0.M1[R1] + L0.M2[R1]：加载引导上下文、长期记忆和当前历史，能输出稳定 system prompt 和调用消息列表。来源：`C1 + WORKROOT + CTXSET`。
- `B2` Provider 注册表与选择器：L0.M1[R2]：登记 provider 注册元数据、模型前缀规则和 API 入口绑定，能根据模型配置定位唯一模型入口。来源：`C2 + PROVIDERSET`。
- `B3` 工具注册表与执行器：L0.M1[R3]：登记工具 schema、工具名和执行入口，能根据 tool call 名称和参数执行工具并回填结果。来源：`C3 + TOOLSET + ITERMAX`。
- `B4` Agent 主循环调度器：L0.M1[R4]：推进当前消息序列、迭代状态和停止条件，能根据模型响应输出下一轮调用或最终回复文本。来源：`C2 + C3 + C4 + PROVIDERSET + TOOLSET + ITERMAX`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 上下文成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定引导文件、长期记忆、运行时元数据和历史顺序，形成一次完整调用上下文。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`WORKROOT/CTXSET`。
- `R2` 模型调用成立
  - `R2.1` 参与基：`B1 + B2 + B4`。
  - `R2.2` 组合方式：先由提示上下文装配器生成调用消息，再由 Provider 注册表与选择器定位唯一模型入口，最后由 Agent 主循环调度器发起一次模型调用。
  - `R2.3` 输出能力：`C1 + C2 + C4`。
  - `R2.4` 边界绑定：`WORKROOT/CTXSET/PROVIDERSET/ITERMAX`。
- `R3` 工具循环成立
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：当模型返回 tool call 时，由工具注册表与执行器校验并执行工具，再由 Agent 主循环调度器把工具结果回填到消息序列后继续推进。
  - `R3.3` 输出能力：`C3 + C4`。
  - `R3.4` 边界绑定：`TOOLSET/ITERMAX`。
- `R4` 主循环闭环成立
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：把上下文生成、模型调用、工具回填和停止条件接成一条完整 agent 主循环。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`WORKROOT/CTXSET/PROVIDERSET/TOOLSET/ITERMAX`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：上下文越出工作区、未注册 provider 仍被调用、未注册工具仍被执行或超过 `ITERMAX` 仍继续推进的组合无效。
  - `R5.3` 输出能力：`C5`。
  - `R5.4` 边界绑定：`WORKROOT/CTXSET/PROVIDERSET/TOOLSET/ITERMAX`。

## 5. 验证（Verification）

- `V1` 上下文生成有效性：提示上下文装配器必须满足 `WORKROOT/CTXSET` 约束。
- `V2` Provider 入口有效性：Provider 注册表与选择器必须只能输出 `PROVIDERSET` 内一个明确模型入口。
- `V3` 工具回填有效性：工具注册表与执行器必须只处理 `TOOLSET` 内工具，且结果必须能稳定回填。
- `V4` 主循环结束有效性：同一轮次必须在 `ITERMAX` 内得到下一轮调用或最终回复文本。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 消息事件与会话记录原子模块:MessageEventSessionRecordAtoms

@framework

本模块定义 agent bot 最底层的消息事件和会话记录部件。它只回答 5 件事：入站事件怎样被表达、出站事件怎样被表达、消息队列怎样承接收发、会话键怎样稳定归属、会话记录怎样持续追加；不负责提示上下文装配、模型调用和后台任务触发。

## 1. 能力声明（Capability Statement）

- `C1` 入站事件定义能力：定义一条渠道消息怎样被稳定表达为入站事件。
- `C2` 出站事件定义能力：定义一条回复结果怎样被稳定表达为出站事件。
- `C3` 队列收发定义能力：定义一条消息怎样进入入站队列或出站队列。
- `C4` 会话键定义能力：定义一条消息怎样稳定归入一个会话键。
- `C5` 会话记录定义能力：定义一条可追加的 JSONL 会话记录怎样被稳定表达。
- `C6` 非能力项：不负责提示上下文装配、模型入口定位、工具调用和后台任务触发。

## 2. 边界定义（Boundary / 参数）

- `CHANNELSET` 渠道集合边界：消息事件中的渠道标识只能来自声明过的渠道集合。来源：`C1 + C2 + C3`。
- `SESSIONKEY` 会话键边界：会话键只能由 `channel:chat_id` 或显式 override 稳定给出。来源：`C4 + C5`。
- `MSGROLE` 消息角色边界：会话记录角色只能来自 `user/assistant/tool/system` 这组声明过的角色集合。来源：`C5`。
- `HISTORYFMT` 会话记录格式边界：会话记录必须能稳定写成 JSONL metadata line 和 message line。来源：`C5`。
- `SENDTARGET` 出站目标边界：任一出站事件都必须绑定一个明确渠道和一个明确聊天目标。来源：`C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 入站事件记录：记录渠道标识、发送者标识、聊天标识、正文、媒体和元数据，能表达一条标准入站消息。来源：`C1 + CHANNELSET`。
- `B2` 出站事件记录：记录渠道标识、聊天标识、回复正文、`reply_to`、媒体和元数据，能表达一条标准出站消息。来源：`C2 + CHANNELSET + SENDTARGET`。
- `B3` 消息队列槽：挂接 inbound/outbound 队列位置和消费顺序，能表达一条消息当前处于待处理还是待发送。来源：`C3 + CHANNELSET`。
- `B4` 会话键映射项：绑定 `channel:chat_id` 组合键和显式 override，能表达一条消息归属的唯一会话键。来源：`C4 + SESSIONKEY`。
- `B5` 会话记录条目：记录角色、正文、时间戳、工具调用结果和整理游标，能表达一条可追加的 JSONL 会话记录。来源：`C5 + SESSIONKEY + MSGROLE + HISTORYFMT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 事件总线成立
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：先固定入站事件记录和出站事件记录，再把它们挂到消息队列槽上，形成可发布和可消费的消息事件。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`CHANNELSET/SENDTARGET`。
- `R2` 会话键成立
  - `R2.1` 参与基：`B1 + B2 + B4`。
  - `R2.2` 组合方式：先固定消息来源或目标，再由会话键映射项为其给出稳定会话归属。
  - `R2.3` 输出能力：`C4`。
  - `R2.4` 边界绑定：`SESSIONKEY/SENDTARGET`。
- `R3` 会话记录成立
  - `R3.1` 参与基：`B4 + B5`。
  - `R3.2` 组合方式：先固定会话键，再把角色、正文和时序记录追加到同一会话键下，形成可续写会话记录。
  - `R3.3` 输出能力：`C4 + C5`。
  - `R3.4` 边界绑定：`SESSIONKEY/MSGROLE/HISTORYFMT`。
- `R4` 渠道收发会话事实成立
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：把入站事件、出站事件、队列槽、会话键和会话记录接到同一条消息收发线上，形成完整会话事实。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R4.4` 边界绑定：`CHANNELSET/SESSIONKEY/MSGROLE/HISTORYFMT/SENDTARGET`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：渠道标识未声明、会话键不稳定、消息角色不在声明集合或会话记录无法稳定写成 JSONL 的组合无效。
  - `R5.3` 输出能力：`C6`。
  - `R5.4` 边界绑定：`CHANNELSET/SESSIONKEY/MSGROLE/HISTORYFMT/SENDTARGET`。

## 5. 验证（Verification）

- `V1` 事件总线有效性：入站事件记录、出站事件记录和消息队列槽必须满足 `CHANNELSET/SENDTARGET` 约束。
- `V2` 会话键有效性：同一来源消息必须稳定映到同一 `SESSIONKEY`。
- `V3` 会话记录有效性：会话记录条目必须满足 `MSGROLE/HISTORYFMT` 约束，并能稳定追加到同一会话键下。
- `V4` 收发会话事实有效性：同一会话必须能同时回看入站消息、出站消息和追加记录。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 记忆整理与后台任务原子模块:MemoryConsolidationBackgroundTaskAtoms

@framework

本模块定义 agent bot 最底层的记忆整理和后台任务部件。它只回答 4 件事：长期记忆怎样被表达、历史归档怎样被记录、后台任务怎样被表达、回投目标怎样被定位；不负责渠道入站和主循环推理。

## 1. 能力声明（Capability Statement）

- `C1` 长期记忆定义能力：定义 `MEMORY.md` 和长期记忆版本怎样被稳定表达。
- `C2` 历史归档定义能力：定义 `HISTORY.md` 条目、归档范围和游标推进怎样被稳定表达。
- `C3` 后台任务定义能力：定义子代理、cron 和 heartbeat 任务怎样被稳定表达。
- `C4` 回投目标定义能力：定义后台结果怎样稳定回到来源渠道和聊天目标。
- `C5` 非能力项：不负责渠道入站、提示上下文装配和主循环推理。

## 2. 边界定义（Boundary / 参数）

- `WORKROOT` 工作区边界：`MEMORY.md`、`HISTORY.md` 和后台任务文件只能来自声明过的 workspace 根目录。来源：`C1 + C2 + C3`。
- `MEMWINDOW` 记忆窗口边界：归档范围、保留窗口和 `last_consolidated` 推进规则必须显式声明。来源：`C1 + C2`。
- `TASKSET` 后台任务边界：后台任务类型只能来自子代理、cron 和 heartbeat 三类声明过的任务集合。来源：`C3 + C4`。
- `TASKSCHED` 任务调度边界：cron 表达式、一次性任务和心跳间隔只能来自声明过的调度规则。来源：`C3`。
- `TASKTARGET` 回投目标边界：后台结果必须绑定一个明确来源渠道和一个明确聊天目标。来源：`C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 长期记忆记录：记录 `MEMORY.md` 完整记忆文本和最近更新结果，能表达当前长期记忆版本。来源：`C1 + WORKROOT + MEMWINDOW`。
- `B2` 历史归档记录：记录 `HISTORY.md` 条目、归档范围和 `last_consolidated` 游标，能表达已整理历史和待整理范围。来源：`C2 + WORKROOT + MEMWINDOW`。
- `B3` 后台任务条目：记录任务类型、任务载荷、任务状态和调度条件，能表达一个子代理、cron 或 heartbeat 任务事实。来源：`C3 + TASKSET + TASKSCHED`。
- `B4` 结果回投目标：绑定来源渠道、聊天标识、deliver 标志和通知正文，能表达一个可回投的结果目标。来源：`C4 + TASKTARGET`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 记忆整理成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定长期记忆文本，再补齐历史条目、归档范围和游标推进，形成一条完整记忆整理结果。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`WORKROOT/MEMWINDOW`。
- `R2` 后台任务成立
  - `R2.1` 参与基：`B3 + B4`。
  - `R2.2` 组合方式：先固定后台任务类型、调度条件和任务载荷，再绑定来源渠道和结果回投目标，形成一条可执行后台任务链。
  - `R2.3` 输出能力：`C3 + C4`。
  - `R2.4` 边界绑定：`TASKSET/TASKSCHED/TASKTARGET`。
- `R3` 后台延续事实成立
  - `R3.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R3.2` 组合方式：把长期记忆、历史归档、后台任务和回投目标接到同一条后台延续线上，形成完整延续事实。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 边界绑定：`WORKROOT/MEMWINDOW/TASKSET/TASKSCHED/TASKTARGET`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：记忆文本无工作区归属、游标不推进、任务类型未声明或结果没有明确回投目标的组合无效。
  - `R4.3` 输出能力：`C5`。
  - `R4.4` 边界绑定：`WORKROOT/MEMWINDOW/TASKSET/TASKSCHED/TASKTARGET`。

## 5. 验证（Verification）

- `V1` 记忆整理有效性：长期记忆记录和历史归档记录必须满足 `WORKROOT/MEMWINDOW` 约束。
- `V2` 后台任务有效性：后台任务条目必须满足 `TASKSET/TASKSCHED` 约束。
- `V3` 回投目标有效性：结果回投目标必须满足 `TASKTARGET` 约束。
- `V4` 后台延续事实有效性：同一条后台任务必须能同时回看当前记忆整理结果和明确回投目标。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

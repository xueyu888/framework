# 账号密码存取模块:PasswordStorageModule

@framework

## 0. 模块身份（Module Identity）

    模块名 := 账号密码存取模块:PasswordStorageModule
    问题域 := 账号到密码的映射关系域，以及基于该映射的存取处理域
    目标   := 刻画基于哈希表的账号密码写入与按账号读取的稳定结构
    模块型 := 处理型

## 1. Base

    B1 HashTable := 支持基于字符串账号键进行定位、写入、读取映射项的存储结构

## 2. Combination Principles

只写原则，不写具体结果类。
记号：`x ≡ y` 表示同一结果类；`nf(x)` 表示 `x` 的规范形。

    R1 CredentialWritePrinciple:
      formation     := combinable(account, password, HashTable) iff
                       account 为字符串，password 为字符串，且 HashTable 支持基于账号键写入键值映射
      satisfaction  := satisfied(write(account, password, HashTable)) iff
                       account 长度满足 P1，
                       password 长度满足 P2，
                       账号定位过程遵守 P3，
                       重复账号处理遵守 P4，
                       哈希冲突处理遵守 P5
      identity      := write(account, password, HashTable)_x ≡ write(account, password, HashTable)_y iff
                       两者对同一 account 的最终可观察存储结果一致
      normalization := nf(write(account, password, HashTable)) =
                       一个唯一确定的 account -> password 映射结果 iff
                       对该 account 的重复处理与冲突处理均已完成，且最终仅保留一个有效值

    R2 CredentialLookupPrinciple:
      formation     := combinable(account, HashTable) iff
                       account 为字符串，且 HashTable 支持基于账号键进行查询
      satisfaction  := satisfied(lookup(account, HashTable)) iff
                       account 长度满足 P1，
                       查询定位过程遵守 P3，
                       冲突查找过程遵守 P5，
                       且查询结果唯一确定或明确未命中
      identity      := lookup(account, HashTable)_x ≡ lookup(account, HashTable)_y iff
                       两者对同一 account 返回相同结果
      normalization := nf(lookup(account, HashTable)) =
                       password 或 miss iff
                       查询过程中所有中间定位路径被折叠，只保留最终查询结论

    R3 StoreObservationalEquivalencePrinciple:
      formation     := combinable(HashTable_x, HashTable_y) iff
                       HashTable_x 与 HashTable_y 均为合法账号密码存储结构
      satisfaction  := satisfied(HashTable_x, HashTable_y) iff
                       对任意满足 P1 的合法 account，
                       两者查询结果一致
      identity      := HashTable_x ≡ HashTable_y iff
                       对任意合法 account，
                       两者返回的 password 或 miss 完全一致
      normalization := nf(HashTable) =
                       去除表示差异后的规范映射结构 iff
                       结构中每个 account 至多对应一个最终有效 password，
                       且不同冲突路径不影响最终可观察结果

## 3. Result Space

    A1 SingleCredentialMapping:
      form    := 一个 account 与一个 password 在 HashTable 中形成的一条映射项
      by      := R1
      meaning := 表示单个账号已完成一次合法写入

    A2 CredentialMappingStore:
      form    := 多个 account -> password 映射项组成的可查询哈希表结构
      by      := R1, R2
      meaning := 表示模块已形成可执行写入与读取的账号密码存储结果

    A3 NormalizedCredentialMappingStore:
      form    := 在重复账号策略与冲突处理策略作用下得到的规范化账号密码哈希表
      by      := R1, R2, R3
      meaning := 表示该模块对外稳定暴露的标准结果空间

## 4. Boundary

### 4.1 Input / Output

    main_in:
      payload     := {account: string, password: string}
      cardinality := 1
      to          := B1

    aux_in:
      payload     := {account: string}
      cardinality := 1
      to          := B1

    main_out:
      payload     := {account: string, password: string}
      cardinality := 1
      from        := A1

    trace_out:
      payload     := {action: write|overwrite|reject|lookup, result: hit|miss|stored, collision_resolution: strategy_name}
      cardinality := 1
      from        := A2

### 4.2 Parameter

    P1 AccountLengthRange:
      domain  := [min_account_length, max_account_length]
      affects := R1, R2, IO

    P2 PasswordLengthRange:
      domain  := [min_password_length, max_password_length]
      affects := R1, IO

    P3 AccountHashFunction:
      domain  := 作用于字符串账号的确定性哈希函数
      affects := R1, R2, A2

    P4 DuplicateAccountPolicy:
      domain  := {reject, overwrite}
      affects := R1, A1, A2

    P5 HashCollisionResolutionPolicy:
      domain  := {chaining, open_addressing}
      affects := R1, R2, A2

## 5. Capability Statement

    C1 CredentialWriteCapability:
      requires   := main_in, P1, P2, P3, P4, P5
      guarantees := 对满足长度约束的账号与密码，模块能够基于既定哈希函数、重复账号处理策略与冲突处理策略，将其稳定写入哈希表并形成账号到密码的映射结果

    C2 CredentialLookupCapability:
      requires   := aux_in, A2, P1, P3, P5
      guarantees := 对给定合法账号，模块能够基于已有哈希表结构稳定返回其对应密码，或返回明确未命中结果

# 会话持有映射组件:SessionHoldingMap

@framework

组件目标：为已取得的会话上下文分配稳定句柄，并维护可定位、可原子跃迁的持有关系。

## 1. 能力声明（Capability Statement）

- `C1` 持有登记能力：为一次有效会话持有写入唯一句柄和持有记录。
- `C2` 句柄定位能力：通过稳定句柄定位当前持有关系。
- `C3` 生命周期跃迁能力：把持有关系从 `active` 原子地收敛到 `closed`。
- `C4` 并发隔离能力：隔离同一句柄上的并发竞态，保证生命周期跃迁不会被并发撕裂。
- `C5` 非能力项：不负责创建连接载体，不负责组织会话上下文。

## 2. 边界定义（Boundary / 参数）

- `HP` 句柄标识（handle）：每次登记都必须生成唯一句柄。来源：`C1 + C2 + C3`。
- `SR` 会话引用（session_ref）：持有记录必须指向一个已存在的会话上下文引用。来源：`C1 + C2`。
- `LS` 生命周期槽位（lifecycle_state）：只允许 `active | closed`。来源：`C2 + C3 + C4`。
- `AG` 原子守卫约束（atomic_guard）：同一句柄的生命周期跃迁必须一次完成。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 句柄索引位（自有基）：句柄到记录入口的索引结构，由句柄键和值入口构成。来源：`C1 + C2 + HP`。
- `B2` 持有记录单元（自有基）：当前持有关系的承载结构，由会话引用和生命周期槽位构成。来源：`C1 + C2 + C3 + SR + LS`。
- `B3` 原子跃迁点（自有基）：同一句柄生命周期一次性切换的守卫结构，由跃迁入口和守卫位构成。来源：`C3 + C4 + AG + LS`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 持有登记组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：所有持有关系都必须先写入持有记录，再由句柄索引位建立唯一入口；没有持有记录不承认有效句柄。
  - `R1.3` 输出结构：`HANDLE_ENTRY`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`HP/SR/LS`。
- `R2` 句柄定位组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：定位动作只能从句柄索引位进入，再落到当前持有记录；句柄不存在或记录已无效时不承认合法定位。
  - `R2.3` 输出结构：`HOLDING_REF`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`HP/SR/LS`。
- `R3` 生命周期跃迁组合
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：关闭动作必须先命中当前持有记录，再通过原子跃迁点把状态从活跃态一次性切到关闭态；非原子跃迁不承认有效。
  - `R3.3` 输出结构：`LIFECYCLE_TRANSITION_RESULT`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`LS/AG`。
- `R4` 并发隔离组合
  - `R4.1` 参与基：`B2 + B3`。
  - `R4.2` 组合方式：同一句柄上的并发定位和关闭都必须服从原子守卫；任何可能同时写入多个关闭结果的组合均无效。
  - `R4.3` 输出结构：`GUARDED_HOLDING_SET`。
  - `R4.4` 输出能力：`C4`。
  - `R4.5` 边界绑定：`HP/LS/AG`。
- `R5` 完整持有映射组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：先登记持有关系，再允许按句柄定位，并在需要时通过原子跃迁关闭持有关系；整个过程都不得绕开句柄索引和守卫位。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R5.4` 边界绑定：`HP/SR/LS/AG`。
- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：没有句柄直接定位、没有持有记录直接关闭、绕过原子守卫直接修改生命周期、把连接建立或上下文创建逻辑混入本层的组合均无效。
  - `R6.3` 输出能力：`C5`。
  - `R6.4` 边界绑定：`HP/SR/LS/AG`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R5` 输出能力并集必须覆盖 `C1~C4`。
- `V4` 边界符合性：所有有效持有关系都必须满足句柄、会话引用、生命周期和原子守卫边界。
- `V5` 最小必要性：移除任一 `B*` 后，持有登记、句柄定位、生命周期跃迁或并发隔离至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。

# 数据库会话管理组件:DatabaseSessionManager

@framework

组件目标：围绕同一数据库连接对象，统一管理数据库会话的获取、定位、复用和关闭。

## 1. 能力声明（Capability Statement）

- `C1` 会话获取能力：基于连接信息取得受管数据库会话。
- `C2` 会话定位能力：通过稳定句柄定位已取得的数据库会话。
- `C3` 会话复用能力：在同一连接对象上复用会话上下文，或按独立方式展开新的持有链。
- `C4` 会话关闭能力：按关闭责任关闭单个已持有会话，或清理全部活跃持有关系。
- `C5` 非能力项：不负责事务提交与回滚，不负责 ORM 映射，不负责业务对象装配。

## 2. 边界定义（Boundary / 参数）

- `CI` 连接输入约束（constraint）：连接对象必须先在下层形成完整连接规格，并承接为可继续上推的连接载体。来源：`C1`。
- `SX` 会话展开约束（constraint）：会话上下文只能在合法连接载体上按复用方式与关闭责任展开。来源：`C1 + C3 + C4`。
- `HM` 持有映射约束（constraint）：所有受管会话都必须登记为可定位、可关闭的持有关系。来源：`C1 + C2 + C4`。
- `AI` 管理入口约束（interface）：连接进入、会话获取、句柄定位和关闭都必须经由本组件稳定公开入口进入。来源：`C1 + C2 + C3 + C4`。
- `IN` 组合范围限制（constraint）：本层只组合连接准备、会话上下文和持有映射，不直接承接事务或 ORM 语义。来源：`C1 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 连接准备基：L0.M10[R2] + L0.M11[R2]。来源：`C1 + CI + AI + IN`。
  - 注：该基负责把连接对象描述推进到可用连接载体。
- `B2` 会话上下文基：L0.M12[R2,R3,R4]。来源：`C1 + C3 + C4 + SX + AI`。
  - 注：该基负责把连接载体组织为可复用或独立展开的逻辑会话上下文。
- `B3` 持有映射基：L0.M4[R1,R2,R3,R4]。来源：`C1 + C2 + C4 + HM + AI`。
  - 注：该基负责把已取得的会话上下文登记为受管持有关系，并提供定位与原子关闭前提。

## 4. 基组合原则（Base Combination Principles）

- `R1` 连接准备组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有数据库会话获取都必须先形成完整连接规格，再完成连接承接；没有连接载体不承认进入会话管理路径。
  - `R1.3` 输出结构：`CONNECTION_READY`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`CI/AI/IN`。
- `R2` 会话获取组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：先由连接准备基形成连接载体，再由会话上下文基建立逻辑会话上下文，最后由持有映射基写入唯一句柄和开启态；跳过其中任一层都不承认是合法获取。
  - `R2.3` 输出结构：`MANAGED_SESSION_CONTEXT + MANAGED_HANDLE`。
  - `R2.4` 输出能力：`C1`。
  - `R2.5` 边界绑定：`CI/SX/HM/AI`。
- `R3` 会话定位组合
  - `R3.1` 参与基：`B3`。
  - `R3.2` 组合方式：定位动作只能通过持有映射基的句柄入口命中当前持有关系；没有句柄或句柄已关闭都不承认合法定位。
  - `R3.3` 输出结构：`MANAGED_SESSION_REF`。
  - `R3.4` 输出能力：`C2`。
  - `R3.5` 边界绑定：`HM/AI`。
- `R4` 会话复用组合
  - `R4.1` 参与基：`B2 + B3`。
  - `R4.2` 组合方式：复用或独立展开都必须先服从会话上下文基的作用域索引和收敛责任，再由持有映射基建立新的持有关系；不允许绕过上下文层直接拼接持有记录。
  - `R4.3` 输出结构：`REUSED_SESSION_REF + ISOLATED_SESSION_REF`。
  - `R4.4` 输出能力：`C3`。
  - `R4.5` 边界绑定：`SX/HM/AI`。
- `R5` 会话关闭组合
  - `R5.1` 参与基：`B2 + B3`。
  - `R5.2` 组合方式：关闭动作必须先通过持有映射基命中当前持有关系，再由会话上下文基完成上下文收敛；共享用法只收敛当前持有关系，独立用法必须连同对应上下文一起收敛。
  - `R5.3` 输出结构：`CLOSE_RESULT + CLEANUP_RESULT`。
  - `R5.4` 输出能力：`C4`。
  - `R5.5` 边界绑定：`SX/HM/AI`。
- `R6` 完整数据库会话管理组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：只承认这样一条完整路径：先准备连接对象，再建立会话上下文，再写入持有映射，并在该映射上完成获取、定位、复用和关闭；任何绕开这条路径的用法都不属于本组件承认的数据库会话管理。
  - `R6.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R6.4` 边界绑定：`CI/SX/HM/AI/IN`。
- `R7` 禁止组合
  - `R7.1` 参与基：`B1 + B2 + B3`。
  - `R7.2` 组合方式：没有连接准备直接获取会话、绕过会话上下文层直接伪造持有映射、没有句柄直接定位或关闭会话、把事务语义或 ORM 语义混入本组件的组合均无效。
  - `R7.3` 输出能力：`C5`。
  - `R7.4` 边界绑定：`CI/SX/HM/AI/IN`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R6` 输出能力并集必须覆盖 `C1~C4`。
- `V4` 边界符合性：所有合法用法都必须满足连接输入、会话展开、持有映射和管理入口约束。
- `V5` 最小必要性：移除任一 `B*` 后，连接准备、会话上下文组织或持有映射管理至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
- `V7` 能力落点一致性：`L3` 文档能力声明、`package.py capabilities`、`api.py/component.py` 中的公开出口必须一一对应。

# 会话上下文组件:SessionContext

@framework

组件目标：承接连接载体并形成可复用或独立展开的逻辑会话上下文。

## 1. 能力声明（Capability Statement）

- `C1` 上下文建立能力：把连接载体组织为带生命周期标识的逻辑会话上下文。
- `C2` 上下文复用能力：让多个持有关系按同一作用域锚点落回同一上下文来源，或按独立方式展开新上下文。
- `C3` 上下文收敛能力：在上下文结束时按既定收敛序列收敛资源关系。
- `C4` 非能力项：不负责句柄登记，不负责并发持有映射。

## 2. 边界定义（Boundary / 参数）

- `CC` 连接载体（connection_carrier）：输入必须是已承接的连接载体。来源：`C1 + C2`。
- `SK` 作用域键（scope_key）：必须能稳定标识一次复用范围。来源：`C2`。
- `PO` 复用方式（policy）：只允许 `shared | one-shot`。来源：`C2 + C3`。
- `CO` 收敛责任（owner）：只允许 `component | caller`。来源：`C2 + C3`。
- `LS` 上下文状态（context_state）：只允许 `active | closed`。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 作用域索引单元（自有基）：作用域到上下文舱引用的索引结构，由作用域键和值引用构成。来源：`C2 + SK + PO`。
- `B2` 上下文承载舱单元（自有基）：逻辑会话上下文的承载结构，由连接载体引用、上下文实例位和状态位构成。来源：`C1 + CC + LS`。
- `B3` 收敛序列单元（自有基）：上下文结束时的资源收敛结构，由收敛项序列和收敛挂点构成。来源：`C3 + CO + LS`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 载体承接组合
  - `R1.1` 参与基：`B2`。
  - `R1.2` 组合方式：所有会话上下文都必须先承接连接载体，再形成上下文舱；没有连接载体不承认建立上下文。
  - `R1.3` 输出结构：`CONTEXT_CABIN`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`CC/LS`。
- `R2` 上下文建立组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：上下文建立必须先把作用域键落入索引结构，再把连接载体组织为上下文舱；作用域键缺失时只允许独立展开。
  - `R2.3` 输出结构：`SESSION_CONTEXT`。
  - `R2.4` 输出能力：`C1`。
  - `R2.5` 边界绑定：`CC/SK/PO/LS`。
- `R3` 上下文复用组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：当复用方式为共享时，多个作用域键可以落回同一上下文舱；当复用方式为独立展开时，每次都必须形成新的上下文舱。
  - `R3.3` 输出结构：`SHARED_CONTEXT_REF + ISOLATED_CONTEXT_REF`。
  - `R3.4` 输出能力：`C2`。
  - `R3.5` 边界绑定：`SK/PO/LS`。
- `R4` 上下文收敛组合
  - `R4.1` 参与基：`B2 + B3`。
  - `R4.2` 组合方式：上下文结束时必须沿收敛序列逐项收敛；收敛责任决定由组件还是调用方触发该序列。
  - `R4.3` 输出结构：`CONTEXT_CLOSE_RESULT`。
  - `R4.4` 输出能力：`C3`。
  - `R4.5` 边界绑定：`PO/CO/LS`。
- `R5` 完整上下文组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：先承接连接载体，再建立上下文舱，随后按复用方式展开或复用上下文，最后按收敛序列结束上下文。
  - `R5.3` 输出能力：`C1 + C2 + C3`。
  - `R5.4` 边界绑定：`CC/SK/PO/CO/LS`。
- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：没有连接载体直接建立上下文、没有作用域索引直接宣称共享复用、没有收敛序列直接关闭上下文的组合均无效。
  - `R6.3` 输出能力：`C4`。
  - `R6.4` 边界绑定：`CC/SK/PO/CO/LS`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R5` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效上下文都必须满足连接载体、作用域键、复用方式、收敛责任和状态边界。
- `V5` 最小必要性：移除任一 `B*` 后，上下文建立、上下文复用或上下文收敛至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。

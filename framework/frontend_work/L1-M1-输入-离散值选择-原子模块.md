# 前端离散值选择原子模块:FrontendDiscreteValueSelectAtoms

@framework


## 1. 能力声明（Capability Statement）

- `C1` 选择入口能力：规定当前值放在哪里、选择入口如何承接，形成清楚的选择入口结构。
- `C2` 候选切换能力：规定候选项如何展开、切换和确认，形成清楚的候选切换结构。
- `C3` 当前值回填能力：规定选中结果如何返回入口，并形成稳定显示与反馈，形成清楚的回填结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责页面级业务流程、任务编排与跨模块选择协调。
- `N2` 不负责具体品牌样式、实例文案与页面布局方案。

## 2. 主边界定义（Primary Boundaries）


- `ENTRY` 入口边界：当前值显示、展开入口与相关说明由哪个位置承接。
- `OPTION` 候选边界：候选项集合、当前候选、候选切换关系与选中确认方式。
- `ACTION` 动作边界：展开、收起、切换、确认、清空等动作如何发出并进入处理。
- `STATE` 状态边界：默认、展开中、已选、禁用、错误等状态集合及其切换关系。
- `VISUAL` 视觉边界：选择入口、候选项与当前值显示如何保持清楚、稳定和可识别。


## 3. 最小可行基（Minimum Viable Bases）

- `B1` 选择入口基：`L0-M0[R3] + L0-M2[R1, R2]`。当前值显示和展开入口，形成稳定可识别的视觉起点。来源：`C1 + ENTRY + VISUAL`。
- `B2` 候选切换基：`L0-M0[R3] + L0-M1[R2]`。候选项展开、切换和确认，动作进入处理过程。来源：`C2 + OPTION + ACTION + STATE`。
- `B3` 当前值回填基：`L0-M0[R3] + L0-M1[R3] + L0-M2[R3, R4]`。选中结果回到入口，并附着到稳定的显示、状态与视觉反馈上。来源：`C3 + ENTRY + VISUAL`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 选择入口先成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先由 `L0-M0[R3] + L0-M2[R1, R2]` 建立稳定选择入口，使当前值显示和展开入口先有清楚的承接位置与基础视觉表达，再允许候选切换和结果回填附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`ENTRY + VISUAL`。

- `R2` 候选切换附着在选择入口上
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已成立的选择入口上，由 `L0-M1[R2]` 进一步建立展开、收起、切换和确认路径，使入口与候选切换形成清楚且稳定的关系。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`ENTRY + OPTION + ACTION + STATE`。

- `R3` 当前值回填附着在选择入口与候选切换上
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在已成立的入口与候选切换结构上，由 `L0-M1[R3] + L0-M2[R3, R4]` 进一步建立选中结果返回、状态变化与视觉反馈，使入口、候选切换和当前值回填形成清楚、稳定且可解释的闭合关系。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`ENTRY + OPTION + ACTION + STATE + VISUAL`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：如果没有先建立选择入口，就直接声明候选切换；或者候选切换发生了但没有选中结果；或者结果返回了但没有回填到入口与视觉反馈上；或者三者之间的对应关系说不清，这样的方案都无效。
  - `R4.3` 失效结论：`N1 + N2`。
  - `R4.4` 边界绑定：`ENTRY + OPTION + ACTION + STATE + VISUAL`。

## 5. 验证（Verification）

- `V1` 选择入口完整性：这个结构必须能说清当前值显示和展开入口放在哪里，由哪一部分承接，以及入口的基础视觉表达如何成立。
- `V2` 候选切换有效性：这个结构必须能说清候选项如何展开、切换和确认，相关动作与状态如何变化。
- `V3` 当前值回填有效性：这个结构必须能说清选中结果如何回到入口，并如何形成稳定显示与反馈。
- `V4` 组合闭合性：当选择入口、候选切换和当前值回填一起成立时，不能出现“有入口但不能展开”“有切换但没有选中结果”“有结果但没有回到入口”这类断链情况。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败，说明这三个基础部分都是必要的。
- `V6` 结论表达：验证结果应逐条说明哪条 `R*` 成立或失败、对应输出了什么能力或失效结论，并附上对应的边界证据。

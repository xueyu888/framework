# 前端容纳与绑定原子模块:FrontendCarrierBindingAtoms

@framework


直观示例：带前后缀与错误提示的输入框外壳

以一个常见的搜索输入框为例，其基础结构可分为主要内容区与辅助说明区两层：

+------------------------------------------------------+
| 输入框外壳                                          |
|                                                      |
|  +------------------------------------------------+  |
|  | 主要内容区                                     |  |
|  |                                                |  |
|  |  [前置内容位] [ 核心位置：输入 / 占位提示 ]   |  |
|  |   图标                                  [清空] |  |
|  |                                      后置动作位 |  |
|  +------------------------------------------------+  |
|                                                      |
|  +------------------------------------------------+  |
|  | 辅助说明区                                     |  |
|  | 帮助文本                                       |  |
|  | 错误文本                                       |  |
|  +------------------------------------------------+  |
+------------------------------------------------------+

这个例子说明：本模块不是规定某个具体组件长什么样，而是在规定一个前端结构至少要先形成“外层包裹 -> 固定位置 -> 核心位置 -> 信息与动作连接”的基础骨架，才能成为可复用的底层结构单元。

这里的“核心位置”，可以先理解为用户真正输入、点击或完成主要操作的地方。

## 1. 能力声明（Capability Statement）

- `C1` 结构分区能力：结构能清楚划分由哪些区域和位置组成，以及它们如何稳定排列，形成清楚的界面外壳。
- `C2` 核心连接能力：结构里真正承接主要操作的位置是什么，外部信息从哪里进入，用户动作从哪里触发，以及这些部分分别对应谁。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责完整交互流程、状态流转与可访问性细则。
- `N2` 不负责具体组件语义、主题值与项目实例布局方案。

## 2. 主边界定义（Primary Boundaries）


- `SURFACE` 分区边界：结构分成哪几个区域，这些区域如何组成一个稳定外壳。
- `SLOT` 位置边界：主要内容前后有哪些固定位置可以放内容或动作，这些位置的顺序如何。
- `HOST` 核心位置边界：结构里真正承接主要输入或主要操作的是哪一部分。
- `PROP` 连接边界：外部信息从哪里进入，结构里的动作从哪里发出，以及相关内容靠什么建立对应关系。


## 3. 最小可行基（Minimum Viable Bases）

- `B1` 外壳结构基：由主要内容区和辅助说明区组成的最小稳定外壳。定义结构的盒子模型。来源：`C1 + SURFACE`。
- `B2` 固定位置结构基：由前置位置、核心位置、后置位置等固定位置组成的最小稳定位置结构(嵌套的必要条件，前置位置，后置位置是否存在取决于位置边界)。来源：`C1 + SLOT`。
- `B3` 信息动作落点基：由核心位置、外部信息进入点、动作触发点和对应关系组成的最小稳定结构。来源：`C2 + HOST + PROP`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 外壳先成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先把主要内容区和辅助说明区组织成稳定外壳，再让更高层结构接在上面。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SURFACE`。

- `R2` 固定位置附着在外壳上
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已经成立的稳定外壳上，进一步说明前置位置、核心位置、后置位置等固定位置如何排列，使主要内容的落点清楚且稳定。
  - `R2.3` 输出能力：`C1`。
  - `R2.4` 边界绑定：`SURFACE + SLOT`。

- `R3` 信息动作落点附着在外壳与固定位置上
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在已经成立的稳定外壳和固定位置结构上，进一步说明哪一部分承接主要操作、外部信息从哪里进入、动作从哪里触发，以及说明和状态分别对应谁，使这些关系清楚、稳定且不混淆。
  - `R3.3` 输出能力：`C1 + C2`。
  - `R3.4` 边界绑定：`SURFACE + SLOT + HOST + PROP`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：如果没有先建立外壳，就直接声明固定位置或核心连接；或者核心位置悬空；或者信息、动作、说明之间的对应关系说不清，这样的方案都无效。
  - `R4.3` 失效结论：`N1 + N2`。
  - `R4.4` 边界绑定：`SURFACE + SLOT + HOST + PROP`。

## 5. 验证（Verification）

- `V1` 外壳完整性：这个结构必须能说清主要内容区和辅助说明区分别在哪里，并且两者组成的整体外壳不混乱。
- `V2` 固定位置完整性：这个结构必须能说清前置位置、核心位置、后置位置等固定位置分别在哪里，顺序不能混乱。
- `V3` 信息动作落点有效性：这个结构必须能说清哪一部分承接主要操作，外部信息从哪里进入，动作从哪里触发，相关说明分别对应谁。
- `V4` 组合闭合性：当外壳、固定位置和信息动作落点一起成立时，不能出现“有内容但没有位置”“有位置但没有承接点”“有动作但不知道作用于谁”“有提示但不知道提示谁”这类悬空情况。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败，说明这三个基础部分都是必要的。
- `V6` 结论表达：验证结果应逐条说明哪条 `R*` 成立或失败、对应输出了什么能力或失效结论，并附上对应的边界证据。

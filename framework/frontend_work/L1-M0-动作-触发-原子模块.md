# 前端动作触发原子模块:FrontendActionTriggerAtoms

@framework


## 1. 能力声明（Capability Statement）

- `C1` 入口承接能力：规定动作入口放在哪里、由什么结构承接，形成清楚的动作入口结构。
- `C2` 动作承接能力：规定用户如何发出动作、动作如何进入处理流程，形成清楚的动作承接结构。
- `C3` 反馈闭合能力：规定动作发出后的状态变化、结果返回与视觉反馈如何回到当前入口，形成清楚的反馈闭合结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责页面级业务流程、任务编排与跨模块动作协调。
- `N2` 不负责具体品牌样式、实例文案与页面布局方案。

## 2. 主边界定义（Primary Boundaries）


- `TRIGGER` 触发入口边界：动作入口由哪个位置承接、与周边结构如何对应，以及以什么方式触发。
- `ACTION` 动作边界：点击、确认、切换、回退等动作如何发出、进入处理过程并形成结果。
- `STATE` 状态边界：默认、悬停、按下、处理中、成功、失败、禁用等状态集合及其切换关系。
- `VISUAL` 视觉边界：动作入口的样式选择、布局组织与主题映射如何保持清楚、稳定和可识别。


## 3. 最小可行基（Minimum Viable Bases）

- `B1` 触发入口基：`L0-M0[R3] + L0-M2[R1]`。形成稳定可识别的视觉起点。来源：`C1 + TRIGGER + VISUAL`。
- `B2` 动作承接基：`L0-M0[R3] + L0-M1[R2]`。入口发出动作，动作进入处理流程。来源：`C2 + ACTION + STATE`。
- `B3` 反馈闭合基：`L0-M0[R3] + L0-M1[R3] + L0-M2[R3]`。动作发出后进入状态变化、返回结果，并附着到稳定视觉表达上。来源：`C3 + STATE + VISUAL`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 触发入口先成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先由 `L0-M0[R3] + L0-M2[R1]` 建立稳定入口结构，使动作入口先有清楚的承接位置和基础视觉表达，再允许动作与反馈附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TRIGGER + VISUAL`。

- `R2` 动作承接附着在触发入口上
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已成立的触发入口上，由 `L0-M1[R2]` 进一步建立动作发出与处理路径，使入口与动作承接形成清楚且稳定的关系。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`TRIGGER + ACTION + STATE`。

- `R3` 反馈闭合附着在触发入口与动作承接上
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在已成立的入口与动作承接结构上，由 `L0-M1[R3] + L0-M2[R3]` 进一步建立状态变化、结果返回与视觉反馈，使入口、动作和反馈形成清楚、稳定且可解释的闭合关系。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`TRIGGER + ACTION + STATE + VISUAL`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：如果没有先建立触发入口，就直接声明动作；或者动作发出了但没有进入状态处理；或者结果返回了但没有附着到入口与视觉反馈上；或者三者之间的对应关系说不清，这样的方案都无效。
  - `R4.3` 失效结论：`N1 + N2`。
  - `R4.4` 边界绑定：`TRIGGER + ACTION + STATE + VISUAL`。

## 5. 验证（Verification）

- `V1` 触发入口完整性：这个结构必须能说清动作入口放在哪里，由哪一部分承接，以及入口的基础视觉表达如何成立。
- `V2` 动作承接有效性：这个结构必须能说清用户如何从入口发出动作，动作如何进入处理流程，动作类型不能混乱。
- `V3` 反馈闭合有效性：这个结构必须能说清动作发出后状态如何变化、结果如何返回，以及反馈如何附着到当前入口上。
- `V4` 组合闭合性：当触发入口、动作承接和反馈闭合一起成立时，不能出现“有入口但不能触发”“有触发但没有状态结果”“有反馈但没有回到入口”这类断链情况。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败，说明这三个基础部分都是必要的。
- `V6` 结论表达：验证结果应逐条说明哪条 `R*` 成立或失败、对应输出了什么能力或失效结论，并附上对应的边界证据。

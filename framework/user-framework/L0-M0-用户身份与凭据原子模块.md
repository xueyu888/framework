# 用户身份与凭据原子模块:UserIdentityCredentialAtoms

@framework

本模块定义用户框架里最底层的身份与凭据部件。它只回答 5 件事：主用户怎么放、登录标识怎么放、用户当前是否可用怎么放、密码规则怎么放、密码证明怎么放；不负责失败锁定、令牌签发和权限判定。

## 1. 能力声明（Capability Statement）

- `C1` 用户定位定义能力：定义一个用户怎样被唯一定位。
- `C2` 密码规则定义能力：定义密码复杂度和复用窗口怎样被稳定表达。
- `C3` 凭据证明定义能力：定义密码摘要、历史摘要与比对结果怎样被稳定表达。
- `C4` 非能力项：不负责失败锁定、令牌签发与角色权限判定。

## 2. 边界定义（Boundary / 参数）

- `USERKEY` 用户标识边界：同一登录标识只能指向一个主用户。来源：`C1`。
- `USERSTATE` 用户状态边界：同一用户只能落在可用、禁用或冻结等声明过的状态位中。来源：`C1`。
- `PWDRULE` 密码规则边界：最小长度、字符种类下限、禁用模式与复用窗口必须显式声明。来源：`C2`。
- `PWDPROOF` 凭据证明边界：当前摘要、历史摘要与比对结果必须能稳定表达。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 用户主键位基：存放用户主标识和登录标识，能根据登录标识指向唯一用户。来源：`C1 + USERKEY`。
- `B2` 用户状态位基：存放可用、禁用和冻结状态，能标记当前用户是否允许继续进入认证链。来源：`C1 + USERSTATE`。
- `B3` 密码规则位基：存放长度阈值、字符种类下限、禁用模式和复用窗口，能表达一套可执行的密码规则。来源：`C2 + PWDRULE`。
- `B4` 密码证明位基：存放当前密码摘要、历史摘要和比对结果，能表达当前密码是否匹配。来源：`C3 + PWDPROOF`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 身份定位成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先把主用户位和登录标识位接成唯一主体，再补齐当前状态位，形成可定位主体。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`USERKEY/USERSTATE`。
- `R2` 规则证明成立
  - `R2.1` 参与基：`B3 + B4`。
  - `R2.2` 组合方式：先固定密码规则位，再把当前摘要和历史摘要槽接上，形成可比对凭据。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`PWDRULE/PWDPROOF`。
- `R3` 主体凭据成立
  - `R3.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R3.2` 组合方式：把可定位主体和可比对凭据接到同一用户记录上，形成完整身份凭据事实。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`USERKEY/USERSTATE/PWDRULE/PWDPROOF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：登录标识不唯一、状态位缺失、密码规则未声明或摘要无法比对的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`USERKEY/USERSTATE/PWDRULE/PWDPROOF`。

## 5. 验证（Verification）

- `V1` 身份定位有效性：主用户位、登录标识位和状态位必须满足 `USERKEY/USERSTATE` 约束。
- `V2` 规则证明有效性：密码规则位和密码证明位必须满足 `PWDRULE/PWDPROOF` 约束。
- `V3` 主体凭据有效性：同一用户记录必须同时具备可定位主体和可比对凭据。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

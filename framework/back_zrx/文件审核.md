# 文件审核:FileReviewer

@framework

## 0. 目标
对数据清洗模块所得的文件内部的分块进行审核，确保其符合预期的格式和内容


## 1. 能力声明（Capability Statement）

- `C1` 文件块存取能力: 能保存并读取文件块的信息
- `C2` 文件块编辑能力: 能编辑(修改保存)文件分块中的内容
- `C3` 文件块与文件对应能力: 能查询某个文件拥有的分块, 或某个文件块所属的文件
- `C4` 文件块状态流转能力: 根据不同的用户操作, 转换文件分块的状态
- `C5` 文件状态流转能力: 能根据文件内部分块的状态, 转换文件的状态
- `C6` 审计能力: 能记录文件审核操作, 并生成审计日志
- `C7` 质量统计能力: 能统计文件分块的质量指标, 如分块数量, 分块大小, 分块状态分布等

## 2. 边界定义（Boundary / 参数）

- `P1` 文件块存取形式边界: 是所有文件块随便存好, 还是按文件分类存储?
- `P2` 单文件块大小边界: 单个文件块所能存储数据的最大和最小空间
- `P3` 整体文件块的大小边界: 单个文件所能存储数据的最大和最小空间
- `P4` 文件块的信息边界: 文件块 id, 文件块对应的文件 id, 文件块原始内容, 原文件中的位置信息, 文件块状态(这个要在这个里面吗?)
- `P5` 文件块编辑边界: 仅能编辑文件分块中的原始内容

## 4. 基组合原则（Base Combination Principles）

- `R1` 结构装配组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定结构接口，再收敛状态通路与装配关系。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`P1/P2/P3`。
- `R2` 上层展开组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：在装配结果上叠加扩展约束，形成可继续生长的模块结构。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`P1/P2/P3`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：脱离下层结构直接按功能清单拼接，或违反边界约束的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`P1/P2/P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力/边界绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效组合必须满足绑定的参数边界。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。

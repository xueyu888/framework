# 文件审核:FileReviewer

@framework

## 0. 目标

对数据清洗模块所得的文件内部的分块进行审核，确保其符合预期的格式和内容

## 1. 能力声明（Capability Statement）

- `C1` 文件块存取能力: 能保存并读取文件块的信息
- `C2` 文件块编辑能力: 能编辑(修改保存)文件分块中的内容
- `C3` 文件块与文件对应能力: 能查询某个文件拥有的分块, 或某个文件块所属的文件
- `C4` 文件块状态流转能力: 根据不同的用户操作, 转换文件分块的状态
- `C5` 文件状态流转能力: 能根据文件内部分块的状态, 转换文件的状态
- `C6` 审计能力
- `C7` 质量统计能力

## 2. 边界定义（Boundary / 参数）

- `P1` 文件块存取形式边界: 逻辑上按文件分类存储, 物理上仅需保证所有文件块在同一个物理数据库实例中. 来源: `C1`
- `P2` 文件块的信息边界: 文件块 id, 文件块对应的文件 id, 文件块原始内容, 原文件中的位置信息. 来源: `C1`
- `P3` 文件块大小边界: 单个文件块所能存储数据存在最大和最小空间. 来源: `C1`
- `P4` 文件块编辑形式边界: 实质是仅能在任务流转中生成独立的修改后内容. 来源: `C2`
- `P5` 单文件内的文件块数量边界: 单个文件所能存储文件块的最大数量和最小数量. 来源: `C3`
- `P6` 文件块状态边界: 自动确认, 待人工审核, 人工编辑过, 人工确认过. 来源: `C4`
- `P7` 文件块状态流转边界: 待人工审核 -> 人工编辑过, 待人工审核 -> 人工确认过. 来源: `C4`
- `P8` 文件状态边界: 待审核, 审核中, 已审核. 来源: `C5`
- `P9` 文件状态流转边界: 待审核 -> 审核中, 审核中 -> 已审核, 审核中 -> 待审核. 来源: `C5`

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件块存取 来源: `C1 + C3 + P1 + P2 + P3 + P5`
- `B2` 文件块状态流转状态机 来源: `C2 + C4 + P4 + P6 + P7`
- `B3` 文件状态流转状态机 来源: `C5 + P8 + P9`

# 文件块存取模块:FileChunkStorageModule

@framework

## 0. 目标
- 设计一个模块, 能存取文件块, 记录了文件和文件块之间的对应关系, 并支持删除一个文件对应的所有文件块

## 1. 能力声明（Capability Statement）

- `C1` 文件块存取: 能够将文件块批量存入存储载体, 或从存储载体中批量取出文件块
- `C2` 文件块与文件对应能力: 能查询某个文件拥有的文件块, 或根据某个文件块查询其所属的文件
- `C3` 集合销毁能力: 可根据文件标识，一次性删除一个文件拥有的所有文件块

### 非职责声明（Non-Responsibility Statement）

- `N1` 不提供更新能力: 不提供对单独某个文件块的更新能力
- `N2` 不提供单点销毁能力: 不提供针对单一文件块的独立删除能力

## 2. 边界定义（Boundary / 参数）

- `P1` 文件块内容大小边界: 每个文件块内容的大小限制. 来源: `C1 + B1`
- `P2` 文件块容器类型边界: 关系型数据库(VARCHAR, TEXT, MEDIUMTEXT), minio, 内存. 来源: `C1 + B8`
- `P3` 文件块批量存放数量边界: 每批次存放的文件块数量存在限制. 来源: `C1`
- `P4` 文件块批量取出数量边界: 每批次取出的文件块数量存在限制. 来源: `C1`
- `P5` 获取文件中的块索引数量边界: 每次获取一个文件中拥有的文件块的索引的数量存在限制. 来源: `C2`
- `P6` 销毁粒度边界: 执行销毁操作时，入参必须为文件标识，不可为单一分块标识. 来源: `C3`
- `P7` 文件块总数量边界: 此模块能存储的文件块总数存在限制. 来源: `C1 + B2`
- `P8` 文件总数量边界: 此模块能存储的文件总数存在限制. 来源: `C1 + B3`

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件块内容
- `B2` 文件块索引号
- `B3` 文件块所属文件号
- `B4` 查找器
- `B5` 删除器
- `B6` 提取器
- `B7` 存入器
- `B8` 文件块内容容器
- `B9` 文件块索引号容器
- `B10` 文件块所属文件号容器

## 4. 基组合原则（Base Combination Principles）

- `R1` 形成文件
    - `R1.1` 参与基：`B1`
    - `R1.2` 组合方式: 重复堆叠 `B1`
    - `R1.3` 边界绑定: `P1`
    - `R1.4` 输出能力: 文件
- `R2` 无索引文件块存储仓
    - `R2.1` 参与基：`B8`
    - `R2.2` 组合方式: 重复堆叠 `B8` 
    - `R2.3` 边界绑定: `P2`
    - `R2.4` 输出能力: 无索引文件块存储仓
- `R3` 带索引文件块存储仓
    - `R3.1` 参与基：`B8 + B2`
    - `R3.2` 组合方式: 一个 `B8` 对应一个 `B2`
    - `R3.3` 边界绑定: `P2`
    - `R3.4` 输出能力: 带索引文件块存储仓
- `R4` 带索引的文件块
    - `R4.1` 参与基：`B1 + B2`
    - `R4.2` 组合方式: 一个 `B1` 对应一个 `B2`
    - `R4.3` 边界绑定: `P1 + P7`
    - `R4.4` 输出能力: 带索引的文件块
- `R5` 带归属的文件块
    - `R5.1` 参与基：`B1 + B3`
    - `R5.2` 组合方式: 一个 `B1` 对应一个 `B3`, 一个 `B3` 对应多个 `B1`
    - `R5.3` 边界绑定: `P1 + P8`
    - `R5.4` 输出能力: 带归属的文件块

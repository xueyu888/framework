# 文件块存取模块:FileChunkStorageModule

@framework

## 0. 目标

- 设计一个模块, 能存取文件块, 记录了文件和文件块之间的对应关系, 并支持删除一个文件对应的所有文件块

## 1. 能力声明（Capability Statement）

- `C1` 文件块存取: 能够将文件块批量存入存储载体, 或从存储载体中批量取出文件块
- `C2` 文件块与文件对应能力: 能查询某个文件拥有的文件块, 或根据某个文件块查询其所属的文件
- `C3` 集合销毁能力: 可根据文件标识，一次性删除一个文件拥有的所有文件块

### 非职责声明（Non-Responsibility Statement）

- `N1` 不提供更新能力: 不提供对单独某个文件块的更新能力
- `N2` 不提供单点销毁能力: 不提供针对单一文件块的独立删除能力

## 2. 边界定义（Boundary / 参数）

- 文件块内容边界:
    - 文件块内容本身:
        - `P1` 文件块内容大小边界: 每个文件块内容的大小限制. 来源: `C1 + B1`
        - `P2` 文件块总数量边界: 此模块能存储的文件块总数存在限制. 来源: `C1 + B1 + B2`
        - `P3` 文件块批量存放数量边界: 每批次存放的文件块数量存在限制. 来源: `C1`
        - `P4` 文件块批量取出数量边界: 每批次取出的文件块数量存在限制. 来源: `C1`
    - 文件块内容容器:
        - `P5` 文件块内容容器类型边界: 关系型数据库(VARCHAR, TEXT, MEDIUMTEXT), minio, 内存. 来源: `C1 + B1`
    
- 文件块索引边界:
    - 文件块索引本身:
        - `P6` 文件块索引号大小边界: 每个文件块索引号的大小限制. 来源: `C1 + B2`
        - `P7` 文件块索引算法边界: 用于生成文件块索引号的算法. 来源: `C1 + B2`
    - 文件块索引号容器:
        - `P8` 文件块索引号容器类型边界: 关系型数据库(INT, VARCHAR) 内存. 来源: `C1 + B2`
    
- 文件号边界:
    - 文件号本身:
        - `P9` 文件总数量边界: 此模块能存储的文件号总数存在限制. 来源: `C2 + B3`
        - `P10` 文件号算法边界: 用于生成文件号的算法. 来源: `C2 + B3`
        - `P11` 文件可对应块数量边界: 一个文件可对应的文件块 来源: `C2 + B3`
        - `P12` 每次获取一个文件对应的文件块的数存在限制. 来源: `C2`
    - 文件号容器:
        - `P13` 文件号容器类型边界: 关系型数据库(INT, VARCHAR), 内存. 来源: `C2 + B3`
    
- `P14` 销毁粒度边界: 执行销毁操作时，入参必须为文件标识，不可为单一分块标识. 来源: `C3`

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件块内容
- `B2` 文件块索引号
- `B3` 文件块所属文件号
- `B4` b+树索引结构

## 4. 基组合原则（Base Combination 

- `R1` 文件
    - `R1.1` 参与基：`B1`
    - `R1.2` 组合方式: `B1` 相互组合
    - `R1.3` 边界绑定: `P1 + P2 + P5`
    - `R1.4` 输出能力: 文件
- `R2` 带索引的文件块
    - `R2.1` 参与基：`B1 + B2 + B4`
    - `R2.2` 组合方式: 使用一棵 `B4`, 将 `B2` 填充到非叶子节点中作为索引, 在叶子节点中存储 `B1`
    - `R2.3` 边界绑定: `P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8`
    - `R2.4` 输出能力: 带索引的文件块
- `R3` 带索引和归属的文件块
    - `R3.1` 参与基：`B1 + B2 + B3 + B4`
    - `R3.2` 组合方式: 将 `B2` 填充到 `B4` 中的非叶子节点中作为索引, 在叶子节点中存储 `B1` 和 `B3`, 形成一棵 `B4`, 再将 `B3` 和 `B2` 拼接, 填充到另一个 `B4` 的非叶子节点中作为索引, 在叶子节点中存储 `B1` 和 `B2`
    - `R3.3` 边界绑定: `P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11 + P12 + P13 + P14`
    - `R3.4` 输出能力: 带索引和归属的文件块

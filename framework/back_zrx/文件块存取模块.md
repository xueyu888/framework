# 文件块存取模块:FileChunkStorageModule

@framework

## 0. 目标
- 设计一个模块, 能存取文件块, 记录了文件和文件块之间的对应关系, 并支持删除一个文件对应的所有文件块

## 1. 能力声明（Capability Statement）

- `C1` 文件块存取: 能够将文件块批量存入存储载体, 或从存储载体中批量取出文件块
- `C2` 文件块与文件对应能力: 能查询某个文件拥有的文件块, 或根据某个文件块查询其所属的文件
- `C3` 集合销毁能力: 可根据文件标识，一次性删除一个文件拥有的所有文件块

### 非职责声明（Non-Responsibility Statement）

- `N1` 不提供更新能力: 不提供对单独某个文件块的更新能力
- `N2` 不提供单点销毁能力: 不提供针对单一文件块的独立删除能力

## 2. 边界定义（Boundary / 参数）
- 文件块内容边界:
    - 文件块内容本身:
        - `FILE_CHUNK_SIZE` 文件块内容大小边界: 每个文件块内容的大小限制. 来源: `C1 + B1`
        - `FILE_CHUNK_TOTAL_NUM` 文件块总数量边界: 此模块能存储的文件块总数存在限制. 来源: `C1 + B1 + B2`
        - `FILE_CHUNK_BATCH_PUT_NUM` 文件块批量存放数量边界: 每批次存放的文件块数量存在限制. 来源: `C1`
        - `FILE_CHUNK_BATCH_GET_NUM` 文件块批量取出数量边界: 每批次取出的文件块数量存在限制. 来源: `C1`
    - 文件块内容容器:
        - `FILE_CHUNK_CONTENT_CONTAINER_TYPE` 文件块内容容器类型边界: 关系型数据库(VARCHAR, TEXT, MEDIUMTEXT), minio, 内存. 来源: `C1 + B5`
    
- 文件块索引边界:
    - 文件块索引本身:
        - `FILE_CHUNK_INDEX_SIZE` 文件块索引号大小边界: 每个文件块索引号的大小限制. 来源: `C1 + B2`
        - `FILE_CHUNK_INDEX_ALGO` 文件块索引算法边界: 用于生成文件块索引号的算法. 来源: `C1 + B2`
    - 文件块索引号容器:
        - `FILE_CHUNK_INDEX_CONTAINER_TYPE` 文件块索引号容器类型边界: 关系型数据库(INT, VARCHAR) 内存. 来源: `C1 + B6`
    
- 文件号边界:
    - 文件号本身:
        - `FILE_TOTAL_NUM` 文件总数量边界: 此模块能存储的文件号总数存在限制. 来源: `C2 + B3`
        - `FILE_ID_ALGO` 文件号算法边界: 用于生成文件号的算法. 来源: `C2 + B3`
        - `FILE_CHUNK_IN_FILE_NUM` 文件可对应块数量边界: 一个文件可对应的文件块 来源: 来源: `C2 + B3`
        - `BATCH_GET_FILE_CHUNK_IN_FILE_NUM` 每次获取一个文件对应的文件块的数存在限制. 来源: `C2`
    - 文件号容器:
        - `FILE_ID_CONTAINER_TYPE` 文件号容器类型边界: 关系型数据库(INT, VARCHAR), 内存. 来源: `C2 + B7`
    
- `P6` 销毁粒度边界: 执行销毁操作时，入参必须为文件标识，不可为单一分块标识. 来源: `C3`

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件块内容
- `B2` 文件块索引号
- `B3` 文件块所属文件号
- `B4` 查找器
- `B5` 文件块内容容器
- `B6` 文件块索引号容器
- `B7` 文件块所属文件号容器

## 4. 基组合原则（Base Combination Principles）
- 容器:
    - `R2` 无索引文件块存储仓
        - `R2.1` 参与基：`B5`
        - `R2.2` 组合方式: 重复堆叠 `B5` 
        - `R2.3` 边界绑定: `FILE_CHUNK_SIZE + FILE_CHUNK_TOTAL_NUM`
        - `R2.4` 输出能力: 无索引文件块存储仓
    - `R3` 带索引的文件块存储仓
        - `R3.1` 参与基：`B5 + B2 + B6`
        - `R3.2` 组合方式: 一个 `B2` 置于一个 `B6`之中, `B2` 与 `B5` 一一对应
        - `R3.3` 边界绑定: `FILE_CHUNK_SIZE + FILE_CHUNK_TOTAL_NUM + FILE_CHUNK_INDEX_SIZE + FILE_CHUNK_INDEX_ALGO + FILE_CHUNK_INDEX_CONTAINER_TYPE`
        - `R3.4` 输出能力: 带索引的文件块存储仓
    - `R8` 带归属的文件块存储仓
        - `R8.1` 参与基：`B5 + B3 + B7`
        - `R8.2` 组合方式: 一个 `B3` 置于一个 `B7` 之中, 一个 `B3` 对应多个 `B3`
        - `R8.3` 边界绑定: `FILE_CHUNK_SIZE + FILE_CHUNK_TOTAL_NUM + FILE_TOTAL_NUM + FILE_ID_ALGO + FILE_CHUNK_IN_FILE_NUM + FILE_ID_CONTAINER_TYPE`
        - `R8.4` 输出能力: 带归属的文件块存储仓
    - `R4` 带索引和归属的文件块存储仓
        - `R4.1` 参与基：`B5 + B2 + B6 + B3 + B7`
        - `R4.2` 组合方式: 一个 `B2` 置于一个 `B6`之中, 一个 `B3` 置于一个 `B7` 之中, `B2` 与 `B5` 一一对应, 一个 `B3` 对应多个 `B5`
        - `R4.3` 边界绑定: `FILE_CHUNK_SIZE + FILE_CHUNK_TOTAL_NUM + FILE_CHUNK_INDEX_SIZE + FILE_CHUNK_INDEX_ALGO + FILE_CHUNK_INDEX_CONTAINER_TYPE + FILE_TOTAL_NUM + FILE_ID_ALGO + FILE_CHUNK_IN_FILE_NUM + FILE_ID_CONTAINER_TYPE`
        - `R4.4` 输出能力: 带索引和归属的文件块存储仓

    - `R5` 带索引的文件块
        - `R5.1` 参与基：`B1 + B5 + B2 + B6`

        - `R5.3` 边界绑定:

# 文件审核前端交互:FileAuditFrontend

@framework

## 0. 目标（Goal）

本模块专门处理文件审核中各个模块的连接, 并在前端下发不同 api 时, 按一定的逻辑和规则调用相应的模块, 并返回结果给前端.

## 1. 能力声明（Capability Statement）

- `C1` 文件状态查询: 前端下发文件 id 时, 返回此文件对应的状态.
- `C2` 审核文件内容获取: 前端下发文件 id 时, 返回此文件中所有待审文件块及自动确认文件块.
- `C3` 审核文件块: 前端下发文件块 id 及编辑的信息, 记录或更新此文件块的审核结果.
- `C4` 开始审核: 前端下发文件 id 时, 记录此文件已开始审核.

### 非职责声明（Non-Responsibility Statement）

- `N1` 不提供已归档文件的状态查询
- `N2` 不提供已归档文件的文件块内容
- `N3` 此模块不自己记录任何信息, 记录相关操作均借由其他模块完成

# 运行环境检测模块:RuntimeEnvDetector

@framework

## 0. 目标（Goal）
- 检测一个 python 程序的运行环境是开发环境还是打包环境, 输出判断结果和程序所在文件夹路径

## 1. 能力声明（Capability Statement）
- `C1` 注入变量读取能力: 能够读取程序启动时被注入的变量.
- `C2` 程序路径读取能力: 能够读取程序所在文件夹的路径.
- `C3` 环境定义能力: 定义了开发环境和打包环境两种环境.
- `C4` 环境判断能力: 能判断程序运行所在的环境是定义的哪种环境.


## 2. 边界定义（Boundary / Parameter 参数）
- `VAR` 被注入变量边界: 需要读取的被注入变量有哪些. 来源: `C1`
- `PATH` 路径长度边界: 读取程序所在文件夹的路径的长度存在上限 Lmax 和下限 Lmin. 来源: `C2`
- `ENV` 环境数量边界: 定义了 N 个环境种类. 来源: `C3`
- `ENV2` 可判断环境数量边界: 能判断的环境数量为 M, M < N. 来源: `C4`

## 3. 最小结构基（Minimal Structural Bases）
- `B1` 程序事实提取器: 借由系统/库函数实现, 能够从运行程序的环境中提取出相关的事实. 来源: `C1 + C2 + PATH + VAR`
- `B2` 环境种类-程序事实映射表: [环境种类及其对应的程序事实映射表](../../docs/back_zrx/环境种类-程序事实映射表.md). 来源: `C3 + ENV`


## 4. 基组合原则（Base Combination Principles）
- `R1` 运行环境检测
    - `R1.1` 参与基: `B1 + B2`
    - `R1.2` 组合方式: 用 `B1` 提取出事实, 与 `B2` 中的程序事实进行比较, 可输出判断结果.
    - `R1.4` 边界绑定: `ENV + ENV2`
    - `R1.3` 输出能力: `C4`
- `R2` 事实提取
    - `R2.1` 参与基: `B1`
    - `R2.2` 组合方式: 用 `B1` 提取出程序所在文件夹的路径.
    - `R2.4` 边界绑定: `VAR + PATH`
    - `R2.3` 输出能力: `C1 + C2`

## 5. 验证（Verification）
- `V1` 注入变量能力验证: 在不同环境下, 用 `R2` 对程序运行环境中的事实进行检测, 验证是否能正确读取程序启动时被注入的变量.
- `V2` 路径读取能力验证: 将程序预先置于各种不同路径中(须在设定的路径长度上下限范围内), 用 `R2` 提取出程序的运行路径, 并与真实路径进行对比, 验证是否能正确输出程序所在文件夹的路径.
- `V3` 环境判断能力验证: 用 `R1` 对程序运行环境中的事实进行检测, 验证是否能正确判断出环境种类.

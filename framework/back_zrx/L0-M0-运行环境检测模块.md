# 运行环境检测模块:RuntimeEnvDetector

@framework

## 0. 目标（Goal）
- 检测一个 python 程序的运行环境是开发环境还是打包环境, 输出判断结果和程序所在文件夹路径

## 1. 能力声明（Capability Statement）
- `C1` 环境判定能力: 若预定义好了 a1, a2, ..., an 共 n 种环境, 能以布尔形式输出 is_a1, is_a2, ..., is_an 的判断结果
- `C2` 路径识别能力: 输出程序所在文件夹的路径
- `C3` 环境比较能力: 能对提取出的环境事实进行比较, 输出判断结果.
- `C4` 路径长度异常处理能力: 若程序所在文件夹的路径长度超出了预定义的上下限, 能抛出异常, 确保程序不会崩溃.

## 2. 边界定义（Boundary / 参数）
- `NUM` 环境数量边界: 可检测的环境种类数 N, N >= 2. 来源: `C1`
- `PATH` 路径长度边界: 输出路径的长度存在上下限. 来源: `C2`

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 环境事实提取器: 借由系统/库函数实现, 能够从运行程序的环境中提取出相关的事实, 如代表了当前环境的特殊变量, 程序所在文件夹的路径, 程序的父进程 id 等. 来源: `C2 + PATH`
- `B2` 环境映射表: 环境种类及其对应的预期特征映射表. 来源: `C1 + NUM`
- `B3` 比较器: 借由系统/库函数实现, 能对两个变量进行比较, 输出布尔值. 来源: `C3 + PATH`
- `B4` 路径长度异常处理器: 借由系统/库函数实现, 能对程序所在文件夹的路径长度进行判断, 若超出了预定义的上下限, 则抛出异常. 来源: `C4 + PATH`

## 4. 基组合原则（Base Combination Principles）
- `R1` 环境事实比较
    - `R1.1` 参与基: `B1 + B3`
    - `R1.2` 组合方式: 用 `B1` 提取出多份事实, 相互之间用 `B3` 比较, 输出判断结果.
    - `R1.3` 输出能力: `C3`
    - `R1.4` 边界绑定: `NUM`
- `R2` 无异常处理的运行环境检测
    - `R2.1` 参与基: `B1 + B2 + B3`
    - `R2.2` 组合方式: 用 `B1` 提取出事实, 与 `B2` 中的预期特征用 `B3` 进行比较, 输出判断结果.
    - `R2.4` 边界绑定: `NUM`
    - `R2.3` 输出能力: `C1`
- `R3` 无异常处理的路径识别
    - `R3.1` 参与基: `B1`
    - `R3.2` 组合方式: 用 `B1` 提取出程序所在文件夹的路径.
    - `R3.3` 输出能力: `C2`
    - `R3.4` 边界绑定: `PATH`
- `R4` 带异常处理的路径识别
    - `R4.1` 参与基: `B1 + B4`
    - `R4.2` 组合方式: 用 `B1` 提取出程序所在文件夹的路径, 并与 `B4` 中的路径长度上下限进行比较.
    - `R4.3` 输出能力: `C2 + C4`
    - `R4.4` 边界绑定: `PATH`
- `R5` 带路径异常处理的运行环境检测
    - `R5.1` 参与基: `B1 + B2 + B3 + B4`
    - `R5.2` 组合方式: 用 `B1` 提取出事实, 与 `B2` 中的预期特征用 `B3` 进行比较, 若路径长度超出了 `B4` 中的上下限, 则抛出异常.
    - `R5.3` 输出能力: `C1 + C4`
    - `R5.4` 边界绑定: `NUM + PATH`

## 5. 验证（Verification）
- `V1` 环境判定能力验证: 在已明确环境映射表的情况下, 用 `R2` 对程序运行环境进行检测, 验证是否能正确判断出环境种类.
- `V2` 路径识别能力验证: 用 `R3` 对程序运行路径进行检测, 验证是否能正确输出程序所在文件夹的路径.
- `V3` 环境比较能力验证: 用 `R1` 对程序运行环境中的事实进行比较, 验证环境事实的比较结果是否符合预期.
- `V4` 路径长度边界验证: 在设定的上下限路径长度路径下, 用 `R3` 对程序运行路径进行检测, 验证是否能正确输出路径.
- `V5` 路径长度异常处理能力验证: 在超出预定义路径长度上下限的路径下, 用 `R4` 和 `R5` 对程序运行路径进行检测, 验证程序是否能正确抛出异常.

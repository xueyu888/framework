# 父子任务状态管理模块:TaskStateTransformer

@framework

## 0. 目标

设计一个模块, 此模块初始接受传入一个父任务的所有子任务的数量, 并不断接收某个子任务完成后的事件, 记录每个子任务的完成状态, 并根据子任务的完成状态, 更新父任务的状态

## 1. 能力声明（Capability Statement）

- `C1` 记录某个父任务的所有子任务的数量, 父任务的完成状态名, 子任务的完成状态名
- `C2` 传入父任务 id, 可查询父任务相关信息
- `C3` 传入子任务 id, 可查询子任务相关信息
- `C4` 传入一条子任务记录时, 可以新增或更新此条子任务的记录
- `C5` 传入一条父任务记录时, 可以新增此条父任务的记录
- `C5` 新增或更新子任务记录时, 能根据子任务的状态, 自动更新父任务中已完成的子任务数量和父任务状态
- `C6` 传入父任务 id, 可删除该父任务的记录信息, 并删除其对应的所有子任务的信息


### 非职责声明（Non-Responsibility Statement）

- `N1` 不能单独删除某一子任务的记录
- `N2` 不提供更新父任务记录的能力


## 2. 边界定义（Boundary / Parameter 参数）
- 输入参数:
    - 更新一条子任务信息:
        - 子任务 id
        - 对应父任务 id
        - 子任务状态
        - 子任务关心的其他信息
    - 清除父任务:
        - 父任务 id
    - 查询父任务:
        - 父任务 id
    - 查询子任务:
        - 子任务 id
- 输出参数:
    - 更新一条子任务信息:
        - true / false: 代表更新成功或失败
    - 清除父任务:
        - true / false: 代表清除成功或失败
    - 查询父任务:
        - 父任务 id
        - 总子任务数
        - 已完成子任务数
        - 父任务状态
        - 父任务关心的其他信息
    - 查询子任务:
        - 子任务 id
        - 对应父任务 id
        - 子任务状态
        - 子任务关心的其他信息

- `P` 父任务的子任务数量边界
- `P` 父任务状态数量边界
- `P` 子任务状态数量边界

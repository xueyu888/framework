# 父子任务状态管理模块:TaskStateTransformer

@framework

## 0. 目标

设计一个模块, 此模块初始接受传入一个父任务的所有子任务的数量, 并不断接收某个子任务完成后的事件, 记录每个子任务的完成状态, 并根据子任务的完成状态, 更新父任务的状态

## 1. 能力声明（Capability Statement）

- `C1` 记录某个父任务的所有子任务的数量, 父任务的完成状态名, 子任务的完成状态名
- `C2` 传入父任务或子任务 id, 可查询其对应的状态
- `C3` 传入子任务 id, 对应父任务 id, 子任务状态, 更新信息时, 保存此子任务的状态记录, 并能自动更新父任务中已完成的子任务数量和父任务状态
- `C4` 传入父任务 id, 可清空该父任务的所有子任务的状态记录, 并删除此父任务的记录


### 非职责声明（Non-Responsibility Statement）

- `N1` 不能单独删除某一子任务的记录


## 2. 边界定义（Boundary / Parameter 参数）

- `P` 父任务的子任务数量边界
- `P` 父任务状态数量边界
- `P` 子任务状态数量边界

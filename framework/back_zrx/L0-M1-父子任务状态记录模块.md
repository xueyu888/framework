# 父子任务状态记录模块:TaskStateRecorder

@framework

## 0. 目标

设计一个模块, 此模块初始接受传入一个父任务的所有子任务的数量, 并不断接收某个子任务完成后的事件, 记录每个子任务的完成状态, 并根据子任务的完成状态, 更新父任务的状态

## 1. 能力声明（Capability Statement）

- `C2` 传入父任务 id, 可查询父任务相关信息
- `C3` 传入子任务 id, 可查询子任务相关信息
- `C4` 传入一条子任务记录时, 可以新增或更新此条子任务的记录
- `C5` 传入一条父任务记录时, 可以新增此条父任务的记录
- `C6` 新增或更新子任务记录时, 能根据子任务的状态, 自动更新父任务中已完成的子任务数量和父任务状态
- `C7` 传入父任务 id, 可删除该父任务的记录信息, 并删除其对应的所有子任务的信息


### 非职责声明（Non-Responsibility Statement）

- `N1` 不能单独删除某一子任务的记录
- `N2` 不提供更新父任务记录的能力


## 2. 边界定义（Boundary / Parameter 参数）
- 输入参数:
    - 父任务 id ∈ `B1`, `B3`, `B5`
    - 子任务 id ∈ `B2`, `B4`
    - 对应父任务 id ∈ `B2`
    - 子任务状态 ∈ `B2`
    - 是否初次完成 ∈ `B2`
    - 子任务关心的其他信息 ∈ `B2`
- 输出参数:
    - true / false ∈ `B1`, `B2`, `B3`
    - 总子任务数 ∈ `B5`
    - 已完成子任务数 ∈ `B5`
    - 父任务状态 ∈ `B5`
    - 父任务关心的其他信息 ∈ `B5`
    - 对应父任务 id ∈ `B4`
    - 子任务状态 ∈ `B4`
    - 子任务关心的其他信息 ∈ `B4`
- 配置:
    - 父任务记录表的表名 ∈ `B5`
    - 子任务记录表的表名 ∈ `B4`
    - 父任务状态代表"完成"和"未完成"的名字列表 ∈ `B5`
    - 子任务状态代表"完成"和"未完成"的名字列表 ∈ `B4`
    - 父任务插入时使用的默认状态名 ∈ `B5`
    - 子任务插入时使用的默认状态名 ∈ `B4`
    - 父任务的子任务数量边界 ∈ `B5`
    - 父任务关心的其他信息的字段名 ∈ `B5`
    - 子任务关心的其他信息的字段名 ∈ `B4`

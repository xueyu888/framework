# 文档分块原子模块:DocumentChunkingAtoms

@framework

本模块定义文档分块的最小原子结构，只回答“单文档纯文本如何稳定落成可排序的段落块”。它不直接负责标题正文归组策略、跨文档聚类或下游检索编排。

## 1. 能力声明（Capability Statement）

- `C1` 归一文本承接能力：把单文档纯文本、扫描起点与输入锚点收敛为稳定切块输入结构。
- `C2` 分界命中能力：把可执行的分界信号、命中位置与顺序约束收敛为稳定分界结构。
- `C3` 段落块承载能力：把段落文本、顺序编号、原文偏移与候选角色位收敛为稳定段落块结构。
- `C4` 非能力项：不负责标题正文归组策略、跨文档聚类与下游检索排序。

## 2. 边界定义（Boundary / 参数）

- `DOC` 文档边界：输入必须是单文档纯文本，且已完成编码、空白与换行归一。来源：`C1`。
- `DELIM` 分界边界：分界信号、命中规则与扫描起点必须明确。来源：`C1 + C2`。
- `ORDER` 顺序边界：段落块顺序编号必须稳定且不可跳序。来源：`C2 + C3`。
- `OFFSET` 定位边界：每个分界命中与段落块都必须可回连到原文偏移区间。来源：`C2 + C3`。
- `ROLE` 角色边界：段落块必须预留可判型的角色位，但本层不定义具体判型规则。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 归一文本输入基：由单文档纯文本、扫描起点与输入锚点构成的切块输入结构。来源：`C1 + DOC + DELIM`。
- `B2` 分界命中结构基：由分界信号、命中位置、顺序编号与偏移区间构成的分界命中结构。来源：`C1 + C2 + DELIM + ORDER + OFFSET`。
- `B3` 段落块载荷基：由段落文本、顺序编号、偏移区间与候选角色位构成的段落块结构。来源：`C1 + C2 + C3 + ORDER + OFFSET + ROLE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 分界先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定归一文本输入，再对其执行稳定分界命中；任何脱离单文档输入上下文的命中结果都不构成有效组合。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`DOC/DELIM/ORDER/OFFSET`。
- `R2` 段落块闭合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：每个合法分界命中都必须生成一个与原文偏移可回连、顺序稳定且携带候选角色位的段落块；段落块不得重叠、跳序或跨文档混合。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`DOC/DELIM/ORDER/OFFSET/ROLE`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：未归一化即切块、分界信号不可执行、顺序编号不稳定、偏移不可回连或段落块缺失角色位的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`DOC/DELIM/ORDER/OFFSET/ROLE`。

## 5. 验erification）

- `V1` 输入有效性：进入 `R1` 的文本必须满足 `DOC/DELIM` 约束。
- `V2` 命中有效性：分界命中必须满足 `ORDER/OFFSET` 约束，且命中序列不可跳序。
- `V3` 段落块合法性：`R2` 产出的段落块必须同时保留文本、顺序、偏移与角色位。
- `V4` 禁止组合可检出性：`R3` 列举的无效组合必须能够被稳定判定。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

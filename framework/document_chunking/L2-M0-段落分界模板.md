# 段落分界模板模块:ParagraphBoundaryTemplate

@framework

本模块位于文档分块原子层之上，负责把“可执行分界信号”表达成稳定模板，并输出可被上层编排消费的命中结果结构。

## 1. 能力声明（Capability Statement）

- `C1` 模板声明能力：把分界模板名称、信号集合与模板约束收敛为稳定模板结构。
- `C2` 分界命中能力：根据模板对归一文本执行分界命中，并产出稳定命中序列。
- `C3` 命中结果结构化能力：把命中结果、失败原因与回放窗口收敛为统一结果结构。
- `C4` 非能力项：不负责段落块角色判型、标题正文归组与下游知识检索。

## 2. 边界定义（Boundary / 参数）

- `TEMPLATE` 模板边界：模板名称、版本与启停状态必须稳定。来源：`C1 + C2`。
- `SIGNAL` 信号边界：分界信号、命中条件与信号优先级必须明确。来源：`C1 + C2`。
- `WINDOW` 窗口边界：命中前后查看窗口、偏移校验与冲突裁决范围必须明确。来源：`C2 + C3`。
- `RESULT` 结果边界：命中结果、失败原因与命中偏移结构必须统一。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 模板定义结构基：L0.M0[R1]。来源：`C1 + TEMPLATE + SIGNAL`。
- `B2` 命中判定结构基：L0.M0[R1,R2]。来源：`C2 + SIGNAL + WINDOW`。
- `B3` 命中结果结构基：L0.M0[R2]。来源：`C2 + C3 + WINDOW + RESULT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 模板命中组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定模板定义，再按模板信号执行命中判定；脱离模板声明上下文的命中判定不构成合法组合。
  - `R1.3` 输出结构：`TemplateHit_set`（元素：`TH`，每个 `TH` 必须包含 `template_id`、`signal_id`、`hit_offset`）。
  - `R1.4` 输出能力：`C1 + C2`。
  - `R1.5` 边界绑定：`TEMPLATE/SIGNAL/WINDOW`。
- `R2` 结果结构收敛
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：每次命中判定都必须输出统一结果结构，显式记录命中偏移、失败原因与回放窗口；缺失任一结果字段的方案均不合法。
  - `R2.3` 输出结构：`BoundaryResult_set`（元素：`BR`，每个 `BR` 必须包含 `template_id`、`hit_offset`、`window_range`、`result_status`）。
  - `R2.4` 输出能力：`C1 + C2 + C3`。
  - `R2.5` 边界绑定：`TEMPLATE/SIGNAL/WINDOW/RESULT`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：模板未命名、信号不可执行、窗口边界缺失或结果结构不可统一解析的方案均无效。
  - `R3.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效模板、失效关系与失效原因）。
  - `R3.4` 输出能力：`C4`。
  - `R3.5` 边界绑定：`TEMPLATE/SIGNAL/WINDOW/RESULT`。

## 5. 验证（Verification）

- `V1` 模板完整性：模板定义必须同时满足 `TEMPLATE/SIGNAL` 约束。
- `V2` 命中有效性：命中判定必须满足 `WINDOW` 约束，且命中偏移可稳定回放。
- `V3` 结果统一性：输出结果必须满足 `RESULT` 约束，并统一包含命中与失败两类表达。
- `V4` 禁止组合可检出性：`R3` 列举的无效组合必须能够被稳定判定。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 段落块模块:ParagraphBlock

@framework

本模块位于文档分块原子层之上，负责把分界命中后的文本段稳定承接为可排序、可判型、可登记归属的段落块结构。

## 1. 能力声明（Capability Statement）

- `C1` 段落块承载能力：把段落文本、顺序编号与原文定位收敛为稳定段落块结构。
- `C2` 角色标注能力：把段落块的候选角色、判型状态与角色约束收敛为稳定角色结构。
- `C3` 归属登记能力：把段落块与上层文本块之间的归属关系、块内位置与回溯标识收敛为稳定登记结构。
- `C4` 非能力项：不负责上层文本块聚合策略、跨文档关系与检索排序。

## 2. 边界定义（Boundary / 参数）

- `BLOCK` 段落块边界：段落文本、块标识与定位信息必须完整。来源：`C1`。
- `ORDER` 顺序边界：段落块顺序必须稳定且不可跳序。来源：`C1 + C2`。
- `ROLE` 角色边界：角色值、判型状态与角色解释必须稳定。来源：`C1 + C2`。
- `OWNERSHIP` 归属边界：段落块归属到上层文本块的关系必须唯一且可回放。来源：`C2 + C3`。
- `TRACE` 回溯边界：块来源、角色变化与归属变化必须可追踪。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 段落块承载基：L0.M0[R2]。来源：`C1 + BLOCK + ORDER`。
- `B2` 角色标注结构基：L0.M0[R2]。来源：`C1 + C2 + ROLE + TRACE`。
- `B3` 归属登记结构基：L0.M0[R2]。来源：`C1 + C2 + C3 + OWNERSHIP + TRACE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 段落块收敛
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：每个段落块在进入上层编排前都必须同时具备稳定顺序与角色状态；未完成角色标注的段落块不得进入后续组合。
  - `R1.3` 输出结构：`LabeledParagraphBlock_set`（元素：`LPB`，每个 `LPB` 必须包含 `block_id`、`order_index`、`block_role`）。
  - `R1.4` 输出能力：`C1 + C2`。
  - `R1.5` 边界绑定：`BLOCK/ORDER/ROLE/TRACE`。
- `R2` 归属登记闭合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：进入上层文本块的每个段落块都必须登记唯一归属，显式记录其块内位置、角色与回溯标识；一块多属或无归属的方案均不合法。
  - `R2.3` 输出结构：`OwnershipRecord_set`（元素：`OR`，每个 `OR` 必须包含 `block_id`、`owner_chunk_id`、`position_in_chunk`、`block_role`）。
  - `R2.4` 输出能力：`C1 + C2 + C3`。
  - `R2.5` 边界绑定：`OWNERSHIP/BLOCK/ORDER/ROLE/TRACE`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：块定位缺失、顺序不稳定、角色不可解释、归属重复登记或回溯信息断裂的方案均无效。
  - `R3.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效块、失效关系与失效原因）。
  - `R3.4` 输出能力：`C4`。
  - `R3.5` 边界绑定：`BLOCK/ORDER/ROLE/OWNERSHIP/TRACE`。

## 5. 验证（Verification）

- `V1` 段落块完整性：段落块必须满足 `BLOCK/ORDER` 约束。
- `V2` 角色一致性：角色标注必须满足 `ROLE/TRACE` 约束，且角色状态可解释。
- `V3` 归属唯一性：归属登记必须满足 `OWNERSHIP` 约束，每个段落块只能属于一个上层文本块。
- `V4` 禁止组合可检出性：`R3` 列举的无效组合必须能够被稳定判定。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 段落块拼接规则:ParagraphBlockMergeRule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 判定能力：判断对象用于判断适用于哪一条规则。
- `C2` 执行能力：能对对象执行操作。

## 2. 边界定义（Boundary / 参数）

- `SUBJECT` 对象边界：规则负责的对象必须明确，不对没有声明的对象执行操作。来源：`C1 + C2`。
- `CLOSURE` 闭合边界：规则必须是有穷尽的。来源：`C2`。
- `TRIGGER` 触发边界：规则可使用的判定条件及其合法取值范围必须预先声明，且触发条件必须明确。来源：`C1 + C2`。
- `ACTION` 作用边界：对对象执行的操作必须明确。来源：`C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 对象：规则约束的对象。来源：`C1 + C2 + SUBJECT`。
- `B2` 判定域：规定规则可使用的判定维度及其合法取值范围。来源：`C1 + C2 + TRIGGER + CLOSURE`。
- `B3` 作用动作：执行的动作，赋值、移位、搬移等。来源：`C2 + SUBJECT + ACTION + CLOSURE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 标题-正文有序拼接组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：`B1` 必须先落入 `B2` 声明的合法判定域，再由 `B3` 对命中对象执行拼接动作。在同一有序对象流中，每个拼接组必须且仅能由 `1` 个标题对象作为起点，其后按原顺序吸收零个或多个连续正文对象；当遇到下一个标题对象或到达对象流末尾时，当前组闭合。任何逆序吸收、跳序吸收、多标题并入同一组、或使用未声明判定值的组合均不构成合法拼接结果。
  - `R1.3` 对象：`B1`。
  - `R1.4` 判定条件：对象的角色判定值只能取已声明的合法值；当判定值为标题时，当前对象可作为新组起点；当判定值为正文且当前已存在开启组时，当前对象可作为该组的后续成员。
  - `R1.5` 作用动作：为标题对象开启新组，按顺序将其后连续正文对象依次并入该组，并在闭合时输出完整的标题-正文组合结果。
  - `R1.6` 输出能力：`C1 + C2`。
  - `R1.7` 输出结构：`ParagraphBlockMergePlan_set`（元素：`MP`，每个 `MP` 必须包含 `plan_id`、`title_object_id`、`body_object_id_set`、`ordered_object_id_set`、`closure_reason`）。
  - `R1.8` 边界绑定：`SUBJECT/TRIGGER/ACTION/CLOSURE`。

- `R2` 禁止组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：无标题起点、正文对象先于标题对象独立成组、同一组内出现多个标题对象、正文对象跳序进入组、或对象角色取值超出 `B2` 声明判定域的方案均无效。
  - `R2.3` 对象：`B1`。
  - `R2.4` 判定条件：命中任一已声明非法条件。
  - `R2.5` 作用动作：拒绝当前拼接方案，并记录失效原因与触发对象。
  - `R2.6` 输出能力：`C1 + C2`。
  - `R2.7` 输出结构：`InvalidMerge_set`（元素：`IM`，每个 `IM` 必须包含 `invalid_candidate_id`、`invalid_reason`、`offending_object_id_set`）。
  - `R2.8` 边界绑定：`SUBJECT/TRIGGER/ACTION/CLOSURE`。

## 5. 验证（Verification）

- `V1` 基完备性：`B1` 必须给出规则作用对象，`B2` 必须显式给出对象可用的判定维度及合法取值，`B3` 必须显式给出命中判定后的拼接动作；缺少任一项都不能构成合法拼接规则。
- `V2` 合法值域约束：`R1` 使用的角色判定值必须全部落在 `B2` 预先声明的合法判定域内；任何超出判定域的取值都必须被 `R2` 识别为无效。
- `V3` 组合合法性：`R1` 产出的每个 `MP` 都必须满足“`1` 个标题对象 + 零个或多个按原顺序排列的正文对象”的结构，不得存在正文先于标题成组、正文逆序、或多个标题进入同一组的情况。
- `V4` 闭合合法性：每个 `MP` 都必须在遇到下一个标题对象或对象流结束时闭合，且闭合后不得继续吸收后续对象。
- `V5` 非法组合可检出性：`R2` 列举的无效方案都必须能够被判定为无效，并输出明确失效原因与触发对象。
- `V6` 最小必要性：移除任一 `B*` 后，`V1~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R1/R2 -> 有效/无效 / 边界 / 失效原因` 的结论。

# 文档分块:DocumentChunking

@framework

本模块位于段落分界模板模块和段落块模块之上，负责把单文档文本切块流程编排为稳定主线，并输出可回溯的分块结果结构。

## 1. 能力声明（Capability Statement）

- `C1` 文本切块编排能力：把归一文本输入、分界模板与段落块生成组织为稳定切块主线。
- `C2` 段落归组能力：把段落块、角色状态与块间归属组织为稳定归组结构。
- `C3` 输出回溯能力：把文本块集合、归属记录与回溯信息封装为稳定输出结构。
- `C4` 非能力项：不负责跨文档聚类、摘要改写、向量化索引与检索排序。

## 2. 边界定义（Boundary / 参数）

- `NORMDOC` 输入边界：输入必须是单文档纯文本，且已完成换行、空白与编码归一。来源：`C1`。
- `CHUNKRULE` 切块边界：切块所依赖的分界模板、窗口约束与顺序规则必须明确。来源：`C1 + C2`。
- `CHUNKSET` 文本块边界：文本块集合必须保持稳定顺序、稳定标识与稳定承载字段。来源：`C1 + C2 + C3`。
- `OWNERSHIP` 归属边界：段落块到文本块的归属关系必须唯一且可回放。来源：`C2 + C3`。
- `TRACE` 回溯边界：切块路径、归组结果与输出状态必须可追踪。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 分界模板编排基：L2.M0[R1,R2]。来源：`C1 + NORMDOC + CHUNKRULE`。
- `B2` 段落块归组编排基：L2.M1[R1,R2]。来源：`C1 + C2 + CHUNKSET + OWNERSHIP`。
- `B3` 回溯输出编排基：L2.M0[R2] + L2.M1[R2]。来源：`C2 + C3 + CHUNKSET + OWNERSHIP + TRACE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 切块主线收敛
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定切块规则，再把段落块生成与归组状态挂接到同一单文档主线；任何脱离归一文本上下文的归组结果都不构成合法组合。
  - `R1.3` 输出结构：`ChunkingFlow`（必须包含 `norm_doc_id`、`boundary_result_set`、`paragraph_block_set`）。
  - `R1.4` 输出能力：`C1 + C2`。
  - `R1.5` 边界绑定：`NORMDOC/CHUNKRULE/CHUNKSET/OWNERSHIP`。
- `R2` 输出回溯闭合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：全部合法文本块都必须输出为稳定序列，并同时保留归属记录与回溯信息，使结果能够回连到分界命中与段落块登记阶段。
  - `R2.3` 输出结构：`DocumentChunkingOutput`（必须包含 `text_chunk_set`、`ownership_record_set`、`trace_meta`）。
  - `R2.4` 输出能力：`C1 + C2 + C3`。
  - `R2.5` 边界绑定：`NORMDOC/CHUNKRULE/CHUNKSET/OWNERSHIP/TRACE`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：未归一化即切块、切块规则不可解释、归属关系不唯一或输出结果无法回溯的方案均无效。
  - `R3.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效基、失效关系与失效原因）。
  - `R3.4` 输出能力：`C4`。
  - `R3.5` 边界绑定：`NORMDOC/CHUNKRULE/CHUNKSET/OWNERSHIP/TRACE`。

## 5. 验证（Verification）

- `V1` 输入有效性：进入 `R1` 的输入必须满足 `NORMDOC/CHUNKRULE` 约束。
- `V2` 归组合法性：文本块集合必须满足 `CHUNKSET/OWNERSHIP` 约束，且归属关系唯一。
- `V3` 回溯完整性：输出结果必须满足 `TRACE` 约束，并能回连到分界与段落块阶段。
- `V4` 禁止组合可检出性：`R3` 列举的无效组合必须能够被稳定判定。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

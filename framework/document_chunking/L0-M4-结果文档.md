# 输出格式:OutputFormat

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文件保存格式声明能力：声明文件保存的格式。
- `C2` 原文本追溯能力：能追溯到原文本ID。
- `C3` 顺序存放能力：能按照顺序存放内容。
- `C4` 文本承载能力：提供稳定的存放文本的能力。

## 2. 边界定义（Boundary / 参数）

- `FORMAT` 格式边界：输出的文档保存格式必须明确。来源：`C1`。
- `TRACE` 追溯边界：追溯的原文本只能有一个。来源：`C2`。
- `CONTENT` 字段边界：文档包含的字段必须明确。来源：`C1 + C2 + C3 + C4`
- `STANDARD` 顺序边界：文档包含的字段顺序必须明确。来源：`C3`。
- `LENGTH` 长度边界：文本长度需要进行限制。来源：`C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文档格式：标注输出文档的格式。来源：`C1 + FORMAT`
- `B2` 文本块ID：用于标识当前输出结果属于哪个文本。来源：`C2 + TRACE`。
- `B3` 文本块编号：用于标识文本块在当前文本中的顺序编号。来源：`C3 + STANDARD`。
- `B4` 文本块内容：用于承载与文本块编号对应的文本内容。来源：`C4 + LENGTH`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 结果文档装配组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：结果文档必须先由 `B1` 明确保存格式，再绑定且仅绑定一个用于追溯原文本的 `B2`；在同一结果文档内部，零个或多个 `B3 + B4` 组合形成按顺序排列的文本块单元，每个文本块单元都必须同时包含一个明确的编号和与之对应的文本内容，且全部文本块单元必须按 `B3` 的顺序稳定出现。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 输出结构：`ResultDocument`（必须包含 `document_format`、`source_text_id`、`ordered_text_block_set`；其中每个 `text_block_item` 都必须包含 `block_order`、`block_content`）。
  - `R1.5` 边界绑定：`FORMAT/TRACE/CONTENT/STANDARD/LENGTH`。

- `R2` 禁止组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R2.2` 组合方式：缺失 `B1`、同一结果文档绑定多个原文本追溯 ID、文本块编号脱离文本块内容单独出现、文本块内容长度超出限制、字段缺失或顺序不稳定的组合均无效。
  - `R2.3` 输出能力：`C2 + C3 + C4`。
  - `R2.4` 输出结构：`InvalidResultDocument_set`（元素：`IRD`，每个 `IRD` 必须包含 `invalid_reason`、`offending_source_text_id_set`、`offending_text_block_item_set`）。
  - `R2.5` 边界绑定：`FORMAT/TRACE/CONTENT/STANDARD/LENGTH`。

## 5. 验证（Verification）

- `V1` 基完备性：`B1` 必须能稳定声明结果文档保存格式，`B2` 必须能稳定标识唯一原文本追溯 ID，`B3` 必须能稳定标识文本块顺序，`B4` 必须能稳定承载与顺序编号对应的文本内容；缺少任一项都不能构成合法结果文档。
- `V2` 结构合法性：`R1` 产出的每个 `ResultDocument` 都必须满足“一个 `document_format` + 一个 `source_text_id` + 零个或多个按顺序排列的文本块单元”的结构。
- `V3` 追溯与顺序合法性：同一 `ResultDocument` 内只能出现一个 `source_text_id`；每个 `text_block_item` 都必须同时包含 `block_order` 与 `block_content`，且 `block_order` 不得重复、跳序或脱离对应内容独立存在。
- `V4` 内容合法性：`R1` 产出的每个 `block_content` 都必须满足 `LENGTH` 约束，结果文档中出现的字段与字段顺序都必须满足 `CONTENT/STANDARD` 约束。
- `V5` 非法组合可检出性：`R2` 列举的无效方案都必须能够被识别为无效，并输出明确失效原因与触发原文本 ID 或文本块单元。
- `V6` 最小必要性：移除任一 `B*` 后，`V1~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R1/R2 -> 有效/无效 / 边界 / 失效原因` 的结论。

# 账号密码登录校验组件:AccountPasswordLoginVerificationComponent

@framework

<!-- 组件目标：组合字符串格式校验模块和身份校验模块，完成账号密码登录校验，并输出最终成功或失败结果。 -->

## 1. 能力声明（Capability Statement）

- `C1` 账号格式校验编排能力：组织账号输入进入账号格式校验实例。
- `C2` 身份校验编排能力：在账号格式校验通过后，组织账号定位与凭证校验进入身份校验模块。
- `C3` 最终结果收敛能力：根据账号格式校验结论和身份校验结论收敛为最终成功或失败结果。

## 2. 非职责声明（Non-Capability Statement）

- `N1` 不负责账户创建、账户资料维护和登录后信息修改。
- `N2` 不负责密码存储方式的设计与持久化实现。
- `N3` 不负责 token、会话与权限控制。
- `N4` 不在本组件中单独实例化密码格式校验模块。

## 3. 边界定义（Boundary）

- `D1` 登录输入范围：账号输入与密码输入必须预先确定。来源：`C1 + C2`
- `D2` 账号格式校验实例范围：账号格式校验实例必须预先确定。来源：`C1`
- `D3` 凭证记录范围：账号必须能够稳定定位到对应的已存密码凭证记录。来源：`C2`
- `D4` 最终输出范围：最终结果只允许为成功或失败。来源：`C3`

## 4. 最小可行基（Minimum Viable Bases）

- `B1` 登录输入结构：由账号输入和密码输入构成。来源：`C1 + D1`。
- `B2` 账号格式校验模块基：`L0-M0-字符串格式校验模块[R1,R2,R3]`。来源：`C1 + D2`。
- `B3` 身份校验模块基：`L0-M1-身份校验模块[R1,R2,R3]`。来源：`C2 + D3`。
- `B4` 最终结果结构：由账号格式结论、身份校验结论和最终输出值构成。来源：`C3 + D4`。

## 5. 基组合原则（Base Combination Principles）

- `R1` 账号格式校验编排组合
  - `R1.1` 参与基：`B1 + B2 + B4`。
  - `R1.2` 组合方式：账号输入必须进入账号格式校验实例；账号格式校验实例返回的结论写入最终结果结构中的账号格式结论位。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`D1/D2`。

- `R2` 身份校验编排组合
  - `R2.1` 参与基：`B1 + B3 + B4`。
  - `R2.2` 组合方式：仅当账号格式结论为通过时，账号输入与密码输入才允许进入身份校验模块；身份校验模块返回的结论写入最终结果结构中的身份校验结论位。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`D1/D3`。

- `R3` 最终结果收敛组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：账号格式结论为失败时，最终输出直接为失败；账号格式结论为通过且身份校验结论为通过时，最终输出为成功；账号格式结论为通过但身份校验结论为失败时，最终输出为失败。
  - `R3.3` 输出能力：`C3`。
  - `R3.4` 边界绑定：`D4`。

## 6. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个边界项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力和边界绑定。
- `V3` 闭环性：`R4` 必须形成“输入承载 -> 格式校验 -> 凭证定位 -> 结果输出”的完整登录校验链。
- `V4` 边界符合性：所有有效校验组合都必须满足 `D1/D2/D3/D4`。
- `V5` 最小必要性：移除任一 `B*` 后，至少一项能力无法成立。

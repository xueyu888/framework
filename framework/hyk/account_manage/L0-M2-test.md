# 账号密码登录校验模块:AccountPasswordLoginVerificationModule

@framework

组件目标：接收账号与密码输入，完成输入承载、格式匹配、账号凭证定位和校验结果输出

## 1. 能力声明（Capability Statement）

- `C1` 登录输入承载能力：承载账号字符串与密码字符串，作为后续校验链的统一输入。
- `C2` 输入格式校验能力：基于既定格式规则判断账号与密码输入是否合法。
- `C3` 账号凭证定位能力：根据账号定位对应的已存密码凭证。
- `C4` 校验结果输出能力：根据各类校验状态输出稳定结果。

## 2. 非职责声明（Non-Capability Statement）

- `N1` 不负责账户创建、账户资料维护和登录后信息修改。
- `N2` 不负责 token 发放、会话建立与权限控制。
- `N3` 不负责外部持久化实现细节，只约束结构与校验链。

## 3. 边界定义（Boundary）

- `D1` 输入承载空间：账号与密码输入必须以字符串形式进入校验链，且输入槽位必须预先确定。
- `D2` 格式匹配空间：账号与密码允许采用的格式规则表达方式必须预先确定。
- `D3` 凭证定位空间：账号必须能够稳定定位到对应的已存密码凭证。
- `D4` 结果输出空间：格式错误、账号不存在、密码错误和登录成功等输出结果必须预先确定。

## 4. 最小可行基（Minimum Viable Bases）

- `B1` 字符串容器（自有基）：登录输入的承载结构，由账号字符串槽位和密码字符串槽位构成。来源：`C1 + D1`。
- `B2` 账号密码映射表（自有基）：账号到已存密码凭证定位关系的承载结构，由账号键、密码凭证值和记录引用构成。来源：`C3 + D3`。
- `B3` 结果映射表（自有基）：登录校验结果输出的承载结构，由校验状态键和结果输出值构成。来源：`C4 + D4`。
- `B4` 正则表达式（自有基）：输入格式匹配的承载结构，由账号匹配规则和密码匹配规则构成。来源：`C2 + D2`。

## 5. 基组合原则（Base Combination Principles）

- `R1` 输入承载组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有登录校验都必须先进入字符串容器，账号与密码输入缺失、类型不成立或槽位不完整的情况均不得进入后续校验。
  - `R1.3` 输出结构：`LOGIN_INPUT_ACCEPTED`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`D1`。

- `R2` 输入格式校验组合
  - `R2.1` 参与基：`B1 + B4 + B3`。
  - `R2.2` 组合方式：字符串容器中的账号与密码输入必须分别通过正则表达式定义的格式匹配规则；任一输入不满足格式要求时，直接通过结果映射表输出对应失败结果。
  - `R2.3` 输出结构：`LOGIN_INPUT_VALIDATED`。
  - `R2.4` 输出能力：`C2 + C4`。
  - `R2.5` 边界绑定：`D1/D2/D4`。

- `R3` 账号凭证定位组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：账号输入在通过格式校验后，必须通过账号密码映射表定位对应的已存密码凭证；若账号无法定位到记录，则通过结果映射表输出账号不存在结果。
  - `R3.3` 输出结构：`ACCOUNT_CREDENTIAL_LOCATED`。
  - `R3.4` 输出能力：`C3 + C4`。
  - `R3.5` 边界绑定：`D1/D3/D4`。

- `R4` 登录结果判定组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：先以字符串容器承载输入，再执行格式校验，再通过账号密码映射表取得已存密码凭证，最后根据输入密码与已存凭证的对应关系在结果映射表中输出登录成功或密码错误结果；任一前序步骤失败时不得越过当前失败结论继续执行。
  - `R4.3` 输出结构：`LOGIN_VERIFIED`。
  - `R4.4` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.5` 边界绑定：`D1/D2/D3/D4`。

## 6. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个边界项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力和边界绑定。
- `V3` 闭环性：`R4` 必须形成“输入承载 -> 格式校验 -> 凭证定位 -> 结果输出”的完整登录校验链。
- `V4` 边界符合性：所有有效校验组合都必须满足 `D1/D2/D3/D4`。
- `V5` 最小必要性：移除任一 `B*` 后，至少一项能力无法成立。

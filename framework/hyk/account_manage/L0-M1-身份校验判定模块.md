# 身份校验判定模块:IdentityVerificationJudge
@
组件目标：对已取得的输入凭证和已存凭证执行比对，并判定登录是否成立。

## 1. 能力

- `C1` 凭证比对能力
- `C2` 登录结果判定能力

## 2. 边界

- `D1` 凭证比对空间：明文比对、哈希比对、归一化后比对等方式必须预先确定。
- `D2` 校验结果判定空间：登录成功条件、失败条件和失败输出方式必须预先确定。

## 3. 最小可行基

- `B1` 凭证比对器：由输入凭证读取位、已存凭证读取位、比对方式和失败输出位构成。
- `B2` 校验结果判定器：由成功条件、失败条件和结果输出位构成。
- `B3` 校验规则装配表：由规则项标识、比对器引用和判定器引用构成。

## 4. 组合原则

- `R1` 凭证比对组合
  - `R1.1` 参与基：`B1 + B3`
  - `R1.2` 组合方式：按装配表装入比对规则，对输入凭证与已存凭证执行比对。
  - `R1.3` 输出结构：`CREDENTIAL_COMPARED`
  - `R1.4` 输出能力：`C1`
  - `R1.5` 边界绑定：`D1`

- `R2` 登录结果判定组合
  - `R2.1` 参与基：`B1 + B2 + B3`
  - `R2.2` 组合方式：基于凭证比对结果和判定条件输出登录成功或失败结论。
  - `R2.3` 输出结构：`LOGIN_JUDGED`
  - `R2.4` 输出能力：`C2`
  - `R2.5` 边界绑定：`D1 + D2`

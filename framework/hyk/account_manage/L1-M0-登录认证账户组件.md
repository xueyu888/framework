## 1. 能力

- `C1` 登录账户创建能力：按既定输入规则建立可登录账户。
- `C2` 登录核心信息变更能力：对已确认账户的登录核心信息进行修改，并保持既定规则一致。
- `C3` 登录身份校验能力：根据登录标识定位记录，并按凭证比对规则完成身份校验。

## 2. 非职责

- `N1` 不负责姓名、头像、简介等非登录资料管理。
- `N2` 不负责 token 发放、会话续期与权限控制。

## 3. 边界

- `D1` 登录记录承载空间：登录账户需要承载哪些核心字段必须预先确定。
- `D2` 登录输入合法空间：登录标识与登录凭证的输入格式合法范围必须预先确定。
- `D3` 身份校验成立空间：凭证比对方式和登录成功判定条件必须预先确定。
- `D4` 登录信息变更空间：允许修改的登录字段范围和联动限制必须预先确定。

## 4. 最小可行基

- `B1` 登录账户实体基：登录账户本体的承载结构，由账户引用和登录核心槽位构成。
- `B2` 账号凭证映射基：登录标识到凭证表示与账户引用的定位关系承载结构。
- `B3` 登录输入格式校验模块基：`L0-M0-登录输入格式校验模块[R1,R2]`
- `B4` 身份校验判定模块基：`L0-M1-身份校验判定模块[R1,R2]`

## 5. 组合原则

- `R1` 登录账户创建组合
  - `R1.1` 参与基：`B1 + B2 + B3`
  - `R1.2` 组合方式：创建时，必须先通过输入格式校验，再写入登录账户实体，并登记账号凭证映射关系。
  - `R1.3` 输出结构：`LOGIN_ACCOUNT_CREATED`
  - `R1.4` 输出能力：`C1`
  - `R1.5` 边界绑定：`D1 + D2`

- `R2` 登录身份校验组合
  - `R2.1` 参与基：`B2 + B3 + B4`
  - `R2.2` 组合方式：校验时，必须先通过输入格式校验，再通过账号凭证映射取得已存凭证，最后按身份校验判定模块完成比对和结果判定。
  - `R2.3` 输出结构：`LOGIN_ACCOUNT_VERIFIED`
  - `R2.4` 输出能力：`C3`
  - `R2.5` 边界绑定：`D2 + D3`

- `R3` 登录核心信息变更组合
  - `R3.1` 参与基：`B1 + B2 + B3`
  - `R3.2` 组合方式：变更时，必须先满足变更范围约束并通过输入格式校验，再更新登录账户实体；若变更涉及登录标识或凭证，则同步更新账号凭证映射关系。
  - `R3.3` 输出结构：`LOGIN_ACCOUNT_UPDATED`
  - `R3.4` 输出能力：`C2`
  - `R3.5` 边界绑定：`D1 + D2 + D4`

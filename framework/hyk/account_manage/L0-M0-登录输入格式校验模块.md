# 登录输入格式校验模块:LoginInputFormatValidator

@framework

## 1. 能力声明（Capability Statement）

- `C1` 长度合规判定能力：基于数值边界判定输入长度是否合法。
- `C2` 格式合规判定能力：基于字符模式判定输入内容是否合法。
- `C3` 校验结果映射能力：当校验不通过时，输出对应的校验结果。

## 2. 边界定义（Boundary / 参数）

- `P1` 长度界限：最小字符数与最大字符数。来源：`C1`。
- `P2` 字符模式：仅字母/仅数字/字母数字混合。来源：`C2`。
- `P3` 输出值域：映射表允许返回的具体结果集合。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 长度区间约束（Length Range Constraint）：由 `[min, max]` 数值对构成的标量边界结构。来源：`C1 + P1`。字符串容器，就可以获取长度
- `B2` 正则表达式（Regular Expression）：描述字符组合与限制模式的形式语法结构。来源：`C2 + P2`。
- `B3` 结果输出映射表（Result Mapping Table）：
校验字段 ：用户名/密码
校验方向：长度(范围)、格式(字符模式)
校验器：正则表达式
输出：校验结果，来源：`C3 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 标准拦截与映射组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：将 `B1` 与 `B2` 挂载至 `B4` 顺序校验链。输入数据穿透 `B4` 时，若未通过某个节点的约束，则提取该节点的违规类型，在 `B3` 结果映射表中检索对应的输出标识并返回；若全部穿透，则返回合法标识。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`P1/P2/P3`。

- `R2` 孤立条件校验组合（特定场景降级）
  - `R2.1` 参与基：`B1 + B3 + B4` 或 `B2 + B3 + B4`。
  - `R2.2` 组合方式：在仅需长度限制或仅需格式限制的单一场景中，`B4` 链上仅挂载 `B1` 或 `B2`，校验失败时同样通过 `B3` 映射输出。
  - `R2.3` 输出能力：`C1 + C3` 或 `C2 + C3`。
  - `R2.4` 边界绑定：`P1/P3` 或 `P2/P3`。

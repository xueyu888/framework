# 值结构:Value

@framework

## 1. 能力声明（Capability Statement）

- `C1` 值形成能力：让前端单元能够形成值并显示。
- `C2` 值保持能力：让前端单元在一个阶段内持续持有该值，使值不会在形成后立刻丢失。
- `C3` 值更新能力：让前端单元能够更新值并显示，使值可以随过程变化。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责交互：不负责点击、输入、拖拽、悬停等操作行为。
- `N2` 不负责承载：不负责对象的存在基础、界面落位和容纳结构。
- `N3` 不负责状态变化：不负责正常、禁用、只读、报错、占位等状态变化。
- `N4` 不负责业务实现：不负责业务含义、业务规则、校验逻辑、提交逻辑和具体代码实现。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 值对象边界：值在形成时必须对应明确的值对象；值对象可以是文本值、数值、文件对象、时间值、搜索条件或其他可形成值的内容。来源：`C1`。
- `P2` 值保持边界：值结构必须允许值在一个阶段内被保持，直到新的输入使其更新。来源：`C2`。
- `P3` 值更新参数：值结构必须允许值被更新。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）
  
- `B1` 形成保持结构：用于确定值如何被形成，使输入结果能够落到明确值对象上并被保持。来源：`C1 + P1`。
- `B2` 更新结构：用于确定值在输入继续发生时如何被更新，使值能够随着过程变化。来源：`C3 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基本值形成组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先形成明确的值，使输入结果能够从动作过程进入值对象。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1`。

- `R2` 值保持组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在值已形成的基础上，增加值保持能力，使该值在一个阶段内持续成立。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 参数绑定：`P1 + P2`。

- `R3` 完整值组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在值已形成并可被保持的基础上，引入值更新能力，使值能够随着输入过程继续变化。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 参数绑定：`P1 + P2 + P3`。

- `R4` 无效值悬空组合
  - `R4.1` 参与基：`B2` 或 `B3`。
  - `R4.2` 组合方式：如果没有先形成明确值，而只有保持或更新，则值没有成立对象，组合无效。
  - `R4.3` 失效结论：`N4`。
  - `R4.4` 参数绑定：`P1 + P2 + P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R3` 输出能力并集必须覆盖 `C1 ~ C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

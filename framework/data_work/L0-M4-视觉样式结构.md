# 视觉样式结构:VisualStyle

@framework

视觉样式结构用于让前端对象具备明确且可识别的默认视觉基线，使对象在默认条件下能够以稳定外观被呈现和区分。

## 1. 能力声明（Capability Statement）

- `C1` 样式绑定能力：让默认视觉基线绑定到明确对象上，使样式不悬空。
- `C2` 内容表现能力：让对象中的文本、图标、图片或其他展示内容具备明确的默认视觉表现。
- `C3` 表面样式能力：让对象具备颜色、背景、边框、阴影、圆角等默认表面表现。
- `C4` 内部尺度能力：让对象具备明确的内部密度、内容尺寸和内部留白方式，使对象在默认条件下形成稳定视觉尺度。
- `C5` 默认层级能力：让对象在默认条件下具备可识别的主次、轻重或标题/正文/辅助等层级差异。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责承载成立：视觉样式结构不负责对象是否具备存在基础，也不负责对象是否能够承接内容或下层结构。
- `N2` 不负责布局组织：视觉样式结构不负责对象之间的排列、分区、对齐、对象间外部间距、空间分配或层次落位关系。
- `N3` 不负责状态变化：视觉样式结构不负责显示、隐藏、禁用、选中、报错、处理中等条件变化下的状态差异。
- `N4` 不负责值处理：视觉样式结构不负责值的绑定、更新、校验和提交。
- `N5` 不负责交互行为：视觉样式结构不负责点击、输入、拖拽、悬停等交互行为本身。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 样式对象边界：视觉样式结构必须绑定到明确对象上，不能悬空存在。来源：`C1`。
- `P2` 默认条件边界：视觉样式结构必须用于说明对象在默认条件下的视觉基线，而不是说明条件变化下的状态差异。来源：`C1`。
- `P3` 内容表现边界：视觉样式结构必须允许对象中的文本、图标、图片或其他展示内容形成可识别的默认表现，但不扩展为业务文案规则或状态语义。来源：`C2`。
- `P4` 表面样式边界：视觉样式结构必须允许对象形成可识别的默认表面外观，例如颜色、背景、边框、阴影、圆角等，但这些表现只说明对象默认怎么呈现，不说明状态变化。来源：`C3`。
- `P5` 内部尺度边界：视觉样式结构必须允许对象形成可识别的内部密度、内容尺寸和内部留白方式，但这些关系只说明对象自身的默认视觉尺度，不替代对象间外部间距或布局组织关系。来源：`C4`。
- `P6` 默认层级边界：视觉样式结构必须允许对象形成默认主次、轻重或标题/正文/辅助等层级差异，但这些层级只用于默认识别，不扩展为强调/弱化、选中/未选中、禁用/启用等状态语义。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 样式绑定结构：用于将默认视觉基线绑定到明确对象上，使样式不悬空。来源：`C1 + P1 + P2`。
- `B2` 内容表现结构：用于确定对象中内容的默认视觉表现。来源：`C2 + P3`。
- `B3` 表面样式结构：用于确定对象的默认表面外观。来源：`C3 + P4`。
- `B4` 内部尺度结构：用于确定对象的默认内部密度、内容尺寸和内部留白方式。来源：`C4 + P5`。
- `B5` 默认层级结构：用于确定对象在默认条件下的主次、轻重或标题/正文/辅助等层级差异。来源：`C5 + P6`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 内容默认样式组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先将默认视觉基线绑定到明确对象上，再建立内容表现方式，使对象具备最基本的默认视觉识别能力。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 参数绑定：`P1 + P2 + P3`。

- `R2` 表面默认样式组合
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：先将默认视觉基线绑定到明确对象上，再建立默认表面样式，使对象具备最基本的默认外观识别能力。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 参数绑定：`P1 + P2 + P4`。

- `R3` 内部尺度组合
  - `R3.1` 参与基：`B1 + (B2 或 B3) + B4`。
  - `R3.2` 组合方式：在已具备内容表现或表面样式基础的前提下，引入内部尺度结构，使对象形成稳定的默认视觉尺度。
  - `R3.3` 输出能力：`C1 + C4`。
  - `R3.4` 参数绑定：`P1 + P2 + P5`。

- `R4` 默认层级组合
  - `R4.1` 参与基：`B1 + (B2 或 B3) + B5`。
  - `R4.2` 组合方式：在已具备内容表现或表面样式基础的前提下，引入默认层级结构，使对象形成可识别的默认主次差异。
  - `R4.3` 输出能力：`C1 + C5`。
  - `R4.4` 参数绑定：`P1 + P2 + P6`。

- `R5` 完整视觉样式组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：在默认视觉基线绑定成立的前提下，同时建立内容表现、表面样式、内部尺度和默认层级，使对象形成完整视觉样式结构。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`。

- `R6` 无效悬空组合
  - `R6.1` 参与基：`B2` 或 `B3` 或 `B4` 或 `B5`。
  - `R6.2` 组合方式：只有内容表现、表面样式、内部尺度或默认层级结构，而没有样式绑定结构。
  - `R6.3` 失效结论：如果没有默认视觉绑定基础，则局部样式维度没有绑定对象，不能独立构成合法视觉样式结构。
  - `R6.4` 参数绑定：缺失 `P1` 或 `P2`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R5` 输出能力并集必须覆盖 `C1 ~ C5`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

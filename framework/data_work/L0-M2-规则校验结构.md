# 校验结构:Validation

@framework

校验结构用于让结构化数据能够定义校验规则、绑定校验触发时机，并在校验执行后形成明确结果输出。

## 1. 能力声明（Capability Statement）

- `C1` 校验规则定义能力：让数据具备明确可执行的校验项与校验条件。
- `C2` 校验触发能力：让校验在明确时机被触发，例如输入后、失焦后、提交前或提交中。
- `C3` 校验结果输出能力：让校验执行后形成明确通过/失败结果与失败输出方式。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责值绑定：不负责值从对象中产生和更新。
- `N2` 不负责字段组织：不负责数据字段命名、层级关系与集合关系形成。
- `N3` 不负责提交流程：不负责提交动作发起与重复提交限制。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 规则定义边界：校验结构必须允许定义明确校验项、校验条件与判定依据。来源：`C1`。
- `P2` 触发时机边界：校验结构必须允许定义明确触发时机与触发范围。来源：`C2`。
- `P3` 结果输出边界：校验结构必须允许定义校验结果的输出方式，例如错误项、错误信息或失败标记。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 规则定义基：用于确定校验项、判定条件与作用对象。来源：`C1 + P1`。
- `B2` 校验触发基：用于确定校验在什么时机、对什么范围发生。来源：`C2 + P2`。
- `B3` 结果输出基：用于确定校验通过/失败后的输出形态。来源：`C3 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 规则定义组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先定义明确校验项与判定依据，使数据具备最基本的校验可描述性。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1`。

- `R2` 可触发校验组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在规则定义成立的前提下，引入校验触发机制，使校验能够在明确上下文中执行。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 参数绑定：`P1 + P2`。

- `R3` 完整校验组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在规则定义与校验触发机制成立的前提下，引入结果输出基，使校验执行后形成可消费结果。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 参数绑定：`P1 + P2 + P3`。

- `R4` 无效悬空组合
  - `R4.1` 参与基：`B2` 或 `B3`。
  - `R4.2` 组合方式：只有校验触发基或结果输出基，而没有规则定义基。
  - `R4.3` 失效结论：如果没有明确校验规则，则触发动作与结果输出失去判定依据，不能构成合法校验结构。
  - `R4.4` 参数绑定：缺失 `P1`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R3` 输出能力并集必须覆盖 `C1 ~ C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

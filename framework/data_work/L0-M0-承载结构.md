# 承载结构:Carrier

@framework

## 1. 能力声明（Capability Statement）

- `C1` 承载能力：为前端单元提供最基本的存在基础、位置关系和内容承接空间，使其能够在界面中立住，并接住内容或下层结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责交互：不负责点击、输入、拖拽、悬停等操作行为。
- `N2` 不负责值处理：不负责值的接收、保存、修改、校验和提交。
- `N3` 不负责状态变化：不负责显示、隐藏、强调、弱化、占位等状态变化。
- `N4` 不负责业务实现：不负责业务文案、业务规则、样式写法和组件实现方式。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 承接对象参数：承载结构必须承接明确对象；该对象可以是内容、下层结构或组合后的最小单元。来源：`C1`。
- `P2` 位置关系参数：承载结构必须处于明确的界面位置关系中，不能脱离界面上下文独立成立。来源：`C1`。
- `P3` 容纳范围参数：承载结构允许接住内容或下层结构，但只负责“接住”，不负责定义这些内容的状态、值、交互或业务语义。来源：`C1`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 落位结构：用于确定单元在界面中的最基本存在位置。来源：`C1 + P2`。
- `B2` 容纳结构：用于确定单元能接住什么并保持基本成立。来源：`C1 + P1 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基本承载组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先给单元一个明确的位置关系，使其在界面中具备最基本的承载基础。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1 + P2`。

- `R2` 完整承载组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已具备位置关系的基础上，再提供最基本的容纳能力，使其能够接住内容或下层结构。
  - `R2.3` 输出能力：`C1`。
  - `R2.4` 参数绑定：`P1 + P2 + P3`。

- `R3` 无效悬空组合
  - `R3.1` 参与基：`B2`。
  - `R3.2` 组合方式：如果只有容纳而没有位置关系，则结构没有成立基础，组合无效。
  - `R3.3` 失效结论：`N4`。
  - `R3.4` 参数绑定：`P1 + P2 + P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R2` 输出能力并集必须覆盖 `C1`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

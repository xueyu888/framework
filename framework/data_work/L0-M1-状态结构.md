# 状态结构:State

@framework

状态结构用于让前端对象能够绑定并更新状态，并使状态差异能够被识别和区分。

## 1. 能力声明（Capability Statement）

- `C1` 状态绑定能力：允许对象携带状态，并绑定到具体对象上。
- `C2` 状态更新能力：允许对象在不同状态之间更新，并且更新后能够形成可区分的表现差异。
- `C3` 视觉状态能力：允许对象的视觉状态在界面上呈现不同的效果。
- `C4` 操作状态能力：允许对象的可交互状态在界面上呈现不同的操作方式。
- `C5` 选择状态能力：允许对象的可选择状态在界面上呈现不同的选择结果。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责承载：状态结构不负责对象的存在基础、界面落位或容纳关系。
- `N2` 不负责形成：状态的形成需要外部输入或条件，不由状态结构直接负责。
- `N3` 不负责交互行为：状态结构不负责具体的交互行为实现，例如点击、输入等。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 绑定边界：状态必须绑定到明确对象上，不能悬空。来源：`C1`。
- `P2` 更新边界：状态能够在不同状态之间更新，并且必须能够形成至少一种可区分差异。来源：`C2`。
- `P3` 视觉边界：视觉状态可覆盖正常显示、隐藏、强调、弱化、占位等多种视觉表现。来源：`C3`。
- `P4` 操作边界：操作状态可覆盖可操作、禁用、只读、报错等交互表现。来源：`C4`。
- `P5` 选择边界：选择状态可覆盖未选中、已选中、部分选中等选择结果。来源：`C5`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 绑定更新结构：支持绑定和更新。来源：`C1 + C2 + P1 + P2`。
- `B2` 视觉结构：用于确定状态更新前后，对象在界面上的不同视觉效果，例如正常显示、隐藏、强调、弱化、占位显示。来源：`C3 + P3`。
- `B3` 操作结构：用于确定状态更新前后，对象在交互上的不同操作方式，例如可操作、禁用、只读、报错。来源：`C4 + P4`。
- `B4` 选择结构：用于确定状态更新前后，对象在选择上的不同表现结果，例如未选中、已选中、部分选中。来源：`C5 + P5`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 绑定更新组合  
  - `R1.1` 参与基：`B1`  
  - `R1.2` 组合方式：先将状态绑定到明确对象上，再使该状态能够随条件变化而更新。
  - `R1.3` 输出能力：`C1 + C2`  
  - `R1.4` 参数绑定：`P1 + P2`  

- `R2` 视觉状态组合  
  - `R2.1` 参与基：`B1 + B2`  
  - `R2.2` 组合方式：通过组合视觉结构形成视觉呈现维度  
  - `R2.3` 输出能力：`C1 + C2 + C3`  
  - `R2.4` 参数绑定：`P1 + P2 + P3`  

- `R3` 操作状态组合  
  - `R3.1` 参与基：`B1 + B3`  
  - `R3.2` 组合方式：通过组合操作结构形成操作行为维度  
  - `R3.3` 输出能力：`C1 + C2 + C4`  
  - `R3.4` 参数绑定：`P1 + P2 + P4`  

- `R4` 选择状态组合  
  - `R4.1` 参与基：`B1 + B4`  
  - `R4.2` 组合方式：通过组合选择结构形成选择维度  
  - `R4.3` 输出能力：`C1 + C2 + C5`  
  - `R4.4` 参数绑定：`P1 + P2 + P5`  

-  `R5` 无效悬空组合  
  - `R5.1` 参与基：`B2` 或 `B3` 或 `B4`  
  - `R5.2` 组合方式：具备视觉或操作或选择结构。  
  - `R5.3` 失效结论：如果只有视觉或操作或选择结构而没有绑定更新结构，则状态结构缺少形成基础，组合无效。  
  - `R5.4` 参数绑定：缺失 `P1` 和 `P2`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 B* 必须能由至少一个 C* 和对应参数推导。  
- `V2` 规则一致性：每个 R* 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。  
- `V3` 结构闭环性：R1~R4 输出能力并集必须覆盖 C1~C5。  
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。  
- `V5` 最小必要性：移除任一 B* 后，R4 或 V4 至少一项失败。  
- `V6` 结论表达：逐条输出 R* -> C* + 参数绑定 或 R* -> N* + 参数绑定 的结论。

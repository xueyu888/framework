# 状态结构:State

@framework

## 1. 能力声明（Capability Statement）

- `C1` 状态绑定能力：为前端单元提供状态绑定基础，使其能够带有当前状态。
- `C2` 状态区分能力：为前端单元提供可区分的状态差异，使其能够在不同情况下表现出不同方式。
- `C3` 状态切换能力：允许前端单元在不改变对象本体的前提下切换不同状态表现。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责交互：不负责点击、输入、拖拽、悬停等操作行为。
- `N2` 不负责值处理：不负责值的接收、保存、修改、校验和提交。
- `N3` 不负责对象承接：不负责对象本身的存在基础与界面落位。
- `N4` 不负责业务实现：不负责具体业务规则、状态命名、样式写法和组件实现方式。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 状态对象参数：状态结构必须绑定在明确对象上，不能脱离对象单独成立。来源：`C1`。
- `P2` 状态差异参数：状态结构必须能够形成至少一种可区分的状态差异，但差异类型不在本层固定。来源：`C2`。
- `P3` 状态切换参数：状态结构允许对象在不同情况下切换状态，但切换只改变表现方式，不改变对象本体。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 绑定结构：用于确定状态必须绑定在哪个对象上才能成立。来源：`C1 + P1`。
- `B2` 差异结构：用于确定状态变化后如何形成可区分的表现差异。来源：`C2 + C3 + P2 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基本状态绑定组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先将状态绑定到明确对象上，使状态不再悬空。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1`。

- `R2` 完整状态组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在状态已绑定到对象的基础上，建立状态差异与切换关系，使对象能够在不同情况下表现出不同方式。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 参数绑定：`P1 + P2 + P3`。

- `R3` 无效状态悬空组合
  - `R3.1` 参与基：`B2`。
  - `R3.2` 组合方式：如果只有状态差异，而没有绑定对象，则状态没有绑定对象，组合无效。
  - `R3.3` 失效结论：`N4`。
  - `R3.4` 参数绑定：`P1 + P2 + P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R2` 输出能力并集必须覆盖 `C1 ~ C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

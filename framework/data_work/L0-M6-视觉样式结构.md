# 视觉样式结构:VisualStyle

@framework

视觉样式结构用于让对象具备明确且可识别的默认视觉基线，使对象在默认条件下能够以稳定外观被呈现和区分。

## 1. 能力声明（Capability Statement）

- `C1` 样式绑定能力：让默认视觉基线绑定到明确对象上，使样式不悬空。
- `C2` 边界与表面能力：让对象具备边界、背景、边框等默认表面外观。
- `C3` 层级强调能力：让对象具备标题、主次或默认强调关系。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责布局组织：不负责对象之间的排列、分组、对齐和层次关系。
- `N2` 不负责状态变化：不负责禁用、选中、提交中等状态差异。
- `N3` 不负责值与流程：不负责值处理、校验和提交流程。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 样式对象边界：视觉样式结构必须绑定到明确对象上，不能悬空存在。来源：`C1`。
- `P2` 表面外观边界：视觉样式结构必须允许对象形成默认边界、背景、边框或壳层外观。来源：`C2`。
- `P3` 层级强调边界：视觉样式结构必须允许对象形成默认主次、标题或强调关系。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 样式绑定结构：用于将默认视觉基线绑定到明确对象上。来源：`C1 + P1`。
- `B2` 表面外观结构：用于确定对象的默认边界与表面外观。来源：`C2 + P2`。
- `B3` 层级强调结构：用于确定对象默认主次与强调关系。来源：`C3 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基本视觉样式组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先将默认视觉基线绑定到明确对象上，使样式具备附着对象。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1`。

- `R2` 表面视觉样式组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在样式绑定成立的前提下，引入表面外观结构，使对象具备最基本的默认视觉识别能力。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 参数绑定：`P1 + P2`。

- `R3` 完整视觉样式组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在表面视觉样式成立的前提下，引入层级强调结构，使对象形成稳定的默认主次关系。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 参数绑定：`P1 + P2 + P3`。

- `R4` 无效悬空组合
  - `R4.1` 参与基：`B2` 或 `B3`。
  - `R4.2` 组合方式：只有表面外观或层级强调，而没有样式绑定结构。
  - `R4.3` 失效结论：如果没有默认视觉绑定基础，则局部样式维度没有绑定对象，不能独立构成合法视觉样式结构。
  - `R4.4` 参数绑定：缺失 `P1`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R3` 输出能力并集必须覆盖 `C1 ~ C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 反馈过程结构:FeedbackProcess

@framework

反馈过程结构用于让反馈能够表达明确阶段、落在明确位置层级，并在持续一段时间后完成结束或消退。

## 1. 能力声明（Capability Statement）

- `C1` 反馈阶段表达能力：让反馈能够表达处理中、成功、失败、提示等明确阶段。
- `C2` 反馈落位能力：让反馈具备明确位置与层级关系。
- `C3` 反馈持续与结束能力：让反馈具备持续时机、结束方式或撤销方式。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责动作发起：不负责决定提交或其他动作是否开始。
- `N2` 不负责校验规则：不负责校验规则的定义与执行。
- `N3` 不负责默认样式：不负责边界、颜色和标题层级等默认视觉基线。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 反馈内容边界：反馈过程结构必须允许定义明确反馈内容与反馈阶段。来源：`C1`。
- `P2` 反馈位置边界：反馈过程结构必须允许定义反馈位置、反馈层级与依附对象。来源：`C2`。
- `P3` 反馈时机边界：反馈过程结构必须允许定义反馈持续时长、结束方式或消退条件。来源：`C3`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 阶段表达结构：用于确定反馈的阶段类型与反馈内容。来源：`C1 + P1`。
- `B2` 位置层级结构：用于确定反馈的落位位置与界面层级。来源：`C2 + P2`。
- `B3` 持续结束结构：用于确定反馈的持续时机与结束方式。来源：`C3 + P3`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基本反馈过程组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先为反馈定义明确阶段与内容，使对象具备最基本反馈表达能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1`。

- `R2` 定位反馈过程组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在反馈阶段表达成立的前提下，引入位置层级结构，使反馈落在明确位置并具备层级关系。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 参数绑定：`P1 + P2`。

- `R3` 完整反馈过程组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在阶段表达与位置层级成立的前提下，引入持续结束结构，使反馈能够明确维持、结束或消退。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 参数绑定：`P1 + P2 + P3`。

- `R4` 无效悬空组合
  - `R4.1` 参与基：`B2` 或 `B3`。
  - `R4.2` 组合方式：只有反馈位置层级或持续结束结构，而没有阶段表达结构。
  - `R4.3` 失效结论：如果没有明确反馈内容与阶段，则反馈位置与持续时机失去语义对象，不能构成合法反馈过程结构。
  - `R4.4` 参数绑定：缺失 `P1`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R3` 输出能力并集必须覆盖 `C1 ~ C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

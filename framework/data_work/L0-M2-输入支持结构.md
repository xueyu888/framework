# 输入支持结构:InputSupport

@framework

## 0. 目标 (Goal)

- 输入支持结构负责让一个已经成立的界面单元具备输入能力。
- 该结构聚焦“谁可以输入 -> 如何输入”的结构成立过程，不承担校验、提交、状态反馈等职责。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 输入行为归属：输入行为需要归属于明确界面单元，使输入行为不悬空。
- `B2` 聚焦支持：界面单元需要具备进入输入状态的条件。
- `B3` 接收输入：需要能够接收用户输入内容。


## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `输入准备成立`：由 `{B1, B2}` 形成界面单元进入输入状态的基础。
- `R2` `输入支持成立`：由 `{B1, B2, B3}` 形成界面单元完整的输入支持能力。

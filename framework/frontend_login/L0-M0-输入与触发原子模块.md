# 输入与触发原子模块:InputTriggerAtomicModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 输入结构抽象能力：定义输入容器、校验位和状态位的稳定结构。
- `C2` 触发结构抽象能力：定义提交触发与禁用切换的稳定结构。
- `C3` 原子兼容能力：保证输入与触发结构在上层组合时接口一致。
- `C4` 非能力项：不负责图标反馈结构和业务流程编排。

## 2. 边界定义（Boundary / 参数）

- `INPUT` 输入槽位约束：输入容器和校验插槽职责需固定。来源：`C1 + C3`。
- `TRIGGER` 触发槽位约束：触发节点、禁用态和提交态需可判定。来源：`C2 + C3`。
- `ACCESS` 可访问约束：焦点顺序和语义标签需完整。来源：`C1 + C2`。
- `TOKEN` 令牌映射约束：原子样式映射需依赖语义令牌。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入原子基：由输入容器、校验插槽和事件口构成。来源：`C1 + INPUT + ACCESS`。
- `B2` 触发原子基：由提交触发面、禁用态和状态回写口构成。来源：`C2 + TRIGGER + TOKEN`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 原子定义组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定输入槽位结构，再挂接触发结构和状态回写口。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`INPUT/TRIGGER/ACCESS/TOKEN`。
- `R2` 禁止组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将颜色值、业务文案等实例参数固化进原子定义的方案无效。
  - `R2.3` 输出能力：`C4`。
  - `R2.4` 边界绑定：`INPUT/TRIGGER/ACCESS/TOKEN`。

## 5. 验证（Verification）

- `V1` 结构完整性：输入与触发结构必须可独立复用。
- `V2` 接口一致性：组合后输入和触发结构接口语义必须一致。
- `V3` 约束符合性：实例参数不得固化在 `B*` 中。

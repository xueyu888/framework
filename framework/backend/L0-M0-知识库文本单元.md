# 知识库文本单元模块: KnowledgeBaseTextUnitModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 单元定位能力: 支持通过定位键唯一定位文本单元
- `C2` 原文读取能力: 支持读取文本单元的文本块内容
- `C3` 归属追溯能力: 支持将文本单元追溯到其所属文档索引
- `N1` 不提供预处理能力: 不负责原始文档清洗、切块、纠错、去噪
- `N2` 不提供语义检索能。力: 不负责自然语言查询理解、内容匹配、向量召回、排序、重排
- `N3` 不提供复杂追溯能力: 不负责版本链、修改历史、来源血缘、页码偏移
- `N4` 不提供原始载体恢复能力: 不负责恢复清洗前的 PDF、HTML、表格、图片或版式结构

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 序号值域: 无符号64位整型
- `P2` 文档归属索引值域: 无符号64位整型
- `P3` 文本长度: 文本的最大字数限制，或者token的最大长度限制
- `P4` 文本语言: 中文或者英文

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：文本的具体内容
- `B2` 序号(seq)：文本块在所属范围内的顺序位置
- `B3` 文档归属索引(doc_id)：文本块所属的文档索引

## 4. 基组合原则（Base Combination Principles）

- `R1` 必须满足的原则：
  - `R1.1` 定位唯一性原则: 文本单元必须满足 `(doc_id, seq)` 联合唯一，即任意两个文本单元若具有相同的 `doc_id` 和 `seq`，则它们必须是同一个文本单元
  - `R1.2` 顺序连续性原则: 同一 `doc_id` 下的文本单元应形成一个以 `seq` 为序号的连续序列，即不存在缺失的 `seq` 值
  - `R1.3` 归属明确性原则: 每个文本单元必须且仅能归属于一个文档索引，即每个文本单元的 `doc_id` 必须存在且唯一
- `R2` 内容单基退化组合: 若仅存在
- `R3` 完整定位组合: 若同时存在
  - `C1` 单元定位能力
  - `C2` 原文读取能力
  - `C3` 最小归属追溯能力

- `R4` 基排列无关原则: `B1`、`B2`、`B3` 的排列顺序不影响它们的组合能力，即 `B1 + B2 + B3`、`B2 + B3 + B1`、`B3 + B1 + B2` 等不同排列方式的组合在能力上等价，均能导出 `C1`、`C2`、`C3` 

## 5. 验证（Verification）
- `V1` 规则一致性: 每个 `R-` 必须明确给出参与基、组合条件、输出能力或失效结论，以及绑定的参数边界
- `V2` 结构闭环性: 合法组合规则 `R2 ~ R4` 的输出能力并集必须覆盖所有声明的能力，且不得包含未声明的能力
- `V3` 边界符合性: 任意有效文本单元必须同时满足所有边界
- `V4` 最小必要性: 移除任一 `B-` 后，`C1 ~ C3` 中至少一项失败
- `V5` 结论表达: 逐条输出

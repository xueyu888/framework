# 回答文本单元:AnswerTextUnit

@framework
## 0. 目标
AI根据证据文本和提问文本最终生成的答案文本单元。

## 1. 能力声明（Capability Statement）

- `C1` 单元定位能力: 支持通过定位键唯一定位文本单元

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责具体实例值、业务文案与外部实现细节。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 编号值域: 无符号64位整型
- `P2` 文本块长度: 文本的最大字数限制，或者token的最大长度限制
- `P3` 文本块语言: 中文或者英文


## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：具体的提问内容
- `B2` 序号：文本块在所属范围内的顺序位置
- `B3` 用户溯源：文本块所属的用户
- `B4` 会话溯源：文本块所属的会话
- `B5` 时间戳：文本块被创建或修改的时间信息

## 4. 基组合原则（Base Combination Principles）
- `R1` 必须满足的原则：
  - `R1.1` 定位唯一性原则: 文本单元必须满足 `(用户溯源, 会话溯源, seq)` 联合唯一，即任意两个文本单元若具有相同的 `用户溯源`、`会话溯源` 和 `seq`，则它们必须是同一个文本单元
  - `R1.2` 顺序连续性原则: 同一 `用户溯源` 和 `会话溯源` 下的文本单元应形成一个以 `seq` 为序号的连续序列，即不存在缺失的 `seq` 值
  - `R1.3` 归属明确性原则: 每个文本单元必须且仅能归属于一个用户和一个会话，即每个文本单元的 `用户溯源` 和 `会话溯源` 必须存在且唯一
- `R2` 完整定位组合: 满足`R1 + B1 + B2 + B3 + B4 + B5` 的组合能够导出 `C1`、`C2`、`C3`

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

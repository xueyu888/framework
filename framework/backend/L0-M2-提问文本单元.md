# 提问文本单元:QueryTextUnit

@framework

## 0. 目标 (Goal)
- 提问文本单元定义“单次问答请求中的提问文本对象”，用于承载可读取的提问文本内容与可定位标识。
- 该单元只回答两件事：提问文本是什么、如何稳定定位与读取；不承担向量化、检索、重排或生成职责。

## 1. 能力声明（Capability Statement）

- `C1` 单元定位能力：支持通过提问绑定键唯一定位提问文本单元。
- `C2` 原文读取能力：支持读取提问文本单元的文本内容。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责向量化、检索命中计算、重排排序与回答生成。
- `N2` 非职责声明：不负责规定下游必须以绑定键或文本原文消费本单元；下游消费协议由下游模块边界定义。

## 2. 边界定义（Boundary / Parameter 参数）

- `query_text_length` 提问文本长度：对应 `B1`。必须给出正整数上限与计量单位（字符或 token）；有效文本长度必须落在闭区间 `[1, 上限]`。
- `query_text_lang` 提问文本语言：对应 `B1`。必须给出允许语言的有限枚举集合；不在集合内的文本视为非法。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 提问文本块：单次提问对应的文本内容。
- `B2` 提问绑定键：唯一定位 `B1` 的绑定键。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法提问单元组合：
  - `R1.1` 参与基：`B1 + B2`
  - `R1.2` 组合方式：`B1` 必须满足 `query_text_length + query_text_lang`，`B2` 必须满足 `query_binding_key_uniqueness`，从而形成可定位且可读取的提问文本单元。
  - `R1.3` 输出能力：`C1 + C2`
  - `R1.4` 参数绑定：`query_binding_key_uniqueness + query_text_length + query_text_lang`

- `R2` 非法组合：
  - `R2.1` 若提问文本长度或语言不满足边界约束，则组合非法。
  - `R2.2` 若任一提问绑定键无法唯一定位一个提问文本单元，则组合非法。
  - `R2.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1` 的输出能力必须覆盖 `C1 ~ C2`，且不得包含未声明能力。
- `V3` 参数符合性：任意有效提问文本单元必须同时满足 `query_binding_key_uniqueness + query_text_length + query_text_lang`。
- `V4` 可度量性：每个边界项都必须可落到数值、有限枚举或唯一性断言；禁止不可判定表述。
- `V5` 可测试性：每个边界项都必须对应至少一个可执行通过/失败断言，例如长度断言、语言断言、唯一性断言。
- `V6` 最小必要性：移除任一 `B*` 后，`C1 ~ C2` 中至少一项无法成立。
- `V7` 非职责隔离性：提问文本单元不得直接承担向量化、检索、重排或生成职责。
- `V8` 结论表达：逐条输出 `R1 -> C* + 参数绑定` 或 `R2 -> 非法组合` 的通过或失败结论。

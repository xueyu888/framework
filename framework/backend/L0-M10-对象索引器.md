# 对象索引器:ObjectIndexer

@framework

## 0. 目标 (Goal)
- 对象索引器负责持有对象本体、对象键与索引值之间的对应关系，并提供候选召回与按键读取能力。
- 该索引器不限定对象一定是文本；对象可以是文本、图片、片段或其他可被读取的业务对象。
- 该索引器不负责生成索引值与精确打分；索引值由上游模块提供，分值由独立比较器负责。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 对象键：用于稳定标识对象本体的键。
- `B2` 存储对象：被写入与读取的对象本体。
- `B3` 索引值：附着在对象上的可召回表示。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `索引写入`：由 `{B1, B2, B3}` 形成合法索引写入，导出 `C1`。
- `R2` `候选召回`：由 `{B1, B3}` 形成合法候选召回，导出 `C2`。
- `R3` `按键读取`：由 `{B1, B2}` 形成合法按键读取，导出 `C3`。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `object_write_in`：运行时输入接口，接收 `{object_key, object_payload, index_value}` 并写入对象索引条目。
- `index_query_in`：运行时输入接口，接收 `{query_index_value}` 并召回候选键值对。
- `object_key_list_in`：运行时输入接口，接收 `1..N` 个 `{object_key}` 并读取对象本体。
- `candidate_pair_list_out`：运行时输出接口，导出 `1..N` 个 `{candidate_key, candidate_vector}`。
- `object_pair_list_out`：运行时输出接口，导出 `1..N` 个 `{object_key, object_payload}`。

### 3.2 参数边界（Parameter Constraints）

- `object_key_uniqueness` 对象键唯一性：约束任意两个不同对象不得共享同一 `object_key`。
- `index_value_kind` 索引值类型：约束 `index_value` 的类型为声明允许的有限集合之一；当前检索链路中通常为向量。
- `recall_count` 召回数量：约束单次召回允许返回的候选数量范围。

## 4. 能力声明（Capability Statement）

- `C1` 索引写入能力：支持按 `{object_key, object_payload, index_value}` 写入对象索引条目。
- `C2` 候选召回能力：支持按 `query_index_value` 召回带键候选索引值列表。
- `C3` 按键读取能力：支持按 `object_key` 读取对象本体。
- `N1` 非职责声明：不负责生成索引值、精确打分、阈值过滤与 Top-K 裁剪。
- `N2` 非职责声明：不负责回答生成、证据确认与其他领域结论。

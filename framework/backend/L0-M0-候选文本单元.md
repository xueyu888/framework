# 候选文本单元:CandidateKnowledgeTextUnit

@framework
## 0. 目标
候选文本单元从向量数据库中检索出来的候选证据，但并不一定是最终的证据。一般来说，最终的证据还需要经过重排的流程。当然也可以不重排，直接用，不过效果会打折扣。

## 1. 能力声明（Capability Statement）

- `C1` 单元定位能力: 支持通过定位键唯一定位文本单元

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责具体实例值、业务文案与外部实现细节。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 编号值域: 无符号64位整型
- `P2` 文本块长度: 文本的最大字数限制，或者token的最大长度限制
- `P3` 文本块语言: 中文或者英文

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：文本的具体内容
- `B2` 编号：候选文本的编号

## 4. 基组合原则（Base Combination Principles）

- `R1` 必须满足的原则：
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：`B1` 和 `B2` 同时存在
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 参数绑定：`P1 + P2 + P3`。


## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

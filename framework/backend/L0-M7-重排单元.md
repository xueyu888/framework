# 重排单元:RerankUnit

@framework

## 0. 目标

提问文本单元是知识问答系统中用于承载用户查询的基本结构单元。它的设计目标是提供一个清晰、稳定、可复用的结构基础，使得系统能够准确理解和处理用户的查询意图，并为后续的语义检索、内容匹配、向量召回等能力提供可靠的输入。

## 1. 能力声明（Capability Statement）

- `C1` 单元定位能力: 支持通过定位键唯一定位文本单元
- `C2` 原文读取能力: 支持读取文本单元的文本块内容
- `C3` 归属追溯能力: 支持将文本单元追溯到其所属用户

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 序号值域: 无符号64位整型
- `P2` 文档归属索引值域: 无符号64位整型
- `P3` 文本长度: 文本的最大字数限制，或者token的最大长度限制
- `P4` 文本语言: 中文或者英文

## 4. 基组合原则（Base Combination Principles）
- `R1` 必须满足的原则：
  - `R1.1` 定位唯一性原则: 文本单元必须满足 `(用户溯源, 会话溯源, seq)` 联合唯一，即任意两个文本单元若具有相同的 `用户溯源`、`会话溯源` 和 `seq`，则它们必须是同一个文本单元
  - `R1.2` 顺序连续性原则: 同一 `用户溯源` 和 `会话溯源` 下的文本单元应形成一个以 `seq` 为序号的连续序列，即不存在缺失的 `seq` 值
  - `R1.3` 归属明确性原则: 每个文本单元必须且仅能归属于一个用户和一个会话，即每个文本单元的 `用户溯源` 和 `会话溯源` 必须存在且唯一
- `R2` 完整定位组合: 满足`R1 + B1 + B2 + B3 + B4 + B5` 的组合能够导出 `C1`、`C2`、`C3`

## 5. 验证（Verification）
- `V1` 规则一致性: 每个 `R-` 必须明确给出参与基、组合条件、输出能力或失效结论，以及绑定的参数边界
- `V2` 结构闭环性: 合法组合规则 `R2 ~ R4` 的输出能力并集必须覆盖所有声明的能力，且不得包含未声明的能力
- `V3` 边界符合性: 任意有效文本单元必须同时满足所有边界
- `V4` 最小必要性: 移除任一 `B-` 后，`C1 ~ C3` 中至少一项失败
- `V5` 结论表达: 逐条输出

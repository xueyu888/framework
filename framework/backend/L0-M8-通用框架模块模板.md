# 通用框架模块模板:GenericFrameworkModuleTemplate

@framework

## 0. 目标 (Goal)
- 本模板用于描述通用框架模块，不预设模块一定是对象型、关系型、容器型、判定型、处理型或编排型。
- 本模板的重点是把 `最小结构基（Base）`、`基排列组合（Rule）`、`接口定义（IO / Ports）`、`参数边界（Boundary / Parameter）` 显式分开，避免把输入输出误写成静态配置，或把配置误写成运行时载荷。
- 本模板只提供共同结构写法，不预先规定具体模块属于哪一种型。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` `<结构基 1>`：当前模块内部的一个最小结构件。
- `B2` `<结构基 2>`：当前模块内部的一个最小结构件。
- `B3` `<结构基 3>`：当前模块内部的一个最小结构件。
- `B4` `<结构基 4>`：当前模块内部的一个最小结构件。

## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `规则 1 名称`：由 `{B1, B2}` 形成 `结果 1`，导出 `C1`。
- `R2` `规则 2 名称`：由 `<B1, B3, B4>` 形成 `结果 2`，导出 `C2`。
- `R3` `规则 3 名称`：由 `{B2, B4}` 或 `<B2, B4>` 形成 `结果 3`，导出 `C*` 或中间结果。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `main_in`：运行时输入接口。载荷类型为 `<文本 / 向量 / 集合 / 结构化对象>`；基数为 `<1 / 0..N / 1..N>`；语义为 `<一句话说明该输入是什么>`。
- `aux_in`：运行时输入接口。仅当 `<条件>` 成立时出现；若缺失则 `<对应行为>`。
- `main_out`：运行时输出接口。载荷类型为 `<文本 / 向量 / 集合 / 结构化对象>`；基数为 `<1 / 0..N / 1..N>`；语义为 `<一句话说明该输出是什么>`。
- `trace_out`：运行时输出接口。必须与其它接口或 `B*` 在 `<顺序 / 数量 / 键位 / 索引>` 上保持一致。

### 3.2 参数边界（Parameter Constraints）

- `P1` `<参数 1>`：约束 `<输入接口 / 输出接口 / 结构基>` 的合法变化范围；必须可表达为 `<数值区间 / 有限枚举 / 阈值 / 窗口 / 判定条件>`。
- `P2` `<参数 2>`：约束 `<输入接口 / 输出接口 / 结构基>` 的合法作用域。
- `P3` `<参数 3>`：约束 `<结构一致性 / 对齐关系 / 维度 / 格式 / 语言 / 归一化规则>`；必须可执行校验。
- `P4` `<参数 4>`：约束 `<排序 / 截断 / 过滤 / 比较 / 选择策略>`；不得直接写死某个项目实例值。

## 4. 能力声明（Capability Statement）

- `C1` `<能力 1 名称>`：在 `<输入条件>` 与 `<内部结构条件>` 成立时，能够稳定导出 `<输出结果>`。
- `C2` `<能力 2 名称>`：在 `<输入条件>` 与 `<组合条件>` 成立时，能够稳定导出 `<第二类结果>`。
- `N1` 非职责声明：不负责 `<上游真相源 / 下游最终结论 / 相邻模块职责>`。
- `N2` 非职责声明：不负责在外部接口暴露 `<内部标识 / 中间结构 / 非稳定实现细节>`。
- `N3` 非职责声明：不负责声明跨模块拓扑；编排关系由更高层编排模块统一表达。

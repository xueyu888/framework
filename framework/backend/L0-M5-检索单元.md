# 检索单元:RetrievalUnit

@framework

## 0. 目标 (Goal)
- 检索单元负责在明确的文本输入输出契约与内部向量执行约束下，完成提问文本接收、查询向量生成、文本向量索引检索、命中裁剪及候选文本导出。
- 该单元对外只暴露“提问文本 -> `0..N` 个候选文本 + 分值”，不暴露绑定键、来源定位键或内部索引标识。
- 该单元的内部核心载体为“文本向量索引”：候选文本可入索引、提问文本可转向量、命中向量可稳定回指候选文本。

## 1. 能力声明（Capability Statement）

- `C1` 候选文本导出能力：在提问文本输入与文本向量索引存在时，能够导出 `0..N` 个候选文本。
- `C2` 分值伴随导出能力：能够形成可比较分值，并在候选文本导出时一一伴随输出。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责候选文本生成、最终证据确认、重排排序与回答生成。
- `N2` 非职责声明：不负责在外部边界暴露绑定键、来源定位键或内部索引标识。
- `N3` 非职责声明：不负责声明上游模块来源或跨模块拓扑；编排关系由编排模块统一表达。

## 2. 边界定义（Boundary / Parameter 参数）

### 2.1 外部边界（External Boundary）

- `query_text_in` 提问文本输入：对应 `B1 + B2`。必须是单个非空文本，且必须能按 `B1` 的统一表示规则转换为唯一 `B2`。
- `candidate_text_out` 候选文本输出：对应 `B1 + B3 + B5`。每个被保留命中项都必须能经 `B1` 从命中向量回指到候选文本；输出数量必须与保留命中数一致。
- `candidate_hit_count` 候选命中输出基数：对应 `B3 + B5`。必须落在闭区间 `[0, top_k]`。
- `score_out` 分值输出：对应 `B5`。每个 `candidate_text_out` 项都必须携带且只携带一个数值分值，顺序必须一一对齐。
- `top_k` 候选结果数量上限：对应 `B3 + B5`。必须给出正整数；最终导出数量不得超过该值。
- `score_order` 分值优先方向：对应 `B5`。必须明确为 `desc` 或 `asc`。
- `min_score` 最低保留阈值：对应 `B5`。必须给出数值阈值，或显式声明为空；若非空，未达到阈值的候选不得导出。

### 2.2 内部边界（Internal Boundary）

- `vector_dimension` 向量维度：对应 `B1 + B2 + B3`。必须给出正整数维度；`B2` 与 `B3` 中每个向量维度都必须严格相等。
- `vector_norm` 向量归一化规则：对应 `B2 + B3`。取值必须为 `none` 或 `l2`；若为 `l2`，则向量范数必须在实现容差内满足约束。
- `retrieval_scope` 检索作用域：对应 `B1 + B3`。必须可解析为有限索引子集、命名索引集合或确定性过滤表达式；超出作用域的向量不得参与比较。
- `comparator_kind` 比较器类型：对应 `B2 + B3 + B4 + B5`。必须明确为 `cosine_similarity`、`dot_product` 或 `l2_distance` 之一。
- `text_vector_index_traceability` 文本向量索引可追溯性：对应 `B1`。必须同时满足：文本可入索引、文本可转向量、命中向量可唯一回指候选文本。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本向量索引：内部结构，承载文本入索引、文本到向量映射、命中向量到候选文本回指。
- `B2` 查询向量：由输入提问文本按 `B1` 规则生成的查询向量。
- `B3` 候选向量集：`retrieval_scope` 内由 `B1` 暴露的可比较候选向量集合。
- `B4` 相似度比较器：定义在 `B2` 与 `B3` 所在向量空间上的合法比较器。
- `B5` 比较分值集：`B2` 与 `B3` 经 `B4` 计算后的数值结果集合。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法比较组合：
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5`
  - `R1.2` 组合方式：`query_text_in` 经 `B1` 转换得到 `B2`；`B1` 在 `retrieval_scope` 内暴露 `B3`；`B4` 对 `B2` 与 `B3` 执行比较，形成 `B5`。
  - `R1.3` 输出能力：`C2`
  - `R1.4` 参数绑定：`query_text_in + vector_dimension + vector_norm + retrieval_scope + comparator_kind + text_vector_index_traceability`

- `R2` 合法导出组合：
  - `R2.1` 参与基：`B1 + B3 + B5`
  - `R2.2` 组合方式：对 `R1` 导出的 `B5` 按 `score_order` 排序、按 `min_score` 过滤、按 `top_k` 裁剪；每个保留命中向量都必须经 `B1` 唯一回指候选文本；最终导出 `candidate_text_out + score_out`，并令 `candidate_hit_count` 等于保留结果数量。
  - `R2.3` 输出能力：`C1 + C2`
  - `R2.4` 参数绑定：`candidate_text_out + candidate_hit_count + score_out + top_k + score_order + min_score + text_vector_index_traceability`

- `R3` 非法组合：
  - `R3.1` 若 `query_text_in` 不能被确定性转换为唯一 `B2`，则组合非法。
  - `R3.2` 若 `retrieval_scope` 不能导出可比较 `B3`，或 `B4` 不是合法比较器，则组合非法。
  - `R3.3` 若排序、阈值过滤、`top_k` 裁剪不可确定执行，或任一保留命中向量不能唯一回指候选文本，则组合非法。
  - `R3.4` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定参数边界。
- `V2` 结构闭环性：合法组合规则 `R1 ~ R2` 的输出能力并集必须覆盖 `C1 ~ C2`，且不得包含未声明能力。
- `V3` 外部边界符合性：任意有效检索组合必须同时满足 `query_text_in + candidate_text_out + candidate_hit_count + score_out + top_k + score_order + min_score`。
- `V4` 内部边界符合性：任意有效检索组合必须同时满足 `vector_dimension + vector_norm + retrieval_scope + comparator_kind + text_vector_index_traceability`。
- `V5` 可度量性：每个边界项都必须可落到数值、有限枚举、非空文本或确定性过滤表达式；禁止不可判定表述。
- `V6` 可测试性：每个边界项都必须对应至少一个可执行通过/失败断言，例如非空文本断言、维度断言、归一化断言、比较器断言、排序断言、阈值断言、数量断言、文本回指断言。
- `V7` 输出完备性：`R2` 导出的每个候选文本都必须携带且只携带一个分值，且 `candidate_text_out`、`score_out`、`candidate_hit_count` 三者必须一一对应。
- `V8` 最小必要性：移除任一 `B*` 后，`C1 ~ C2` 中至少一项无法成立。
- `V9` 非职责隔离性：检索单元不得直接导出最终证据结论或回答文本。
- `V10` 边界职责隔离性：文本输入输出接口仅出现在边界定义与参数绑定；`B*` 仅声明内部向量结构与比较载体。
- `V11` 接口去键化：外部边界不得要求或导出绑定键、来源定位键或内部索引标识。
- `V12` 结论表达：逐条输出 `R1/R2 -> C* + 参数绑定` 或 `R3 -> 非法组合` 的通过或失败结论。

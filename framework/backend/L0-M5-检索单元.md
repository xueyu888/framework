# 检索单元:RetrievalUnit

@framework

## 0. 目标 (Goal)
- 检索单元负责在合法向量空间内，以查询向量为比较轴，对索引向量集进行相似度比较、命中裁剪与来源定位导出。

## 1. 能力声明（Capability Statement）

- `C1` 比较结果导出能力：在查询向量、索引向量集与合法比较器存在时，能够导出候选向量的比较结果集合。
- `C2` 候选召回能力：在比较结果存在时，能够按召回窗口导出有限命中集合。
- `C3` 来源定位导出能力：在命中集合存在时，能够导出对应来源文本单元的定位键集合。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责候选文本原文载入、最终证据确认、重排排序与回答生成。

## 2. 边界定义（Boundary / Parameter 参数）

- `retrieval_scope` 检索作用域：对应 `B3`。`B3` 中允许参与本次检索的向量集合范围、过滤窗口或索引子集；超出该作用域的向量不得进入本次比较。
- `score_semantics` 比较分值语义：对应 `B1 + B2 + B3 + B4`。`B4` 必须是在 `B2` 上合法定义的比较器；比较结果必须可比较，并明确分值的优先方向，例如“分值越大越优”或“分值越小越优”。
- `recall_window` 召回窗口：对应 `B3 + B4` 的命中裁剪组合。比较结果中允许保留的命中数量上限、分值阈值，或二者组合，必须在本参数内明确。
- `source_key` 来源定位键：对应 `B5`。`B5` 

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 查询向量：用于发起检索的向量，是本次比较的查询轴。
- `B2` 向量空间：`B1` 与 `B3` 中元素共同所属的可比较向量域。
- `B3` 索引向量集：被检索的一组候选向量，且每个候选向量都属于 `B2`。
- `B4` 相似度比较器：定义在 `B2` 上的比较器，用于计算 `B1` 与 `B3` 中各向量的比较结果。
- `B5` 来源定位键：把 `B3` 中每个候选向量映射到其来源文本单元的定位键（必须唯一）。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法比较组合：
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`
  - `R1.2` 组合方式：`B1` 必须属于 `B2`，`B3` 中每个元素都必须属于同一 `B2`，`B4` 对 `B1` 与 `B3` 中各元素进行比较，形成比较结果集合。
  - `R1.3` 输出能力：`C1`
  - `R1.4` 参数绑定：`retrieval_scope + score_semantics`

- `R2` 合法召回组合：
  - `R2.1` 参与基：`B3 + B4`
  - `R2.2` 组合方式：对 `R1` 导出的比较结果集合，依据 `score_semantics` 中声明的分值优先方向进行排序，并按 `recall_window` 裁剪为有限命中集合。
  - `R2.3` 输出能力：`C2`
  - `R2.4` 参数绑定：`score_semantics + recall_window`

- `R3` 合法定位组合：
  - `R3.1` 参与基：`B5`
  - `R3.2` 组合方式：`R2` 导出的每个命中项都必须能够通过 `B5` 映射为一个来源定位键，形成来源定位结果集合。
  - `R3.3` 输出能力：`C3`
  - `R3.4` 参数绑定：`source_key`

- `R4` 非法组合：
  - `R4.1` 若 `B1` 不属于 `B2`，或 `B3` 中存在元素不属于同一 `B2`，或 `B4` 不是定义在该 `B2` 上的合法比较器，则组合非法。
  - `R4.2` 若 `R2` 导出的任一命中项无法通过 `B5` 得到满足 `source_key` 的来源定位键，则组合非法。
  - `R4.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1 ~ R3` 的输出能力并集必须覆盖 `C1 ~ C3`，且不得包含未声明的能力。
- `V3` 参数符合性：任意有效检索组合必须同时满足 `retrieval_scope ~ source_key`。
- `V4` 映射完备性：`R2` 导出的每个命中项都必须能通过 `B5` 导出唯一来源定位键。
- `V5` 最小必要性：移除任一 `B*` 后，`C1 ~ C3` 中至少一项无法成立。
- `V6` 非职责隔离性：检索单元不得直接导出候选文本原文、最终证据结论或回答文本。
- `V7` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R4 -> 非法组合` 的通过或失败结论。

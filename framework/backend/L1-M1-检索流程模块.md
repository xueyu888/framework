# 检索流程模块:RetrievalFlowModule

@framework

## 0. 目标 (Goal)
- 检索流程模块负责把单个提问文本收敛为 `0..N` 个候选文本及其分值。
- 本模块不重新定义嵌入、对象索引、相似度比较与过滤的内部算法；只定义这些 `L0` 模块如何按固定顺序闭合成检索链。

## 1. 最小结构基（Minimal Structural Bases）

- `B1` 嵌入单元：`L0.M9[R1]`。
- `B2` 向量对象索引器（检索）：`L0.M10[R2]`。
- `B3` 相似度比较器：`L0.M11[R1]`。
- `B4` 过滤器：`L0.M12[R1]`。
- `B5` 对象索引器（读写）：`L0.M10[R3]`。
- `B6` 候选文本回指器：`L0.M0[R1]`。


## 2. 基排列组合（Base Arrangement / Combination）

- `R1` `检索闭环`：`QUERY_TEXT_IN -> B1 -> B2 -> B3 -> B4 -> B5 -> B6`，导出 `C1` 与 `C2`。
- `R2` `分值来源规则`：`SCORE_OUT` 只允许来自 `B4` 保留后的分值结果。

## 3. 边界定义（Boundary）

### 3.1 接口定义（IO / Ports）

- `QUERY_TEXT_IN`：运行时输入接口，`text_in ∈ L0.M9[R1]`，基数为 `1`。
- `CANDIDATE_TEXT_OUT`：运行时输出接口，`candidate_text_out ∈ L0.M0[R1]`，基数为 `0..N`。
- `SCORE_OUT`：运行时输出接口，`score_out ∈ L0.M12[R1]`，基数为 `0..N`，并与候选文本一一对齐。

### 3.2 参数边界（Parameter Constraints）

- `query_cardinality` 提问基数：约束检索链必须且只能接收一个提问文本。
- `candidate_cardinality` 候选基数：约束候选输出数量在 `0..N` 范围内。
- `score_alignment` 分值对齐：约束 `SCORE_OUT` 与 `CANDIDATE_TEXT_OUT` 必须一一对应。
- `stage_order` 阶段顺序：约束检索链路必须保持 `B1 -> B2 -> B3 -> B4 -> B5 -> B6` 的执行顺序。

## 4. 能力声明（Capability Statement）

- `C1` 候选文本导出能力：在提问输入存在时，能够导出一个或多个候选文本单元。
- `C2` 分值伴随导出能力：在候选文本导出时，能够同时导出与候选文本一一对齐的分值列表。
- `N1` 非职责声明：不负责重定义嵌入、索引、比较器与过滤器内部算法。
- `N2` 非职责声明：不负责证据确认、回答生成与编排层拓扑管理。

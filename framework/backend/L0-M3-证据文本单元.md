# 证据文本单元:EvidenceTextUnit

@framework
## 0. 目标
最终确认的需要被本次问答引用的文本单元。它是经过清洗、切块、纠错、去噪等预处理后的文本单元，或者是经过语义检索、内容匹配、向量召回、排序、重排等处理后的文本单元。它是最终被问答系统用来生成答案的文本单元。

## 1. 能力声明（Capability Statement）

- `C1` 单元定位能力: 支持通过定位键唯一定位文本单元

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责具体实例值、业务文案与外部实现细节。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 编号值域: 无符号64位整型
- `P2` 文本块长度: 文本的最大字数限制，或者token的最大长度限制
- `P3` 文本块语言: 中文或者英文
- `P4` 相关度分数值域: 相关度分数的取值范围为0到1之间的浮点数

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文本块：文本的具体内容
- `B2` 序号：候选文本的顺序编号，按照相关度进行排序（由重排模型来输出顺序到本模块）
- `B3` 相关度分数：候选文本与查询的相关度分数，取值范围为0到1之间的浮点数（由重排模型来输出分数到本模块）

## 4. 基组合原则（Base Combination Principles）

- `R1` 必须满足的原则：
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：`B1` 与 `B2`、`B3` 以文本块为核心进行组合，形成一个完整的文本单元。
  - `R1.3` 输出能力：`C1 `。
  - `R1.4` 参数绑定：`P1 + P2 + P3`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 编排模块:OrchestrationModule

@framework

## 0. 目标 (Goal)

- 编排模块负责将不同类型的文本单元按照一定的顺序和规则进行组合，以实现从提问到检索、重排、生成等能力的有机衔接。编排模块的设计目标是提供一个灵活、可扩展的结构框架，使得系统能够根据不同的查询需求和上下文信息，动态调整文本单元的组合方式，从而优化整体的问答效果。

## 1. 能力声明（Capability Statement）

- `C1` 候选导出能力：在提问输入存在时，能够导出候选文本单元集合。
- `C2` 证据导出能力：在提问输入与候选输入存在时，能够导出证据文本单元集合。
- `C3` 回答导出能力：在提问输入与证据输入存在时，能够导出回答文本单元。
- `C4` 闭环问答编排能力：在检索、重排、生成节点按合法顺序连接时，能够形成从提问到回答的完整编排链。


## 2. 边界定义（Boundary / Parameter 参数）

- `QUERY_IN` 提问输入：`L0.M2[R1] ∈ B1,B2,B3` 基数 = `1`；检索节点、重排节点、生成节点都必须且只能接收一个提问文本单元作为查询输入。
- `CANDIDATE_OUT` 候选输出：`L0.M0[R1] ∈ B1` 基数 = `1..*`；检索节点输出一个或多个候选文本单元。
- `CANDIDATE_IN` 候选输入：`L0.M0[R1] ∈ B2` 基数 = `1..*`；重排节点输入一个或多个候选文本单元。
- `EVIDENCE_OUT` 证据输出：`L0.M3[R1] ∈ B2` 基数 = `1..*`；重排节点输出一个或多个证据文本单元。
- `EVIDENCE_IN` 证据输入：`L0.M3[R1] ∈ B3` 基数 = `1..*`；生成节点输入一个或多个证据文本单元作为上下文。
- `ANSWER_OUT` 回答输出：`L0.M4[R1] ∈ B3` 基数 = `1`；生成节点输出且只能输出一个回答文本单元。

## 3. 最小结构基（Minimal Structural Bases）
  <!-- 表示“提问输入 + 检索处理 + 候选输出”闭合成一个检索阶段节点。 -->
- `B1` 检索节点: L0.M2[R1] + L0.M5[R1] + L0.M0[R1]

  <!-- 表示“提问输入 + 候选输入 + 重排处理 + 证据输出”闭合成一个证据阶段节点。 -->
- `B2` 重排节点: L0.M2[R1] + L0.M0[R1] + L0.M7[R1] + L0.M3[R1]。
    
  <!-- 表示“提问输入 + 证据输入 + 生成处理 + 回答输出”闭合成一个回答阶段节点。 -->
- `B3` 生成节点: L0.M2[R1] + L0.M3[R1] + L0.M6[R1] + L0.M4[R1]

## 4. 基组合原则（Base Combination Principles）
- `R1` 检索节点必须消费 `QUERY_IN`，并产出 `CANDIDATE_OUT`。
- `R2` 重排节点必须消费 `QUERY_IN + CANDIDATE_IN`，并产出 `EVIDENCE_OUT`。
- `R3` 生成节点必须消费 `QUERY_IN + EVIDENCE_IN`，并产出 `ANSWER_OUT`。
- `R4` 只有 `CANDIDATE_OUT -> CANDIDATE_IN` 与 `EVIDENCE_OUT -> EVIDENCE_IN` 属于合法连接。
- `R5` 检索节点必须位于重排节点之前，重排节点必须位于生成节点之前，生成节点是回答链终端节点。


## 5. 验证（Verification）
- `V1` 规则一致性: 每个 `R-` 必须明确给出参与基、组合条件、输出能力或失效结论，以及绑定的参数边界
- `V2` 结构闭环性: 合法组合规则 `R2 ~ R4` 的输出能力并集必须覆盖所有声明的能力，且不得包含未声明的能力
- `V3` 边界符合性: 任意有效文本单元必须同时满足所有边界
- `V4` 最小必要性: 移除任一 `B-` 后，`C1 ~ C3` 中至少一项失败
- `V5` 结论表达: 逐条输出

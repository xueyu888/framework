# 上传契约模块:UploadContractModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 请求契约定义能力：提供上传请求字段与请求组织方式定义能力。
- `C2` 响应契约定义能力：提供上传结果字段与结果组织方式定义能力。
- `C3` 错误表达统一能力：提供错误码、错误消息与拒绝原因的一致性表达能力。
- `C4` 契约兼容能力：提供前后端共享契约稳定演进能力。
- `C5` 非能力项：不负责页面交互、服务端文件接收与共享治理实现。

## 2. 边界定义（Boundary / 参数）

- `REQUEST` 请求边界：上传请求字段、字段语义与字段位置必须明确。来源：`C1 + C4`。
- `RESPONSE` 响应边界：上传结果字段、成功结果与失败结果结构必须明确。来源：`C2 + C4`。
- `ERROR` 错误边界：错误码、错误消息与拒绝原因必须统一。来源：`C2 + C3 + C4`。
- `FIELD` 字段边界：字段命名、字段可选性与字段兼容规则必须明确。来源：`C1 + C2 + C4`。
- `LIMIT` 约束边界：文件大小、文件类型与基础输入约束表达必须统一。来源：`C1 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 请求结构契约基：L1.M0[R1,R2]。来源：`C1 + REQUEST + FIELD + LIMIT`。
- `B2` 响应结构契约基：L1.M0[R1,R2,R3]。来源：`C2 + RESPONSE + FIELD + ERROR`。
- `B3` 错误兼容契约基：L1.M0[R1,R3]。来源：`C3 + C4 + ERROR + LIMIT + RESPONSE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 请求契约组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：统一定义上传请求字段、字段命名与基础输入约束表达。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`REQUEST/FIELD/LIMIT`。
- `R2` 响应契约组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在请求契约基础上统一定义成功、失败与拒绝结果结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`REQUEST/RESPONSE/FIELD/ERROR`。
- `R3` 错误兼容组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将错误表达与输入约束统一收敛到稳定契约，保证前后端兼容演进。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 边界绑定：`ERROR/LIMIT/RESPONSE/FIELD`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：前后端字段命名不一致、错误表达不一致或约束语义不明确的契约方案无效。
  - `R4.3` 输出能力：`C5`。
  - `R4.4` 边界绑定：`REQUEST/RESPONSE/ERROR/FIELD/LIMIT`。

## 5. 验证（Verification）

- `V1` 请求契约一致性：上传请求结构必须满足 `REQUEST/FIELD/LIMIT` 约束。
- `V2` 响应契约一致性：上传响应结构必须满足 `RESPONSE/FIELD/ERROR` 约束。
- `V3` 错误表达一致性：错误表达必须满足 `ERROR` 约束。
- `V4` 契约兼容性：契约演进必须满足 `FIELD/LIMIT` 兼容约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

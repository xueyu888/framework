# 结构化处理服务模块:StructuredProcessingServiceModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 内容提取能力：提供原始内容提取与提取完成判定能力。
- `C2` 结果归一能力：提供处理中间结果与处理结果归一能力。
- `C3` 处理结果生成能力：提供结构化处理结果生成与结果交接能力。
- `C4` 非能力项：不负责处理输入准备与审核回写。

## 2. 边界定义（Boundary / 参数）

- `CONTENT` 内容边界：参与提取的原始内容与内容范围必须明确。来源：`C1 + C3`。
- `PROCESS` 处理边界：处理步骤、处理中间结果与处理完成条件必须明确。来源：`C1 + C2`。
- `NORMALIZE` 归一边界：处理结果字段、字段语义与字段完整性必须明确。来源：`C2 + C3`。
- `OUTPUT` 输出边界：结构化处理结果与结果输出语义必须明确。来源：`C2 + C3`。
- `QUALITY` 质量边界：处理结果达标条件与拒绝条件必须明确。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 内容提取结构基：L1.M4[R1,R2,R3]。来源：`C1 + CONTENT + PROCESS`。
  说明：负责对原始内容执行提取并形成处理中间结果。
- `B2` 结果归一结构基：L1.M4[R1,R2,R3]。来源：`C1 + C2 + NORMALIZE + QUALITY`。
  说明：负责对处理中间结果执行归一并形成统一处理结果。
- `B3` 处理结果生成结构基：L1.M4[R2,R3]。来源：`C2 + C3 + OUTPUT + NORMALIZE`。
  说明：负责生成结构化处理结果并输出统一结果表达。

## 4. 基组合原则（Base Combination Principles）

- `R1` 提取归一组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先完成原始内容提取，再将处理中间结果归一为统一结果表达。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`CONTENT/PROCESS/NORMALIZE/QUALITY`。
- `R2` 结果输出组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：基于归一结果生成结构化处理结果并输出统一结果表达。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`NORMALIZE/OUTPUT`。
- `R3` 完整结构化处理组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将内容提取、结果归一与处理结果生成收敛为完整结构化处理服务结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CONTENT/PROCESS/NORMALIZE/OUTPUT/QUALITY`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：提取结果不可归一、处理结果不可输出或质量约束不明确的方案无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`CONTENT/NORMALIZE/OUTPUT/QUALITY`。

## 5. 验证（Verification）

- `V1` 内容提取有效性：内容提取必须满足 `CONTENT/PROCESS` 约束。
- `V2` 结果归一有效性：结果归一必须满足 `NORMALIZE/QUALITY` 约束。
- `V3` 结果输出一致性：处理结果输出必须满足 `OUTPUT` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

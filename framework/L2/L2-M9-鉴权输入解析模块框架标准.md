# 鉴权输入解析模块:AuthorizationInputParsingModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 主体动作整理能力：提供主体与动作字段整理能力。
- `C2` 资源上下文整理能力：提供资源与上下文字段整理能力。
- `C3` 输入归一交接能力：提供鉴权输入归一与交接能力。
- `C4` 非能力项：不负责鉴权决策生成与范围治理。

## 2. 边界定义（Boundary / 参数）

- `SUBJECT` 主体边界：主体字段与主体来源规则必须明确。来源：`C1 + C3`。
- `ACTION` 动作边界：动作字段与动作语义规则必须明确。来源：`C1 + C3`。
- `RESOURCE` 资源边界：资源字段与资源语义规则必须明确。来源：`C2 + C3`。
- `CONTEXT` 上下文边界：上下文字段与上下文完整性规则必须明确。来源：`C2 + C3`。
- `HANDOFF` 交接边界：归一后的鉴权输入交接结果必须明确。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 主体动作整理结构基：L1.M3[R1,R2,R3]。来源：`C1 + SUBJECT + ACTION`。
  说明：负责整理主体与动作输入字段。
- `B2` 资源上下文整理结构基：L1.M3[R1,R2,R3]。来源：`C2 + RESOURCE + CONTEXT`。
  说明：负责整理资源与上下文输入字段。
- `B3` 输入归一交接结构基：L1.M3[R1,R2,R3]。来源：`C1 + C2 + C3 + HANDOFF + CONTEXT`。
  说明：负责输出统一鉴权输入并形成交接结果。

## 4. 基组合原则（Base Combination Principles）

- `R1` 输入整理组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先整理主体动作输入，再整理资源上下文输入。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SUBJECT/ACTION/RESOURCE/CONTEXT`。
- `R2` 输入交接组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：基于整理后的输入输出统一鉴权输入并形成交接结果。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`RESOURCE/CONTEXT/HANDOFF`。
- `R3` 完整输入解析组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将主体动作整理、资源上下文整理与输入归一交接收敛为完整鉴权输入解析结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SUBJECT/ACTION/RESOURCE/CONTEXT/HANDOFF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：主体、动作、资源或上下文输入不完整，或交接结果不可判定的方案无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SUBJECT/ACTION/RESOURCE/CONTEXT/HANDOFF`。

## 5. 验证（Verification）

- `V1` 主体动作整理有效性：主体动作整理必须满足 `SUBJECT/ACTION` 约束。
- `V2` 资源上下文整理有效性：资源上下文整理必须满足 `RESOURCE/CONTEXT` 约束。
- `V3` 输入交接一致性：鉴权输入交接必须满足 `HANDOFF` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

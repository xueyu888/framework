# 鉴权决策生成模块:AuthorizationDecisionGenerationModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 策略匹配能力：提供鉴权策略匹配与策略命中能力。
- `C2` 原因组装能力：提供允许或拒绝原因组装能力。
- `C3` 决策交接能力：提供鉴权决策输出与交接能力。
- `C4` 非能力项：不负责鉴权输入解析与范围治理。

## 2. 边界定义（Boundary / 参数）

- `POLICY` 策略边界：鉴权策略集合与策略匹配规则必须明确。来源：`C1 + C3`。
- `DECISION` 决策边界：允许或拒绝决策的表达规则必须明确。来源：`C1 + C3`。
- `REASON` 原因边界：原因码、原因消息与原因语义必须明确。来源：`C2 + C3`。
- `ACTION` 动作边界：参与决策的动作语义必须明确。来源：`C1 + C2`。
- `HANDOFF` 交接边界：决策输出的交接结果必须明确。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 策略匹配结构基：L1.M3[R2,R3]。来源：`C1 + POLICY + ACTION`。
  说明：负责匹配鉴权策略并形成策略命中结果。
- `B2` 原因组装结构基：L1.M3[R2,R3]。来源：`C1 + C2 + DECISION + REASON`。
  说明：负责为允许或拒绝决策组装原因说明。
- `B3` 决策交接结构基：L1.M3[R2,R3]。来源：`C2 + C3 + DECISION + HANDOFF`。
  说明：负责输出鉴权决策并形成统一交接结果。

## 4. 基组合原则（Base Combination Principles）

- `R1` 策略决策组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先匹配鉴权策略，再组装允许或拒绝原因。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`POLICY/DECISION/REASON`。
- `R2` 决策交接组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：基于决策与原因输出统一鉴权结果并形成交接结果。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`DECISION/REASON/HANDOFF`。
- `R3` 完整决策生成组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将策略匹配、原因组装与决策交接收敛为完整鉴权决策生成结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`POLICY/DECISION/REASON/HANDOFF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：策略命中不可判定、原因表达不一致或决策结果不可交接的方案无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`POLICY/DECISION/REASON/HANDOFF`。

## 5. 验证（Verification）

- `V1` 策略匹配有效性：策略匹配必须满足 `POLICY` 约束。
- `V2` 原因组装有效性：原因组装必须满足 `DECISION/REASON` 约束。
- `V3` 决策交接一致性：鉴权决策必须满足 `HANDOFF` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

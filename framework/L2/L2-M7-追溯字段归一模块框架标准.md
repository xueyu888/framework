# 追溯字段归一模块:TraceFieldNormalizationModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 字段提取能力：提供追溯字段提取与字段可用性识别能力。
- `C2` 语义归一能力：提供结果、原因与版本等字段语义归一能力。
- `C3` 归一拒绝交接能力：提供字段归一失败时的统一拒绝交接能力。
- `C4` 非能力项：不负责事件接入与记录输出。

## 2. 边界定义（Boundary / 参数）

- `FIELD` 字段边界：需提取的追溯字段与字段完整性规则必须明确。来源：`C1 + C3`。
- `RESULT` 结果边界：结果字段与结果语义规则必须明确。来源：`C1 + C2`。
- `REASON` 原因边界：原因字段与原因表达规则必须明确。来源：`C2 + C3`。
- `VERSION` 版本边界：版本字段与版本语义规则必须明确。来源：`C2 + C3`。
- `HANDOFF` 交接边界：字段归一成功或失败后的交接结果必须明确。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 字段提取结构基：L1.M2[R2,R3]。来源：`C1 + FIELD + RESULT`。
  说明：负责提取追溯事件中的关键字段。
- `B2` 语义归一结构基：L1.M2[R2,R3]。来源：`C1 + C2 + REASON + VERSION`。
  说明：负责对结果、原因与版本等字段执行语义归一。
- `B3` 归一拒绝交接结构基：L1.M2[R2,R4]。来源：`C2 + C3 + HANDOFF + REASON`。
  说明：负责在字段归一失败时输出统一拒绝交接结果。

## 4. 基组合原则（Base Combination Principles）

- `R1` 提取归一组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先提取关键字段，再完成结果、原因与版本字段语义归一。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`FIELD/RESULT/REASON/VERSION`。
- `R2` 拒绝交接组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：当字段归一失败时，输出统一拒绝交接结果。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`REASON/VERSION/HANDOFF`。
- `R3` 完整字段归一组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将字段提取、语义归一与拒绝交接收敛为完整字段归一结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`FIELD/RESULT/REASON/VERSION/HANDOFF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：关键字段缺失、语义归一不一致或拒绝结果不可交接的方案无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`FIELD/REASON/VERSION/HANDOFF`。

## 5. 验证（Verification）

- `V1` 字段提取有效性：字段提取必须满足 `FIELD` 约束。
- `V2` 语义归一有效性：语义归一必须满足 `RESULT/REASON/VERSION` 约束。
- `V3` 拒绝交接一致性：归一拒绝结果必须满足 `HANDOFF` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

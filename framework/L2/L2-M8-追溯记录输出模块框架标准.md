# 追溯记录输出模块:TraceRecordOutputModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 记录组装能力：提供追溯记录内容组装与记录结构建立能力。
- `C2` 记录写入能力：提供追溯记录持久化写入能力。
- `C3` 引用交接能力：提供记录引用输出与交接能力。
- `C4` 非能力项：不负责事件接入与字段归一。

## 2. 边界定义（Boundary / 参数）

- `RECORD` 记录边界：追溯记录字段、记录结构与记录完整性必须明确。来源：`C1 + C3`。
- `STORAGE` 存储边界：追溯记录存储对象、写入规则与写入完成条件必须明确。来源：`C1 + C2`。
- `REFERENCE` 引用边界：记录引用字段、引用语义与引用稳定性必须明确。来源：`C2 + C3`。
- `REQUEST` 请求边界：请求关联字段与请求关联规则必须明确。来源：`C1 + C2`。
- `HANDOFF` 交接边界：记录输出结果与引用交接结果必须明确。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 记录组装结构基：L1.M2[R3,R4]。来源：`C1 + RECORD + REQUEST`。
  说明：负责组装待输出的追溯记录内容。
- `B2` 记录写入结构基：L1.M2[R3,R4]。来源：`C1 + C2 + STORAGE + RECORD`。
  说明：负责将追溯记录写入目标存储。
- `B3` 引用交接结构基：L1.M2[R3,R4]。来源：`C2 + C3 + REFERENCE + HANDOFF`。
  说明：负责输出记录引用并形成统一交接结果。

## 4. 基组合原则（Base Combination Principles）

- `R1` 记录写入组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先组装追溯记录内容，再执行记录写入。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`RECORD/STORAGE/REQUEST`。
- `R2` 引用交接组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：基于写入结果输出记录引用并形成统一交接结果。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`STORAGE/REFERENCE/HANDOFF`。
- `R3` 完整记录输出组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将记录组装、记录写入与引用交接收敛为完整记录输出结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`RECORD/STORAGE/REFERENCE/REQUEST/HANDOFF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：记录结构不完整、写入不可判定或引用结果不可交接的方案无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`RECORD/STORAGE/REFERENCE/HANDOFF`。

## 5. 验证（Verification）

- `V1` 记录组装有效性：追溯记录组装必须满足 `RECORD/REQUEST` 约束。
- `V2` 记录写入有效性：追溯记录写入必须满足 `STORAGE` 约束。
- `V3` 引用交接一致性：记录引用输出必须满足 `REFERENCE/HANDOFF` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

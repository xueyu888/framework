# 状态流转控制模块:StateTransitionControlModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 当前状态读取能力：提供共享对象当前状态读取与当前状态识别能力。
- `C2` 流转规则匹配能力：提供目标状态流转规则匹配与合法性判断能力。
- `C3` 控制决策交接能力：提供允许或拒绝流转的统一交接结果能力。
- `C4` 非能力项：不负责初始状态登记与版本幂等判定。

## 2. 边界定义（Boundary / 参数）

- `CURRENT` 当前状态边界：当前状态读取字段与当前状态表示规则必须明确。来源：`C1 + C3`。
- `TARGET` 目标状态边界：目标状态字段与目标状态表达规则必须明确。来源：`C1 + C2`。
- `LEGAL` 合法边界：状态流转合法性规则与拒绝条件必须明确。来源：`C2 + C3`。
- `DECISION` 决策边界：流转允许或拒绝决策及原因表达必须明确。来源：`C2 + C3`。
- `HANDOFF` 交接边界：流转控制结果的交接对象与交接状态必须明确。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 当前状态读取结构基：L1.M1[R2,R3]。来源：`C1 + CURRENT + TARGET`。
  说明：负责读取共享对象当前状态与目标状态输入。
- `B2` 流转规则匹配结构基：L1.M1[R2,R3]。来源：`C1 + C2 + LEGAL + DECISION`。
  说明：负责匹配状态流转规则并形成合法或非法结论。
- `B3` 控制决策交接结构基：L1.M1[R2,R3]。来源：`C2 + C3 + DECISION + HANDOFF`。
  说明：负责输出流转控制决策并形成统一交接结果。

## 4. 基组合原则（Base Combination Principles）

- `R1` 状态判定组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先读取当前状态与目标状态，再匹配流转规则并形成合法性结论。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`CURRENT/TARGET/LEGAL`。
- `R2` 决策交接组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：基于合法性结论输出允许或拒绝决策，并形成统一交接结果。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`LEGAL/DECISION/HANDOFF`。
- `R3` 完整流转控制组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将当前状态读取、流转规则匹配与控制决策交接收敛为完整流转控制结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CURRENT/TARGET/LEGAL/DECISION/HANDOFF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：当前状态不可识别、流转规则不可判定或控制结果无法交接的方案无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`CURRENT/LEGAL/DECISION/HANDOFF`。

## 5. 验证（Verification）

- `V1` 状态读取有效性：当前状态读取必须满足 `CURRENT/TARGET` 约束。
- `V2` 流转规则有效性：流转规则匹配必须满足 `LEGAL` 约束。
- `V3` 决策交接一致性：控制决策必须满足 `DECISION/HANDOFF` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

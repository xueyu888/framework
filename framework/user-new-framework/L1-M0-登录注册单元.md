# 登录注册单元:LoginRegistrationUnit

@framework

## 0. 目标 (Goal)
- 登录注册单元负责在用户存取单元与密码摘要单元的约束下，完成新用户注册和登录校验。
- 该单元对外只暴露“登录名 + 密码 -> 注册结果或用户编号”，不暴露摘要内部写入细节。

## 1. 能力声明（Capability Statement）

- `C1` 注册能力：在登录名未占用且密码通过规则校验时，支持写入新用户和对应密码摘要。
- `C2` 登录能力：在登录名存在且密码比对通过时，支持导出用户编号。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责访问令牌的生成与解析。
- `N2` 非职责声明：不负责失败次数、锁定状态、角色权限或访问控制。

## 2. 边界定义（Boundary / Parameter 参数）

- `login_name_uniqueness` 登录名唯一性：对应 `B1`。注册时新登录名不得重复；登录时一个登录名只能命中一个用户。
- `password_rule_set` 密码规则集合：对应 `B2`。注册前输入密码必须满足规则表。
- `password_hash_kind` 摘要算法：对应 `B2`。注册写入与登录比对必须使用声明过的同一摘要算法。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 用户存取单元：`L0.M0[R1]`：提供用户写入、登录名定位和用户读取。
- `B2` 密码摘要单元：`L0.M1[R1]`：提供密码规则校验、摘要写入和摘要比对。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法注册组合：
  - `R1.1` 参与基：`B1 + B2`
  - `R1.2` 组合方式：先用 `B1` 判断登录名未被占用，再用 `B2` 校验输入密码并写入新摘要，最后把新用户写入 `B1`，从而形成可登录的新用户。
  - `R1.3` 输出能力：`C1`
  - `R1.4` 参数绑定：`login_name_uniqueness + password_rule_set + password_hash_kind`

- `R2` 合法登录组合：
  - `R2.1` 参与基：`B1 + B2`
  - `R2.2` 组合方式：先用登录名在 `B1` 中定位用户，再按 `B2` 读取该用户当前摘要并比对输入密码；比对通过后导出用户编号。
  - `R2.3` 输出能力：`C2`
  - `R2.4` 参数绑定：`login_name_uniqueness + password_hash_kind`

- `R3` 非法组合：
  - `R3.1` 若注册时登录名已被占用，或输入密码不满足规则表，则组合非法。
  - `R3.2` 若登录时登录名不存在或密码比对失败，则组合非法。
  - `R3.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1 ~ R2` 的输出能力并集必须覆盖 `C1 ~ C2`，且不得包含未声明能力。
- `V3` 参数符合性：任意有效登录注册组合必须同时满足 `login_name_uniqueness + password_rule_set + password_hash_kind`。
- `V4` 可测试性：每个边界项都必须对应至少一个可执行通过/失败断言，例如登录名占用断言、规则通过断言、算法标记断言、密码比对断言。
- `V5` 最小必要性：移除任一 `B*` 后，`C1 ~ C2` 中至少一项无法成立。
- `V6` 非职责隔离性：登录注册单元不得直接承担令牌生成、令牌解析或权限判断职责。
- `V7` 结论表达：逐条输出 `R1/R2 -> C* + 参数绑定` 或 `R3 -> 非法组合` 的通过或失败结论。

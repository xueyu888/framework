# 令牌单元:TokenUnit

@framework

## 0. 目标 (Goal)
- 令牌单元负责在用户存取单元与访问令牌单元的约束下，完成访问令牌生成与访问令牌解析。
- 该单元对外只暴露“用户编号 -> 访问令牌”与“令牌文本 -> 用户编号 + 到期时间”，不暴露内部签名细节。

## 1. 能力声明（Capability Statement）

- `C1` 令牌生成能力：支持按用户编号生成访问令牌。
- `C2` 令牌解析能力：支持按访问令牌文本解析用户编号和访问到期时间。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责登录名注册、密码规则校验或密码比对。
- `N2` 非职责声明：不负责权限判断、失败锁定或安全事件记账。

## 2. 边界定义（Boundary / Parameter 参数）

- `user_id_uniqueness` 用户编号唯一性：对应 `B1`。任一用户编号只能回指一条用户记录。
- `access_token_ttl` 访问令牌时长：对应 `B2`。必须给出正时长。
- `access_token_sign_kind` 访问令牌签名方式：对应 `B2`。必须给出有限枚举集合。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 用户存取单元：`L0.M0[R1]`：提供用户定位和用户编号读取。
- `B2` 访问令牌单元：`L0.M2[R1]`：提供令牌生成、令牌解析和令牌验签。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法令牌生成组合：
  - `R1.1` 参与基：`B1 + B2`
  - `R1.2` 组合方式：先确认 `B1` 中存在该用户编号，再按 `B2` 的时长和签名方式生成访问令牌。
  - `R1.3` 输出能力：`C1`
  - `R1.4` 参数绑定：`user_id_uniqueness + access_token_ttl + access_token_sign_kind`

- `R2` 合法令牌解析组合：
  - `R2.1` 参与基：`B2`
  - `R2.2` 组合方式：按 `B2` 的签名方式和载荷规则解析访问令牌，读出用户编号和访问到期时间。
  - `R2.3` 输出能力：`C2`
  - `R2.4` 参数绑定：`access_token_ttl + access_token_sign_kind`

- `R3` 非法组合：
  - `R3.1` 若用户编号无法回指一条用户记录，则组合非法。
  - `R3.2` 若访问令牌不满足时长约束，或签名方式不满足边界约束，则组合非法。
  - `R3.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1 ~ R2` 的输出能力并集必须覆盖 `C1 ~ C2`，且不得包含未声明能力。
- `V3` 参数符合性：任意有效令牌组合必须同时满足 `user_id_uniqueness + access_token_ttl + access_token_sign_kind`。
- `V4` 可测试性：每个边界项都必须对应至少一个可执行通过/失败断言，例如用户编号回指断言、访问令牌时长断言、签名方式断言。
- `V5` 最小必要性：移除任一 `B*` 后，`C1 ~ C2` 中至少一项无法成立。
- `V6` 非职责隔离性：令牌单元不得直接承担登录注册、密码校验或权限判断职责。
- `V7` 结论表达：逐条输出 `R1/R2 -> C* + 参数绑定` 或 `R3 -> 非法组合` 的通过或失败结论。

# 用户管理模块:UserManagementModule

@framework

## 0. 目标 (Goal)
- 用户管理模块负责在登录注册单元与令牌单元的约束下，完成用户注册、登录校验、令牌生成与令牌解析。
- 该模块对外只暴露“登录名 + 密码 -> 注册结果或用户编号”与“用户编号/令牌文本 -> 令牌或解析结果”，不承担权限判断、失败锁定或第三方身份接入职责。

## 1. 能力声明（Capability Statement）

- `C1` 注册能力：支持创建新用户和对应密码摘要。
- `C2` 登录能力：支持校验登录名和密码，并导出用户编号。
- `C3` 令牌生成能力：支持按用户编号生成访问令牌。
- `C4` 令牌解析能力：支持按令牌文本解析用户编号和到期时间。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责失败次数、锁定状态或任何风控状态。
- `N2` 非职责声明：不负责角色、权限、访问控制或第三方身份接入。

## 2. 边界定义（Boundary / Parameter 参数）

- `login_name_uniqueness` 登录名唯一性：对应 `B1`。注册时新登录名不得重复；登录时一个登录名只能命中一个用户。
- `password_rule_set` 密码规则集合：对应 `B1`。注册前输入密码必须满足规则表。
- `password_hash_kind` 摘要算法：对应 `B1`。注册写入与登录比对必须使用声明过的同一摘要算法。
- `access_token_ttl` 访问令牌时长：对应 `B2`。必须给出正时长。
- `access_token_sign_kind` 访问令牌签名方式：对应 `B2`。必须给出有限枚举集合。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 登录注册单元：`L1.M0[R1] + L1.M0[R2]`：提供用户注册和登录校验。
- `B2` 令牌单元：`L1.M1[R1] + L1.M1[R2]`：提供访问令牌生成和访问令牌解析。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法注册组合：
  - `R1.1` 参与基：`B1`
  - `R1.2` 组合方式：按 `B1` 的注册规则写入新用户和对应密码摘要。
  - `R1.3` 输出能力：`C1`
  - `R1.4` 参数绑定：`login_name_uniqueness + password_rule_set + password_hash_kind`

- `R2` 合法登录组合：
  - `R2.1` 参与基：`B1`
  - `R2.2` 组合方式：按 `B1` 的登录规则校验登录名和密码，并导出用户编号。
  - `R2.3` 输出能力：`C2`
  - `R2.4` 参数绑定：`login_name_uniqueness + password_hash_kind`

- `R3` 合法令牌生成组合：
  - `R3.1` 参与基：`B1 + B2`
  - `R3.2` 组合方式：登录成功后把用户编号交给 `B2`，生成访问令牌。
  - `R3.3` 输出能力：`C3`
  - `R3.4` 参数绑定：`access_token_ttl + access_token_sign_kind`

- `R4` 合法令牌解析组合：
  - `R4.1` 参与基：`B2`
  - `R4.2` 组合方式：把访问令牌文本交给 `B2`，解析用户编号和到期时间。
  - `R4.3` 输出能力：`C4`
  - `R4.4` 参数绑定：`access_token_ttl + access_token_sign_kind`

- `R5` 非法组合：
  - `R5.1` 若注册时登录名已被占用，或输入密码不满足规则表，则组合非法。
  - `R5.2` 若登录时密码比对失败，或访问令牌不满足时长或签名方式约束，则组合非法。
  - `R5.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1 ~ R4` 的输出能力并集必须覆盖 `C1 ~ C4`，且不得包含未声明能力。
- `V3` 参数符合性：任意有效用户管理组合必须同时满足 `login_name_uniqueness + password_rule_set + password_hash_kind + access_token_ttl + access_token_sign_kind`。
- `V4` 可测试性：每个边界项都必须对应至少一个可执行通过/失败断言，例如登录名占用断言、规则通过断言、密码比对断言、访问令牌时长断言、签名方式断言。
- `V5` 最小必要性：移除任一 `B*` 后，`C1 ~ C4` 中至少一项无法成立。
- `V6` 非职责隔离性：用户管理模块不得直接承担权限判断、失败锁定或第三方身份接入职责。
- `V7` 结论表达：逐条输出 `R1/R2/R3/R4 -> C* + 参数绑定` 或 `R5 -> 非法组合` 的通过或失败结论。

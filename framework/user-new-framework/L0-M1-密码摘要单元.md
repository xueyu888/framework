# 密码摘要单元:PasswordDigestUnit

@framework

## 0. 目标 (Goal)
- 密码摘要单元定义“密码字符串、正则表达式和摘要转换器这一组最小结构”，用于承载密码校验和摘要生成。
- 该单元只回答三件事：输入密码是什么、按什么表达式通过、如何把输入密码转换成摘要；不承担用户定位或令牌处理职责。

## 1. 能力声明（Capability Statement）

- `C1` 密码规则校验能力：支持按正则表达式判断输入密码是否可用。
- `C2` 摘要生成能力：支持按声明算法把输入密码转换成摘要。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责登录名定位、用户写入或用户读取。
- `N2` 非职责声明：不负责访问令牌生成、令牌解析或权限判断。

## 2. 边界定义（Boundary / Parameter 参数）

- `password_text_length` 密码长度：对应 `B1`。有效密码长度必须落在闭区间 `[1, 上限]`。
- `password_rule_pattern` 密码规则表达式：对应 `B2`。必须给出可执行的正则表达式。
- `password_hash_kind` 摘要算法：对应 `B3`。必须给出有限枚举集合，如 `MD5`、`SHA-1`、`SHA-256` 等。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 密码字符串：用户的原始密码。
- `B2` 正则表达式：校验密码长度和字符特征要求。
- `B3` 摘要转换器：根据输入密码，转换成边界`password_hash_kind`对应的加密算法的摘要。

## 4. 基排列组合（Base Arrangement / Combination）

- `R1` `密码校验组合`：由 `{B1, B2}` 形成密码校验路径，导出 `C1`。
- `R2` `密码摘要组合`：由 `{B1, B2, B3}` 形成密码摘要路径，导出 `C1` 与 `C2`。输入密码先经过 `B2` 的规则匹配；匹配通过后，`B3` 再把同一份输入转成摘要结果。
- `R3` `非法组合`：密码长度不落在边界内、输入文本不匹配正则表达式或摘要算法未声明时，不导出 `C1 ~ C2`。

## 5. 验证（Verification）

- `V1` 校验路径验证：`R1` 必须只由 `B1 + B2` 导出 `C1`。
- `V2` 摘要路径验证：`R2` 必须同时经过规则匹配和摘要转换，导出 `C1` 与 `C2`。
- `V3` 参数符合性：有效组合受 `password_text_length + password_rule_pattern + password_hash_kind` 约束。

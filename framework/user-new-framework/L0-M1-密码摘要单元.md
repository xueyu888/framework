# 密码摘要单元:PasswordDigestUnit

@framework

## 0. 目标 (Goal)
- 密码摘要单元定义“密码规则表和摘要条目这一对最小结构”，用于承载注册写入和登录比对时的密码处理。
- 该单元只回答三件事：密码按什么规则通过、摘要写到哪里、如何拿已有摘要和输入密码做比对；不承担用户定位或令牌处理职责。

## 1. 能力声明（Capability Statement）

- `C1` 密码规则校验能力：支持按密码规则表判断输入密码是否可用。
- `C2` 摘要写入能力：支持按声明算法为一个用户写入新的密码摘要条目。
- `C3` 摘要比对能力：支持按已有摘要条目比较输入密码是否匹配。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责登录名定位、用户写入或用户读取。
- `N2` 非职责声明：不负责访问令牌生成、令牌解析或权限判断。

## 2. 边界定义（Boundary / Parameter 参数）

- `password_rule_set` 密码规则集合：对应 `B1`。必须给出长度和字符要求的有限规则集合；不满足任一规则的输入密码视为非法。
- `password_hash_kind` 摘要算法：对应 `B2`。必须给出有限枚举集合；不在集合内的算法视为非法。
- `password_salt_length` 盐值长度：对应 `B2`。必须给出正整数长度或固定格式；不满足约束的盐值视为非法。
- `password_owner_uniqueness` 密码归属唯一性：对应 `B2`。任一用户编号只能绑定一条当前密码摘要条目。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 密码规则表：写明密码长度和字符要求，能根据输入密码输出是否通过规则校验。
- `B2` 摘要条目：写入用户编号对应的摘要正文、盐值和算法标记，能根据用户编号写入当前摘要，并与输入密码做比对。

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法密码摘要组合：
  - `R1.1` 参与基：`B1 + B2`
  - `R1.2` 组合方式：`B1` 必须满足 `password_rule_set`，`B2` 必须满足 `password_hash_kind + password_salt_length + password_owner_uniqueness`，从而形成可校验规则、可写入摘要、可比对摘要的密码摘要单元。
  - `R1.3` 输出能力：`C1 + C2 + C3`
  - `R1.4` 参数绑定：`password_rule_set + password_hash_kind + password_salt_length + password_owner_uniqueness`

- `R2` 非法组合：
  - `R2.1` 若输入密码不满足规则表，或摘要算法不在声明集合内，则组合非法。
  - `R2.2` 若盐值长度不满足边界约束，或一个用户编号命中多条当前摘要条目，则组合非法。
  - `R2.3` 非法组合不导出任何 `C*`。

## 5. 验证（Verification）

- `V1` 规则一致性：每个 `R*` 必须明确给出参与基、组合方式、输出能力或失效结论，以及绑定的参数边界。
- `V2` 结构闭环性：合法组合规则 `R1` 的输出能力必须覆盖 `C1 ~ C3`，且不得包含未声明能力。
- `V3` 参数符合性：任意有效密码摘要单元必须同时满足 `password_rule_set + password_hash_kind + password_salt_length + password_owner_uniqueness`。
- `V4` 可度量性：每个边界项都必须可落到规则集合、有限枚举、长度或唯一性断言；禁止不可判定表述。
- `V5` 可测试性：每个边界项都必须对应至少一个可执行通过/失败断言，例如规则通过断言、算法枚举断言、盐值长度断言、摘要归属唯一性断言。
- `V6` 最小必要性：移除任一 `B*` 后，`C1 ~ C3` 中至少一项无法成立。
- `V7` 非职责隔离性：密码摘要单元不得直接承担用户定位、令牌生成或令牌解析职责。
- `V8` 结论表达：逐条输出 `R1 -> C* + 参数绑定` 或 `R2 -> 非法组合` 的通过或失败结论。

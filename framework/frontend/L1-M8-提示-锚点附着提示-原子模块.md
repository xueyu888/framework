# 前端锚点附着提示原子模块:FrontendAnchorAttachedHintAtoms

@framework

模块短标签：附着提示
典型落地对象：tooltip / popover hint / annotation bubble
不包括：临时展开选择面、纯动作入口、页面级通知流

本模块定义前端锚点附着提示原子。它承接工具提示、锚点提示泡和附着信息面这类“围绕稳定锚点短时展开提示或辅助说明，并以说明感知而非候选选择结果闭合为主”的结构，不把独立任务容器、长期驻留说明区、候选选择闭合面或页面级反馈流程混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 锚点附着提示能力：把锚点、附着提示面与提示内容收敛为稳定附着结构。
- `C2` 提示出现消退能力：把进入触发、离开关闭与短时可见关系收敛为稳定提示相位结构。
- `C3` 提示可达能力：把朗读顺序、提示说明与锚点关系收敛为稳定可访问提示结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责局部任务接管、候选项选择面与长期驻留帮助面板。
- `N2` 非职责声明：不负责只承接点击、确认、切换或即时动作结果的离散触发入口结构。
- `N3` 非职责声明：不负责由候选项顺序、当前项与选择后闭合共同成立的临时展开选择面结构。

## 2. 边界定义（Boundary / 参数）

- `ANCHOR` 锚点边界：提示锚点、锚点摘要、锚点关系说明与锚点到提示面的附着关系需稳定。来源：`C1 + C3`。
- `HINT` 提示边界：提示文本、提示外壳、附着说明位与提示内容归属关系需稳定。来源：`C1 + C2`。
- `TRIGGER` 触发边界：进入触发、离开关闭、再次显示出口与触发成立条件需稳定。来源：`C2 + C3`。
- `PHASE` 相位边界：隐藏、显示与消退中的状态集合、相位迁移关系与短时可见窗口需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：聚焦锚点、提示面可读顺序、退出回退落点与进入退出条件需稳定。来源：`C1 + C3`。
- `A11Y` 可访问边界：提示朗读、锚点说明、状态播报与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 锚点到提示附着基：L0.M0[R2] + L0.M2[R1,R2]。来源：`C1 + ANCHOR + HINT + A11Y`。
- `B2` 说明感知相位基：L0.M0[R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C2 + C3 + TRIGGER + PHASE + FOCUS`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 锚点附着先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定提示锚点、附着提示面、提示内容与锚点到提示面的附着关系，形成稳定“锚点 -> 说明”附着结构，再允许提示相位接入。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`ANCHOR + HINT + A11Y`。
- `R2` 提示相位闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：提示必须围绕稳定锚点，以进入触发、离开关闭、短时显示窗口与相位迁移关系形成统一附着提示结构；其成立目标是说明感知与关系提示，而不是候选选择结果闭合。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`TRIGGER + PHASE + FOCUS + A11Y + ANCHOR + HINT`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：提示内容脱离锚点存在、锚点到提示面的附着关系不可解释、显示与消退相位不可解释，或把离散动作入口、以候选项到结果映射和选择后闭合为主的临时展开选择面、长期驻留帮助面板误写为锚点附着提示原子的方案均无效。
  - `R3.3` 失效结论：`N1 + N2 + N3`。
  - `R3.4` 边界绑定：`ANCHOR + HINT + TRIGGER + PHASE + FOCUS + A11Y`。

## 5. 验证（Verification）

- `V1` 锚点完整性：锚点附着结构必须满足 `ANCHOR + HINT` 约束；提示锚点、提示内容与锚点说明完整，且锚点到提示面的附着关系可判定。
- `V2` 相位闭合有效性：提示相位结构必须满足 `TRIGGER + PHASE` 约束；显示、隐藏与消退关系可解释，且短时可见窗口与相位迁移关系完整。
- `V3` 可达一致性：可访问提示结构必须满足 `FOCUS + A11Y` 约束；朗读顺序、锚点关系与退出回退完整，且对辅助技术可感知。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

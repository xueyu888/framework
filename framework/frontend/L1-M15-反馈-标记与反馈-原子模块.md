# 前端标记与反馈原子模块:FrontendMarkerFeedbackAtoms

@framework

模块短标签：附着反馈
典型落地对象：badge / tag / inline status / empty hint / alert bubble
不包括：锚点附着提示、输入内反馈闭环、文件摄取反馈闭环、媒体预览反馈闭环

本模块定义前端最小标记与反馈原子。它承接标签、状态标记、摘要提示、空态说明与轻量反馈气泡这类“附着在稳定主体结构上、对主体当前状态或结果回流形成可观察表达，但不改写主体主关系”的反馈结构，不把锚点提示面、主体结构专属状态闭环或页面级通知流程混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 稳定标记承接能力：把主体上的身份标记、状态标记与摘要标记收敛为稳定标记结构。
- `C2` 附着回流闭合能力：把结果反馈、空态说明与轻量动作出口收敛为稳定附着回流结构。
- `C3` 可达反馈能力：把朗读关系、提示顺序与动作说明收敛为稳定可访问反馈结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责文本编辑、集合浏览与页面路由编排。
- `N2` 非职责声明：不负责围绕稳定锚点短时附着的提示、说明与注释结构。
- `N3` 非职责声明：不负责输入、文件摄取、日期时间输入与媒体预览等主体结构内部的专属状态闭环。
- `N4` 非职责声明：不负责局部任务接管、页面级通知流与长期驻留帮助面板。

## 2. 边界定义（Boundary / 参数）

- `MARK` 标记边界：身份标记、状态标记、摘要标记与标记归属关系需稳定。来源：`C1 + C3`。
- `ATTACH` 附着边界：反馈附着位置、附着对象、附着持续窗口与附着后不改写主体主关系的条件需稳定。来源：`C2 + C3`。
- `RETURN` 回流边界：动作结果、状态结果、空态结果与反馈回流到主体边缘的映射关系需稳定。来源：`C1 + C2`。
- `STATUS` 状态边界：成功、失败、处理中、禁用说明等状态集合、状态映射关系与反馈窗口需稳定。来源：`C1 + C2`。
- `EVENT` 事件边界：关闭、确认、撤销、重试等附着反馈出口与事件结果类型需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：状态说明、动作说明、朗读顺序与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 稳定标记结构基：L0.M0[R1,R3] + L0.M2[R1,R3]。来源：`C1 + MARK + STATUS + A11Y`。
- `B2` 附着回流结构基：L0.M0[R3,R4] + L0.M1[R4] + L0.M2[R1,R3]。来源：`C2 + C3 + ATTACH + RETURN + EVENT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 标记先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定主体上的身份标记、状态标记、摘要标记与标记归属关系，形成稳定标记结构，再允许结果反馈附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`MARK + STATUS + A11Y`。
- `R2` 附着回流成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：动作结果、状态结果、空态说明与轻量动作出口必须附着在稳定标记结构或主体结构边缘，并通过反馈回流关系保持可观察，但不得替代主体结构本身或改写主体主关系。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`ATTACH + RETURN + EVENT + A11Y + MARK + STATUS`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：反馈脱离主体附着存在、标记归属关系不可解释、结果无法回流到主体边缘，或把锚点提示面、主体结构内部专属状态闭环、页面级通知流误写为标记反馈原子的方案均无效。
  - `R3.3` 失效结论：`N2 + N3 + N4`。
  - `R3.4` 边界绑定：`MARK + ATTACH + RETURN + STATUS + EVENT + A11Y`。

## 5. 验证（Verification）

- `V1` 标记完整性：稳定标记结构必须满足 `MARK + STATUS` 约束；身份标记、状态标记与摘要标记完整，且标记归属关系与状态映射关系可判定。
- `V2` 附着回流有效性：附着回流结构必须满足 `ATTACH + RETURN + EVENT` 约束；结果反馈、空态说明与轻量动作出口可解释，且反馈能够回流到主体边缘而不改写主体主关系。
- `V3` 反馈可达一致性：可访问反馈结构必须满足 `A11Y + STATUS` 约束；状态说明、动作说明与朗读顺序完整，且对辅助技术可感知。
- `V4` 主体附着稳定性：反馈必须附着于稳定主体结构，而不是替代主体结构本身；主体仍负责主关系，反馈只负责状态或结果表达。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

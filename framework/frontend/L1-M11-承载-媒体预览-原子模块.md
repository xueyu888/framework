# 前端媒体预览原子模块:FrontendMediaPreviewAtoms

@framework

模块短标签：媒体预览
典型落地对象：image preview / video preview / audio preview / document page preview
不包括：普通展示容器、局部任务接管、媒体编辑器、资源库管理

本模块定义前端媒体预览原子。它承接图片预览、视频预览、音频预览、文档页预览与资源缩略预览这类“围绕当前媒体对象、预览视口、最小控制与状态说明回流形成稳定查看闭环”的原子，不把完整媒体编辑器、资源库管理后台或后端转码解码实现混进同一模块。
本模块位于 `frontend` 的界面原子层。它在展示与容器、稳定标记与附着回流反馈等更基础前端原子之上，把媒体主内容、预览视口、预览控制与状态说明回流沉成共享结构，为附件查看、图库预览、素材浏览、文档页查看与工作台内嵌媒体查看提供统一底座。

## 1. 能力声明（Capability Statement）

- `C1` 媒体承载能力：把当前媒体对象、媒体主内容与预览视口收敛为稳定媒体承载结构。
- `C2` 预览闭环能力：把播放、暂停、切换、放大、关闭与当前媒体切换关系收敛为稳定预览闭环结构。
- `C3` 状态说明回流能力：把加载、失败、不可预览、当前状态播报、媒体元信息说明与状态附着回流关系收敛为稳定状态说明回流结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责媒体编辑、资源目录治理、后端转码解码实现与领域业务加工流程。
- `N2` 非职责声明：不负责只承载正文、说明、面板与阅读视口而不围绕当前媒体焦点形成预览闭环的普通展示容器结构。
- `N3` 非职责声明：不负责进入、退出、返回落点与背景约束共同形成的局部任务接管结构。

## 2. 边界定义（Boundary / 参数）

- `MEDIA` 媒体边界：当前媒体对象、媒体类型、媒体主内容、当前聚焦媒体与媒体切换归属关系需稳定。来源：`C1 + C2`。
- `VIEWPORT` 视口边界：预览宿主、可见范围、适配策略、缩放后可见区与视口到媒体内容的映射关系需稳定。来源：`C1 + C2`。
- `CONTROL` 控制边界：播放、暂停、切换、放大、关闭、跳转控制位与控制结果类型需稳定。来源：`C2 + C3`。
- `STATE` 状态边界：加载中、可预览、不可预览、失败、播放中与暂停中等状态集合、状态迁移关系、状态附着位置与反馈窗口需稳定。来源：`C1 + C3`。
- `META` 元信息边界：文件名、时长、尺寸、页数、来源摘要、媒体说明、元信息归属关系与元信息附着位置需稳定。来源：`C3`。
- `A11Y` 可访问边界：当前媒体提示、控制说明、状态播报、切换说明与辅助技术可感知关系需可达。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 媒体承载基：L0.M0[R1,R3] + L0.M1[R3] + L0.M2[R1,R2]。来源：`C1 + MEDIA + VIEWPORT + STATE`。
- `B2` 预览控制闭环基：L0.M0[R4] + L0.M1[R4] + L0.M2[R1,R3]。来源：`C2 + CONTROL + MEDIA + VIEWPORT + A11Y`。
- `B3` 状态说明回流基：L0.M0[R3,R4] + L0.M1[R3] + L0.M2[R1,R3]。来源：`C3 + STATE + META + A11Y + MEDIA + VIEWPORT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 媒体承载先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定当前媒体对象、媒体主内容、预览视口与视口到媒体内容的映射关系，形成稳定“当前媒体 -> 预览视口”承载结构，再允许控制位与状态反馈附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`MEDIA + VIEWPORT + STATE`。
- `R2` 预览控制闭环成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：播放、暂停、切换、放大与关闭等控制必须附着稳定媒体承载面，并通过媒体切换归属关系、当前媒体焦点与控制结果类型形成统一媒体预览闭环；其主结果是围绕当前媒体焦点的查看成立，而不是任务流接管或普通阅读承载。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CONTROL + MEDIA + VIEWPORT + A11Y`。
- `R3` 状态说明回流成立
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：加载、失败、不可预览、播放状态、媒体元信息说明与元信息归属关系必须附着在当前媒体预览结构或其边缘，并随当前媒体与预览控制关系回流，形成稳定状态说明回流结构，但不得替代媒体承载与预览控制闭环本身。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`STATE + META + A11Y + MEDIA + VIEWPORT + CONTROL`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：只有媒体壳没有当前媒体对象、存在播放或切换控制却无视口承载、状态失败却无说明，或把只承载普通阅读展示的展示容器、把以进入退出和返回落点为主的局部任务接管、把完整编辑器误写为预览原子的方案均无效。
  - `R4.3` 失效结论：`N1 + N2 + N3`。
  - `R4.4` 边界绑定：`MEDIA + VIEWPORT + CONTROL + STATE + META + A11Y`。

## 5. 验证（Verification）

- `V1` 媒体承载完整性：媒体承载结构必须满足 `MEDIA + VIEWPORT` 约束；当前媒体对象、媒体主内容与预览视口完整，且视口到媒体内容的映射关系可判定。
- `V2` 预览控制闭环有效性：预览控制闭环结构必须满足 `CONTROL + MEDIA + VIEWPORT` 约束；播放、暂停、切换、放大或关闭等最小控制可解释，且当前媒体焦点、媒体切换归属关系与控制结果类型完整。
- `V3` 状态说明回流有效性：状态说明回流结构必须满足 `STATE + META + A11Y` 约束；状态播报、媒体说明与控制说明可解释，且状态与元信息必须附着于当前媒体预览结构或其边缘，并对辅助技术可感知。
- `V4` 共享原子有效性：本模块必须能被图片预览、视频预览、音频预览、文档页预览与资源缩略预览复用，而不是被误写成某一种媒体播放器的唯一真相源。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

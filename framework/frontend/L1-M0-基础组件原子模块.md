# 前端基础组件原子模块:FrontendPrimitiveComponents

@framework

## 1. 能力声明（Capability Statement）

- `C1` 输入触发原子沉淀能力：沉淀按钮、输入框、文本域、文件选择器等最小输入结构。
- `C2` 展示承载原子沉淀能力：沉淀面板、列表、树、卡片、文本块等最小展示结构。
- `C3` 导航反馈原子沉淀能力：沉淀标签、提示、气泡、状态条与导航触发结构。
- `C4` 非能力项：不负责领域对象语义、页面路由与后端契约执行。

## 2. 边界定义（Boundary / 参数）

- `INPUTATOM` 输入原子边界：按钮、输入框、文本域与文件选择器的输入职责需稳定。来源：`C1 + C3`。
- `DISPLAYATOM` 展示原子边界：面板、列表、树、卡片与文本块的承载职责需稳定。来源：`C2 + C3`。
- `NAVATOM` 导航原子边界：页签、链接、切换器与折叠入口的导航职责需稳定。来源：`C2 + C3`。
- `FEEDBACKATOM` 反馈原子边界：提示条、占位态、加载态与消息气泡的表达职责需统一。来源：`C1 + C3`。
- `EVENT` 事件边界：点击、提交、选择、折叠与上传等事件出口需一致。来源：`C1 + C3`。
- `TOKEN` 令牌边界：原子组件只能消费语义令牌，不得写死实例样式。来源：`C2 + C3`。
- `A11Y` 可访问边界：原子层的焦点、语义标签与提示关系需达标。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入触发原子基：L0.M0[R1,R3] + L0.M1[R1,R2]。来源：`C1 + INPUTATOM + EVENT + A11Y`。
- `B2` 展示承载原子基：L0.M0[R1] + L0.M1[R1,R3]。来源：`C2 + DISPLAYATOM + TOKEN + A11Y`。
- `B3` 导航反馈原子基：L0.M0[R3] + L0.M1[R2,R3]。来源：`C2 + C3 + NAVATOM + FEEDBACKATOM + EVENT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 原子职责分层
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：将输入触发原子与展示承载原子分层组织，避免交互出口污染承载面。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`INPUTATOM/DISPLAYATOM/TOKEN/A11Y`。
- `R2` 令牌消费统一
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：所有原子都通过统一令牌与事件出口工作，不在原子层固化实例主题和领域语义。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`TOKEN/EVENT/NAVATOM/FEEDBACKATOM`。
- `R3` 插接契约输出
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：输出稳定输入口、展示口、导航口与反馈口，供更上层结构直接编排。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`INPUTATOM/DISPLAYATOM/NAVATOM/FEEDBACKATOM/EVENT/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：让原子层直接绑定领域对象、绕过令牌体系或破坏事件出口一致性的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`INPUTATOM/DISPLAYATOM/NAVATOM/FEEDBACKATOM/EVENT/TOKEN/A11Y`。

## 5. 验证（Verification）

- `V1` 原子覆盖性：输入、展示、导航和反馈四类原子必须全部具备。
- `V2` 事件一致性：同类交互事件在不同原子中的输入输出契约必须一致。
- `V3` 令牌符合性：所有原子必须通过 `TOKEN` 体系消费样式语义。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

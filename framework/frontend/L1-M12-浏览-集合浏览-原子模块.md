# 前端集合浏览原子模块:FrontendCollectionBrowsingAtoms

@framework

模块短标签：集合内浏览
典型落地对象：list / tree / item browser / side collection
不包括：页序列导航、路径链导航、临时展开选择面

本模块定义前端最小集合浏览原子。它承接列表、树、节点组与集合项浏览这类“围绕集合承载、当前项定位与当前项浏览关系持续成立”的原子，不把页序列窗口、路径链回溯、临时展开选择面或页面级路由编排混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 集合承载能力：把列表、树与节点分组关系收敛为稳定集合承载结构。
- `C2` 当前项浏览闭合能力：把当前项定位、当前项切换、展开关系与集合内移动收敛为稳定当前项浏览闭合结构。
- `C3` 浏览可达能力：把节点焦点、键盘移动与层级阅读关系收敛为稳定浏览可达结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责文本编辑、反馈提示与页面级场景流程。
- `N2` 非职责声明：不负责页序列窗口、前后页跳转与跳页输入形成的页序列导航结构。
- `N3` 非职责声明：不负责当前位置路径链、上游回溯与层级压缩表达形成的路径链导航结构。
- `N4` 非职责声明：不负责由触发入口唤起并在短时展开面中完成候选选择的临时展开选择面结构。

## 2. 边界定义（Boundary / 参数）

- `COLLECTION` 集合边界：集合顺序、分组关系、当前项落位与集合项归属关系需稳定。来源：`C1 + C2`。
- `HIERARCHY` 层级边界：层级深度、展开顺序、父子关系与层级映射关系需稳定。来源：`C1 + C2`。
- `ITEM` 项边界：集合项外壳、集合项摘要、项级动作位与项到集合关系需稳定。来源：`C1 + C2`。
- `CURRENT` 当前项边界：当前项、展开项、集合内移动结果与当前项切换条件需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：项焦点、列表焦点、层级阅读顺序与焦点回路需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：键盘移动、层级播报、当前项提示与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 集合承载基：L0.M0[R1,R3] + L0.M1[R3] + L0.M2[R1,R2]。来源：`C1 + COLLECTION + HIERARCHY + FOCUS`。
- `B2` 当前项浏览闭合基：L0.M0[R4] + L0.M1[R4] + L0.M2[R1,R3]。来源：`C2 + C3 + ITEM + CURRENT + FOCUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 集合承载先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定集合分组、层级深度、项顺序与集合项归属关系，再允许当前项与集合内移动接入。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`COLLECTION + HIERARCHY + FOCUS`。
- `R2` 当前项浏览闭合成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：当前项必须能在集合中被稳定定位，并通过当前项切换、展开关系、集合内移动与层级提示形成统一当前项浏览结构；若不能围绕当前项形成持续浏览关系，或退化为页序列窗口、路径链回溯或短时展开候选选择，则该结构不成立。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`ITEM + CURRENT + FOCUS + A11Y + COLLECTION + HIERARCHY`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：缺失父子关系、当前项不可解释、集合内移动无法回收，或把页序列、路径链、临时展开选择面误写为集合浏览原子的方案均无效。
  - `R3.3` 失效结论：`N2 + N3 + N4`。
  - `R3.4` 边界绑定：`COLLECTION + HIERARCHY + ITEM + CURRENT + FOCUS + A11Y`。

## 5. 验证（Verification）

- `V1` 集合承载完整性：集合结构必须满足 `COLLECTION + HIERARCHY` 约束；集合顺序、层级深度与当前项落位完整，且集合项归属关系与层级映射关系可判定。
- `V2` 当前项浏览闭合有效性：当前项浏览结构必须满足 `CURRENT + ITEM` 约束；当前项定位、展开关系与集合内移动可解释，且当前项切换条件完整并能持续维持当前项到集合的浏览关系。
- `V3` 浏览可达一致性：可达结构必须满足 `FOCUS + A11Y` 约束；键盘移动、层级播报与当前项提示可解释，且对辅助技术可感知。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

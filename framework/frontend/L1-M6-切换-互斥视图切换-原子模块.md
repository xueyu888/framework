# 前端互斥视图切换原子模块:FrontendMutuallyExclusiveViewSwitchAtoms

@framework

模块短标签：互斥切换
典型落地对象：tabs / segmented control / view switcher
不包括：页面级路由跳转、展开披露、一般集合浏览、离散值选择

本模块定义前端互斥视图切换原子。它承接页签、分段切换和同位视图切换栏这类“多个内容面共处同一工作面且同一时刻只激活一个视图，并围绕入口到内容面的唯一激活配对成立”的结构，不把页面级跳转、层级导航、普通按钮组或有限候选域上的值状态选择混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 切换入口承接能力：把切换入口集合、排列顺序与当前切换入口收敛为稳定切换入口结构。
- `C2` 配对激活承接能力：把切换入口与内容面的唯一配对关系、当前入口与当前激活面的对应关系收敛为稳定配对激活结构。
- `C3` 互斥切换可达能力：把唯一激活、焦点移动、当前项提示与切换播报收敛为稳定互斥切换可达结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责页面级路由跳转、层级导航树与任意自由排布的按钮组。
- `N2` 非职责声明：不负责有限候选域上的当前值、选择约束与值反馈共同形成的离散值选择结构。

## 2. 边界定义（Boundary / 参数）

- `SWITCH` 切换边界：切换入口集合、排列顺序、当前切换入口与入口切换方式需稳定。来源：`C1 + C3`。
- `PANEL` 视图面边界：被切换内容面的承载区、激活面、非激活面与视图面切换窗口需稳定。来源：`C2 + C3`。
- `PAIR` 配对边界：切换入口与内容面的唯一对应关系、当前入口到当前激活面的对应关系必须可追溯且唯一。来源：`C2 + C3`。
- `STATE` 状态边界：默认、激活、禁用与切换中等状态集合、状态迁移关系与反馈窗口需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：键盘切换顺序、当前聚焦入口、激活面进入退出路径与焦点回路需稳定。来源：`C3`。
- `A11Y` 可访问边界：当前项提示、配对关系、激活面播报与辅助技术可感知关系需可达。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 切换入口基：L0.M0[R1,R4] + L0.M1[R4] + L0.M2[R1,R4]。来源：`C1 + SWITCH + FOCUS`。
- `B2` 配对激活基：L0.M0[R1,R3] + L0.M1[R4] + L0.M2[R1,R4]。来源：`C2 + C3 + PAIR + PANEL + STATE + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 切换入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定切换入口集合、排列顺序、当前切换入口与焦点移动方式，形成稳定切换入口结构。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SWITCH + FOCUS`。
- `R2` 配对激活成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：每个切换入口都必须能追溯到唯一内容面，且当前入口必须对应唯一激活面；激活态、焦点移动、当前项提示与切换播报必须围绕该配对关系保持一致，形成稳定配对激活结构，不得退化为单纯值反馈或按钮触发。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`PAIR + PANEL + STATE + FOCUS + A11Y + SWITCH`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：只有切换入口没有配对内容面、存在多个激活面、当前入口与当前激活面不一致、配对关系不可追溯，或把页面级路由跳转、展开披露、一般集合浏览、以候选域上的当前值归属与值反馈为主的离散值选择误写为互斥视图切换原子的方案均无效。
  - `R3.3` 失效结论：`N1 + N2`。
  - `R3.4` 边界绑定：`SWITCH + PANEL + PAIR + STATE + FOCUS + A11Y`。

## 5. 验证（Verification）

- `V1` 切换入口完整性：切换入口集合、排列顺序、当前切换入口与焦点移动路径必须满足 `SWITCH + FOCUS` 约束。
- `V2` 配对激活有效性：切换入口与内容面的对应关系必须满足 `PAIR + PANEL + STATE` 约束；每次切换只能得到唯一激活面，且当前入口与当前激活面一致。
- `V3` 互斥切换可达一致性：当前项提示、配对关系、焦点移动与切换播报必须满足 `FOCUS + A11Y` 约束，并围绕当前配对激活关系保持一致。
- `V4` 共享原子有效性：本模块必须能被页签、分段切换和同位视图切换栏等场景复用，而不是被误写成某一个具体视觉控件的唯一真相源。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

# 前端互斥视图切换原子模块:FrontendMutuallyExclusiveViewSwitchAtoms

@framework

模块短标签：互斥切换
典型落地对象：tabs / segmented control / view switcher
不包括：页面级路由跳转、展开披露、一般集合浏览、离散值选择

本模块定义前端互斥视图切换原子。它承接页签、分段切换和同位视图切换栏这类“多个内容面共处同一工作面且同一时刻只激活一个视图，并围绕入口到内容面的唯一激活配对成立”的结构，不把页面级跳转、层级导航、普通按钮组或有限候选域上的值状态选择混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 互斥切换承接能力：把多个切换入口收敛为稳定互斥视图切换结构。
- `C2` 当前视图绑定能力：把当前激活项与对应内容面收敛为稳定配对结构。
- `C3` 切换可达能力：把切换顺序、焦点移动、当前项提示与切换反馈收敛为稳定可达结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责页面级路由跳转、层级导航树与任意自由排布的按钮组。
- `N2` 非职责声明：不负责有限候选域上的当前值、选择约束与值反馈共同形成的离散值选择结构。

## 2. 边界定义（Boundary / 参数）

- `SWITCH` 切换边界：切换入口集合、排列顺序、当前激活项与切换入口到视图面的映射关系需稳定。来源：`C1 + C3`。
- `PANEL` 视图面边界：被切换内容面的承载区、激活面、隐藏面关系与视图面层级关系需稳定。来源：`C2 + C3`。
- `PAIR` 配对边界：切换入口与内容面的配对关系、唯一激活关系与配对判定条件需稳定。来源：`C1 + C2`。
- `STATE` 状态边界：默认、激活、禁用与切换中等状态集合、状态迁移关系与反馈窗口需稳定。来源：`C1 + C3`。
- `FOCUS` 焦点边界：键盘切换顺序、当前聚焦项、面板内外回退路径与焦点回路需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：当前项提示、面板关系、切换播报与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 切换入口到当前激活项基：L0.M0[R1,R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C1 + SWITCH + STATE + FOCUS + A11Y`。
- `B2` 入口到视图面唯一配对基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C2 + C3 + PANEL + PAIR + STATE + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 切换入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定切换入口集合、排列顺序、当前激活项与切换入口到视图面的映射关系，形成稳定“入口 -> 当前激活项”结构，再允许更高层结构挂接对应内容面。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SWITCH + STATE + FOCUS + A11Y`。
- `R2` 入口与视图面配对
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：每个切换入口都必须能解释其对应内容面，且当前激活项必须指向唯一激活面，并满足配对判定条件，形成稳定的入口-视图面唯一配对关系；其主结果是内容面激活切换，而不是值归属变化。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`PAIR + PANEL + SWITCH + STATE`。
- `R3` 互斥激活闭合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：同一时刻只能有一个当前激活面，切换顺序、焦点移动与当前项提示必须与激活面状态保持一致，形成稳定可达切换结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SWITCH + PANEL + PAIR + STATE + FOCUS + A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2`。
  - `R4.2` 组合方式：只有切换头没有内容面、存在多个当前激活面、入口与内容面关系不可追溯，或把页面级路由跳转、展开披露、一般集合浏览、以候选域上的当前值归属与值反馈为主的离散值选择误写为互斥视图切换原子的方案均无效。
  - `R4.3` 失效结论：`N1 + N2`。
  - `R4.4` 边界绑定：`SWITCH + PANEL + PAIR + STATE + FOCUS + A11Y`。

## 5. 验证（Verification）

- `V1` 切换入口完整性：切换结构必须满足 `SWITCH + PANEL` 约束；切换入口集合、排列顺序与当前激活项完整，且切换入口到视图面的映射关系可判定。
- `V2` 配对完整性：配对结构必须满足 `PAIR + PANEL` 约束；入口与视图面的关系可解释，且唯一激活关系与配对判定条件完整。
- `V3` 唯一激活性：任一时刻只能存在一个当前激活面，且其状态必须满足 `STATE` 约束；状态迁移关系与反馈窗口可解释。
- `V4` 切换可达性：键盘切换顺序、当前项提示与切换播报必须满足 `FOCUS + A11Y` 约束；并对辅助技术可感知。
- `V5` 共享原子有效性：本模块必须能被页签、分段切换和同位视图切换栏等场景复用，而不是被误写成某一个具体视觉控件的唯一真相源。
- `V6` 最小必要性：移除任一 `B*` 后，`V1 ~ V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

# 前端局部任务接管容器原子模块:FrontendLocalTaskCaptureContainerAtoms

@framework

模块短标签：任务接管
典型落地对象：dialog / modal / side sheet / confirm surface
不包括：普通展示容器、媒体预览、页面级路由流程

本模块定义前端局部任务接管容器原子。它承接对话框、模态任务面、侧滑任务面和阶段性确认面这类“在当前上下文中临时接管焦点与局部任务流，并在退出后把结果回流到原上下文或新落点”的结构，不把普通展示容器、长期驻留布局或页面级路由流程混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 局部任务接管能力：把当前页面上下文中的临时任务面收敛为稳定接管结构。
- `C2` 进入退出闭合能力：把打开、关闭、取消、确认与返回落点收敛为稳定退出结构。
- `C3` 焦点与背景约束能力：把焦点进入、焦点停留、背景区可操作性与提示关系收敛为稳定可达结构。
- `C4` 任务闭合反馈能力：把提交、取消、关闭后的状态回流与结果提示收敛为稳定任务反馈结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责页面级路由切换、长期驻留展示布局与具体业务字段定义。
- `N2` 非职责声明：不负责只承载正文、说明、面板或阅读视口而不接管当前任务流的普通展示容器结构。
- `N3` 非职责声明：不负责围绕当前媒体对象、预览控制与媒体状态闭环形成的媒体预览结构。

## 2. 边界定义（Boundary / 参数）

- `ENTRY` 进入边界：打开入口、来源位置、进入方式与入口到接管面的映射关系需稳定。来源：`C1 + C2`。
- `TASK_SURFACE` 接管面边界：任务面外壳、主体区、辅助区、关闭位与接管面层级关系需稳定。来源：`C1 + C3`。
- `FOCUS` 焦点边界：焦点进入、焦点停留、焦点退出、回退落点与焦点回路需稳定。来源：`C2 + C3`。
- `BACKGROUND` 背景边界：背景区在接管期间的可读、可点、可聚焦状态与前后景约束关系需可判定。来源：`C1 + C3`。
- `EXIT` 退出边界：关闭、取消、确认、提交等退出路径与退出结果类型需稳定。来源：`C2 + C4`。
- `RETURN` 返回边界：退出后原位置恢复、状态回流、再进入落点与返回判定条件需稳定。来源：`C2 + C4`。
- `A11Y` 可访问边界：标题关系、关闭提示、键盘退出、状态播报与辅助技术可感知关系需可达。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 任务面与入口接管基：L0.M0[R1,R2] + L0.M2[R1,R2]。来源：`C1 + TASK_SURFACE + ENTRY + A11Y`。
- `B2` 进入退出与回流闭合基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1]。来源：`C2 + EXIT + RETURN + FOCUS`。
- `B3` 背景约束与反馈回流基：L0.M0[R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C3 + C4 + BACKGROUND + FOCUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 接管面先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定局部任务面的外壳、主体区、辅助区、关闭位与入口到接管面的映射关系，形成稳定任务接管面，再允许更高层结构把具体任务内容装入该接管面。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`ENTRY + TASK_SURFACE + A11Y`。
- `R2` 进入退出闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：任务面打开后必须存在可解释焦点落点，关闭、取消、确认或提交后必须存在可解释退出结果、返回落点与返回判定条件，形成稳定局部任务闭环。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`ENTRY + EXIT + RETURN + FOCUS + TASK_SURFACE`。
- `R3` 背景关系闭合
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：任务面打开后，背景区是否可读、可点、可聚焦必须一致可解释，且前后景约束关系、焦点回路与任务结果回流不得互相冲突。
  - `R3.3` 输出能力：`C1 + C3 + C4`。
  - `R3.4` 边界绑定：`BACKGROUND + FOCUS + A11Y + TASK_SURFACE`。
- `R4` 局部任务接管闭环
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：接管面承载、进入退出闭合与背景关系约束必须共同组成“进入局部任务面 -> 在接管面内完成局部任务 -> 退出并返回原上下文或新落点”的稳定结构。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`ENTRY + TASK_SURFACE + FOCUS + BACKGROUND + EXIT + RETURN + A11Y`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：只有遮罩没有接管面、打开后焦点游离、退出后无返回落点、视觉上像模态但背景仍可任意交互，或把只承载普通阅读展示的展示容器、把围绕当前媒体对象和预览控制闭环成立的媒体预览误写为局部任务接管容器的方案均无效。
  - `R5.3` 失效结论：`N1 + N2 + N3`。
  - `R5.4` 边界绑定：`ENTRY + TASK_SURFACE + FOCUS + BACKGROUND + EXIT + RETURN + A11Y`。

## 5. 验证（Verification）

- `V1` 接管面可识别性：局部任务面必须满足 `ENTRY + TASK_SURFACE` 约束；接管边界可识别，且进入来源、接管面层级关系与入口到接管面的映射关系可判定。
- `V2` 进入退出闭合性：进入退出结构必须满足 `ENTRY + EXIT + RETURN` 约束；打开、关闭、取消、确认与提交后的退出结果可解释，且返回落点与返回判定条件完整。
- `V3` 焦点与背景一致性：焦点与背景结构必须满足 `FOCUS + BACKGROUND + A11Y` 约束；焦点进入、停留、退出与背景区可操作性可解释，且对辅助技术可感知。
- `V4` 任务反馈有效性：退出后原上下文或新落点中必须存在可解释状态回流、结果提示或恢复落点。
- `V5` 共享原子有效性：本模块必须能被确认框、表单任务面、侧滑任务面等局部任务场景复用，而不是被误写成某一种具体视觉壳体的唯一真相源。
- `V6` 最小必要性：移除任一 `B*` 后，`V1 ~ V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

# 前端触发原子模块:FrontendTriggerAtoms

@framework

模块短标签：动作入口
典型落地对象：button / icon action / submit trigger / confirm trigger
不包括：临时展开选择面、锚点附着提示、文件摄取、集合浏览、离散值选择

本模块定义前端离散动作入口原子。它承接按钮、确认入口与即时动作入口这类“触发一次、产生一次动作结果”的原子，不把文本编辑、临时展开选择面、文件摄取闭环、锚点提示、集合浏览或离散值选择混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 触发入口承接能力：把点击、确认与即时动作入口收敛为稳定触发入口结构。
- `C2` 动作结果闭合能力：把提交、回退与确认出口收敛为稳定动作结果闭合结构。
- `C3` 触发反馈可达能力：把状态反馈、禁用关系与可访问提示收敛为稳定触发反馈结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责连续文本编辑、长文阅读与页面级流程编排。
- `N2` 非职责声明：不负责由触发入口唤起、在短时展开面中完成候选选择并闭合的临时选择面结构。
- `N3` 非职责声明：不负责围绕稳定锚点短时附着的提示、说明与注释结构。
- `N4` 非职责声明：不负责文件接纳入口、文件队列、摄取约束与摄取状态回流形成的文件摄取结构。
- `N5` 非职责声明：不负责候选项顺序、当前项关系与集合浏览移动形成的导航或选择结构。
- `N6` 非职责声明：不负责有限候选域上的当前值、选择约束与值反馈共同形成的离散值选择结构。

## 2. 边界定义（Boundary / 参数）

- `TRIGGER` 触发边界：触发入口集合、入口到动作意图的唯一映射关系、重复触发窗口与二次触发成立条件需稳定。来源：`C1 + C3`。
- `ACTION` 动作边界：提交、确认、切换与回退等动作类型集合、动作结果类型与动作终止条件需稳定。来源：`C1 + C2`。
- `STATUS` 状态边界：默认、悬停、聚焦、按下、处理中、成功、失败与禁用等状态集合、状态迁移关系与反馈窗口需稳定；其中交互前态、操作中态与结果态的切换条件需可判定。来源：`C2 + C3`。
- `A11Y` 可访问边界：键盘触发、聚焦可见性、禁用说明与动作提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 触发入口基：L0.M0[R2] + L0.M1[R4] + L0.M2[R1,R2]。来源：`C1 + TRIGGER + STATUS`。
- `B2` 动作反馈闭合基：L0.M0[R4] + L0.M1[R4] + L0.M2[R1,R3]。来源：`C2 + C3 + ACTION + STATUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 触发入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定触发入口集合、入口到动作意图的唯一映射关系与重复触发控制，形成稳定离散动作入口。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TRIGGER + STATUS`。
- `R2` 动作反馈闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在稳定触发入口上接入动作结果、状态迁移、禁用关系与可访问提示，形成统一离散触发闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`TRIGGER + ACTION + STATUS + A11Y`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：入口无法唯一映射动作意图、动作结果不可解释、状态反馈不附着稳定触发入口，或把临时展开选择面、锚点附着提示、文件摄取闭环、集合浏览、离散值选择误写为触发原子的方案均无效。
  - `R3.3` 失效结论：`N2 + N3 + N4 + N5 + N6`。
  - `R3.4` 边界绑定：`TRIGGER + ACTION + STATUS + A11Y`。

## 5. 验证（Verification）

- `V1` 触发入口完整性：触发入口必须满足 `TRIGGER` 约束；入口集合可枚举、入口到动作意图映射唯一，且重复触发控制与二次触发条件完整。
- `V2` 动作结果闭合有效性：动作结果必须满足 `ACTION + STATUS` 约束；动作类型可解释、动作结果可观察，且动作发出后必须进入合法状态集合。
- `V3` 触发反馈可达一致性：状态反馈与可访问能力必须满足 `STATUS + A11Y` 约束；默认、悬停、聚焦、按下、处理中与结果态之间的迁移可解释，键盘路径完整、聚焦可见、禁用说明可达，且状态变化可被感知。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

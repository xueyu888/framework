# 前端设计令牌与主题模块:FrontendTokenThemeFoundation

@framework

## 1. 能力声明（Capability Statement）

- `C1` 语义令牌沉淀能力：把排版、色彩、间距、层级与状态抽象为稳定令牌结构。
- `C2` 主题映射能力：把同一组语义令牌映射到多主题、多密度与多状态输出。
- `C3` 覆盖继承能力：为局部覆盖、默认回退与可访问性治理提供稳定结构。
- `C4` 非能力项：不负责领域文案、业务状态流与页面路由定义。

## 2. 边界定义（Boundary / 参数）

- `TOKEN` 令牌边界：排版、色彩、间距和层级需按语义命名。来源：`C1 + C2`。
- `STATE` 状态边界：默认、悬停、激活、禁用与错误等状态映射需统一。来源：`C1 + C2`。
- `THEME` 主题边界：亮色、暗色与高对比输出需可治理。来源：`C2 + C3`。
- `DENSITY` 密度边界：舒展、标准与紧凑三类密度需稳定切换。来源：`C2 + C3`。
- `OVERRIDE` 覆盖边界：局部覆盖顺序与默认回退关系需明确。来源：`C2 + C3`。
- `A11Y` 可访问边界：对比度、焦点可见性与阅读可达性需达标。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 语义令牌表结构基：由排版、色彩、间距与层级语义键构成的令牌结构。来源：`C1 + TOKEN + STATE`。
- `B2` 主题映射矩阵基：由主题集、密度集与状态映射表构成的输出结构。来源：`C1 + C2 + THEME + DENSITY`。
- `B3` 覆盖继承结构基：由局部覆盖表、默认回退链与可访问约束构成的治理结构。来源：`C2 + C3 + OVERRIDE + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 语义先于实例
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定语义令牌键，再允许具体主题和实例值展开。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TOKEN/STATE`。
- `R2` 主题密度映射收敛
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将语义令牌绑定到主题与密度矩阵，使不同输出共享同一语义骨架。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`THEME/DENSITY/STATE/TOKEN`。
- `R3` 覆盖继承闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在映射矩阵上叠加局部覆盖与回退顺序，形成可治理主题结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`OVERRIDE/A11Y/THEME/DENSITY/TOKEN`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：直接把实例样式写入上层结构、跳过令牌层或破坏覆盖回退链的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TOKEN/STATE/THEME/DENSITY/OVERRIDE/A11Y`。

## 5. 验证（Verification）

- `V1` 语义完整性：令牌键集必须覆盖排版、色彩、间距和状态四类语义。
- `V2` 映射一致性：同一语义在不同主题与密度下映射必须可解释且稳定。
- `V3` 覆盖有效性：局部覆盖必须具备明确优先级和回退路径。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 前端日期时间输入原子模块:FrontendDateTimeInputAtoms

@framework

模块短标签：时间值输入
典型落地对象：date picker / time picker / range picker / datetime input
不包括：普通连续文本编辑、任意候选选择面、领域调度规则

本模块定义前端日期时间输入原子。它承接日期输入、时间输入、日期时间选择面和范围时间选择这类“围绕时间粒度、时间值落点、选择面确认与范围判定共同形成稳定输入闭环”的原子，不把仅依赖字符连续编辑即可成立的普通文本输入、任意候选选择面或页面级调度流程混进同一模块。

本模块位于 `frontend` 的界面原子层。它在承载绑定、状态交互与视觉语义等更基础前端原子之上，把时间值落点、时间粒度切面、日期时间选择面、确认关闭位与范围反馈沉成共享结构，为筛选时间、计划时间、预约时间和范围时间输入提供统一底座。

## 1. 能力声明（Capability Statement）

- `C1` 时间值承接能力：把日期值、时间值与日期时间值收敛为稳定时间输入结构。
- `C2` 时间粒度与选择闭合能力：把时间粒度、当前选择面与确认闭合结果收敛为稳定时间选择结构。
- `C3` 时间反馈与可达能力：把范围校验、禁用时间、错误提示与时间说明收敛为稳定可达反馈结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责调度算法、时区业务规则真相源与任意领域计划流程。
- `N2` 非职责声明：不负责不含时间粒度、日期时间选择面与范围关系的普通连续文本编辑结构。
- `N3` 非职责声明：不负责长文本滚动扩展、自由文本连续输入与仅依赖字符编辑形成的文本输入闭环。

## 2. 边界定义（Boundary / 参数）

- `VALUE` 值边界：日期值、时间值、日期时间值、范围值与值同步关系需稳定。来源：`C1 + C2`。
- `GRANULARITY` 粒度边界：年、月、日、时、分、秒与范围粒度集合、粒度切换条件与粒度到选择面的映射关系需稳定。来源：`C1 + C2`。
- `PICKER` 选择面边界：日期时间选择面、时间槽位、确认关闭位与选择面附着范围需稳定。来源：`C2 + C3`。
- `RANGE` 范围边界：开始值、结束值、范围关系、非法范围提示与范围判定条件需稳定。来源：`C1 + C3`。
- `STATE` 状态边界：默认、已选、禁用、校验失败与处理中等状态集合、状态迁移关系与反馈窗口需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：时间说明、当前值提示、粒度切换、错误提示与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 时间值落点与范围锚定基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C1 + VALUE + RANGE + STATE`。
- `B2` 粒度-选择面确认闭合基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C2 + PICKER + GRANULARITY + A11Y`。
- `B3` 范围与禁用反馈回流基：L0.M0[R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C3 + RANGE + STATE + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 时间值输入先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定日期值、时间值或范围值的当前落点、值同步关系与范围锚定关系，形成稳定时间值入口，再允许时间粒度与选择面附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`VALUE + RANGE + STATE`。
- `R2` 时间选择闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：时间粒度、当前选择面、确认关闭位与粒度到选择面的映射关系必须共同附着稳定时间值落点，才能形成统一日期时间选择结构；仅有输入壳而无选择确认闭合不成立。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`GRANULARITY + PICKER + VALUE + RANGE + A11Y`。
- `R3` 时间反馈闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：范围校验、禁用时间、错误提示、当前值说明与范围判定条件必须回流到已确认的时间值或范围关系之上，形成稳定日期时间输入闭环，不得退化为普通字符编辑反馈。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`RANGE + STATE + A11Y + VALUE + PICKER + GRANULARITY`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：只有文本壳没有时间粒度、存在选择面却无法说明当前值、范围关系不可解释、错误提示缺位，或把仅依赖字符连续编辑即可成立的普通文本输入、把不要求粒度切换与确认闭合的任意候选选择面、以及把领域调度规则误写为日期时间输入原子的方案均无效。
  - `R4.3` 失效结论：`N1 + N2 + N3`。
  - `R4.4` 边界绑定：`VALUE + GRANULARITY + PICKER + RANGE + STATE + A11Y`。

## 5. 验证（Verification）

- `V1` 时间值完整性：时间值输入结构必须满足 `VALUE + RANGE` 约束；日期值、时间值或范围值输入壳完整，且值同步关系与范围判定条件可判定。
- `V2` 时间选择完整性：时间选择结构必须满足 `GRANULARITY + PICKER` 约束；时间粒度、选择面与确认闭合关系可解释，且粒度到选择面的映射关系完整。
- `V3` 反馈可达性：时间反馈结构必须满足 `RANGE + STATE + A11Y` 约束；范围校验、禁用时间、当前值说明与错误提示可解释，且对辅助技术可感知。
- `V4` 共享原子有效性：本模块必须能被筛选时间、预约时间和范围时间输入等场景复用，而不是被误写成某一个具体日期控件的唯一真相源。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

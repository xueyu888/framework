# 前端临时展开选择面原子模块:FrontendTemporaryExpandedSelectionSurfaceAtoms

@framework

模块短标签：选择闭合面
典型落地对象：dropdown / menu / select panel / temporary option surface
不包括：纯动作入口、锚点附着提示、长期驻留导航区、离散值选择

本模块定义前端临时展开选择面原子。它承接下拉选择面、菜单面和临时展开候选面这类“由稳定触发入口唤起、在短时展开面中完成候选选择并以选择结果闭合”的结构，不把以说明附着为主的提示结构、长期驻留导航区、页面级路由切换、普通展示面或不依赖展开面的离散值选择混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 展开入口承接能力：把触发入口、再次展开关系与展开面附着入口收敛为稳定展开入口结构。
- `C2` 候选选择闭合能力：把候选项选择、候选项到结果的映射关系与选择后闭合结果收敛为稳定候选选择闭合结构。
- `C3` 焦点可达承接能力：把展开顺序、焦点进入退出、关闭路径与提示关系收敛为稳定可达结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责长期驻留集合浏览、页面级路由与任意自由浮层展示。
- `N2` 非职责声明：不负责仅承接离散动作出口而不形成展开候选面闭合的触发入口结构。
- `N3` 非职责声明：不负责以说明、提示或注释为主而不以候选选择闭合为主的锚点附着提示结构。
- `N4` 非职责声明：不负责不依赖短时展开面、而以有限候选域上的当前值与选择约束为主的离散值选择结构。

## 2. 边界定义（Boundary / 参数）

- `TRIGGER` 触发边界：展开入口、当前触发项、再次触发出口与触发入口到展开面的映射关系需稳定。来源：`C1 + C3`。
- `SURFACE` 展开面边界：临时展开面外壳、候选区、关闭位与展开面附着范围需稳定。来源：`C1 + C2`。
- `OPTION` 候选边界：候选项顺序、当前项、禁用项、候选摘要与候选项到结果的映射关系需稳定。来源：`C1 + C2`。
- `CLOSE` 闭合边界：选择后闭合、取消闭合、外部关闭出口与闭合成立条件需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：焦点进入、候选移动、焦点退出、回退落点与焦点回路需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：当前项提示、候选说明、展开状态播报与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 展开入口基：L0.M0[R4] + L0.M1[R4] + L0.M2[R1,R4]。来源：`C1 + TRIGGER + CLOSE + FOCUS`。
- `B2` 候选选择闭合基：L0.M0[R1,R4] + L0.M1[R4] + L0.M2[R1,R4]。来源：`C2 + C3 + SURFACE + OPTION + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 展开入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定展开入口、再次触发出口、触发入口到展开面的映射关系与焦点进入方式，形成稳定“触发 -> 展开”入口，再允许候选展开面附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TRIGGER + CLOSE + FOCUS`。
- `R2` 候选选择闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：候选项必须附着稳定展开面，并通过当前项、候选移动、候选项到结果的映射关系与选择后闭合形成统一临时展开选择结构；若不能以候选结果闭合，或退化为纯提示附着、纯动作出口或长期驻留导航，则该结构不成立。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`SURFACE + OPTION + CLOSE + FOCUS + A11Y + TRIGGER`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：只有触发入口没有候选面、候选项脱离展开面存在、候选项到结果的映射关系不可解释、关闭路径不可解释，或把纯动作入口、以说明附着和短时感知为主的锚点附着提示、长期驻留导航区、离散值选择误写为临时展开选择面的方案均无效。
  - `R3.3` 失效结论：`N1 + N2 + N3 + N4`。
  - `R3.4` 边界绑定：`TRIGGER + SURFACE + OPTION + CLOSE + FOCUS + A11Y`。

## 5. 验证（Verification）

- `V1` 展开入口完整性：展开入口结构必须满足 `TRIGGER + SURFACE` 约束；展开入口、再次触发出口与展开面附着范围完整，且触发入口到展开面的映射关系可判定。
- `V2` 候选选择闭合有效性：候选选择闭合结构必须满足 `OPTION + CLOSE` 约束；候选项、当前项与禁用项可解释，且候选项到结果的映射关系与选择后闭合条件完整。
- `V3` 焦点可达一致性：可达结构必须满足 `FOCUS + A11Y` 约束；焦点进入、候选移动、焦点回退与状态播报完整，且对辅助技术可感知。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

# 前端离散值选择原子模块:FrontendDiscreteValueSelectionAtoms

@framework

模块短标签：值选择
典型落地对象：`radio group` / `checkbox group` / `switch` / `slider` / `stepper` / `rating`
不包括：纯动作触发、临时展开选择面、互斥视图切换、日期时间输入、结构化字段工作面

本模块定义前端离散值选择原子。它承接单选组、复选组、开关、步进值选择和评分选择这类“围绕有限候选域上的当前值归属、选择约束与值反馈形成稳定值状态”的结构，不把一次性动作触发、短时展开选择面、视图切换栏或日期时间输入闭环混进同一模块。

本模块位于 `frontend` 的界面原子层。它在承载绑定、状态交互与视觉语义等更底层前端原子之上，把候选值集合、当前值、值选择约束与值反馈沉成共享结构，为布尔设置、枚举选择、评分录入、范围步进与轻量参数选择提供统一底座。

## 1. 能力声明（Capability Statement）

- `C1` 值归属入口承接能力：把候选值集合、当前值与值变化顺序收敛为稳定值选择入口结构。
- `C2` 选择约束闭合能力：把单选、多选、互斥、最小最大数量与步进约束收敛为稳定值选择规则闭合结构。
- `C3` 值反馈播报能力：把选中状态、约束提示、错误提示与当前值播报收敛为稳定反馈播报结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责点击一次即产生动作结果的纯触发入口与一次性动作闭环。
- `N2` 非职责声明：不负责由稳定入口唤起、在短时展开面中完成候选选择并闭合的临时展开选择面结构。
- `N3` 非职责声明：不负责多个内容面共处同一工作面且同一时刻只激活一个视图的互斥视图切换结构。
- `N4` 非职责声明：不负责时间粒度、日期时间选择面、范围时间关系与领域调度规则共同形成的日期时间输入结构。
- `N5` 非职责声明：不负责字段提交工作面、条件组合检索工作面与任意领域级规则真相源。

## 2. 边界定义（Boundary / 参数）

- `OPTION` 候选边界：候选值集合、候选顺序、候选标签、禁用项与候选值归属关系需稳定。来源：`C1 + C2`。
- `VALUE` 值边界：默认值、当前值、已选值组、值同步关系与值变化顺序需稳定。来源：`C1 + C2`。
- `RULE` 约束边界：单选、多选、互斥、最小最大数量、步进粒度与非法值判定条件需稳定。来源：`C2 + C3`。
- `STATE` 状态边界：未选、已选、半选、禁用、校验失败与处理中等状态集合、状态迁移关系与反馈窗口需稳定。来源：`C1 + C3`。
- `FEEDBACK` 反馈边界：值变化提示、约束提示、错误提示与提示归属关系需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：键盘切换路径与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 值归属入口基：L0.M0[R2] + L0.M1[R3] + L0.M2[R1,R2]。来源：`C1 + OPTION + VALUE + STATE`。
- `B2` 选择约束闭合基：L0.M0[R2] + L0.M1[R5] + L0.M2[R1,R2]。来源：`C2 + RULE + VALUE + STATE`。
- `B3` 值反馈播报基：L0.M0[R4] + L0.M1[R6] + L0.M2[R1,R3]。来源：`C3 + FEEDBACK + A11Y + STATE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 值归属入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定候选值集合、默认值、当前值与值变化顺序，形成稳定“候选域 -> 当前值”的归属入口，再允许更高层结构附着选择约束与反馈关系。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`OPTION + VALUE + STATE`。
- `R2` 选择约束闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：单选、多选、互斥、步进与最小最大数量约束必须附着稳定候选值结构，并通过当前值与已选值组回流形成统一值选择闭环；其主结果是值归属变化，而不是内容面切换。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`RULE + VALUE + OPTION + STATE`。
- `R3` 值反馈播报闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：选中状态、约束提示、错误提示与当前值播报必须随值变化回流到对应候选值或值位，形成稳定离散值选择反馈播报结构，不得替代内容面的唯一激活配对结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`FEEDBACK + A11Y + STATE + RULE + VALUE + OPTION`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：只有候选值没有当前值、值约束不可解释、错误提示无法回流到值选择结构，或把纯动作触发、临时展开选择面、以入口到内容面的唯一激活配对为主的互斥视图切换、日期时间输入与字段工作面误写为离散值选择原子的方案均无效。
  - `R4.3` 失效结论：`N1 + N2 + N3 + N4 + N5`。
  - `R4.4` 边界绑定：`OPTION + VALUE + RULE + STATE + FEEDBACK + A11Y`。

## 5. 验证（Verification）

- `V1` 值归属入口完整性：值选择入口必须满足 `OPTION + VALUE + STATE` 约束；候选值集合、当前值与值同步关系完整，且状态集合可判定。
- `V2` 选择约束闭合有效性：值选择规则结构必须满足 `RULE + VALUE` 约束；单选、多选、互斥、步进与最小最大数量约束可解释，且非法值判定条件完整。
- `V3` 值反馈播报一致性：值反馈播报结构必须满足 `FEEDBACK + A11Y + STATE` 约束；选中状态、约束提示、错误提示与当前值播报可解释，且对辅助技术可感知。
- `V4` 共享原子有效性：本模块必须能被枚举选择、布尔设置、评分录入、步进值选择与轻量参数选择等场景复用，而不是被误写成某一个具体控件的唯一真相源。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

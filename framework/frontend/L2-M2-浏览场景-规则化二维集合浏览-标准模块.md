# 前端规则化二维集合浏览标准模块:FrontendRegulatedTwoDimensionalCollectionBrowsingStandard

@framework

本模块定义前端规则化二维集合浏览标准。它承接表格、数据网格和二维对比浏览面这类“以行列坐标、列语义与单元格关系稳定组织集合内容”的复合结构，不把后端查询契约、项目实例字段或任意自由排布的集合展示混进同一模块。

本模块位于 `frontend` 的复合结构层。它建立在触发、展示容器、集合浏览与稳定标记、附着回流反馈等 `L1` 原子之上，把“行集合 + 列定义 + 单元格关系 + 浏览操作 + 状态反馈”收敛为稳定二维浏览结构，为管理后台、审阅工作台、对比浏览面和结构化列表工作面提供可复用底座。

## 1. 能力声明（Capability Statement）

- `C1` 二维集合承载能力：把行集合、列定义与网格化承载面收敛为稳定二维浏览结构。
- `C2` 列语义与单元格映射能力：把表头、列语义、单元格坐标与行列对应关系收敛为稳定映射结构。
- `C3` 二维浏览操作能力：把排序、筛选、行选择、列操作与局部跳转出口收敛为稳定浏览交互结构。
- `C4` 二维浏览反馈与可达能力：把空态、加载、错误、选中提示与行列阅读关系收敛为稳定反馈可达结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责后端查询协议、项目实例字段命名、虚拟滚动实现技巧与自由排版可视化画布。

## 2. 边界定义（Boundary / 参数）

- `ROW` 行边界：行集合、当前行、分组关系与行顺序需稳定。来源：`C1 + C4`。
- `COLUMN` 列边界：列定义、列顺序、列宽策略与列语义需稳定。来源：`C1 + C2`。
- `HEADER` 表头边界：表头区、列标题、列说明与列操作位需稳定。来源：`C2 + C3`。
- `CELL` 单元格边界：单元格坐标、值承载、单元格摘要与行列映射关系需稳定。来源：`C1 + C2`。
- `VIEWPORT` 视口边界：表头表体分区、滚动视口、冻结区与空态落点需稳定。来源：`C1 + C4`。
- `ACTION` 操作边界：排序、筛选、行选择、列操作与局部跳转出口需稳定。来源：`C3 + C4`。
- `STATE` 状态边界：加载、空态、错误、选中与局部处理中状态需稳定。来源：`C3 + C4`。
- `A11Y` 可访问边界：表头与单元格关系、当前坐标提示、键盘移动与操作说明需可达。来源：`C2 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 二维承载结构基：L1.M9[R2] + L1.M12[R1,R2]。来源：`C1 + ROW + COLUMN + CELL + VIEWPORT`。
- `B2` 列语义操作结构基：L1.M4[R2] + L1.M12[R2]。来源：`C2 + C3 + HEADER + ACTION + A11Y`。
- `B3` 视口附着反馈结构基：L1.M9[R2] + L1.M15[R1,R2]。来源：`C3 + C4 + STATE + VIEWPORT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 二维承载先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定行集合、列定义、单元格坐标与表体视口，再允许列语义和浏览操作接入该二维承载面。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`ROW + COLUMN + CELL + VIEWPORT`。
- `R2` 列语义映射闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：表头区必须能解释列语义与列操作位，每个单元格都必须能追溯到对应行列坐标，形成稳定表头-列-单元格映射结构。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`COLUMN + HEADER + CELL + ACTION + A11Y`。
- `R3` 浏览操作闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：排序、筛选、行选择、列操作与局部跳转出口必须附着稳定二维承载面，并通过附着于视口或网格边缘的状态反馈、空态说明与错误说明保持可解释浏览闭环，但不得替代二维浏览内部的行列关系与视口状态结构。
  - `R3.3` 输出能力：`C1 + C3 + C4`。
  - `R3.4` 边界绑定：`ACTION + STATE + ROW + VIEWPORT + A11Y`。
- `R4` 二维浏览标准闭环
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：二维承载、列语义映射与状态反馈闭合必须共同组成“识别行列关系 -> 浏览表头与单元格 -> 执行排序/筛选/选择 -> 看到稳定反馈”的复合结构。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`ROW + COLUMN + HEADER + CELL + VIEWPORT + ACTION + STATE + A11Y`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：只有列表行没有列定义、存在列标题但无法追溯到单元格坐标、排序筛选入口脱离二维承载面、空态与错误态无法落到表格视口、把二维浏览内部状态闭环下沉为附着反馈表达，或表头与单元格关系不可达的方案均无效。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`ROW + COLUMN + HEADER + CELL + VIEWPORT + ACTION + STATE + A11Y`。

## 5. 验证（Verification）

- `V1` 二维承载完整性：行集合、列定义、单元格坐标与视口分区必须完整。
- `V2` 列语义映射有效性：表头、列语义与单元格关系必须满足 `COLUMN + HEADER + CELL` 约束。
- `V3` 浏览操作完整性：排序、筛选、行选择、列操作与局部跳转出口必须满足 `ACTION + ROW` 约束。
- `V4` 状态反馈与可达性：加载、空态、错误、选中提示以及表头单元格关系必须满足 `STATE + A11Y` 约束；附着反馈必须落在视口或网格边缘，且不得替代二维浏览内部的行列关系与状态结构。
- `V5` 共享复合结构有效性：本模块必须能被管理后台数据表、审阅工作台列表面和二维对比浏览面等场景复用，而不是被误写成某一个具体产品表格的唯一真相源。
- `V6` 最小必要性：移除任一 `B*` 后，`V2~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论。

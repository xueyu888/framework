# 前端文本输入原子模块:FrontendTextInputAtoms

@framework

模块短标签：连续文本编辑
典型落地对象：input / textarea / inline text editor
不包括：日期时间输入、离散选择控件、集合浏览、页面流程编排

本模块定义前端连续文本编辑原子。它承接输入框和文本域这类“围绕字符连续编辑、值流持续同步、承载形态扩展与编辑中回退链稳定成立”的文本输入结构，不把依赖候选确认、时间粒度切换或文件接纳闭合的输入结构混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 连续编辑入口承接能力：把单值连续编辑入口、值流同步与基础状态反馈收敛为稳定文本输入入口结构。
- `C2` 多行承载扩展能力：把多行内容区、滚动窗口、自适应高度与长文本承载收敛为稳定多行编辑扩展结构。
- `C3` 提交回退闭合能力：把提交出口、回退出口、错误提示、帮助信息与可访问说明收敛为稳定文本输入闭合结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责离散选择控件、集合浏览与页面流程编排。
- `N2` 非职责声明：不负责围绕时间粒度、日期时间选择面与范围关系共同形成的日期时间输入结构。
- `N3` 非职责声明：不负责依赖时间值确认闭合、时间范围校验与时间说明回流的时间选择闭环。
- `N4` 非职责声明：不负责由稳定入口唤起、在短时展开面中完成候选选择或说明附着的临时展开选择面与锚点附着提示结构。
- `N5` 非职责声明：不负责文件接纳入口、文件队列、摄取约束与摄取状态回流共同形成的文件摄取结构。

## 2. 边界定义（Boundary / 参数）

- `FIELD` 单行边界：单行输入位、占位文本、前后缀附着关系与即时校验触发条件需稳定。来源：`C1 + C3`。
- `AREA` 多行边界：多行内容区、滚动窗口、自适应高度、换行策略与长文本承载条件需稳定。来源：`C2 + C3`。
- `VALUE` 值流边界：默认值、当前值、变更同步关系与值更新顺序需稳定。来源：`C1 + C2 + C3`。
- `SUBMIT` 提交边界：提交出口、快捷提交条件、清空或回退出口与提交后回流条件需稳定。来源：`C1 + C3`。
- `STATUS` 状态边界：聚焦、失焦、处理中、成功、失败与禁用等状态集合、状态迁移关系与反馈窗口需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：标签关系、可访问名称、帮助信息、错误提示与状态变化提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 连续编辑入口基：L0.M0[R2] + L0.M1[R3] + L0.M2[R1,R2]。来源：`C1 + FIELD + VALUE + STATUS`。
- `B2` 多行承载扩展基：L0.M0[R1,R3] + L0.M1[R3] + L0.M2[R1,R2]。来源：`C2 + AREA + VALUE + STATUS`。
- `B3` 提交回退闭合基：L0.M0[R4] + L0.M1[R6] + L0.M2[R1,R3]。来源：`C3 + VALUE + SUBMIT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 连续编辑入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定以字符连续编辑为核心的单值输入位、值流同步关系、编辑中状态与回退链，形成稳定连续编辑入口。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`FIELD + VALUE + STATUS`。
- `R2` 多行编辑扩展
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在稳定连续编辑入口之上附着多行内容区、滚动窗口、自适应高度、换行策略与长文本承载条件，形成多行连续编辑扩展；该扩展改变承载形态，不改写入口与基础状态闭合。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`AREA + VALUE + STATUS + FIELD`。
- `R3` 单行提交回退闭合
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：在稳定连续编辑入口上接入提交出口、回退出口、错误提示、帮助信息与可访问说明，使提交、回退与说明附着在单行连续编辑闭环之上。
  - `R3.3` 输出能力：`C1 + C3`。
  - `R3.4` 边界绑定：`FIELD + VALUE + SUBMIT + STATUS + A11Y`。
- `R4` 多行提交回退闭合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：在稳定多行连续编辑结构上接入提交出口、回退出口、错误提示、帮助信息与可访问说明，使提交、回退与说明附着在多行连续编辑闭环之上，但不得替代多行承载扩展本身。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`AREA + VALUE + SUBMIT + STATUS + A11Y + FIELD`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：值流不可解释、提交出口脱离连续编辑结构、错误提示不附着输入关系，或把离散值选择、日期时间输入、临时展开选择面、文件摄取等其他输入结构误写为连续文本编辑原子的方案均无效。
  - `R5.3` 失效结论：`N1 + N2 + N3 + N4 + N5`。
  - `R5.4` 边界绑定：`FIELD + AREA + VALUE + SUBMIT + STATUS + A11Y`。

## 5. 验证（Verification）

- `V1` 连续编辑入口完整性：连续编辑入口必须满足 `FIELD + VALUE + STATUS` 约束；输入位可定位、值流同步关系明确，且基础状态反馈完整。
- `V2` 多行承载扩展有效性：多行扩展必须满足 `AREA + VALUE + STATUS` 约束；滚动窗口、自适应高度、换行策略与长文本承载条件可解释，且不破坏入口与基础状态闭合。
- `V3` 单行提交回退一致性：单行提交回退闭合必须满足 `FIELD + SUBMIT + STATUS + A11Y` 约束；提交条件、回退关系、错误提示与帮助信息可解释，且对辅助技术可达。
- `V4` 多行提交回退一致性：多行提交回退闭合必须满足 `AREA + SUBMIT + STATUS + A11Y` 约束；多行提交、回退与说明关系可解释，且不破坏多行承载扩展与辅助技术可感知关系。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

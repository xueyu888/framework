# 前端文本输入原子模块:FrontendTextInputAtoms

@framework

模块短标签：连续文本编辑
典型落地对象：input / textarea / inline text editor
不包括：日期时间输入、离散选择控件、集合浏览、页面流程编排

本模块定义前端连续文本编辑原子。它承接输入框和文本域这类“围绕字符连续编辑、值流持续同步与编辑中回退链稳定成立”的文本输入结构，不把依赖候选确认、时间粒度切换或文件接纳闭合的输入结构混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 单行编辑承载能力：把单值输入、占位文本与即时校验收敛为稳定编辑结构。
- `C2` 多行编辑承载能力：把多行内容、滚动策略与高度扩展收敛为稳定编辑结构。
- `C3` 绑定同步能力：把值流、提交出口与可访问提示收敛为稳定文本输入接口。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责离散选择控件、集合浏览与页面流程编排。
- `N2` 非职责声明：不负责围绕时间粒度、日期时间选择面与范围关系共同形成的日期时间输入结构。
- `N3` 非职责声明：不负责依赖时间值确认闭合、时间范围校验与时间说明回流的时间选择闭环。
- `N4` 非职责声明：不负责由稳定入口唤起、在短时展开面中完成候选选择或说明附着的临时展开选择面与锚点附着提示结构。
- `N5` 非职责声明：不负责文件接纳入口、文件队列、摄取约束与摄取状态回流共同形成的文件摄取结构。

## 2. 边界定义（Boundary / 参数）

- `FIELD` 单行边界：单行输入位、占位文本、前后缀附着关系与即时校验触发条件需稳定。来源：`C1 + C3`。
- `AREA` 多行边界：多行内容区、滚动窗口、自适应高度与长文本承载条件需稳定。来源：`C2 + C3`。
- `VALUE` 值流边界：默认值、当前值、变更同步关系与值更新顺序需稳定。来源：`C1 + C2 + C3`。
- `SUBMIT` 提交边界：提交出口、快捷提交条件、清空或回退出口与提交后回流条件需稳定。来源：`C1 + C3`。
- `STATUS` 状态边界：聚焦、失焦、处理中、成功、失败与禁用等状态集合、状态迁移关系与反馈窗口需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：标签关系、可访问名称、帮助信息、错误提示与状态变化提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 单行连续编辑结构基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C1 + FIELD + VALUE + STATUS`。
- `B2` 多行连续编辑扩展基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C2 + AREA + VALUE + STATUS`。
- `B3` 编辑提交回退闭合基：L0.M0[R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C3 + VALUE + SUBMIT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单行编辑先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定以字符连续编辑为核心的单行输入位、值流同步关系、编辑中状态与回退链，形成稳定单行连续编辑入口。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`FIELD + VALUE + STATUS`。
- `R2` 多行编辑扩展
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在稳定单行连续编辑入口之上，把多行内容区、滚动窗口、自适应高度与长文本承载条件收敛为同一连续编辑结构；多行扩展只改变内容承载形态，不改写字符编辑、值流同步与提交回退主闭环。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`AREA + VALUE + STATUS`。
- `R3` 绑定提交闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在稳定连续编辑结构上接入提交出口、错误提示、帮助信息与状态回流，使提交、回退与可访问说明全部附着在字符连续编辑闭环之上，形成可提交、可反馈、可访问的文本输入原子。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`VALUE + SUBMIT + STATUS + A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：值流不可解释、提交出口脱离连续编辑结构、错误提示不附着输入关系、把离散值选择或集合浏览误写为连续文本编辑，或把依赖粒度切换、选择面确认、时间范围判定、短时展开候选闭合、文件摄取状态回流或锚点说明附着才能成立的结构误写为文本输入原子的方案均无效。
  - `R4.3` 失效结论：`N1 + N2 + N3 + N4 + N5`。
  - `R4.4` 边界绑定：`FIELD + AREA + VALUE + SUBMIT + STATUS + A11Y`。

## 5. 验证（Verification）

- `V1` 单行编辑完整性：单行输入结构必须满足 `FIELD + VALUE + STATUS` 约束；输入位可定位、值流同步关系明确，且基础状态反馈完整。
- `V2` 多行扩展有效性：多行输入结构必须满足 `AREA + VALUE + STATUS` 约束；滚动窗口、高度扩展与长文本承载条件可解释，且不破坏值流与状态闭合。
- `V3` 提交与可达一致性：提交出口、错误提示与帮助关系必须满足 `SUBMIT + STATUS + A11Y` 约束；提交条件明确、反馈可观察，且标签、错误与状态变化对辅助技术可达。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

# 前端展示与容器原子模块:FrontendDisplayContainerAtoms

@framework

模块短标签：展示承载
典型落地对象：panel / drawer shell / content surface / reading viewport
不包括：局部任务接管、媒体预览、临时展开选择面

本模块定义前端最小展示与容器原子。它把文本展示、面板、抽屉和阅读视口放在同一结构族中，因为这些对象都属于“承载内容、组织阅读顺序、提供普通展示视口，而不接管任务流或围绕媒体焦点形成闭环”的展示面。

## 1. 能力声明（Capability Statement）

- `C1` 展示承接能力：把正文、标题、说明与元信息承载收敛为稳定展示承载结构。
- `C2` 容器视口闭合能力：把面板、抽屉、分区视口与空态承载收敛为稳定容器视口闭合结构。
- `C3` 阅读可达能力：把展示内容、容器分区与阅读路径收敛为稳定阅读可达结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责文本编辑、集合浏览与页面级流程。
- `N2` 非职责声明：不负责进入、退出、返回落点与背景约束共同形成的局部任务接管结构。
- `N3` 非职责声明：不负责围绕当前媒体对象、预览控制与预览状态闭环形成的媒体预览结构。

## 2. 边界定义（Boundary / 参数）

- `TEXT` 文本边界：标题、正文、说明、截断策略与文本到阅读顺序的映射关系需稳定。来源：`C1 + C3`。
- `PANEL` 容器边界：面板外壳、抽屉轮廓、分区边界与容器层级关系需稳定。来源：`C2 + C3`。
- `VIEWPORT` 视口边界：滚动区、分节顺序、当前阅读视口与视口到内容区的映射关系需稳定。来源：`C2 + C3`。
- `META` 元信息边界：标签、更新时间、来源说明、辅助信息与元信息归属关系需稳定。来源：`C1 + C2`。
- `EMPTY` 空态边界：无内容、无选择、加载占位结构与空态落位条件需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：阅读顺序、标题关系、说明提示与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 展示承载基：L0.M0[R1,R3] + L0.M2[R1,R3]。来源：`C1 + TEXT + META + A11Y`。
- `B2` 容器视口闭合基：L0.M0[R1,R3] + L0.M1[R3] + L0.M2[R1,R3]。来源：`C2 + C3 + PANEL + VIEWPORT + EMPTY + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 展示承载先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定标题、正文、说明、元信息与文本到阅读顺序的映射关系，形成稳定展示承载，再允许普通容器结构附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TEXT + META + A11Y`。
- `R2` 容器视口闭合成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将面板轮廓、视口分节、空态承载与视口到内容区的映射关系并入文本展示结构，形成统一普通阅读容器；其主结果是展示与阅读承接，而不是任务接管或媒体焦点预览。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`PANEL + VIEWPORT + EMPTY + TEXT + META + A11Y`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：缺失文本阅读顺序、容器分区不可解释、空态无法落位，或把以进入退出和返回落点为主的局部任务接管、把以当前媒体对象和预览控制闭环为主的媒体预览误写为展示与容器原子的方案均无效。
  - `R3.3` 失效结论：`N1 + N2 + N3`。
  - `R3.4` 边界绑定：`TEXT + PANEL + VIEWPORT + META + EMPTY + A11Y`。

## 5. 验证（Verification）

- `V1` 展示承载完整性：展示承载结构必须满足 `TEXT + META` 约束；标题、正文与说明关系完整，且文本到阅读顺序的映射关系与元信息归属关系可判定。
- `V2` 容器视口闭合有效性：容器视口结构必须满足 `PANEL + VIEWPORT + EMPTY` 约束；面板轮廓、视口分节与空态承载可解释，且视口到内容区的映射关系完整。
- `V3` 阅读可达一致性：阅读结构必须满足 `TEXT + A11Y` 约束；阅读顺序、标题关系与说明提示可解释，且对辅助技术可感知。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

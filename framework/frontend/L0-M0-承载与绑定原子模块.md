# 前端承载与绑定原子模块:FrontendCarrierBindingAtoms

@framework

本模块定义前端框架最底层的承载与绑定结构。它只负责回答“界面如何承载内容、如何落到宿主、如何保持可达”，并把这些能力收敛为承载壳、宿主绑定与可达闭合三条最小结构轴，而不是直接上升到具体组件壳体或项目实例布局。

直观示例：带前后缀与错误提示的输入壳

以一个常见的搜索输入框为例，其基础结构可分为主承载区与辅助承载区两层：

[label/div]
├─ main surface
│  ├─ prefix slot -> icon
│  ├─ main slot   -> input
│  ├─ suffix slot -> clear button
│  └─ empty slot  -> placeholder
└─ assist surface
   ├─ help text
   └─ error text

B1：结构承载
- main_surface
- assist_surface
- prefix/main/suffix/empty slots

B2：宿主绑定
- input 作为宿主
- clear button 作为事件出口
- label/help/error 与 input 建立绑定关系

B3：可达闭合
- 焦点优先进入 input，再到 clear button
- label/help/error 形成可阅读提示
- error text 形成状态变化的可感知表达

这个例子说明：本模块不是规定某个具体组件长什么样，而是在规定一个前端组件至少要先形成“承载区 -> 槽位 -> 宿主绑定 -> 可达闭合”的基础结构，才能成为可复用的前端底层原子。

## 1. 能力声明（Capability Statement）

- `C1` 承载面组织能力：把主承载区、辅助承载区与槽位顺序收敛为稳定视图壳结构。
- `C2` 宿主绑定能力：把宿主元素、属性入口、事件出口与引用句柄收敛为稳定绑定结构。
- `C3` 可达闭合能力：把焦点顺序、阅读提示与状态播报关系收敛为稳定可访问结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责具体组件语义、主题值与项目实例布局方案。

## 2. 边界定义（Boundary / 参数）

- `SURFACE` 承载面边界：主承载区、辅助承载区、阅读分区与承载层级关系需稳定。来源：`C1 + C3`。
- `SLOT` 槽位边界：前缀槽位、主体槽位、后缀槽位、空态占位与槽位顺序关系需稳定。来源：`C1 + C2`。
- `HOST` 宿主边界：宿主元素集合、语义角色映射关系、宿主替换条件与挂载作用域需稳定。来源：`C2 + C3`。
- `PROP` 绑定边界：属性入口、事件出口、引用句柄与绑定方向关系需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：焦点顺序、焦点回路、回退落点与进入退出条件需稳定。来源：`C1 + C3`。
- `A11Y` 可访问边界：阅读提示、关系说明、状态播报位与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 视图壳与槽位结构基：由主承载区、辅助承载区、顺序槽位与空态占位构成的稳定闭环的最小视图壳结构。来源：`C1 + SURFACE + SLOT`。
- `B2` 宿主绑定结构基：由宿主元素、属性入口、事件出口与引用句柄构成的稳定宿主绑定结构。来源：`C2 + HOST + PROP`。
- `B3` 可达闭合结构基：由焦点顺序、焦点回路、阅读提示、关系说明与状态播报位构成的稳定可达闭合结构。来源：`C3 + FOCUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 承载壳先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定承载面层级关系与槽位顺序关系，再允许更高层结构附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SURFACE + SLOT`。
- `R2` 宿主绑定附着
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在稳定承载壳上对齐宿主元素、属性入口、事件出口与引用句柄，使宿主角色、挂载作用域与绑定方向形成可解释的稳定绑定结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`SURFACE + SLOT + HOST + PROP`。
- `R3` 可达闭合附着
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：在稳定承载壳上对齐焦点进入退出、焦点回路、阅读提示、关系说明与状态播报位，使可达关系附着于承载壳并形成可闭合结构。
  - `R3.3` 输出能力：`C1 + C3`。
  - `R3.4` 边界绑定：`SURFACE + SLOT + FOCUS + A11Y`。
- `R4` 承载与绑定可达闭环
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：承载壳、宿主绑定与可达闭合必须共同组成“承载区 -> 槽位 -> 宿主绑定 -> 可达提示与焦点闭环”的稳定基础结构，形成可直接被前端实现承接的底层原子。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`SURFACE + SLOT + HOST + PROP + FOCUS + A11Y`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：跳过承载壳直接声明宿主绑定、有宿主绑定但无可解释焦点进入退出、宿主角色与绑定方向不可解释、焦点回路不可闭合或阅读提示缺位的方案均无效。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`SURFACE + SLOT + HOST + PROP + FOCUS + A11Y`。

## 5. 验证（Verification）

- `V1` 承载完整性：承载壳必须满足 `SURFACE + SLOT` 约束；主承载区、辅助承载区与槽位顺序关系可判定，且空态占位不破坏承载层级关系。
- `V2` 宿主绑定有效性：宿主绑定结构必须满足 `HOST + PROP` 约束；宿主元素可定位、语义角色映射明确，且属性入口、事件出口与引用句柄之间的绑定方向可解释。
- `V3` 可达一致性：可达闭合结构必须满足 `FOCUS + A11Y` 约束；焦点回路可闭合、进入退出条件明确，且阅读提示、关系说明与状态播报对辅助技术可感知。
- `V4` 组合闭合性：承载壳、宿主绑定与可达闭合共同参与时，必须能满足 `SURFACE + SLOT + HOST + PROP + FOCUS + A11Y` 约束；三者之间不存在宿主绑定悬空、焦点脱离承载壳或提示关系无附着面的情况。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

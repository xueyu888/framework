# 前端场景与流程编排模块:FrontendSceneFlowOrchestration

@framework

## 1. 能力声明（Capability Statement）

- `C1` 场景链路编排能力：把浏览、创建、编辑、上传、对话等场景收敛为稳定流程结构。
- `C2` 上下文保持能力：让筛选条件、当前焦点、草稿内容与返回路径在流程中稳定传递。
- `C3` 回退恢复能力：为失败、取消与重新进入提供稳定回退结构。
- `C4` 非能力项：不负责具体领域规则本身与后端事务策略。

## 2. 边界定义（Boundary / 参数）

- `SCENE` 场景边界：每类流程至少要有明确入口、主链和退出口。来源：`C1 + C2`。
- `STEP` 步骤边界：每个场景的关键步骤与切换点需可枚举。来源：`C1 + C2`。
- `CONTEXT` 上下文边界：筛选条件、当前焦点、草稿与选择域需稳定传递。来源：`C2 + C3`。
- `ENTRY` 入口边界：首页进入、深链进入、创建进入与对话进入需统一。来源：`C1 + C2`。
- `RETURN` 返回边界：取消、完成、失败后的返回路径需明确。来源：`C2 + C3`。
- `FALLBACK` 回退边界：失败场景必须具备重试、回退或保留上下文的稳定出口。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 主链流程结构基：L4.M0[R1,R3]。来源：`C1 + SCENE + STEP + ENTRY`。
- `B2` 上下文传递结构基：L4.M0[R2,R3]。来源：`C2 + CONTEXT + RETURN + STEP`。
- `B3` 回退恢复结构基：L4.M0[R1,R2]。来源：`C2 + C3 + FALLBACK + RETURN + ENTRY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 主链先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定场景主链和上下文传递，再允许插入领域动作与反馈。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SCENE/STEP/ENTRY/CONTEXT`。
- `R2` 回退并轨
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将失败、取消与重新进入路径并入主链，形成稳定流程闭环。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`RETURN/FALLBACK/ENTRY/SCENE`。
- `R3` 场景闭环编排
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合主链、上下文传递和回退恢复结构，形成可观测、可回退的场景流程。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SCENE/STEP/CONTEXT/ENTRY/RETURN/FALLBACK`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失上下文传递、失败路径不可回退或入口与退出不对齐的流程均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SCENE/STEP/CONTEXT/ENTRY/RETURN/FALLBACK`。

## 5. 验证（Verification）

- `V1` 场景覆盖性：`SCENE` 中每类流程必须具备主链定义。
- `V2` 上下文一致性：焦点、草稿与选择域必须可追踪且一致。
- `V3` 回退有效性：所有失败场景必须存在可执行回退路径。
- `V4` 入口返回符合性：有效组合必须满足 `ENTRY/RETURN` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

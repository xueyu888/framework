# 前端视觉语义原子模块:FrontendVisualSemanticAtoms

@framework

本模块定义前端最底层的视觉语义结构。它只负责令牌、主题、密度和可读对比等视觉基础，不再把启动、交付和治理这类运行对象混进同一层。它关注的不是某个项目的品牌视觉资源清单，而是高层组件消费视觉语义时共同依赖的底层结构语言，把“设计 token、主题映射、密度档位、状态色调、可读回退”收敛成前端可以稳定消费的最小视觉语法。这里的 `TOKEN` 指跨主题稳定存在的视觉语义键，这里的 `THEME` 指这些语义键在主题上下文中的映射、局部覆盖与回退关系，不应把两者混写为同一层具体取值表。

直观示例：按钮的视觉语义闭环

以一个常见的主按钮为例：

- 文本、背景、边框、间距与层级先对应稳定语义令牌。

例如，可先抽出如下稳定视觉语义键：

- 文本语义：`text.primary`、`text.secondary`、`text.disabled`
- 背景语义：`bg.canvas`、`bg.surface`、`bg.elevated`
- 边框语义：`border.default`、`border.strong`
- 间距语义：`space.inline.sm`、`space.block.md`
- 层级语义：`layer.base`、`layer.overlay`
- 控件尺寸语义：`control.height.md`

上述语义键先回答“界面里有哪些稳定视觉位”，而不直接回答“它们在某个主题下具体长什么样” 

- 同一组令牌可在浅色主题、深色主题与局部覆盖中映射为不同取值。
- 在紧凑、常规与宽松密度下，按钮尺寸与间距按稳定密度档变化。
- 在默认、悬停、按下、禁用与危险态下，色调映射关系需稳定。
- 在不同主题与密度组合下，对比、字号、焦点可见与视觉回退链都应满足可读约束。

选择语义令牌
  -> 映射到当前主题
  -> 按密度档调整尺寸
  -> 附着状态色调
  -> 校验可读性

这个例子说明：本模块不是规定某个组件最终长什么样，而是在规定高层组件至少要先形成“令牌、主题、密度、色调与可读性”共同闭合的视觉基础结构，才能稳定复用，让前端的视觉从“每个组件各写各的样式”，变成“大家在消费同一套可解释、可组合、可验证的结构语言”。

## 1. 能力声明（Capability Statement）

- `C1` 语义令牌组织能力：把排版、色彩、间距与层级语义键收敛为稳定视觉结构。
- `C2` 主题配方映射能力：把主题集、状态色调与变体配方收敛为稳定主题结构。
- `C3` 可读约束承接能力：把密度、对比与可读性回退链收敛为稳定视觉约束结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责宿主挂载、页面流程与实例品牌文案。

## 2. 边界定义（Boundary / 参数）

- `TOKEN` 令牌边界：排版、色彩、间距、层级语义键与令牌映射关系需稳定。来源：`C1 + C3`。
- `THEME` 主题边界：主题集、状态映射、局部覆盖作用域与回退链需稳定。来源：`C1 + C2`。
- `DENSITY` 密度边界：紧凑、常规与宽松密度档位、密度切换条件与密度对布局影响范围需稳定。来源：`C1 + C3`。
- `TONE` 色调边界：主色调、辅助色调、状态色调关系与色调变体映射条件需稳定。来源：`C2 + C3`。
- `A11Y` 可读边界：对比约束、可读字号、焦点可见提示与视觉回退链需满足可访问要求；本边界只负责视觉可读约束，不替代承载与绑定层的结构可达定义。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 语义令牌结构基：由排版、色彩、间距与层级语义键构成的稳定视觉语义结构（主要回答：把视觉里那些会被反复消费的“位置”命名出来）。来源：`C1 + TOKEN`。
- `B2` 主题配方映射结构基：由主题集、状态色调、变体配方与局部覆盖作用域构成的稳定主题映射结构（主要回答：视觉语义位，在不同主题/状态/tone 下到底怎么取值）。来源：`C2 + THEME + TONE`。
- `B3` 可读约束回退结构基：由密度档位、对比约束、可读字号、焦点可见提示与视觉回退链构成的稳定可读约束结构（主要回答：就算主题、状态、密度变化了，这个界面是否仍然可读、可看、可聚焦、可回退）。来源：`C3 + DENSITY + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 令牌先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定令牌键与令牌映射关系，再允许更高层组件消费这些稳定视觉语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TOKEN`。
- `R2` 主题映射附着
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在令牌语义之上叠加主题配方、状态色调、变体映射与局部覆盖作用域，形成稳定主题映射层，但不把可读约束提前混入同一基。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`TOKEN + THEME + TONE`。
- `R3` 可读约束附着
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：在令牌语义之上叠加密度档、对比约束、可读字号、焦点可见提示与视觉回退链，形成稳定可读约束层，但不要求主题配方已经整包闭合。
  - `R3.3` 输出能力：`C1 + C3`。
  - `R3.4` 边界绑定：`TOKEN + DENSITY + A11Y`。
- `R4` 视觉语义闭环
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：把令牌语义、主题配方映射与可读约束回退统一闭合，形成前端可稳定复用的完整视觉语义基础层。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`TOKEN + THEME + DENSITY + TONE + A11Y`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：没有令牌语义就直接声明主题变体、主题映射与色调关系不可解释、局部覆盖作用域失控，或密度档、对比约束、焦点可见提示与视觉回退链缺位的方案均无效。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`TOKEN + THEME + DENSITY + TONE + A11Y`。

## 5. 验证（Verification）

- `V1` 令牌完整性：视觉令牌结构必须满足 `TOKEN` 约束；排版、色彩、间距与层级语义键完整，且令牌映射关系可判定。
- `V2` 主题映射有效性：主题映射结构必须满足 `THEME + TONE` 约束；主题集、状态色调、变体配方与局部覆盖作用域可解释。
- `V3` 可读一致性：可读约束结构必须满足 `DENSITY + A11Y` 约束；密度切换条件明确，对比与字号满足可访问要求，且焦点可见提示与视觉回退链可解释。
- `V4` 视觉闭环有效性：完整视觉语义结构必须同时满足 `TOKEN + THEME + DENSITY + TONE + A11Y` 约束；主题映射与可读约束共同附着于同一令牌语义之上。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。

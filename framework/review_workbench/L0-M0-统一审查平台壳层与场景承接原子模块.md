# 统一审查平台壳层与场景承接原子模块:ReviewWorkbenchPlatformShellAndSceneMountAtoms

@framework

本模块定义 `review_workbench` 作为统一审查平台时最小的壳层、场景导航与顶部全局工具位结构。推导链为：`framework/frontend/L0-M0-承载与绑定原子模块.md` + `framework/frontend/L1-M4-动作-触发-原子模块.md` + `framework/frontend/L1-M9-承载-展示与容器-原子模块.md` + `framework/frontend/L1-M12-浏览-集合浏览-原子模块.md` + `framework/frontend/L1-M15-反馈-标记与反馈-原子模块.md` -> 本模块。也就是说，`review_workbench` 不单独重写一套平台壳与导航基础，而是在前端已有的承载壳、触发、展示容器、集合浏览与标记反馈结构之上，收束出“统一平台壳 + 左侧场景入口 + 当前激活场景 + 顶部全局工具位”这组领域原子。

本模块位于 `review_workbench` 的原子层。它负责把统一审查平台最前端的壳层、场景入口和顶部工具位表达清楚，为上层的文件库处理、审核提交、历史追溯以及未来新增场景提供稳定承载面；它不在这里展开具体场景内部的范围树、查询、审核动作或历史差异本身。

## 1. 能力声明（Capability Statement）
- `C1` 平台壳层表达能力：将顶栏、底栏、左侧导航区和主工作区组织成统一平台外壳。
- `C2` 场景入口集合表达能力：将平台可进入的一级场景入口以及它们的稳定识别方式组织成集合结构。
- `C3` 当前激活场景表达能力：将默认场景、当前激活场景和切换结果组织成稳定状态结构。
- `C4` 顶部全局工具位表达能力：将全局搜索、主题切换、用户入口或其他平台级工具位组织成稳定结构。
- `N1` 非职责声明：不负责场景内部的范围树、查询、结果列表、审核结论和历史差异内容。

## 2. 边界定义（Boundary / 参数）
- `PLATFORM_SHELL` 平台壳层边界：顶栏、底栏、左侧导航区和主工作区允许采用不同承载形式，但四者必须同时存在于同一平台壳层中，并保持稳定承接关系。
- `SCENE_SET` 场景集合边界：一级场景入口的数量、排序和展示方式允许变化，但每个入口都必须具备稳定标识，并进入同一可切换场景集合。
- `SCENE_STATE` 场景状态边界：默认场景、当前激活场景和切换结果允许变化，但任一时刻必须存在唯一当前激活场景，且切换前后结果必须可解释。
- `GLOBAL_TOOLS` 顶部全局工具位边界：工具位种类允许变化，但必须稳定挂接在平台顶栏承载面上，对整个平台生效，且不能被误写成某个具体场景内部控件。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 平台壳层结构基：frontend.L0.M0[R1,R2]。由主承载区、辅助承载区、宿主绑定和可达结构约束成统一审查平台的顶栏、底栏、左侧导航区与主工作区。来源：`C1 + PLATFORM_SHELL`。
- `B2` 场景入口集合基：frontend.L1.M12[R1,R2]。由集合层级、当前项和集合内切换关系约束成平台一级场景入口集合。来源：`C2 + SCENE_SET`。
- `B3` 当前场景状态基：frontend.L1.M4[R1,R2] + frontend.L1.M12[R2]。由切换入口、当前激活场景标识和切换结果约束成默认场景、当前激活场景与切换后的状态结构。来源：`C3 + SCENE_STATE`。
- `B4` 顶部全局工具位基：frontend.L1.M4[R1,R2] + frontend.L1.M9[R2] + frontend.L1.M15[R1]。由工具入口、顶部承接面和工具位状态标记约束成全局搜索、主题切换、用户入口与其他平台级工具位结构。来源：`C4 + GLOBAL_TOOLS`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 统一平台必须先有稳定壳层
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先承接 frontend 的承载壳、宿主绑定与可达结构，再把它们收束成统一平台语义下的顶栏、底栏、左侧导航区和主工作区；后续任何场景都应承接在这个外壳里。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`PLATFORM_SHELL`。
- `R2` 场景入口切换必须可解释
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：在 frontend 的集合浏览与触发结构上，明确平台有哪些一级场景入口、当前激活的是哪一个，以及切换之后进入了哪个场景；不能只有图标入口而没有场景状态语义。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`SCENE_SET/SCENE_STATE`。
- `R3` 顶部全局工具位必须挂接在平台壳层里
  - `R3.1` 参与基：`B1 + B4`。
  - `R3.2` 组合方式：顶部全局搜索、主题切换、用户入口和其他平台级工具位必须挂接在统一平台外壳中的稳定顶栏承载面上，并对整个平台生效；不能把它们写成某个具体场景内部的局部控件。
  - `R3.3` 输出能力：`C1 + C4`。
  - `R3.4` 边界绑定：`PLATFORM_SHELL/GLOBAL_TOOLS`。
- `R4` 场景导航与顶部全局工具位必须共同附着在平台壳层里
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：场景入口集合、当前激活场景和顶部全局工具位必须共同挂接在统一平台外壳中的稳定承载面上，形成“壳层先行，场景附着，平台工具位持续可见”的领域结构；不能跳过 frontend 的承载壳，直接把场景切换或全局工具写成脱离平台语境的孤立动作。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`PLATFORM_SHELL/SCENE_SET/SCENE_STATE/GLOBAL_TOOLS`。
- `R5` 无效方案
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：以下方案无效：跳过 frontend 承载壳直接声明统一平台场景；只有图标或文字入口而无场景语义；只有平台容器而无当前激活场景；顶部存在搜索或主题入口但无法说明其平台级作用范围；把平台级工具位误写成文件库、审核或历史场景内部控件。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`PLATFORM_SHELL/SCENE_SET/SCENE_STATE/GLOBAL_TOOLS`。

## 5. 验证（Verification）
- `V1` 平台壳层完整性：顶栏、底栏、左侧导航区和主工作区必须同时存在于同一平台壳层中，并能解释它们的承接关系。
- `V2` 场景入口可解释性：平台中必须能识别可切换场景集合与唯一当前激活场景。
- `V3` 顶部全局工具位可追溯性：任一全局搜索、主题切换或用户入口都必须能追溯到平台顶栏承载面，而不是某个具体场景局部结构。
- `V4` 前端承接有效性：统一平台壳层、场景导航与顶部全局工具位必须能追溯到 frontend 的承载绑定、触发、展示容器、集合浏览和标记反馈结构，而不是并行重写一套基础原子。
- `V5` 最小必要性：移除任一 `B*` 后，`V1~V4` 至少一项失效。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

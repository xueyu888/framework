# 范围上下文与局部摘要原子模块:ReviewWorkbenchScopeContextAndLocalSummaryAtom

@framework

本模块定义 `review_workbench` 中用于表达范围节点集合、唯一当前范围、局部概览、附着当前范围的局部动作入口以及范围展示承接面的最小语义原子。它关注的不是查询结果、文件清单、对象工作资源或具体场景实例主线如何展开，而是范围上下文如何成立，并如何形成可被上层编排稳定消费的范围展示承接面。

本模块位于 `review_workbench` 的原子层。向上它可被文件库范围查询编排、历史追溯编排或其他需要范围上下文的模块复用；向下它只承接“范围节点集合、当前范围、局部概览、局部动作入口与范围展示承接面”这一最小语义，不扩写查询条件、文件处理或审核流转。

本模块不直接定义查询条件、文件清单结果、对象工作资源、审核结论或具体页面实现，只回答：什么样的范围上下文才算成立，且能够稳定驱动后续主线。

## 1. 能力声明（Capability Statement）
- `C1` 范围节点组织能力：支持以树形或等价层级结构表达范围节点集合，并支持在该集合中定位当前工作范围。
- `C2` 唯一当前范围导出能力：支持范围节点集合在任一时刻导出唯一当前范围，供后续查询、对象列表与局部概览承接。
- `C3` 局部概览附着能力：支持局部概览附着当前范围成立，并随当前范围变化而重新对齐。
- `C4` 范围级局部控制入口附着能力：支持与当前范围直接相关的局部控制入口附着当前范围成立，并随当前范围变化而重新对齐。
- `C5` 范围展示承接能力：支持把唯一当前范围、局部概览与局部动作入口组织成稳定的范围展示承接面，供上层编排消费。
- `C6` 范围切换回稳能力：支持当前范围在节点切换后完成替换，并使旧范围退出当前有效上下文。
- `N1` 非职责声明：不直接定义查询条件、文件清单结果、对象工作资源、审核结论或具体场景实例主线，只负责范围上下文成立。

## 2. 边界定义（Boundary / 参数）
- `CURRENT_RANGE_SCOPE` 当前范围边界：范围节点组织、层级深度与局部概览内容允许变化，但任一时刻只允许一个当前范围作为后续工作面的有效上下文。来源：`C1 + C2 + C3`。
- `RANGE_SWITCH_SCOPE` 范围切换边界：允许当前范围在节点切换后被替换，并允许局部概览与局部动作入口随之更新，但不允许旧范围与新范围并行作为当前有效工作上下文。来源：`C2 + C3 + C4 + C6`。
- `OVERVIEW_ATTACHMENT_SCOPE` 局部概览附着边界：局部概览的字段、统计或摘要形式允许变化，但它们必须继续附着当前范围成立；不允许脱离当前范围形成独立真相源。来源：`C3`。
- `SCOPE_CONTROL_SCOPE` 范围级局部控制边界：范围级局部控制入口的种类、命名与表现允许变化，但它们必须附着当前范围成立；不允许脱离当前范围形成独立入口上下文。来源：`C4`。
- `CONTEXT_SURFACE_SCOPE` 范围展示承接边界：范围展示承接面允许有不同布局、区域组织或视图表达，但它必须继续由唯一当前范围统一约束。来源：`C5 + C6`。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 范围节点集合基：能够表达范围节点集合及其层级或等价组织方式。来源：`C1 + CURRENT_RANGE_SCOPE`。
- `B2` 当前范围定位基：能够从范围节点集合中定位唯一当前范围。来源：`C2 + CURRENT_RANGE_SCOPE + RANGE_SWITCH_SCOPE`。
- `B3` 局部概览附着基：能够把局部概览附着到当前范围。来源：`C3 + OVERVIEW_ATTACHMENT_SCOPE`。
- `B4` 范围级局部控制入口附着基：能够把与当前范围直接相关的局部控制入口附着到当前范围。来源：`C4 + SCOPE_CONTROL_SCOPE`。
- `B5` 范围展示承接基：能够把唯一当前范围、局部概览与附着当前范围的局部动作入口组织成稳定的范围展示承接面。来源：`C2 + C3 + C4 + C5 + CONTEXT_SURFACE_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 范围节点集合必须先导出唯一当前范围
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：后续查询、对象列表、局部反馈、局部动作入口与范围展示承接面只有在唯一当前范围被定位后才能成立；不允许多个范围同时充当当前有效上下文。
  - `R1.3` 输出能力：`C1 + C2 + C5`。
  - `R1.4` 边界绑定：`CURRENT_RANGE_SCOPE/CONTEXT_SURFACE_SCOPE`。

- `R2` 局部概览必须附着当前范围
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：局部概览只能作为当前范围的附着摘要或反馈承接面存在；不允许概览脱离当前范围单独成立。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`CURRENT_RANGE_SCOPE/OVERVIEW_ATTACHMENT_SCOPE`。

- `R3` 范围级局部控制入口必须附着当前范围
  - `R3.1` 参与基：`B2 + B4`。
  - `R3.2` 组合方式：与当前范围直接相关的局部控制入口只能在当前范围上下文中成立；不允许这些控制入口脱离当前范围形成独立控制面。
  - `R3.3` 输出能力：`C2 + C4`。
  - `R3.4` 边界绑定：`CURRENT_RANGE_SCOPE/SCOPE_CONTROL_SCOPE`。

- `R4` 范围切换后必须完成上下文替换
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：当前范围切换后，新的当前范围必须替代旧范围成为唯一有效上下文，旧范围相关概览、局部动作入口与范围展示承接面只能保留为历史或失效状态，不能继续作为当前承接面。
  - `R4.3` 输出能力：`C2 + C3 + C4 + C5 + C6`。
  - `R4.4` 边界绑定：`RANGE_SWITCH_SCOPE/OVERVIEW_ATTACHMENT_SCOPE/SCOPE_CONTROL_SCOPE/CONTEXT_SURFACE_SCOPE`。

- `R5` 缺失当前范围时，下游承接不得视为成立
  - `R5.1` 参与基：`B1 + B2 + B5`。
  - `R5.2` 组合方式：若范围节点集合存在但未能定位唯一当前范围，则依附该范围的查询、对象列表、局部反馈、局部动作入口与范围展示承接面不得误判为已经成立。
  - `R5.3` 输出能力：`C2 + C5 + C6`。
  - `R5.4` 边界绑定：`CURRENT_RANGE_SCOPE/RANGE_SWITCH_SCOPE/CONTEXT_SURFACE_SCOPE`。

- `R6` 无效方案
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：以下方案无效：只有范围节点集合没有唯一当前范围；多个范围并行充当当前有效上下文；局部概览脱离当前范围独立成立；局部动作入口脱离当前范围独立成立；范围展示承接面脱离唯一当前范围独立成立；范围切换后旧范围仍被误当作当前上下文；缺失当前范围却仍把下游承接面当作已成立状态。
  - `R6.3` 失效结论：`N1`。
  - `R6.4` 边界绑定：`CURRENT_RANGE_SCOPE/RANGE_SWITCH_SCOPE/OVERVIEW_ATTACHMENT_SCOPE/SCOPE_CONTROL_SCOPE/CONTEXT_SURFACE_SCOPE`。

## 5. 验证（Verification）
- `V1` 当前范围有效性：任一时刻都必须能定位唯一当前范围。
- `V2` 概览附着有效性：任一局部概览都必须能追溯到当前范围。
- `V3` 局部动作附着有效性：任一局部动作入口都必须能追溯到当前范围。
- `V4` 展示承接有效性：任一范围展示承接面都必须能追溯到唯一当前范围。
- `V5` 范围切换有效性：范围切换后，下游承接必须能追溯到新的当前范围，而不是停留在旧范围。
- `V6` 失效保护：缺失唯一当前范围时，不得误判依附该范围的下游工作面与范围展示承接面已经成立。
- `V7` 最小必要性：移除任一 `B*` 后，`V1~V6` 至少一项失效。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

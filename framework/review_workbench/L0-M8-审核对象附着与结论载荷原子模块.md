# 审核对象附着与结论载荷原子模块:ReviewWorkbenchReviewObjectAndDecisionPayloadAtoms

@framework

本模块定义 `review_workbench` 中“当前审核对象如何作为结论输入的附着对象、一次审核结论最少由什么组成”的共享结构原子。它只回答审核对象附着关系、结论类型与最小结论载荷这几个最小问题；它不负责审核完成后的状态影响、焦点结果或反馈回流。

本模块位于 `review_workbench` 的原子层。它不依赖本框架其他 `L0` 模块，不在这里形成“识别审核对象 -> 给结论 -> 看到回流”的完整闭环；它只把审核对象附着与结论载荷沉成稳定共享结构，供上层编排消费。凡是“结论怎样映射成动作请求、结果怎样映射成后端契约回流”的问题，统一交给外部领域 `review_workbench_backend.L0.M4`。

## 1. 能力声明（Capability Statement）
- `C1` 当前审核对象附着能力：将提交项、审核对象和审核上下文组织成稳定的“当前结论附着对象”结构。
- `C2` 审核结论载荷表达能力：将通过、驳回、退回、补充意见等审核结论输入组织成稳定结论载荷结构。
- `N1` 非职责声明：不负责具体审核规则、不负责权限细则、不负责历史差异查看。

## 2. 边界定义（Boundary / 参数）
- `REVIEW_OBJECT_SCOPE` 审核对象边界：提交项种类和审核上下文允许变化，但任一审核结论都必须附着稳定当前审核对象身份。
- `DECISION_PAYLOAD_SCOPE` 审核结论载荷边界：结论类型、附加意见形式和输入承载方式允许变化，但每次结论表达都必须具备可解释结论类型与最小结论载荷。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 当前审核对象基：能够表达当前提交项、审核对象与审核上下文的稳定附着关系。来源：`C1 + REVIEW_OBJECT_SCOPE`。
- `B2` 审核结论类型基：能够表达通过、驳回、退回、补充意见等可判定结论类型。来源：`C2 + DECISION_PAYLOAD_SCOPE`。
- `B3` 审核结论载荷基：能够表达附加意见、说明文本或其他最小结论载荷。来源：`C2 + DECISION_PAYLOAD_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 审核结论输入必须附着当前审核对象
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：任何审核结论都必须能说明作用于哪个当前提交项或当前审核对象，不能只有结论类型而没有对象归属；这里约束的是对象附着与结论载荷如何成立，而不是服务动作请求如何编码。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`REVIEW_OBJECT_SCOPE/DECISION_PAYLOAD_SCOPE`。

- `R2` 审核结论必须具备可判定类型与最小载荷
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：任何审核结论都必须同时具备可判定结论类型与最小结论载荷；不能只有一段意见文本而没有明确结论，也不能只有结论类型而没有最小说明载荷。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`DECISION_PAYLOAD_SCOPE`。

- `R3` 审核对象附着与结论载荷原子闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：当前审核对象附着关系、审核结论类型和最小结论载荷必须共同形成“识别当前对象 -> 形成结论载荷”的稳定前端审核结构；其中动作请求与结果回流由上层编排和外部后端契约单独承担。
  - `R3.3` 输出能力：`C1 + C2`。
  - `R3.4` 边界绑定：`REVIEW_OBJECT_SCOPE/DECISION_PAYLOAD_SCOPE`。

- `R4` 无效方案
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：以下方案无效：只有审核触发入口没有当前审核对象；只有结论词没有最小结论载荷；只有意见文本没有明确结论类型；把工作栏队列切换、结果回流或服务动作请求字段直接写回本模块。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`REVIEW_OBJECT_SCOPE/DECISION_PAYLOAD_SCOPE`。

## 5. 验证（Verification）
- `V1` 当前审核对象可识别性：任一审核结论输入都必须能追溯到明确当前审核对象身份。
- `V2` 审核结论类型有效性：任一审核结论表达都必须能说明明确结论类型。
- `V3` 审核结论载荷有效性：任一审核结论表达都必须能说明最小结论载荷。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

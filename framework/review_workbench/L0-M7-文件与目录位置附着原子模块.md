# 文件与目录位置附着原子模块:ReviewWorkbenchFileAndDirectoryPlacementAttachmentAtoms

@framework

本模块定义 `review_workbench` 中“文件与目录对象在当前范围里是谁、挂在哪、位置怎样被持续解释”的共享结构原子。它只回答文件 / 目录对象身份、父子挂接关系、层级位置与位置连续性这几个最小问题；它不负责改动载荷本身，也不负责改动完成后的树刷新、焦点回退或反馈回流。

本模块位于 `review_workbench` 的原子层。它不依赖本框架其他 `L0` 模块，不在这里形成“对象识别 -> 发起改动 -> 回流结果”的完整闭环；它只把文件与目录对象的位置本体沉成稳定共享结构，供上层编排消费。

## 1. 能力声明（Capability Statement）
- `C1` 文件与目录对象身份表达能力：将文件、目录与稳定对象身份组织成稳定对象结构。
- `C2` 父子挂接与层级位置表达能力：将父节点、子节点、层级深度和当前挂接关系组织成稳定位置结构。
- `C3` 位置连续性表达能力：将对象在位置变化前后的连续身份与位置关系组织成稳定追踪结构。
- `N1` 非职责声明：不负责改动类型与改动载荷，不负责树刷新、焦点回退与反馈回流，不负责后端契约字段。

## 2. 边界定义（Boundary / 参数）
- `OBJECT_ID_SCOPE` 对象身份边界：对象类型和展示信息允许变化，但每个文件或目录都必须具备稳定对象标识。
- `CONTAINER_SCOPE` 挂接边界：父子节点数量、层级深度和挂接方式允许变化，但任一对象都必须能解释当前父节点与当前层级位置。
- `PLACEMENT_SCOPE` 位置连续性边界：对象位置、父节点和路径允许变化，但同一对象在变化前后都必须保持连续可追踪关系。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 对象身份基：能够表达文件或目录对象的稳定身份。来源：`C1 + OBJECT_ID_SCOPE`。
- `B2` 父子挂接基：能够表达对象当前父节点、子节点关系与层级位置。来源：`C2 + CONTAINER_SCOPE`。
- `B3` 位置连续性基：能够表达同一对象在位置变化前后的连续身份与位置关系。来源：`C3 + PLACEMENT_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 对象身份必须附着可解释挂接关系
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：任一文件或目录对象都必须同时具备稳定对象身份与可解释父子挂接关系；不能只有对象标识而没有当前挂接位置。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`OBJECT_ID_SCOPE/CONTAINER_SCOPE`。

- `R2` 位置变化前后必须保持对象连续性
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：同一对象在位置变化前后，必须能同时解释原位置关系与当前位置关系；位置变化不得改写对象连续身份。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`OBJECT_ID_SCOPE/CONTAINER_SCOPE/PLACEMENT_SCOPE`。

- `R3` 位置结构不得退化成不可解释路径片段
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：对象位置表达必须能回到父子挂接关系和连续身份解释；不能只剩一个孤立路径、名称片段或不可解释定位串。
  - `R3.3` 输出能力：`C2 + C3`。
  - `R3.4` 边界绑定：`CONTAINER_SCOPE/PLACEMENT_SCOPE`。

- `R4` 无效方案
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：以下方案无效：只有对象名没有稳定对象身份；只有对象标识没有当前父节点与层级位置；位置变化后只能看见新位置看不见原位置关系；把改动类型、改动载荷、树刷新结果或焦点回退结果直接写回本模块。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`OBJECT_ID_SCOPE/CONTAINER_SCOPE/PLACEMENT_SCOPE`。

## 5. 验证（Verification）
- `V1` 对象身份有效性：任一文件或目录都必须能识别稳定对象身份。
- `V2` 挂接关系有效性：任一文件或目录都必须能识别当前父节点与当前挂接关系。
- `V3` 位置连续性有效性：位置变化前后都必须能解释同一对象的连续身份与位置关系。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

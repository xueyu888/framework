# 查询条件与筛选上下文原子模块:ReviewWorkbenchQueryAndFilterContextAtoms

@framework

本模块定义 `review_workbench` 领域最小的查询条件与筛选上下文结构。推导链为：`framework/frontend/L1-M4-动作-触发-原子模块.md` + `framework/frontend/L1-M0-输入-文本输入-原子模块.md` + `framework/frontend/L1-M5-选择-临时展开选择面-原子模块.md` + `framework/frontend/L1-M15-反馈-标记与反馈-原子模块.md` -> 本模块。也就是说，工作台中的关键词输入、筛选条件、重置动作和查询上下文，不单独重写新的输入模型，而是把前端已有的触发、文本输入、临时选择面和标记反馈约束成“范围内查询”这组领域原子。
本模块定义审查处理工作台中的查询条件结构原子。它回答“用户用哪些条件缩小列表项集合”以及“当前查询被如何稳定表达”这两个最小问题。它不负责范围树和列表审查，只负责把搜索词、筛选字段、重置动作与查询摘要沉成稳定查询结构。

## 1. 能力声明（Capability Statement）

- `C1` 查询输入表达能力：将关键词、输入框与基础搜索条件组织成稳定结构。
- `C2` 筛选条件表达能力：将枚举筛选、状态筛选、字段筛选与组合条件组织成稳定结构。
- `C3` 查询摘要表达能力：将当前条件集合、范围内查询说明与清空重置动作组织成稳定摘要结构。
- `N1` 非职责声明：不负责结果展示、不负责范围树、不负责动作回流。

## 2. 边界定义（Boundary / 参数）

- `QUERYINPUT` 查询输入边界：关键词类型、输入值和触发方式允许变化，但必须能稳定识别当前输入值与对应触发关系。
- `FILTERSET` 筛选集合边界：筛选字段、候选值和组合方式允许变化，但每个筛选条件都必须能说明当前选中值及其组合状态。
- `QUERYSUMMARY` 查询摘要边界：摘要项数量、范围说明和重置入口允许变化，但当前查询状态成立后必须至少导出一处可解释摘要或重置入口。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 查询输入基：`frontend.L1.M0[R1,R2] + frontend.L1.M4[R1]`。由关键词输入、输入状态和触发入口约束成查询输入结构。来源：`C1 + QUERYINPUT`。
- `B2` 筛选集合基：`frontend.L1.M5[R2] + frontend.L1.M15[R1]`。由筛选候选项、当前选中值和状态标记约束成筛选字段与当前筛选状态结构。来源：`C2 + FILTERSET`。
- `B3` 查询摘要基：`frontend.L1.M15[R1,R2] + frontend.L1.M4[R1]`。由当前条件摘要、范围说明与清空重置动作约束成查询摘要结构。来源：`C3 + QUERYSUMMARY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 查询输入必须进入筛选状态
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：在 frontend 的文本输入和选择结构上，关键词输入与筛选条件必须共同导出当前查询状态，而不能各自独立存在。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`QUERYINPUT/FILTERSET`。

- `R2` 当前查询必须导出摘要
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：在 frontend 的标记反馈和触发入口结构上，筛选条件在成立后必须导出当前查询摘要、范围说明或重置入口，不得只在隐藏状态中存在。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`FILTERSET/QUERYSUMMARY`。

- `R3` 查询原子闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：查询输入、筛选集合与查询摘要必须共同形成“输入条件 -> 当前条件成立 -> 当前查询可被解释”的稳定结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`QUERYINPUT/FILTERSET/QUERYSUMMARY`。

- `R4` 无效方案
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：以下方案无效：跳过 frontend 输入与触发结构直接声明查询状态；只有输入框没有当前条件；只有下拉筛选没有查询摘要；清空筛选后无法解释当前查询状态。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`QUERYINPUT/FILTERSET/QUERYSUMMARY`。

## 5. 验证（Verification）

- `V1` 查询可识别性：当前查询输入值、筛选条件和组合状态必须可识别。
- `V2` 摘要可解释性：查询状态成立后，必须至少导出一处可解释摘要或重置入口。
- `V3` 前端承接有效性：查询输入、筛选集合和查询摘要必须能追溯到 frontend 的文本输入、触发、临时选择面和标记反馈结构。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

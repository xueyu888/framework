# 工作集与工作面承接原子模块:ReviewWorkbenchWorksetAndWorkspaceMountAtoms

@framework

本模块定义 `review_workbench` 领域最小的工作资源集合与工作面承接结构。推导链为：`framework/frontend/L1-M4-动作-触发-原子模块.md` + `framework/frontend/L1-M12-浏览-集合浏览-原子模块.md` + `framework/frontend/L1-M15-反馈-标记与反馈-原子模块.md` -> 本模块。也就是说，平台中的固定锚点页签、对象工作页签、辅助回看页签、当前活动工作面以及关闭回退或复用关系，不单独重写页签基础，而是把前端已有的集合浏览、离散触发和附着反馈约束成“工作资源承接”这组领域原子。

本模块定义统一审查平台中的工作资源集合、当前活动工作面与工作资源回退结构原子。它回答“当前工作台里有哪些固定锚点工作面、对象工作面或辅助回看工作面”“用户此刻正在处理哪一个工作面内容”以及“关闭或再次打开后如何回到可解释工作状态”这三个最小问题。它不定义对象字段和动作语义，只负责把工作资源类型、工作资源身份、当前活动工作面以及回退/复用关系沉成稳定结构。

## 1. 能力声明（Capability Statement）

- `C1` 工作资源集合表达能力：将固定锚点工作面、对象工作面、辅助回看工作面和其它可切换工作资源组织成稳定集合。
- `C2` 当前活动工作面表达能力：将当前激活工作面、工作资源类型、切换结果与关闭后的回退关系组织成稳定状态。
- `C3` 工作资源身份表达能力：将工作资源来源、工作资源身份与标题外的稳定识别方式组织成稳定结构。
- `C4` 工作资源复用与回退表达能力：将再次打开同一来源对象时的复用关系、关闭当前工作面后的回退结果组织成稳定结构。
- `N1` 非职责声明：不负责对象字段、不负责动作语义、不负责统计反馈。

## 2. 边界定义（Boundary / 参数）

- `WORKSETSET` 工作资源集合边界：固定锚点工作面、对象工作面、辅助工作面和可切换项数量允许变化，但必须共同落在同一工作资源集合中，并能识别当前可切换工作资源集合。
- `WORKSETSTATE` 工作面状态边界：当前激活项、工作资源类型、切换结果和关闭回退关系允许变化，但任一时刻必须存在唯一当前工作面，且关闭后回退结果必须可解释。
- `WORKRESOURCE_IDENTITY` 工作资源身份边界：工作资源标题、来源对象、来源任务或来源场景允许变化，但每个工作资源都必须具备不依赖展示标题的稳定身份，并能解释自己从何而来。
- `WORKRESOURCE_REUSE` 工作资源复用边界：再次打开同一来源对象时允许复用已有工作资源或新建新的工作资源，但必须能够解释采用了哪种复用/新建结果，以及当前活动工作面如何因此变化。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 工作资源集合基：frontend.L1.M12[R1,R2]。由集合结构、当前项和集合内切换关系约束成固定锚点工作面、对象工作面与辅助工作面的集合结构。来源：`C1 + WORKSETSET`。
- `B2` 工作面状态基：frontend.L1.M4[R1,R2] + frontend.L1.M12[R2] + frontend.L1.M15[R1]。由离散切换入口、当前激活工作面标识和状态标记约束成当前激活工作面、工作资源类型、切换结果与关闭回退关系。来源：`C2 + WORKSETSTATE`。
- `B3` 工作资源身份基：frontend.L1.M12[R2] + frontend.L1.M15[R1]。由当前项标识、来源附着关系与状态标记约束成工作资源身份和来源结构。来源：`C3 + WORKRESOURCE_IDENTITY`。
- `B4` 工作资源复用与回退基：frontend.L1.M4[R1,R2] + frontend.L1.M15[R1,R2]。由再次打开触发、关闭触发和结果状态标记约束成复用关系与关闭回退关系。来源：`C4 + WORKRESOURCE_REUSE + WORKSETSTATE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 工作资源集合必须导出唯一当前工作面
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：在 frontend 的集合浏览与触发结构上，工作资源集合必须能够导出唯一当前激活工作面及其工作资源类型，不能只有打开资源集合而没有当前内容。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`WORKSETSET/WORKSETSTATE`。

- `R2` 任一工作面都必须具备稳定工作资源身份
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：固定锚点工作面、对象工作面和辅助工作面都必须能说明自己的来源与稳定身份；不能只靠展示标题或视觉位置识别当前工作面。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`WORKSETSET/WORKRESOURCE_IDENTITY`。

- `R3` 工作面切换和关闭结果必须可解释
  - `R3.1` 参与基：`B2 + B4`。
  - `R3.2` 组合方式：在 frontend 的触发和附着反馈结构上，切换或关闭工作面后，系统必须能解释当前回到了哪个工作资源、为什么回到它，以及该结果属于关闭回退还是集合内切换。
  - `R3.3` 输出能力：`C2 + C4`。
  - `R3.4` 边界绑定：`WORKSETSTATE/WORKRESOURCE_REUSE`。

- `R4` 同一来源对象再次打开时的复用结果必须可解释
  - `R4.1` 参与基：`B3 + B4`。
  - `R4.2` 组合方式：再次打开同一来源对象时，可以复用已有对象工作面，也可以新建新的工作资源；但系统必须能解释采用了哪种结果，以及当前活动工作面如何因此发生变化。
  - `R4.3` 输出能力：`C3 + C4`。
  - `R4.4` 边界绑定：`WORKRESOURCE_IDENTITY/WORKRESOURCE_REUSE`。

- `R5` 无效方案
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：以下方案无效：跳过 frontend 集合浏览直接声明工作资源集合；只有标签可见但无唯一当前工作面；固定锚点工作面、对象工作面和辅助工作面不可区分；工作资源只能靠标题识别；关闭当前工作面后当前内容不可解释；再次打开同一来源对象后无法说明是复用还是新建。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`WORKSETSET/WORKSETSTATE/WORKRESOURCE_IDENTITY/WORKRESOURCE_REUSE`。

## 5. 验证（Verification）

- `V1` 当前工作面可识别性：任一时刻都必须能识别唯一当前激活工作面及其工作资源类型。
- `V2` 工作资源身份有效性：任一工作资源都必须能说明稳定身份与来源，而不是只靠展示标题识别。
- `V3` 切换与回退可解释性：切换或关闭后必须能解释当前回到哪一个工作资源，以及为什么回到它。
- `V4` 复用结果可解释性：再次打开同一来源对象后，必须能解释当前结果是复用已有工作资源还是新建新的工作资源。
- `V5` 前端承接有效性：工作资源集合、状态、身份与复用/回退关系必须能追溯到 frontend 的触发、集合浏览和附着反馈结构。
- `V6` 最小必要性：移除任一 `B*` 后，`V1~V5` 至少一项失效。
- `V7` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 结果影响与可观察回流原子模块:ReviewWorkbenchResultImpactAndObservableReturnAtoms

@framework

本模块定义 `review_workbench` 中“某个动作完成后影响了什么、当前焦点怎样保持可解释、哪里可持续观察到变化”的共享结构原子。它只回答结果影响、焦点结果与可观察回流这三个最小问题；它不负责动作目标本体，也不负责动作载荷本身。

本模块位于 `review_workbench` 的原子层。它不依赖本框架其他 `L0` 模块，不在这里形成完整动作闭环；它只把动作完成后的影响表达与回流表达沉成稳定共享结构，供上层编排复用。这里的动作可以是范围结构改动、审核结论提交或其他会带来结果变化的场景动作。

## 1. 能力声明（Capability Statement）
- `C1` 结果影响表达能力：将动作完成后的对象状态变化、统计变化、范围变化、队列变化或其他影响组织成稳定影响结构。
- `C2` 焦点结果表达能力：将动作完成后的当前焦点保持、回退、切换、重定位或失效解释组织成稳定焦点结果结构。
- `C3` 可观察回流表达能力：将动作完成后的局部反馈、全局反馈、状态提示、摘要变化或刷新结果组织成稳定回流结构。
- `N1` 非职责声明：不负责动作目标本体，不负责动作载荷本体，不负责后端契约字段。

## 2. 边界定义（Boundary / 参数）
- `IMPACT_SCOPE` 结果影响边界：对象状态、统计、范围、队列和其他影响范围允许变化，但任一次动作完成后必须至少能解释一组对象或状态变化。
- `FOCUS_SCOPE` 焦点结果边界：焦点保持、回退、切换、重定位或失效处理允许变化，但动作完成后必须能解释当前焦点落点或失效原因。
- `OBSERVABLE_SCOPE` 回流边界：反馈位置、反馈形式、摘要方式和刷新方式允许变化，但动作完成后必须至少出现一处持续可观察变化。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 结果影响基：能够表达动作完成后至少一组对象状态、统计、范围或队列变化。来源：`C1 + IMPACT_SCOPE`。
- `B2` 焦点结果基：能够表达动作完成后当前焦点保持、回退、切换、重定位或失效解释。来源：`C2 + FOCUS_SCOPE`。
- `B3` 可观察回流基：能够表达动作完成后至少一处持续可观察反馈、摘要、提示或刷新结果。来源：`C3 + OBSERVABLE_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 动作完成后必须存在可解释影响
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：任一次动作完成后，都必须至少能说明一组对象、状态、范围、统计或队列变化；不能只写“完成成功”而没有任何结果影响语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`IMPACT_SCOPE`。
- `R2` 动作完成后必须能解释当前焦点结果
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：任一次动作完成后，都必须能解释当前焦点仍是谁、回退到了哪里、切换到了哪里，或为什么当前焦点失效；不能让焦点处于不可解释状态。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`IMPACT_SCOPE/FOCUS_SCOPE`。
- `R3` 动作完成后必须至少有一处可观察回流
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：任一次动作完成后，至少要有一处持续可观察反馈、摘要变化、状态提示或刷新结果，并且该回流要能解释结果影响与当前焦点结果；不能只有内部状态变化而外部完全不可见。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`IMPACT_SCOPE/FOCUS_SCOPE/OBSERVABLE_SCOPE`。
- `R4` 无效方案
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：以下方案无效：动作完成后无法说明任何结果影响；当前焦点结果不可解释；有内部状态变化但没有任何持续可观察回流；把动作目标本体、动作载荷或后端契约字段直接写回本模块。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`IMPACT_SCOPE/FOCUS_SCOPE/OBSERVABLE_SCOPE`。

## 5. 验证（Verification）
- `V1` 结果影响可识别性：任一次动作完成后都必须至少能识别一组对象、状态、范围、统计或队列变化。
- `V2` 焦点结果有效性：任一次动作完成后都必须能解释当前焦点落点或失效原因。
- `V3` 回流可见性：任一次动作完成后都必须至少有一处持续可观察变化。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 历史序列与差异承接原子模块:ReviewWorkbenchHistorySequenceAndDiffMountAtoms

@framework

本模块定义 `review_workbench` 领域最小的“历史序列前端承接与可解释差异回看”结构。它回答“当前对象有哪些历史版本能够被承接”“一次历史变更如何被工作面打开”和“差异与变更元信息如何稳定表达”这三个最小问题。它不负责具体文件块处理动作，也不负责审核结论本身。

本模块位于 `review_workbench` 的原子层。它不依赖本框架其他 `L0` 模块，不在这里形成“识别对象 -> 打开历史节点 -> 查看差异与元信息”的完整闭环；它只把历史序列、版本节点、差异内容和元信息沉成稳定共享结构，供上层编排消费。凡是“历史请求怎么表达、节点怎样返回、差异内容怎样编码”的问题，统一交给外部领域 `review_workbench_backend.L0.M5`。

## 1. 能力声明（Capability Statement）
- `C1` 历史序列前端承接能力：将对象历史版本、变更节点和顺序关系组织成稳定前端序列结构。
- `C2` 差异回看前端表达能力：将某次变更的前后差异、内容片段和查看入口组织成稳定前端差异结构。
- `C3` 变更元信息前端表达能力：将操作者、时间、变更说明和关联上下文组织成稳定前端元信息结构。
- `N1` 非职责声明：不负责具体恢复策略、不负责文件块纠错动作、不负责审核决策规则。

## 2. 边界定义（Boundary / 参数）
- `HISTORY_SCOPE` 历史序列边界：版本节点数量和展示顺序允许变化，但任一历史序列都必须附着稳定对象身份，并能在前端解释节点顺序关系与当前历史来源。
- `DIFF_VIEW_SCOPE` 差异边界：前后版本组合、差异片段数量和查看方式允许变化，但任一差异查看都必须能说明明确前后版本关系，并稳定落在当前前端查看上下文中。
- `META_VIEW_SCOPE` 元信息边界：操作者、时间、变更说明和关联上下文字段允许变化，但任一历史节点都必须至少保留一组可解释元信息，并能在前端被稳定承接。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 历史序列与节点基：能够表达对象历史版本、节点顺序关系与当前历史来源。来源：`C1 + HISTORY_SCOPE`。
- `B2` 差异回看基：能够表达当前历史节点下的前后版本关系、差异片段与查看落点。来源：`C2 + DIFF_VIEW_SCOPE`。
- `B3` 元信息表达基：能够表达操作者、时间、变更说明和关联上下文等最小元信息。来源：`C3 + META_VIEW_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 差异查看必须附着历史节点
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：任何差异回看都必须能说明来自哪个对象、哪个历史节点以及哪一组前后版本；不能只有一段差异文本而没有历史来源。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`HISTORY_SCOPE/DIFF_VIEW_SCOPE`。
- `R2` 历史节点必须带出元信息
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：任何历史节点都必须能说明由谁改、何时改、为什么改或关联到什么上下文；不能只有版本号而没有可解释元信息。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`HISTORY_SCOPE/META_VIEW_SCOPE`。
- `R3` 历史回看原子闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：历史序列、差异回看和元信息必须共同形成“识别历史节点 -> 打开差异 -> 理解变更上下文”的稳定前端追溯结构；其中历史请求、节点返回和差异编码由 `review_workbench_backend.L0.M5` 单独承担。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`HISTORY_SCOPE/DIFF_VIEW_SCOPE/META_VIEW_SCOPE`。
- `R4` 无效方案
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：以下方案无效：只有历史列表没有当前节点语义；只有差异片段没有历史来源；只有变更时间没有前后版本关系；把对象入口结构、反馈结构或后端历史请求契约字段、差异回流契约字段直接写回本模块。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`HISTORY_SCOPE/DIFF_VIEW_SCOPE/META_VIEW_SCOPE`。

## 5. 验证（Verification）
- `V1` 历史节点可识别性：任一差异回看都必须能追溯到明确历史节点与对应对象。
- `V2` 差异来源有效性：任一差异片段都必须能说明明确前后版本关系。
- `V3` 元信息可解释性：任一历史节点都必须至少带有一组可解释元信息。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

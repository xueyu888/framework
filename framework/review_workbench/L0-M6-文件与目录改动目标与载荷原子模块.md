# 文件与目录改动目标与载荷原子模块:ReviewWorkbenchFileAndDirectoryMutationTargetAndPayloadAtoms

@framework

本模块定义 `review_workbench` 中“文件与目录结构改动最少要改什么、改到哪里”的共享结构原子。它只回答一次结构改动最少需要哪些可判定字段，如何表达目标对象、目标容器、目标名称与目标落点；它不负责对象位置本体如何成立，也不负责改动完成后的树刷新、焦点回退或反馈回流。

本模块位于 `review_workbench` 的原子层。它不依赖本框架其他 `L0` 模块，不在这里形成“识别对象 -> 发起改动 -> 回流”的完整闭环；它只把上传、新建目录、删除、移动与重命名这类结构改动统一沉成可复用的最小改动载荷结构，供上层编排消费。

## 1. 能力声明（Capability Statement）
- `C1` 改动类型表达能力：将上传、新建目录、删除、移动与重命名组织成稳定改动类型结构。
- `C2` 改动目标附着能力：将一次改动稳定附着到明确目标对象、目标集合或目标容器。
- `C3` 改动落点表达能力：将目标父节点、目标名称、目标路径或其他落点信息组织成稳定目标落点结构。
- `N1` 非职责声明：不负责对象位置本体、不负责树刷新、不负责焦点回退、不负责反馈结果。

## 2. 边界定义（Boundary / 参数）
- `MUTATION_TYPE_SCOPE` 改动类型边界：结构改动种类允许变化，但任一次改动都必须具备可判定改动类型。
- `TARGET_SCOPE` 改动目标边界：目标对象、目标集合或目标容器允许变化，但任一次改动都必须能解释“改的是谁”。
- `DESTINATION_SCOPE` 改动落点边界：目标父节点、目标名称、目标路径和目标挂接方式允许变化，但凡涉及新落点的改动都必须能解释“改到哪里”。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 改动类型基：能够表达上传、新建目录、删除、移动与重命名等结构改动类型。来源：`C1 + MUTATION_TYPE_SCOPE`。
- `B2` 改动目标基：能够把一次结构改动附着到明确目标对象、目标集合或目标容器。来源：`C2 + TARGET_SCOPE`。
- `B3` 改动落点基：能够表达目标父节点、目标名称、目标路径或其他最小落点信息。来源：`C3 + DESTINATION_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 改动必须先有可判定类型
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：任一次结构改动都必须先能判定属于哪一类改动；不能只有一段模糊“改一下结构”的描述而没有明确改动类型。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`MUTATION_TYPE_SCOPE`。

- `R2` 改动必须附着明确目标
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：任一次结构改动都必须能说明作用于哪个对象、对象集合或容器；不能只有改动类型而没有目标归属。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`MUTATION_TYPE_SCOPE/TARGET_SCOPE`。

- `R3` 需要落点的改动必须附着最小落点
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：凡是上传、新建目录、移动、重命名或其他涉及新落点的结构改动，都必须能说明目标父节点、目标名称、目标路径或等价落点信息；不能把改动目标和目标落点混成不可解释的黑盒载荷。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`MUTATION_TYPE_SCOPE/TARGET_SCOPE/DESTINATION_SCOPE`。

- `R4` 无效方案
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：以下方案无效：只有“改动请求”没有改动类型；只有移动或重命名语义没有目标对象；需要新落点的改动却没有目标父节点、目标名称或等价落点信息；把树刷新、焦点回退或反馈结果直接写回本模块。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`MUTATION_TYPE_SCOPE/TARGET_SCOPE/DESTINATION_SCOPE`。

## 5. 验证（Verification）
- `V1` 改动类型可识别性：任一次结构改动都必须能判定明确改动类型。
- `V2` 改动目标有效性：任一次结构改动都必须能追溯到明确目标对象、目标集合或目标容器。
- `V3` 改动落点有效性：凡涉及新落点的改动，都必须能说明最小落点信息。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

# 平台导航到场景挂接编排模块:ReviewWorkbenchPlatformNavigationAndSceneMountOrchestration

@framework

本模块定义统一审查平台前半段“如何从平台导航进入某个具体场景，并让该场景获得稳定承载面、顶部全局工具位、平台级统计集合承接面与默认上下文”这条链路。它位于 `review_workbench` 的编排层，关心的是平台壳、场景导航、当前激活场景、顶部全局工具位、平台级统计、平台级反馈和场景承载面如何配合，而不是具体场景内部有哪些目录、审核字段或历史差异项。

本模块建立在 `review_workbench.L0.M0`、`review_workbench.L0.M3`、`review_workbench.L0.M5` 和 `review_workbench.L0.M11` 这些领域原子之上。也就是说，它不重新定义文件库、审核提交或历史追溯内部结构，而是把平台级导航与统一承载关系收束成“进入平台 -> 选场景 -> 获得场景承载面 -> 持续使用平台顶部工具位 -> 持续看到平台级统计与平台状态”的稳定前置编排。

本模块也不负责判断“哪些具体场景属于统一平台中的合法场景实例集合”，也不负责从平台层约束场景内部主线是否被错误上提；这些属于 `L3-M0` 的统一平台承接层职责。本模块只负责把已经允许进入平台的当前场景，稳定接到平台前置承接面中。

## 1. 能力声明（Capability Statement）
- `C1` 平台进入与场景选择能力：把统一平台壳、左侧导航和当前激活场景接成稳定入口。
- `C2` 场景承载能力：让被激活的场景获得稳定主工作区承载面，并与平台壳保持附着关系。
- `C3` 顶部全局工具位持续可用能力：让全局搜索、主题切换、用户入口和其他平台级工具位在场景切换前后仍保持稳定位置与平台语义。
- `C4` 平台级统计持续可见能力：让平台级统计集合在场景切换前后持续可见，并保持平台归属语义。
- `C5` 场景切换可解释能力：让切换前后都能说明当前落在哪个场景，以及回到了哪个场景上下文。
- `C6` 平台级反馈持续可见能力：让平台级状态条、汇总和切换反馈在场景切换过程中持续可观察。
- `N1` 非职责声明：不负责具体场景内部的范围树、对象工作资源、审核结论或历史差异编排，也不负责判断平台合法场景实例集合。

## 2. 边界定义（Boundary / 参数）
- `PLATFORM_SCOPE` 平台边界：平台壳、场景导航、顶部全局工具位、平台级统计集合承接面、主工作区和平台反馈区允许采用不同承载形式，但六者必须共同落在同一统一平台中，并保持稳定承接关系。来源：`C1 + C2 + C3 + C4`。
- `ACTIVE_SCENE_SCOPE` 当前场景边界：场景入口集合、默认场景和当前激活场景允许因实例而变化，但任一时刻必须存在唯一当前场景，且切换前后结果必须可解释；至于该当前场景是否属于统一平台中的合法场景实例集合，由 `L3-M0` 单独约束。来源：`C1 + C5`。
- `MOUNT_SCOPE` 场景挂接边界：具体场景的承载内容和展示方式允许变化，但当前激活场景必须挂接到主工作区中，不能漂浮在平台之外。来源：`C2`。
- `GLOBAL_TOOL_SCOPE` 全局工具位边界：工具位种类允许因平台实例而变化，但每个工具位都必须保留稳定位置、明确作用范围和平台级语义，且场景切换前后不可丢失。来源：`C3`。
- `PLATFORM_STAT_SCOPE` 平台统计边界：平台级统计集合的项种类、排序与表现允许变化，但场景切换前后都必须继续附着统一平台成立，且平台中的动作与状态变化后统计必须保持可解释更新。来源：`C4`。
- `PLATFORM_FEEDBACK_SCOPE` 平台反馈边界：平台级状态、汇总和切换提示允许采用不同反馈组合，但平台状态变化后必须至少有一处持续可观察反馈。来源：`C6`。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 平台壳、场景导航与顶部工具位基：`L0.M0[R4]`。来源：`C1 + C3 + PLATFORM_SCOPE + ACTIVE_SCENE_SCOPE + GLOBAL_TOOL_SCOPE`。
- `B2` 场景承载与工作集基：`L0.M3[R1,R3,R5]`。来源：`C2 + C5 + MOUNT_SCOPE`。
- `B3` 平台反馈基：`L0.M5[R1,R2,R3]`。来源：`C6 + PLATFORM_FEEDBACK_SCOPE`。
- `B4` 平台级统计基：`L0.M11[R2,R3,R4]`。来源：`C4 + PLATFORM_STAT_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 先有平台壳，再有具体场景
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：统一平台要先有稳定壳层、场景导航和顶部全局工具位，再在当前主工作区挂接被激活的具体场景；不能跳过平台层，直接把具体场景写成孤立页面。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`PLATFORM_SCOPE/ACTIVE_SCENE_SCOPE/MOUNT_SCOPE/GLOBAL_TOOL_SCOPE`。
- `R2` 顶部全局工具位必须随着场景切换持续可用
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：切换场景后，全局搜索、主题切换、用户入口和其他平台级工具位必须继续挂接在稳定顶栏中，并保持对整个平台生效；不能在进入文件库、审核或历史场景后消失，或退化成某个场景的局部触发入口。
  - `R2.3` 输出能力：`C2 + C3 + C5`。
  - `R2.4` 边界绑定：`PLATFORM_SCOPE/MOUNT_SCOPE/GLOBAL_TOOL_SCOPE/ACTIVE_SCENE_SCOPE`。
- `R3` 平台级统计必须随着场景切换持续成立
  - `R3.1` 参与基：`B1 + B4`。
  - `R3.2` 组合方式：切换场景后，平台级统计集合必须继续挂接在统一平台中，并保持平台归属语义；不能在进入文件库、审核或历史场景后消失，或退化成某个场景的局部摘要。
  - `R3.3` 输出能力：`C1 + C4 + C5`。
  - `R3.4` 边界绑定：`PLATFORM_SCOPE/PLATFORM_STAT_SCOPE/ACTIVE_SCENE_SCOPE`。
- `R4` 场景切换后当前落点必须可解释
  - `R4.1` 参与基：`B1 + B2`。
  - `R4.2` 组合方式：切换场景后，平台必须能解释当前主工作区落在哪个场景、当前工作集属于哪个场景，以及切换关闭后回到了哪个场景上下文；不能只有视觉变化而没有场景语义。
  - `R4.3` 输出能力：`C1 + C5`。
  - `R4.4` 边界绑定：`ACTIVE_SCENE_SCOPE/MOUNT_SCOPE`。
- `R5` 平台反馈必须随着场景切换持续可见
  - `R5.1` 参与基：`B1 + B3`。
  - `R5.2` 组合方式：场景切换、平台加载、切换失败或状态变化之后，平台级反馈必须持续挂接在稳定位置；不能让反馈停留在不可观察状态。
  - `R5.3` 输出能力：`C1 + C6`。
  - `R5.4` 边界绑定：`PLATFORM_SCOPE/PLATFORM_FEEDBACK_SCOPE`。
- `R6` 平台导航到场景挂接闭环
  - `R6.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R6.2` 组合方式：平台壳、场景导航、顶部全局工具位、平台级统计、场景承载和平台反馈要共同组成“进入统一平台 -> 选场景 -> 获得当前场景承载面 -> 持续使用平台顶部工具位 -> 持续看到平台级统计与平台状态”的完整平台前置闭环。
  - `R6.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  - `R6.4` 边界绑定：`PLATFORM_SCOPE/ACTIVE_SCENE_SCOPE/MOUNT_SCOPE/GLOBAL_TOOL_SCOPE/PLATFORM_STAT_SCOPE/PLATFORM_FEEDBACK_SCOPE`。
- `R7` 无效方案
  - `R7.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R7.2` 组合方式：以下方案无效：只有左侧导航没有统一平台壳；只有主工作区没有当前场景语义；场景切换后顶部搜索和主题入口失去平台语义；场景切换后平台级统计消失或退化成场景局部摘要；切换场景后无法说明当前落点；平台存在状态变化但没有稳定反馈承接面；把本模块扩写成对合法场景实例集合的判定层，或在本模块中直接混入具体场景内部主线。
  - `R7.3` 失效结论：`N1`。
  - `R7.4` 边界绑定：`PLATFORM_SCOPE/ACTIVE_SCENE_SCOPE/MOUNT_SCOPE/GLOBAL_TOOL_SCOPE/PLATFORM_STAT_SCOPE/PLATFORM_FEEDBACK_SCOPE`。

## 5. 验证（Verification）
- `V1` 平台入口有效性：进入平台后必须能识别可切换场景集合与唯一当前激活场景。
- `V2` 顶部全局工具位稳定性：进入任一场景后，全局搜索、主题切换、用户入口等平台级工具位必须仍可定位且仍具平台语义。
- `V3` 平台级统计持续性：进入任一场景后，平台级统计集合必须仍可定位且仍具平台归属语义。
- `V4` 场景挂接有效性：当前激活场景必须挂接在主工作区中，并能解释对应场景归属。
- `V5` 场景切换可解释性：切换或关闭后必须能解释当前回到哪个场景上下文。
- `V6` 平台反馈可见性：平台级状态变化后必须至少有一处持续可观察反馈。
- `V7` 层间职责有效性：本模块只验证平台前置编排是否成立，不直接验证当前激活场景是否属于统一平台中的合法场景实例集合；该项由 `L3-M0` 单独验证。
- `V8` 最小必要性：移除任一 `B*` 后，`V1~V7` 至少一项失效。
- `V9` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

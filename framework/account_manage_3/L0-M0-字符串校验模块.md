# 字符串校验模块:VerificationVerificationModule

@framework

## 0. 目标 (Goal)

- 本模块负责字符串的校验功能，可以按照指定的格式或逐字符进行校验。



## 1. 能力声明（Capability Statement）

- `C1` 字符串校验能力：提供字符串的校验功能
- `C2` 格式匹配校验能力：提供基于正则表达式的字符串格式校验功能
- `C3` 逐字符匹配校验能力：提供基于参考字符串的逐字符校验功能

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责字符串内容转换功能：也就是不负责对输入字符串进行修改、替换或格式化
- `N2` 不负责字符串摘要生成功能：也就是不能将输入字符串转换为摘要字符串
- `N3` 不负责字符串业务语义判断功能：也就是不负责判断字符串是否满足具体业务含义

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 输入类型：字符串。
- `P2` 字符串长度：最短长度与最长长度。
- `P3` 字符串格式：字母、数字、字母+数字、字母+数字+特殊字符，等。
- `P4` 校验输出类型：通过或失败。

## 配置项：
- `P6` 待检验字符串
- `P7` 正则表达式
- `P8` 参考字符串

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 正则匹配器
- `B2` 逐字符匹配器

## 4. 基组合原则（Base Combination Principles）

- `R1` 格式匹配校验组合
  - `R1.1` 参与基：`B1 + B4`。
  - `R1.2` 组合方式：字符串输入进入后，使用 `B1` 按 `p7` 中定义的正则表达式执行格式匹配；匹配成功时输出通过，匹配失败时输出失败。
  - `R1.3` 输出能力：`C1 + C2`
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P6 + P7`

- `R2` 逐字符匹配校验组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：字符串输入进入后，使用 `B2` 将 `P6` 作为待检验字符串，与 `P8` 按字符位置逐一比较；全部字符一致时输出通过，存在任一字符不一致时输出失败。
  - `R2.3` 输出能力：`C1 + C3`
  - `R2.4` 参数绑定：`P1 + P2 + P4 + P6 + P8`

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

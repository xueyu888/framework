# 账号密码存取模块:PasswordStorageModule

@framework

## 0. 目标 (Goal)
- 能够完成账号密码的存储，并且能够通过账号找到对应的密码

## 1. 能力声明（Capability Statement）

- `C1` 账号密码存取能力：提供账号密码的存储和访问功能

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责账号单独修改功能：也就是不能只修改账号

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 账号输入类型：字符串
- `P2` 密码输出类型：字符串
- `P3` 账号长度范围：最短长度与最长长度
- `P4` 密码长度范围：最短长度与最长长度
- `P5` 账号键哈希函数：必须是作用于字符串账号的确定性哈希函数
- `P6` 重复账号处理策略：拒绝写入、覆盖原值
- `P7` 哈希冲突处理策略：拉链法、开放寻址

## 配置项：
- `P8` 账号
- `P9` 密码


## 3. 最小结构基（Minimal Structural Bases）

- `B1` 哈希表

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法存入组合
  - `R1.1` 参与基：`P8 + P9 + B1`
  - `R1.2` 组合方式：将 `p8` 作为键写入 `P8`，并按照 `P5` 的账号键哈希函数定位到哈希表中的存储位置；将 `p9` 作为值存储在 `B1` 中与该 `P8` 对应的位置上，形成一条 `P8` -> `P9` 的账号密码映射；若不同账号在定位过程中发生哈希冲突，则按 `P7` 指定的哈希冲突处理策略继续完成定位与存储；写入前必须先基于 `P8` 在 `B1` 中执行存在性判定，若该账号不存在则建立映射，若该账号已存在则按 `P6` 指定的重复账号处理策略执行拒绝写入或覆盖原值。
  - `R1.3` 输出能力：`C1`
  - `R1.4` 边界绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7`

- `R2` 合法取出组合
  - `R2.1` 参与基：`P8 + P9 + B1`
  - `R2.2` 组合方式：将 `P8` 作为键输入 `B1`，并按照 `P5` 的账号键哈希函数定位到哈希表中的存储位置；若定位过程中存在哈希冲突，则按 `P7` 指定的哈希冲突处理策略继续查找；当找到与 `P8` 对应的存储项后，从 `B1` 中取出对应的 `P9`，作为密码输出。
  - `R2.3` 输出能力：`C1`
  - `R2.4` 边界绑定：`P1 + P2 + P3 + P4 + P5 + P7`

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

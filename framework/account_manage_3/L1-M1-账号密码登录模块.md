# 账号密码登录模块:AccountPasswordLoginModule

@framework
## 0. 目标 (Goal)
- 本模块负责账号密码的登录功能，包括账号格式校验、密码摘要转换、已存密码摘要取出和密码逐字符比较。


## 1. 能力声明（Capability Statement）

- `C1` 账号密码登录能力：提供账号密码登录功能
- `C2` 合法登录处理能力：提供账号格式校验、密码摘要转换、已存密码摘要取出和密码校验的串联处理功能
- `C3` 登录结果输出能力：提供登录成功或登录失败的结果输出功能

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责账号密码注册：也就是不负责新账号创建及初始凭据写入
- `N2` 不负责账号或密码修改/重置/找回：也就是不负责对既有凭据做变更或恢复
- `N3` 不负责会话建立与凭证签发：也就是不负责生成、刷新或撤销 token/session/cookie
- `N4` 不负责权限授权判定：也就是不负责角色、资源访问权限与策略决策

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 账号输入类型：字符串
- `P2` 密码输入类型：字符串
- `P3` 登录结果输出类型：成功或失败
- `P4` 账号长度范围：最短长度与最长长度
- `P5` 密码长度范围：最短长度与最长长度
- `P6` 密码摘要输出类型：字符串

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 账号格式校验器：L0.M0[R1]
- `B2` 密码摘要转换器：L0.M1[R1]
- `B3` 密码摘要取值器：L0.M2[R2]
- `B4` 密码逐字符比较器：L0.M0[R2]

## 4. 基组合原则（Base Combination Principles）

- `R1` 合法登录组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`
  - `R1.2` 组合方式：字符串类型的账号和密码输入进入后，先通过 `B1` 对账号执行格式校验；校验通过后，将密码输入送入 `B2` 执行摘要转换，得到输入密码摘要；再将账号输入送入 `B3` 取出已存密码摘要；最后将输入密码摘要与已存密码摘要送入 `B4` 执行校验，校验通过则登录成功。
  - `R1.3` 输出能力：`C1 + C2 + C3`
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`

- `R2` 非法登录组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4`
  - `R2.2` 组合方式：字符串类型的账号和密码输入进入后，若 `B1` 对账号格式校验失败，则直接输出登录失败；若 `B1` 校验通过但 `B2` 密码摘要转换失败，则输出登录失败；若 `B2` 转换成功但 `B3` 未取出对应的已存密码摘要，则输出登录失败；若 `B3` 取出成功但 `B4` 校验失败，则输出登录失败。
  - `R2.3` 输出能力：`C3`
  - `R2.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基/组合方式/输出能力或失效结论/参数绑定。
- `V3` 结构闭环性：`R1~R2` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

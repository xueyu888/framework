# 标签定义:TagDefinition

@framework

## 1. 能力声明（Capability Statement）

- `C1` 标签定位能力：支持在数据库中通过标签ID唯一定位标签。
- `C2` 标签名承载能力：能为单个标签提供稳定且唯一的标签名。
- `C3` 标签说明承载能力：能体现当前标签表示的内容含义。
- `C4` 正向线索承载能力：能体现支持文本命中该标签的正向线索。
- `C5` 反向线索承载能力：能体现削弱或排除文本命中该标签的反向线索。
- `C6` 示例承载能力：能体现该标签对应内容的示例文本。
- `C7` 领域关联能力：能将标签与对应的领域进行关联的能力。
- `C8` 标签定义导出能力：能将标签名、标签说明、正向线索、反向线索和示例组成一个完整标签定义，供上层评分模块消费。
- `N1` 不提供文本评分能力：不负责对输入文本执行匹配计算或输出最终置信度。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 标签ID类型：字符串类型。来源：`C1`。
- `P2` 标签ID唯一性：标签ID必须唯一。来源：`C1`。
- `P3` 标签名类型：字符串类型。来源：`C2`。
- `P4` 标签名语言：中文。来源：`C2`。
- `P5` 标签名长度：不超过10字符。来源：`C2`。
- `P6` 标签说明类型：字符串类型。来源：`C3`。
- `P7` 标签说明长度：不超过100字符。来源：`C3`。
- `P8` 正向线索类型：列表类型。来源：`C4`。
- `P9` 正向线索内容：列表中的每一项都必须是字符串。来源：`C4`。
- `P10` 反向线索类型：列表类型。来源：`C5`。
- `P11` 反向线索内容：列表中的每一项都必须是字符串。来源：`C5`。
- `P12` 示例类型：列表类型。来源：`C6`。
- `P13` 示例内容：列表中的每一项都必须是字符串。来源：`C6`。
- `P14` 领域ID类型：字符串类型。来源：`C7`。
- `P15` 领域ID关联边界：领域ID必须为单个合法领域标识，并可关联到领域表中的主键。来源：`C7`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 标签ID：主键，标签的唯一标识。来源：`C1 + P1 + P2`。
- `B2` 标签名：标签名。来源：`C2 + P3 + P4 + P5`。
- `B3` 标签说明：当前标签表示的内容说明。来源：`C3 + P6 + P7`。
- `B4` 正向线索：支持文本命中该标签的线索。来源：`C4 + P8 + P9`。
- `B5` 反向线索：削弱或排除文本命中该标签的线索。来源：`C5 + P10 + P11`。
- `B6` 示例：该标签对应内容的示例文本。来源：`C6 + P12 + P13`。
- `B7` 领域ID：外键，指向领域表格中的领域ID。来源：`C7 + P14 + P15`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 完整定义组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6 + B7`。
  - `R1.2` 组合方式：`B1`、`B2`、`B3`、`B4`、`B5`、`B6` 和 `B7` 共同组成一个完整的标签定义，作为数据库表中的一行数据，也作为上层AI评分模块的输入。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6 + C7 + C8`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10 + P11 + P12 + P13 + P14 + P15`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2` 推导得到，`B2` 必须可由 `C2 + P3 + P4 + P5` 推导得到，`B3` 必须可由 `C3 + P6 + P7` 推导得到，`B4` 必须可由 `C4 + P8 + P9` 推导得到，`B5` 必须可由 `C5 + P10 + P11` 推导得到，`B6` 必须可由 `C6 + P12 + P13` 推导得到，`B7` 必须可由 `C7 + P14 + P15` 推导得到。
- `V2` 规则完整性：`R1` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1` 的输出能力必须覆盖 `C1 + C2 + C3 + C4 + C5 + C6 + C7 + C8`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除 `B1`、`B2`、`B3`、`B4`、`B5`、`B6` 或 `B7` 任一结构基后，完整标签定义组合不再成立，且 `C8` 至少无法成立。

# 标注审核流转编排模块:TaggingReviewFlowOrchestration

@framework

## 1. 能力声明（Capability Statement）

- `C1` 标注审核流转入口承接能力：支持以单个文档ID和系统判定阈值作为入口，启动该文档的标注审核流转。
- `C2` 自动标注流转能力：支持在入口约束下先完成文本块标签匹配评分，再完成系统判定分流。
- `C3` 人工审核接续能力：支持将系统判定后仍需人工处理的记录接续到人工审核段，并在收到人工确认结果后完成回写。
- `C4` 标注审核闭环能力：支持将评分、系统判定与人工审核串联为完整的文档级标注审核闭环。

- `N1` 不提供底层数据库实现能力：不负责数据库表结构、ORM 或存储引擎细节实现。
- `N2` 不提供前端交互能力：不负责登录、页面展示、页面跳转或人工审核界面的实现。
- `N3` 不提供评分算法实现能力：不负责实现具体评分模型、提示词或算法本身。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文档级入口边界：流转入口必须为单个合法文档ID。来源：`C1 + C2 + C4`。
- `P2` 阈值输入边界：系统判定阶段必须输入合法的启用阈值与不启用阈值，且启用阈值必须大于不启用阈值。来源：`C1 + C2`。
- `P3` 流转顺序边界：完整流转必须按“评分 -> 系统判定 -> 人工审核”的顺序展开，不得跳过前序阶段直接进入后序阶段。来源：`C2 + C4`。
- `P4` 系统判定分流边界：只有系统判定结果为 `待人工审核` 的记录才允许进入人工审核段。来源：`C2 + C3`。
- `P5` 人工审核作用范围边界：人工审核只作用于 `enable_status = 待人工审核` 且 `review_status = 未审核` 的记录。来源：`C3`。
- `P6` 闭环结果边界：完整流转结束后，每条标注记录都必须落到合法的 `enable_status`、`review_status` 与 `review_source` 状态组合。来源：`C4`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 标注审核流转入口：承接单个文档ID与系统判定阈值输入，并触发文档级标注审核流转的入口结构。来源：`C1 + P1 + P2 + P3`。
- `B2` 文本块标签匹配评分器：L2.M0[R1,R2,R3]。来源：`C2 + P1 + P3`。
- `B3` 系统判定标签启用模块：L2.M1[R1]。来源：`C2 + P2 + P3 + P4`。
- `B4` 人工判定接口模块：L2.M2[R1,R2]。来源：`C3 + P4 + P5 + P6`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 自动标注流转启动组合
  - `R1.1` 参与基：`B1 + B2 + B3`。
  - `R1.2` 组合方式：从标注审核流转入口接收单个文档ID和系统判定阈值后，先完成该文档的文本块标签匹配评分与初始标注记录落库，再对已有标注记录执行系统判定分流，形成 `启用`、`不启用` 或 `待人工审核` 的系统结果。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4`。

- `R2` 人工审核接续组合
  - `R2.1` 参与基：`B1 + B3 + B4`。
  - `R2.2` 组合方式：在系统判定完成后，仅将分流为 `待人工审核` 的记录接续到人工审核段，由人工判定接口模块导出人工审核数据，并在收到人工确认结果后完成结果回写。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 参数绑定：`P1 + P3 + P4 + P5 + P6`。

- `R3` 标注审核闭环编排组合
  - `R3.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R3.2` 组合方式：以单个文档ID为入口，先完成评分与初始标注记录落库，再完成系统判定分流，并对其中需人工审核的记录继续执行人工审核接续与结果回写，形成完整的文档级标注审核闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2 + P3` 推导得到，`B2` 必须可由 `C2 + P1 + P3` 推导得到，`B3` 必须可由 `C2 + P2 + P3 + P4` 推导得到，`B4` 必须可由 `C3 + P4 + P5 + P6` 推导得到。
- `V2` 规则一致性：`R1`、`R2`、`R3` 与 `R4` 必须明确给出参与基、组合方式、输出能力或失效结论与参数绑定。
- `V3` 结构闭环性：`R1 + R2 + R3` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4`。
- `V4` 流转顺序有效性：任意有效闭环都必须满足 `P3`，不得跳过评分或系统判定直接进入人工审核。
- `V5` 分流范围有效性：任意进入人工审核段的记录都必须满足 `P4 + P5`。
- `V6` 闭环结果有效性：任意有效闭环输出都必须满足 `P6`。
- `V7` 最小必要性：移除 `B1`、`B2`、`B3` 或 `B4` 任一结构基后，自动标注流转、人工审核接续或完整闭环至少一项无法成立。
- `V8` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 文本块标签匹配评分器:CTMatchScorer

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文档ID输入能力：支持输入单个需要进行标注的文档ID。
- `C2` 数据库调用能力：支持通过数据库接口查询文本块、查询文档领域、查询标签定义并写入标注记录。
- `C3` 文本块集合承载能力：支持承载当前文档对应的文本块集合。
- `C4` 标签定义集合承载能力：支持承载当前文档领域对应的标签定义集合。
- `C5` 文本块-标签匹配程度评分能力：能对文本块和标签之间的匹配程度进行评分的能力。
- `C6` 标注结果入库能力：能将评分结果补充审核状态后整理为标注记录并写入数据库。
- `N1` 不提供文档生成能力：不负责生成文档内容或创建新文档。
- `N2` 不提供文本块切分能力：不负责将原始文档切分为文本块。
- `N3` 不提供标签创建能力：不负责创建新标签或修改标签定义。
- `N4` 不提供审核判定能力：不负责对“未审核”之后的审核流转进行处理。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 文档ID边界：输入文档ID必须为单个合法文档标识。来源：`C1`。
- `P2` 文档ID数量边界：单次输入的文档ID数量为 `1`。来源：`C1`。
- `P3` 数据库接口边界：输入数据库接口必须满足 `L1.M1[R2]` 的组合要求。来源：`C2`。
- `P4` 文本块集合边界：根据文档ID查询得到的文本块集合中的每个文本块都必须满足 `backend.L0.M0[R3]` 的组合规则。来源：`C3`。
- `P5` 文本块集合数量边界：当前文档对应的文本块数量必须大于等于 `1`。来源：`C3`。
- `P6` 标签定义集合边界：根据文档领域查询得到的标签定义集合中的每个标签定义都必须满足 `L0.M0[R1]` 的组合规则。来源：`C4`。
- `P7` 标签定义集合数量边界：当前文档领域对应的标签定义数量必须大于等于 `1`。来源：`C4`。
- `P8` 单次评分接口边界：单次评分接口必须满足 `L1.M0[R1]` 的组合要求。来源：`C5`。
- `P9` 审核状态边界：写入数据库的每条标注记录都必须补充 `review_status = 未审核`。来源：`C6`。
- `P10` 标注结果边界：写入数据库的标注记录必须至少包含 `doc_id`、`chunk_id`、`tag_name`、`confidence`、`reason` 和 `review_status` 字段。来源：`C6`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 文档ID：单个待标注文档的唯一标识。来源：`C1 + P1 + P2`。
- `B2` 数据库接口：查询文本块、查询文档领域、查询标签定义并写入标注记录的接口 L1.M1[R2]。来源：`C2 + P3`。
- `B3` 文本块集合：从数据库中读取出来的文本块集合。来源：`C3 + P4 + P5`。
- `B4` 标签定义集合：从数据库中读取出来的标签定义集合。来源：`C4 + P6 + P7`。
- `B5` 匹配程度评分接口：对文本块和标签之间的匹配程度的评分接口 L1.M0[R1]。来源：`C5 + P8`。
- `B6` 文本-标签评分记录：最终送入数据库的记录的结构，包括对文本块-标签匹配程度的评分。来源：`C5 + P9 + P10`

## 4. 基组合原则（Base Combination Principles）

- `R1` 查询装配组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：接收单个文档ID后，通过数据库接口查询该文档对应的文本块集合；再查询该文档的领域，并根据领域查询对应的标签定义集合；最后将查询得到的文本块集合与标签定义集合装配为当前文档标注所需的结构。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7`。

- `R2` 文本块-标签匹配程度评分组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R2.2` 组合方式：在当前文档的文本块集合与标签定义集合基础上，遍历文本块集合中的每个文本块，并对每个标签定义调用一次单次评分接口，得到对应的评分结果。
  - `R2.3` 输出能力：`C5`。
  - `R2.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8`。

- `R3` 文档-标签初筛组合
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`
  - `R3.2` 组合方式：每对一个文本块-标签的匹配程度完成评分后，将记录装配为`B6`的格式，并送入数据库中。
  - `R3.3` 输出能力：`C6`。
  - `R3.4` 参数绑定：`P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1` 必须可由 `C1 + P1 + P2` 推导得到，`B2` 必须可由 `C2 + P3` 推导得到，`B3` 必须可由 `C3 + P4 + P5` 推导得到，`B4` 必须可由 `C4 + P6 + P7` 推导得到，`B5` 必须可由 `C5 + P8` 推导得到，`B6` 必须可由 `C5 + P9 + P10` 推导得到。
- `V2` 规则完整性：`R1`、`R2` 与 `R3` 必须明确给出参与基、组合方式、输出能力与参数绑定。
- `V3` 结构闭环性：`R1 + R2 + R3` 的输出能力并集必须覆盖 `C1 + C2 + C3 + C4 + C5 + C6`。
- `V4` 查询结果边界符合性：任意有效标注组合中，文本块集合必须满足 `P4 + P5`，标签定义集合必须满足 `P6 + P7`。
- `V5` 评分结果边界符合性：任意有效评分结果都必须满足 `P8` 对应的接口边界要求。
- `V6` 写入结果合法性：任意有效写入记录都必须满足 `P9 + P10`。
- `V7` 最小必要性：移除 `B1`、`B2`、`B3`、`B4`、`B5` 或 `B6` 任一结构基后，查询装配组合、匹配评分组合或文档-标签初筛组合至少一项无法成立。
- `V8` 结论表达：逐条输出 `R* -> C* + 参数绑定` 的通过或失败结论。

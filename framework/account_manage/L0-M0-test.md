# 账号密码登录校验模块:AccountPasswordLoginVerificationModule

@framework

组件目标：接收账号与密码输入，完成输入格式校验、账号凭证定位、凭证校验和结果输出

## 1. 能力声明（Capability Statement）

- `C1` 输入格式校验能力：对账号输入和密码输入执行格式合法性校验
- `C2` 账号凭证定位能力：根据账号定位对应的已存密码凭证记录
- `C3` 凭证校验能力：根据既定凭证校验方式，对输入密码与已存密码凭证执行校验
- `C4` 校验结果输出能力：根据各类校验状态输出校验结果

## 2. 非职责声明（Non-Capability Statement）

- `N1` 不负责账户创建、账户资料维护和登录后信息修改。
- `N2` 不负责密码存储方式的设计与持久化实现，只消费既有凭证记录并完成校验。
- `N3` 不负责 token 发放、会话建立与权限控制。

## 3. 边界定义（Boundary）

- `d1` 账号长度范围：最短长度与最长长度。来源：`C1`
- `d2` 账号组成方式：只能字符、字符+数字、是否区分大小写。来源：`C1`
- `d3` 密码长度范围：最短长度与最长长度。来源：`C1`
- `d4` 密码组成方式：只能字符、字符+数字、是否要求复杂度。来源：`C1`
- `d5` 凭证校验方式：完全匹配、哈希校验、归一化后校验等。来源：`C3`
- `d6` 凭证表示方式：明文、MD5等    来源：`C3`
- `d7` 账号定位规则：同一账号必须唯一定位到一条有效凭证记录。来源：`C2`
- `d8` 校验结果类型：格式错误、账号不存在、密码错误、登录成功。来源：`C4`

## 4. 最小可行基（Minimum Viable Bases）

- `B1` 字符串容器：由字符串输入项、输入标识，容量构成。来源：`C1 + d1 + d2 + d3 + d4`。
- `B2` 账号密码映射表：由账号、密码凭证值构成。来源：`C2 + C3 + d5 + d6`。
- `B3` 结果映射表：由校验类别(格式、账号存在与否、密码是否正确)和结果输出值构成。来源：`C4 + d8`。
- `B4` 正则表达式：输入格式匹配规则。来源：`C1 + d1 + d2 + d3 + d4`。

差了一点也就是校验内容和校验器的映射
## 5. 基组合原则（Base Combination Principles）

- `R1` 输入格式校验组合
  - `R1.1` 参与基：`B1 + B3 + B4`。
  - `R1.2` 组合方式：字符串容器中的输入项先按输入标识区分为账号输入或密码输入，再分别按正则表达式中的对应匹配规则执行格式校验，若任一输入不满足长度范围或组成方式要求，则直接以格式校验为key查询结果映射表并输出结果，若全部校验通过，则返回成功
  - `R1.3` 输出能力：`C1 + C4`。
  - `R1.4` 边界绑定：`d1/d2/d3/d4/d8`。

- `R2` 账号凭证定位组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：账号输入在通过格式校验后，必须通过账号密码映射表定位对应的已存密码凭证；若若账号无法定位到有效记录，则直接以账号验证为key查询结果映射表并输出结果；若账号能够唯一定位到凭证记录，则返回成功
  - `R2.3` 输出能力：`C2 + C4`。
  - `R2.4` 边界绑定：`d7/d8`。

- `R3` 凭证校验组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在账号已定位到对应凭证后，根据既定凭证表示方式与凭证校验方式，对输入密码和已存密码凭证执行校验；校验失败时产出密码错误状态，校验通过时产出登录成功状态，再通过结果映射表输出结果。
  - `R3.3` 输出能力：`C3 + C4`。
  - `R3.4` 边界绑定：`d5/d6/d8`。

- `R4` 完整登录校验链组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：先执行输入格式校验，再执行账号凭证定位，最后执行凭证校验；任一前序步骤失败时，不得进入后续步骤，并统一通过结果映射表输出最终结果。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`d1/d2/d3/d4/d5/d6/d7/d8`。


## 6. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个边界项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力和边界绑定。
- `V3` 闭环性：`R4` 必须形成“输入格式校验 -> 账号凭证定位 -> 凭证校验 -> 结果输出”的完整登录校验链。
- `V4` 边界符合性：所有有效校验组合都必须满足 `d1~d7`。
- `V5` 最小必要性：移除任一 `B*` 后，至少一项能力无法成立。

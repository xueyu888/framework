# 承载结构:Carrier

@framework

承载结构用于让一个前端对象或区域具备明确落位与承接能力，使对象能够在界面中成立，并承接明确对象、内容或下层单元。

## 1. 能力声明（Capability Statement）

- `C1` 落位能力：让对象具备明确存在基础和位置关系，使其能够在界面中成立。
- `C2` 对象承接能力：让对象能够承接明确对象、内容或下层单元，使承接关系能够成立。

### 非职责声明（Non-Responsibility Statement）

- `N1` 不负责值处理：不负责值的绑定、更新、组织、校验和提交。
- `N2` 不负责布局样式：不负责对象之间的排列、样式和状态差异。
- `N3` 不负责反馈流程：不负责提交结果、提示内容和反馈持续时机。

## 2. 边界定义（Boundary / Parameter 参数）

- `P1` 落位边界：承载结构必须处于明确界面上下文中，不能脱离页面结构悬空存在。来源：`C1`。
- `P2` 承接对象边界：承载结构必须承接明确对象、内容或下层单元；若承接多个对象，仅表示承接关系成立，不自动形成成员归属与集合边界。来源：`C2`。

## 3. 最小结构基（Minimal Structural Bases）

- `B1` 落位基：用于确定对象在界面中的存在基础和位置关系。来源：`C1 + P1`。
- `B2` 对象承接基：用于确定对象能够承接明确对象、内容或下层单元。来源：`C2 + P2`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基本承载组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先为对象建立明确位置关系，使其具备最基本的成立条件。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 参数绑定：`P1`。

- `R2` 完整承载组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在位置关系成立的前提下，再建立对象承接结构，使对象能够承接明确对象、内容或下层单元。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 参数绑定：`P1 + P2`。

- `R3` 无效悬空组合
  - `R3.1` 参与基：`B2`。
  - `R3.2` 组合方式：只有对象承接基，而没有落位基。
  - `R3.3` 失效结论：如果没有位置关系，则承接对象没有依附基础，组合无效。
  - `R3.4` 参数绑定：缺失 `P1`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力或失效结论、参数绑定。
- `V3` 结构闭环性：`R1 ~ R2` 输出能力并集必须覆盖 `C1 ~ C2`。
- `V4` 参数符合性：所有有效组合必须满足绑定的参数约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 参数绑定` 或 `R* -> N* + 参数绑定` 的通过或失败结论。

# 单消息承载链式队列模块:SingleMessageQueueAbstractStructureModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 完整消息暂存能力：导出对单条完整消息的保存、取出与回收能力。
- `C2` 确定性取用能力：导出消息在暂存结构内的确定性排列结构。
- `C3` 并发端点协调能力：导出读写端点相互独立、并可在并发访问下保持状态一致的协调结构。
- `N1` 非职责声明：不负责消息内部外部引用目标的存在性、一致性与时效性。

## 2. 边界定义（Boundary）

<!-- 边界定义了本模块能力的合法变化空间与不变约束：即在实例化时“什么可以变”，但为了维持核心能力“必须满足什么底线”。 -->
- `SCALE` 总容量必须可判定；扩展只能按声明的离散步长发生。。来源：`C1`。
- `LIFETIME` 生命周期边界：消息一旦移出即不得再次参与取用。来源：`C1 + C2`。
- `ORDER` 顺序规则边界：任一时刻下一可取消息必须唯一可判定。来源：`C2`。
- `FLOW` 入出队边界：单实例内入队端与出队端固定，不允许运行时交换。来源：`C3`。
- `PARALLEL` 并行访问边界：必须声明允许的并发拓扑，并保证共享状态更新遵循唯一提交规则。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 承载节点：承载单条完整消息并维护占用/释放状态的承载单元。来源：`C1 + SCALE + LIFETIME`。
- `B2` 顺序定位：定义入/出队方向与下一可取位置的顺序定位结构。来源：`C2 + ORDER + LIFETIME + FLOW`。
- `B3` 并发协调：约束多端对共享队列状态提交顺序的访问协调结构。来源：`C3 + FLOW + PARALLEL`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 节点暂存组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：实例化可申请、可释放的存储节点与容量计量结构，形成仅具备暂存与回收能力的基础存储池。
  - `R1.3` 输出能力：`C1`（暂存）。
  - `R1.4` 边界绑定：`SCALE/LIFETIME`。

- `R2` 链式顺序组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在`B1`之上叠加`B2`，形成具备确定性推进关系的链式队列结构，但尚未开放并发控制。
  - `R2.3` 输出能力：`C1 + C2`（暂存 + 有序）。
  - `R2.4` 边界绑定：`SCALE/LIFETIME/ORDER/FLOW`。

- `R3` 并发缓存组合
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：在`B1`中保存数据，`B3`约束共享状态更新，形成并发访问安全的缓存结构。
  - `R3.3` 输出能力：`C1 + C3`（暂存 + 并发协调）。
  - `R3.4` 边界绑定：`SCALE/LIFETIME/PARALLEL`。

- `R4` 并发队列组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：在`B1`之上叠加`B2`，`B3`约束共享状态更新，形成并发访问安全的标准消息队列。
  - `R4.3` 输出能力：`C1 + C2 + C3`（暂存 + 有序 + 并发协调）。
  - `R4.4` 边界绑定：`SCALE/LIFETIME/ORDER/FLOW/PARALLEL`。

- `NR1` 禁止组合
  - `NR1.1` 参与基：`B1 + B2 + B3`。
  - `NR1.2` 组合方式：缺失链式顺序结构直接暴露端点、或绕过固定入队/出队端规则修改共享状态，或并发访问下无法判定节点追加/摘除结果的方案均无效。


## 5. 验证（Verification）
<!-- 检验边界是都闭合、检验组合能力导出、检验输出格式和证据落点 -->
- `V1` 基切分独立性：`B1` 不得依赖顺序与并发语义，`B2` 不得承担并发提交职责，`B3` 不得解释消息内容；否则结构切分失败。
- `V2` 边界闭合性：每个边界都必须对应一个可判定条件；若无法据此区分合法/非法实例，则该边界无效。来源：`C1 + C2 + C3`。
- `V3` 导出闭合性：`B1 -> C1`，`B1 + B2 -> C1 + C2`，`B1 + B2 + B3 -> C1 + C2 + C3`；若导出链不闭合，则规则失败。来源：`C1 + C2 + C3`。
- `V4` 最小必要性：从 `B1 + B2 + B3` 中移除任一 `B*` 后，至少失去一个正向能力。来源：`C1 + C2 + C3`。

# 异步任务流水线模块:AsyncTaskPipelineModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` **任务缓冲能力**：利用 L0 队列削峰填谷，隔离高并发摄入与慢速处理，确保系统稳定。
- `C2` **流水线编排能力**：将复杂处理流程拆解为多个独立阶段（Stage），各阶段间通过队列解耦，支持并行与异构处理。
- `C3` **处理可靠性能力**：提供任务重试、死信处理与状态追踪机制，确保任务最终一致性。
- `C4` **非能力项**：不负责具体的业务逻辑实现（如解析算法、向量化模型），只负责任务流转与调度。

## 2. 边界定义（Boundary）

- `B_TASK_TYPE` **任务类型边界**
  - **来源**：`C2`
  - **推导**：要编排，必须知晓处理什么。若不定义类型，调度器无法分派。
  - **约束**：必须明确**任务契约**（Payload Schema）与**处理阶段**（Stage）。对于文件解析，需定义支持的文件格式与解析目标。

- `B_THROUGHPUT` **吞吐控制边界**
  - **来源**：`C1`
  - **推导**：要缓冲，必须知晓处理速率。若不控制并发，下游可能被压垮。
  - **约束**：必须明确**并发度**（Concurrency Level）与**速率限制**（Rate Limit）。

- `B_RETRY` **重试策略边界**
  - **来源**：`C3`
  - **推导**：要可靠，必须知晓失败怎么办。若不定义重试，瞬时故障会导致数据丢失。
  - **约束**：必须明确**最大重试次数**、**退避策略**（Backoff）与**死信去向**（DLQ）。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` **任务队列基 (Task Queue Base)**
  - **结构定义**：基于 `L0-M0-消息队列结构模块` 的 **R3 完整队列组合**。
  - **推导**：为实现 `C1`，必须有可靠的缓冲介质。
  - **正交性**：负责 **Buffering（缓冲）**。只管存取任务，不执行业务逻辑。
  - **来源**：`L0-M0.R3 + B_TASK_TYPE`。

- `B2` **工作节点基 (Worker Node Base)**
  - **结构定义**：由**执行线程/协程**（Executor）、**业务处理器**（Handler）与**上下文**（Context）构成的执行单元。
  - **推导**：为实现 `C2`，必须有执行逻辑的载体。
  - **正交性**：负责 **Execution（执行）**。只管处理当前拿到的任务，不关心任务从哪来去哪去。
  - **来源**：`C2 + B_TASK_TYPE`。

- `B3` **调度器基 (Scheduler Base)**
  - **结构定义**：由**分派逻辑**（Dispatcher）、**生命周期监视器**（Monitor）与**错误处理器**（ErrorHandler）构成的控制单元。
  - **推导**：为实现 `C3`，必须有协调队列与Worker的机制。
  - **正交性**：负责 **Coordination（协调）**。只管搬运和监控，不存储任务也不执行业务。
  - **来源**：`C3 + B_THROUGHPUT + B_RETRY`。

## 4. 基组合原则（Base Combination Principles）

本章节定义如何通过基的组合实现不同的流水线模式。

### 4.1 简单生产者-消费者 (Simple Producer-Consumer)
- **参与基**：`B1 + B2 + B3`。
- **组合逻辑**：
  - 单个 `B1` 缓冲任务。
  - `B3` 从 `B1` 拉取任务，分派给 `B2` 执行。
  - 适用于单一类型的异步任务（如单纯的文件上传落盘）。
- **导出能力**：`C1 + C3`。

### 4.2 多级流水线 (Multi-Stage Pipeline)
- **参与基**：`B1(Multiple) + B2(Multiple) + B3`。
- **组合逻辑**：
  - 定义多个阶段（Stage A, Stage B...），每个阶段有独立的 `B1` 和 `B2`。
  - `B2(A)` 执行完后，将结果写入 `B1(B)`，触发下一阶段。
  - 适用于复杂业务链（如：上传 -> 解析 -> 清洗 -> 落库）。
- **导出能力**：`C1 + C2 + C3`。

### 4.3 扇出/扇入 (Fan-out/Fan-in)
- **参与基**：`B1 + B2(Multiple) + B3`。
- **组合逻辑**：
  - **扇出**：`B3` 将一个大任务拆解为多个子任务，推入 `B1`。
  - **并行**：多个 `B2` 并行处理子任务。
  - **扇入**：结果汇总到另一个 `B1` 或聚合器。
  - 适用于大文件切片解析或批量数据处理。
- **导出能力**：`C1 + C2 + C3`（高吞吐）。

## 5. 验证（Verification）

- `V1` **流水线连通性**：验证任务能否从 Stage A 的队列正确流转到 Stage B 的队列。
- `V2` **背压机制**：验证当 B2 处理慢时，B1 是否能积压并在达到边界时触发限流（Backpressure）。
- `V3` **异常恢复**：验证 B2 崩溃或处理失败时，B3 是否能捕获并执行重试/死信逻辑，且不丢失任务。
- `V4` **资源隔离**：验证不同阶段的 B2 是否互不影响（如解析阶段卡死不应影响上传阶段）。

# 文件管理模块

@framework

## 0.目的
文件的存取检索管理单元。
可以保存、取用文件；
可以按制造公司-系列-型号层级信息检索文件；
可以按制造部门-小组-个人层级信息检索文件；
可以按文件名称信息检索文件；
可以按功能标签信息检索文件；

## 最小基结构

- `put_get` 文件存取: [L0-M0.only]。
- `layer_search` 层级索引: [L0-M1.only]。
- `tag_search` tag索引: [L0-M2.only]。

## 基排列组合

- `tag_file` ：结合`put_get`与`tag_search`,可按tag查找并存取文件，增删文件的同时操作关联tag；

- `layer_file` ：结合`put_get`与`layer_search`,可按层级信息查找并存取文件，增删文件时同时操作关联节点；

- `full` ：结合`tag_file`、`layer_file`,可同时按配置层级信息存取文件和配置标签查找并存取文件，增删文件时同时操作关联节点和标签；


## 边界定义（参数）

### I/O定义
`file_key` 文件键：文件的唯一标识。来源：`put_get`。
`file_value` 文件内容：文件的实际内容。来源：`put_get`。

`layer_name` 层级：树的层级名。来源：`layer_search`。
`node_name` 节点名称：。来源：`layer_search`。

`tag_name` 标签名称：。来源：`tag_search`。

### 参数边界/配置边界

#### 文件存取配置
 - `file_base` 文件基础配置：约束`put_get.file_key_type`+`put_get.file_key_length`+`put_get.file_key_count`+`put_get.file_value_length`的范围。

#### tag查找配置
- `tag_file_key_type` 文件键类型配置：约束`put_get.file_key_type`与`tag_search.element_type`为相同的类型。来源：`tag_file` 。

- `tag_base` tag基础配置：约束`tag_search.element_set_type`+`tag_search.tag_name_type`+`tag_name_step`+`tag_name_length`+`tag_count`的范围。

#### 层级查找配置
- `layer_file_key_type`文件键类型配置：约束`put_get.file_key_type`与`layer_search.element_type`为相同的类型。来源：`layer_file`。

- `later_base` 层级基础配置：约束`layer_search.element_set_type`+`layer_search.layer_name_type`+`layer_search.node_name_type`+`layer_search.node_name_length`+`layer_search.children_node_count`的范围。来源：`layer_file`

## 能力申明

- `put_file_tag` 通过`file_key`+`file_value`+`tag_name`存入`file_value`。
- `put_file_layer` 通过`file_key`+`file_value`+`tag_name`存入`file_value`。

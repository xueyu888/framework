# 文件存取单元模块

@framework

## 0.目的
文件的基本存取单元，提供稳定的能查找到文件内容的结构，可通过文件的id直接寻找到文件本体。

## 最小基结构

- `table` 直接寻址表:通key可直接定位使用value的结构。

## 基排列组合

- `only` 模块内有且仅有一个`table`

## 边界定义（参数）

### 接口定义
#### I/O定义
- `file_key` 文件键：文件的唯一ID标识。
- `file_value` 文件内容：文件的实际内容。

### 参数边界/配置边界

- `file_key_type` 文件键类型：约束`file_key`的类型。
- `file_key_length` 文件键长度：约束`file_key`的长度。
- `file_key_count` 文件键数量：约束表中`file_key`的数量。

- `file_value_length` 文件内容长度：约束`file_value`的长度。

## 能力申明
 - `put_file` 通过`file_key`+`file_value`存入`file_value`。
 - `get_file` 通过`file_key`获取`file_value`。
 - `remove_file` 通过`file_key`删除文件`file_key`+`file_value`。

### 非职责声明
- 不负责文件内容本体存放外的其他功能。

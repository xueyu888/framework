# 层级检索单元
@framework

## 目的

一个按层级检索的名称->键的检索单元，可通过层级的具体名称信息检索到其下所有元素集合。

## 最小基结构

- `namespace_tree` 命名空间树:节点含有名称，同层节点名唯一，叶子节点存储元素集合的树。

## 基排列组合

- `only` 模块内有且仅有一个`namespace_tree`

## 边界定义（参数）

### I/O接口定义

  - `layer_name` 层级名称：树的层级名称。
  - `node_name` 节点名称：节点名称。
  - `element` 元素：叶子节点存放的最小单元。 
  - `element_set` 元素集合：存放`element`的集合。

### 参数边界/配置边界

  - `element_type` 叶子存放元素类型：约束叶子节点的`elementset`中的`element`的类型。
  - `element_set_type` 叶子节点元素集合类型：约束叶子节点存放的`element_set`的类型。

  - `layer_name_type` 层级名称类型：约束`layer_name`的类型。
  - `layer_list` 层级列表：树的层级的顺序列表，约束`layer_name`的范围，同时约束`namespace_tree`的深度。

  - `node_name_type` 节点名称类型：约束`node_name`的数据类型。
  - `node_name_length` 节点名称长度：约束`node_name`的长度。

  - `children_node_count` 子节点数量：约束树中的每个节点的子节点数量。

## 能力申明

  - `find_element_set` 通过 [`layer_name`+`node_name`]找到其下关联的所有的 `element_set`。

  - `add_element` 通过 叶子节点`layer_name`+`node_name`把 `element` 添加到对应的`element_set` 中。

  - `remove_element` 通过 `layer`+`name`+`element`删除指定的指定层级和节点名称其下的`element`。

  - `add_node` 通过 `layer`+`name` 添加一个新的节点到树中指定层。

  - `remove_node` 通过 `layer`+`name` 删除树中指定层级和名称的一个节点，会删除其下关联的所有节点。

### 非职责声明

- 不负责层级具体含义。

# tag索引模块

@framework

## 0.目的
可通过标签信息检索文件键集合。

## 最小基结构

- `trie` 前缀/字典树: 将输入标签名称按位拆成节点名的树，节点上保存元素集合。

## 基排列组合

- `only` 模块内有且仅有一个`trie`

## 边界定义（参数）

### 接口定义

 - `element` 文件键：叶子节点存放的元素。
 - `element_set` 文件键集合：存放`element`的集合。

 - `tag_name` tag名称：tag的名称标识。
 - `tag_name_step_value` tag名称步长值：tag名称每一步的值。 
 - `tag_name_step_value_list` tag名称步长值列表：tag名称每一步值的顺序列表。

### 参数边界/配置边界

 - `element_type` 叶子存放元素类型：约束叶子节点的元素集合中的`element`的类型。

 - `tag_name_type` tag类型：约束`tag_name`的数据类型。
 - `tag_name_step` tag名称步长：约束`tag_name`的每一步的长度。
 - `tag_name_length` tag长度：约束`tag_name`的`tag_name_step`长度，同时约束`trie`的深度。
 - `tag_count` tag数量：约束树中tag的数量。

## 能力申明

 - `find_tag_name_step_value_list` 通过`tag_name_step_value_list`找到其下所有的`tag_name`。

 - `find_element_set` 通过`tag_name`找到到达节点的`element_set`
 - `add_element`通过`tag_name`+`element`添加tag下的元素。
 - `remove_element`通过`tag_name`+`element`删除tag下的元素。

 - `add_tag` 通过`tag_name`添加tag
 - `remove_tag` 通过`tag_name`删除tag

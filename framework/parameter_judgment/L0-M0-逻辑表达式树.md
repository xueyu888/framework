# 逻辑表达式树:LogicCalculateTree

@framework

## 0. 目标 (Goal)

对每一个位置i，把所有序列的第i个元素，按照某个逻辑表达式，算出一个值，最终得到一个新的序列


## 1. 最小可行基（Minimum Viable Bases）

- `B1` 叶子节点：持有输入序列元素的引用
- `B2` 非叶子节点：持有一个逻辑运算符（与/或/非）


## 2. 基组合原则（Base Combination Principles）

- `R1` 子节点组合原则：
  - `R1.1`：`B1` 不允许存在子节点
  - `R1.2`：
    - 持有与/或的 `B2`，有且只能有两个子节点
    - 持有非的 `B2`，有且只能有一个子节点
- `R2` 只允许有一个 `B2` 作为根节点
- `R3` 由 `B2` 依次作用于输入序列的每一个元素，向上遍历树。导出 `C1`


## 3. 能力声明（Capability Statement）

- `C1` 多序列逻辑运算能力：将等长的多个序列按索引依次进行逻辑运算

### 非职责声明（Non-Responsibility Statement）


## 4. 边界定义（Boundary / 参数）

### 4.1 接口定义（IO / Ports）

- `P1` 输入约束：N个等长布尔序列
- `P2` 输出约束：一个布尔序列，长度与输入相同

### 4.2 参数边界（Parameter Constraints）

- `P1` 布尔序列个数：uint64
- `P2` 布尔序列长度：uint64

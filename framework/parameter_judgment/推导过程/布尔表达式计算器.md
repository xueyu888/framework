目标：接收一个合法的布尔表达式字符串和 N 个等长布尔序列，按表达式定义的组合方式，逐位置运算，输出一个等长布尔结果序列。

先在这里定义一个合法的布尔表达式，要求进入的布尔表达式必须符合此标准：
1. 操作数：一个或多个连续字母
2. 运算符：&&（与）、||（或）、!（非）
3. 分组符：(、)
4. 分隔符：空格
5. 优先级：! > && > ||，左结合


推导：
首先根据目标，可以知道计算过程有两个对象：布尔表达式字符串、布尔序列。

布尔表达式字符串：一个布尔运算符（与或非）和操作数的排列，例如：A && B || C
要计算这个表达式，必须引入优先级规则，说明每一个操作数的归属关系，比如　&& 计算的是A和B，|| 计算的是（A&&B）的结果和C，这样才能知道先算谁后算谁
单看字符串这个结构，它只是一个从左到右的排列，并不携带归属关系，所以需要另一个包含归属关系的结构来让这个表达式变得可计算：
1. 每个操作数恰好归属一个运算符。
2. 每个运算符恰好有1~2个操作数。
3. 运算符的计算结果本身可以作为另一个运算符的操作数。
4. 表达式最终算完只有一个结果，而这个结果最终只由一个运算符计算得到，这意味着这个运算符的计算结果不会被作为任何其它运算符的操作数。
5. 每个操作数都参与最终的计算，这意味着从任何一个操作数往上走，都能走到最顶层的运算符。

根据上面的推导，可以导出这样一个结构：
1. 每个元素恰好有一个上级，除了最顶层那个
2. 每个元素有1~2个下级（除了最下层的元素，即操作数）
3. 元素之间不会形成循环结构

进一步推导：
1. 除了最下层的节点，每一个节点都有1~2个子节点
2. 除了最顶层的那一个节点，每一个节点都有1个父节点
3. 节点之间无环

这就是二叉树，同时能进一步推导：
1. 所有的操作数都是叶子节点，所有的运算符都是非叶子节点。
2. 对于与操作符，有两个子节点
3. 对于或操作符，有两个子节点
4. 对于非操作符，有一个子节点

这是二叉树的一个实例，即布尔表达式树


接下来继续推导：
现在已经有了这样的结论：需要一个携带归属关系的结构来表示布尔表达式，这样才能使用表达式进行计算。
而我们的输入是一个布尔表达式字符串，所以需要一个将布尔表达式字符串转换为布尔表达式树的结构。

首先需要考虑的是，在布尔表达式树中，所有的操作数都是叶子节点，所有的运算符都是非叶子节点，那么，需要先从字符串中，分辨出哪些是操作数，哪些是运算符。

进一步，分辨哪些是操作数，哪些是运算符，首先的对字符串进行切分，先保证能分辨出一个一个独立的元素，比如：A、B、C、&&、||等，再进一步，就是划分边界，判断两个相邻的字符是否属于同一个元素

这里存在这样一个前提：合法的布尔表达式中不同类型的元素由不同种类的字符组成——操作数由字母组成，运算符由符号字符组成，分组符是括号，分隔符是空格。种类之间没有交叉。

因此，两个相邻字符是否属于同一个元素，可以通过判断它们是否属于同一种类来确定。这就需要一个预先定义好的对应关系：每个字符属于哪个种类。& 属于符号类，B 属于字母类，空格属于空白类。
这个对应关系的性质：给定一个字符，返回一个确定的类别。一个字符只属于一个类别，不会既是符号又是字母。
一个输入对应一个确定的输出，这就是映射。承载映射的最小结构就是一张表——左边是字符，右边是类别。

将字符串切分成一个个元素还不够，还需要判断每个元素的类别，给它们打上标记，对于空格、括号，它们各自就能组成一个类别：空白类、分组符类。那么剩下的就是运算符类和操作数类。而操作数相对于运算符存在更多的可能性，比如：A、B、C等，而运算符只有&&、||、!这三种。所以只需要判断是运算符还是不是运算符就可以确定元素的类别，这需要这样一个结构：持有一组确定的元素，对于任意给定的元素，能判断它在不在其中。这就是集合


接下来继续推导：
现在每个元素都有了类别，组成了一个线性序列，接下来需要沿着序列遍历，确定元素之间的归属关系——每个操作数归哪个运算符管。

影响归属关系的有两个因素：运算符的优先级和括号。

A && B || C，遍历到 && 时，&& 能不能立刻确定自己的归属关系？不能。因为它的右操作数 B 还没看到，而且即使看到了 B，也不知道 B 是不是该归它——万一后面还有一个优先级更高的运算符要把 B 抢走呢。
所以 && 必须先放在某个地方等着。等什么？等后面出现新的运算符，比较优先级之后，才能决定。
这个"放在某个地方等着"的需求，需要一个结构来承载。什么结构？先看看等待的规律。
假设连续出现多个运算符在等待：A && B || C && D。遍历到第二个 && 时，等待中的有 &&（第一个）和 ||。第二个 && 优先级高于 ||，所以 || 还不能确定归属，继续等。但第一个 && 已经在 || 出现时就处理掉了。
再看括号的情况。A && (B || C)，遍历到 || 时，等待中有 && 和 (。|| 不能和 && 比较，因为中间隔了一个 (。( 是一道墙，把内外隔开。遍历到 ) 时，先把 || 处理掉，然后 ( 移除，&& 重新露出来。
观察这个规律：每次需要处理的，都是最近放进去的那个。&& 先放进去，( 后放进去，处理的时候先处理 ( 上面的，再移除 (，最后才轮到 &&。
最近放进去的最先被处理。这就是后进先出。承载这个行为的结构就是栈。

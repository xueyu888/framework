# 输入接入原子模块:InputAccessAtom

@framework

## 1. 能力声明（Capability Statement）

- `C1` 输入源承接能力：提供对输入对象或输入集合执行稳定承接的能力。
- `C2` 读取规则绑定能力：提供将读取规则绑定到输入源并形成稳定读取结果的能力。
- `C3` 变量归一承载能力：提供将读取结果整理为统一变量格式并完成交接的能力。
- `C4` 非能力项：不负责领域规则判定、业务决策与结果输出。

## 2. 边界定义（Boundary / 参数）

- `SOURCE` 输入源边界：输入对象或输入集合的来源标识、对象语义与可用性条件必须明确。来源：`C1 + C2`。
- `READ_RULE` 读取规则边界：字段、路径、筛选条件、读取顺序或读取目标必须明确。来源：`C2 + C3`。
- `READ_VALUE` 读取值边界：按读取规则得到的结果必须稳定、可校验。来源：`C2 + C3`。
- `VAR_MODEL` 变量模型边界：读取结果进入的变量结构、字段组织与容器格式必须明确。来源：`C3`。
- `ORDER` 顺序边界：输入源顺序、读取顺序与变量承载顺序必须保持一致。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入源承载结构基。来源：`C1 + SOURCE + ORDER`。
- `B2` 读取规则绑定结构基。来源：`C2 + SOURCE + READ_RULE + READ_VALUE + ORDER`。
- `B3` 变量归一承载结构基。来源：`C3 + READ_VALUE + VAR_MODEL + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 源规则组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先由输入源承载结构基确认输入对象或输入集合已稳定可消费，再由读取规则绑定结构基依据读取规则生成稳定读取结果。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SOURCE/READ_RULE/READ_VALUE/ORDER`。
- `R2` 读取承载组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：读取规则绑定结构基产出的读取结果必须经过变量归一承载结构基处理后，才能形成统一变量格式并完成交接。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`READ_RULE/READ_VALUE/VAR_MODEL/ORDER`。
- `R3` 完整输入接入组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将输入源承载、读取规则绑定与变量归一承载收敛为完整输入接入闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SOURCE/READ_RULE/READ_VALUE/VAR_MODEL/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`B2` 绕过 `B1` 直接自造输入源、`B3` 绕过 `B2` 直接自造读取结果、以及 `B3` 反向改写 `B1/B2` 已确定的输入来源、读取规则或顺序关系的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SOURCE/READ_RULE/READ_VALUE/VAR_MODEL/ORDER`。

## 5. 验证（Verification）

- `V1` 输入源承载有效性：输入源承载结构必须满足 `SOURCE/ORDER` 约束。
- `V2` 读取规则绑定一致性：读取规则绑定结构必须满足 `SOURCE/READ_RULE/READ_VALUE/ORDER` 约束。
- `V3` 变量归一承载一致性：变量归一承载结构必须满足 `READ_VALUE/VAR_MODEL/ORDER` 约束。
- `V4` 输入接入闭环完整性：完整输入接入流程必须满足从输入源承载到变量归一承载的闭环约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

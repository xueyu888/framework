# 知识库对话与引用原子模块:KnowledgeConversationCitationAtoms

@framework

本模块定义知识库领域的最小对话与引用结构。推导链为：`framework/frontend/L0-M0-工程与运行底座模块.md` + `framework/frontend/L0-M1-设计令牌与主题模块.md` -> `framework/frontend/L1-M0-基础组件原子模块.md` -> 本模块。消息气泡、输入框、按钮、标签与折叠面板等通用前端结构由前端基础组件原子承接，本模块只定义知识库领域的回合、引用与引用域关系。

## 1. 能力声明（Capability Statement）

- `C1` 对话序列承载能力：把用户回合、助手回合与回合顺序抽象为稳定结构。
- `C2` 引用附着承载能力：把回合与文件片段、章节锚点、引用说明之间的关系抽象为稳定结构。
- `C3` 引用域约束能力：把当前允许引用的文件集合与回合上下文外化为稳定边界。
- `C4` 非能力项：不负责文件摄取、文档解析与模型推理实现。

## 2. 边界定义（Boundary / 参数）

- `TURN` 回合边界：用户回合、助手回合与时间顺序需稳定。来源：`C1 + C3`。
- `INPUT` 输入边界：当前输入草稿、提交出口与清空规则需稳定。来源：`C1 + C3`。
- `CITATION` 引用边界：引用文件、章节锚点、片段与说明字段需完整。来源：`C2 + C3`。
- `SCOPE` 引用域边界：当前允许引用的文件集合与默认域需可区分。来源：`C2 + C3`。
- `STATUS` 状态边界：生成中、成功与失败回合需有稳定承载结构。来源：`C1 + C2`。
- `A11Y` 可访问边界：消息阅读、输入提交与引用展开需可达。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 对话序列结构基：frontend.L1.M0[R2,R3]。来源：`C1 + TURN + STATUS`。
- `B2` 输入提交结构基：frontend.L1.M0[R1,R2]。来源：`C1 + INPUT + A11Y`。
- `B3` 引用附着结构基：frontend.L1.M0[R1,R3]。来源：`C2 + C3 + CITATION + SCOPE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 回合先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定回合顺序与输入提交结构，再允许引用附着挂接到回合上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TURN/INPUT/STATUS/A11Y`。
- `R2` 引用附着并轨
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将引用文件、章节锚点与片段说明附着到助手回合上，形成稳定可回溯关系。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CITATION/SCOPE/TURN/STATUS`。
- `R3` 对话引用闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合回合顺序、输入提交和引用附着结构，形成可提交、可引用、可回看对话结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`TURN/INPUT/CITATION/SCOPE/STATUS/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失输入出口、引用不附着到稳定回合或引用域不可解释的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TURN/INPUT/CITATION/SCOPE/STATUS/A11Y`。

## 5. 验证（Verification）

- `V1` 回合完整性：用户回合、助手回合与顺序结构必须完整。
- `V2` 输入完整性：输入草稿与提交出口必须满足 `INPUT` 约束。
- `V3` 引用有效性：引用条目必须满足 `CITATION/SCOPE` 约束并能附着到稳定回合。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。

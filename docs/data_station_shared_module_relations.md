# 数据站共享模块调用关系

## 1. 目的
本文件用于承载数据站功能模块与共享横切模块之间的调用关系。

约束：
- 功能模块文档只描述自身结构、输入输出与局部组合规则。
- 权限治理、状态管理、追溯管理等横切能力单独成模块文档。
- 功能模块对共享模块的调用关系统一在本文件维护，不再回写到各功能模块正文。

## 2. 共享模块清单
- `L1-M1` 状态管理模块：负责共享对象状态登记、合法流转、版本一致性与幂等控制。
- `L1-M2` 追溯管理模块：负责共享事件采集、字段归一、记录持久化与引用输出。
- `L1-M3` 权限治理模块：负责主体/动作/资源/上下文解析、鉴权判定与范围治理。

## 3. 当前功能拆分
- `L1-M0` 接入入口模块由 `L2-M0` 前端上传交互模块、`L2-M1` 后端接入服务模块、`L2-M2` 上传契约模块共同构成。
## 4. 当前调用关系

| 功能模块 | 调用共享模块 | 调用目的 |
| --- | --- | --- |
| `L2-M1` 后端接入服务模块 | `L1-M1` 状态管理模块 | 对上传接入结果执行初始状态登记与后续状态推进 |
| `L2-M1` 后端接入服务模块 | `L1-M2` 追溯管理模块 | 对上传接入关键事件写入统一追溯记录 |
| `L2-M1` 后端接入服务模块 | `L1-M3` 权限治理模块 | 对上传请求执行统一鉴权判定 |

## 5. 预留调用关系
- 后续 `处理审核模块` 预计调用 `L1-M1/L1-M2/L1-M3`。
- 后续 `展示检索模块` 至少预计调用 `L1-M1/L1-M3`；若存在导出、共享、复核动作，则也应调用 `L1-M2`。

## 6. 维护规则
- 当新增共享模块时，必须同步补充“共享模块清单”和“调用关系”。
- 当功能模块新增或删除对共享模块的依赖时，必须同步更新本文件。
- 若共享模块被功能模块内联实现，必须在评审中说明理由，并重新评估是否违反“横切能力独立”约束。

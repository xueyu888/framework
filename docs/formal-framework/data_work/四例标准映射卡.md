# data_work 四例标准映射卡

本文用于对 `framework/data_work/` 的 `L1-M0 ~ L1-M4` 做回归压测。

- 用途：每次调整 `data_work` 的模块边界、能力声明、最小结构基或组合原则后，用这 4 个例子检查结构是否仍然自然成立。
- 判定原则：先判断“根语义属于哪个 L1”，再判断内部是否需要组合其他 `L1` 作为子单元；不要反过来让子单元语义吞掉根单元语义。
- 通过标准：例子应能在不扭曲模块职责的前提下自然落位；若必须靠“把 A 写成 B”才能成立，则说明边界仍未收敛。

## 1. 文本输入框

- 根模块：`L1-M1 输入单元模块`
- 根语义：用户在明确输入位置中录入内容，并形成、保持、更新当前值。
- 结构映射：`输入内容承载基 + 输入操作状态基 + 当前值基`
- 典型子单元：
  - 标签、说明、placeholder、字符计数：可作为展示内容组合到输入单元周边，但不改变根语义仍是“输入”。
  - 校验错误提示、提交后结果消息：若表达结果反馈，应作为反馈子单元，不应写回输入根语义。
- 通过条件：
  - 自由输入内容能形成当前值并持续更新。
  - `正常 / 禁用 / 只读 / 报错` 能被解释为输入状态，而不是反馈结果。
  - 静态文案不会把输入单元误降格为展示单元。
- 回归警戒：
  - 不要让“有展示内容”推导出“输入单元依赖展示单元作为最小结构基”。

## 2. 单选组 / 多选组

- 根模块：`L1-M2 选择单元模块`
- 根语义：多个候选对象在当前边界中形成、保持并改变选择关系。
- 结构映射：`候选对象承载基 + 选择状态基 + 选择关系基`
- 典型子单元：
  - 每个候选项的标题、说明、图标：展示子单元。
  - “请选择一项”这类提示文案：展示子单元。
  - 若带搜索框筛选候选项，应理解为“输入单元 + 选择单元”的组合，而不是把搜索直接并入选择根语义。
- 通过条件：
  - 单选可表达互斥关系；多选可表达并存、增加、减少、取消。
  - 候选对象始终先被承载，再参与选择关系。
  - 选择关系变化不会被偷换成“值更新”来解释。
- 回归警戒：
  - 当前版本已收敛为“选择状态”；若未来又想把禁用、隐藏等状态拉回选择单元，必须同步说明它们由哪个状态维度承接，而不是只在能力声明里扩写。
  - 不要默认把“选中结果”提升成值基；只有当框架明确需要一等导出 `selected value` 时，才考虑新增值维度。

## 3. 按钮

- 根模块：`L1-M3 触发单元模块`
- 根语义：明确对象在当前边界中可发起一个动作，并进入、维持、结束一次触发过程。
- 结构映射：`触发对象承载基 + 触发操作状态基 + 触发过程基`
- 典型子单元：
  - 按钮文案、图标：展示子单元。
  - 点击后的 loading spinner：可以是过程阶段的视觉表达，但根语义仍是触发过程，不是反馈根单元。
  - “确定 / 取消 / 关闭”按钮：多个触发单元并列，不应混成反馈单元。
- 通过条件：
  - `可触发 / 禁用` 能作为操作状态成立。
  - `开始触发 / 处理中 / 完成 / 取消 / 关闭` 能作为触发过程成立。
  - 根语义保持为“发起动作”，不会退化成纯展示或纯反馈。
- 回归警戒：
  - 当前版本已收敛为“触发状态”；若未来又想把视觉状态或反馈结果并回触发单元，必须先判断它属于触发过程，还是应交给展示/反馈子单元。

## 4. Toast / Alert

- 根模块：`L1-M4 反馈单元模块`
- 根语义：明确反馈对象进入某个反馈阶段，并在阶段内维持，直至满足结束条件。
- 结构映射：`反馈对象承载基 + 反馈阶段基`
- 典型子单元：
  - 标题、正文、图标：展示子单元。
  - 右上角关闭按钮、底部确认按钮：触发子单元。
  - 若反馈对象中包含可编辑区或可选区，应视为反馈根单元内部再组合输入/选择子单元，而不是把反馈写成输入或选择。
- 通过条件：
  - `处理中 / 成功 / 失败 / 警告 / 提示` 能作为反馈阶段被识别。
  - `关闭 / 消失 / 确认 / 撤销 / 复位` 能作为反馈过程的结束条件被识别。
  - 反馈单元不会吞掉展示职责和触发职责，只在语义上统领它们。
- 回归警戒：
  - 如果需要进一步区分“反馈阶段”和“视觉呈现”，可以后续补充状态维度；但在此之前，不要把反馈重新写成展示单元。

## 5. 使用方式

- 每次修改 `framework/data_work/` 后，逐个问这 4 个问题：
  - 这个例子的根语义还能否直接落到一个明确的 `L1`？
  - 这个 `L1` 的 `B*` 是否足以支撑它宣称的能力？
  - 哪些内容是根语义，哪些只是子单元组合？
  - 是否出现“为了让例子成立，不得不把 A 写成 B”的情况？
- 若任一问题回答为“不能自然成立”，应优先回到 framework 边界层继续收敛，而不是先去改 config 或 code 层规避。

# data_work L1 一致性表

本文用于固定 `framework/data_work/` 当前版本的 `L1` 骨架，作为回归时的快速检查表。

| L1 模块 | 根语义 | 最小结构基 | 状态维度 | 值维度 | 常见子单元组合 | 最容易越界到 |
| --- | --- | --- | --- | --- | --- | --- |
| 展示单元 | 明确内容在界面中被承载、组织并呈现 | `展示内容承载基 + 视觉呈现状态基` | 视觉状态 | 无 | 可作为输入/选择/触发/反馈内部的展示子单元 | 输入、反馈 |
| 输入单元 | 明确输入内容形成、保持、更新当前值 | `输入内容承载基 + 输入操作状态基 + 当前值基` | 输入操作状态 | 有，一等值维度 | 可组合展示子单元、反馈子单元 | 展示、反馈 |
| 选择单元 | 候选对象形成、保持、改变选择关系 | `候选对象承载基 + 选择状态基 + 选择关系基` | 选择状态 | 默认无 | 可组合展示子单元；带筛选时可组合输入子单元 | 展示、值 |
| 触发单元 | 触发对象发起、维持、结束一次触发过程 | `触发对象承载基 + 触发操作状态基 + 触发过程基` | 触发对象的可区分工作状态 | 无 | 可组合展示子单元；过程中的视觉表达仍不改变根语义 | 展示、反馈 |
| 反馈单元 | 反馈对象进入、维持、结束反馈阶段 | `反馈对象承载基 + 反馈阶段基` | 反馈阶段 | 无 | 可组合展示子单元、触发子单元，必要时再组合输入/选择子单元 | 展示、触发 |

## 1. 当前固定判断

- `展示单元` 是平级模块，不再作为其他 `L1` 的最小结构基来源。
- `输入单元` 是当前唯一明确需要一等值维度的 `L1`。
- `选择单元` 当前只承接“选择状态”，不默认承接一般显示状态或交互状态。
- `触发单元` 当前只承接触发对象的可区分工作状态与触发过程，不默认吞并反馈结果。
- `反馈单元` 当前按“过程阶段”建模，而不是按“纯视觉提示”建模。

## 2. 使用方式

- 每次修改 `framework/data_work/L1-M0 ~ L1-M4` 后，先用本表检查根语义与最小结构基是否仍然一致。
- 若某个例子必须通过“把根语义改写成兄弟模块语义”才能落位，说明边界发生了回退。
- 若某个模块开始稳定复用新的跨模块原语，再考虑是否把该原语上提到新的 `L0`，不要提前抽象。

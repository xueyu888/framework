# 子任务状态记录表

以子任务 id 作为索引, 建立的 b+ 树结构:

| 子任务 id | 对应父任务id | 子任务状态 | 是否已完成过 | 子任务关心的其他信息(必须和子任务id一一对应) |
| :-------: | :----------: | :--------: | :----------: | :------------------------------------------: |
|   st123   |    t1234     |   已完成   |      是      |             如时间, 任务内容...              |

在删除或查询父任务的所有子任务时, 还需要建立一个以(父任务id+子任务id)为索引的b+树:

| 对应父任务id | 子任务id | 子任务状态 | 是否已完成过 | 子任务关心的其他信息(必须和子任务id一一对应) |
| :----------: | :------: | :--------: | :----------: | :------------------------------------------: |
|    t1234     |  st123   |   已完成   |      是      |             如时间, 任务内容...              |

这里的"是否已完成过"表达的是历史事实, 不是"本次是否首次触发完成"这个事件判断。

总言之, 逻辑上只有一张表, 物理上存在两棵树

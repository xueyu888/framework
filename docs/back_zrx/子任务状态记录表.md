# 子任务状态记录表

以子任务 id 作为索引, 建立的 b+ 树结构:

| 子任务 id | 对应父任务id | 子任务状态 | 是否初次完成 | 子任务关心的其他信息(必须和子任务id一一对应) |
| :-------: | :----------: | :--------: | :----------: | :------------------------------------------: |
|   st123   |    t1234     |   待审核   |      是      |             如时间, 任务内容...              |

在删除时, 由于需要用父任务id查询其拥有的所有子任务, 所以还需要建立一个以(父任务id+子任务id)为索引的b+树:

| 对应父任务id | 子任务id | 子任务状态 | 是否初次完成 | 子任务关心的其他信息(必须和子任务id一一对应) |
| :----------: | :------: | :--------: | :----------: | :------------------------------------------: |
|    t1234     |  st123   |   待审核   |      是      |             如时间, 任务内容...              |

总言之, 逻辑上只有一张表, 物理上存在两棵树

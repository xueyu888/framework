1. 检查此条子任务的三个属性, 并根据其状态的排列组合进行不同的操作

需要检查的属性如下:
- A: 此条子任务是否已存在记录表中?
- B: 此次传入的完成状态是否是"已完成"?
- C: 已有记录的历史完成状态是否已是"已完成"?

组合原则:
1. 子任务不在记录表中时, 不考虑历史完成状态
2. 已完成的子任务不可被更新为未完成状态
3. 仅当历史完成状态不是"已完成", 且本次传入状态是"已完成"时, 才视为"本次首次触发完成"

共有 8 种情况:
- A 是:
    - B 是:
        - C 是:
            说明此前已完成过, 本次不是首次触发完成, 更像是更新补充信息, 故需要:
            - 更新子任务关心的其他信息
            - `SUCCESS OUTPUT`
        - C 否:
            说明此前未完成, 本次首次触发完成, 故需要: 
            在子任务状态记录表中:
            - 更新子任务关心的其他信息
            - 更新子任务状态为"已完成"
            - 更新历史完成状态为"已完成"
            在父任务状态记录表中:
            - 对应父任务的已完成子任务数增加 1
            - 父任务的已完成子任务数等于父任务的总子任务数?
                - 是:
                    更新父任务状态为"已完成"
                    - `SUCCESS OUTPUT`
                - 否:
                    - `SUCCESS OUTPUT`
    - B 否:
        - C 是:
            不允许的状态, 历史上已经完成过, 但现在又变成未完成了
            - `ERROR OUTPUT`
        - C 否:
            说明此任务仍没有完成, 故需要:
            - 更新子任务的状态为此次传入的参数
            - 更新子任务关心的其他信息
            - `SUCCESS OUTPUT`
- A 否:
    - B 是:
        - C 是:
            不可能, 之前没有记录时, 历史完成状态不可能已经是"已完成"
            - `ERROR OUTPUT`
        - C 否:
            说明此前没有记录, 本次直接首次触发完成, 故需要: 
            在子任务状态记录表中:
            - 更新子任务关心的其他信息
            - 更新子任务状态为"已完成"
            - 更新历史完成状态为"已完成"
            在父任务状态记录表中:
            - 对应父任务的已完成子任务数增加 1
            - 父任务的已完成子任务数等于父任务的总子任务数?
                - 是:
                    更新父任务状态为"已完成"
                    - `SUCCESS OUTPUT`
                - 否:
                    - `SUCCESS OUTPUT`
    - B 否:
        - C 是:
            不可能, 之前没有记录时, 历史完成状态不可能已经是"已完成"
            - `ERROR OUTPUT`
        - C 否:
            说明此任务还没有完成, 但想要记录信息故需要:
            - 将此次传入的参数插入到子任务状态记录表中
            - `SUCCESS OUTPUT`

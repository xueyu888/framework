1. 检查此条子任务的三个属性, 并根据其状态的排列组合进行不同的操作

需要检查的属性如下:
- A: 此条子任务是否已存在记录表中?
- B: 此次传入的完成状态是否是"已完成"?
- C: 已有记录的初次完成状态是否是"是"?

共有 8 种情况:
- A 是:
    - B 是:
        - C 是:
            说明此前已完成过了, 此次传入可能是想更新信息, 故需要:
            - 更新子任务关心的其他信息
            - `SUCCESS OUTPUT`
        - C 否:
            说明此前未完成, 本次是状态转换为完成, 故需要: 
            在子任务状态记录表中:
            - 更新子任务关心的其他信息
            - 更新子任务状态为"已完成"
            在父任务状态记录表中:
            - 对应父任务的已完成子任务数增加 1
            - 父任务的已完成子任务数等于父任务的总子任务数?
                - 是:
                    更新父任务状态为"已完成"
                    - `SUCCESS OUTPUT`
                - 否:
                    - `SUCCESS OUTPUT`
    - B 否:
        - C 是:
            不允许的状态, 之前是已完成, 但现在又变成未完成了
            - `ERROR OUTPUT`
        - C 否:
            说明此任务仍没有完成, 故需要:
            - 更新子任务的状态为此次传入的参数
            - 更新子任务关心的其他信息
            - `SUCCESS OUTPUT`
- A 否:
    - B 是:
        - C 是:
            不可能, 之前没有记录一定不可能状态是已完成
            - `ERROR OUTPUT`
        - C 否:
            说明此前没有记录, 本次直接完成了, 故需要: 
            在子任务状态记录表中:
            - 更新子任务关心的其他信息
            - 更新子任务状态为"已完成"`
            在父任务状态记录表中:
            - 对应父任务的已完成子任务数增加 1
            - 父任务的已完成子任务数等于父任务的总子任务数?
                - 是:
                    更新父任务状态为"已完成"
                    - `SUCCESS OUTPUT`
                - 否:
                    - `SUCCESS OUTPUT`
    - B 否:
        - C 是:
            不可能, 之前没有记录一定不可能状态是待审核
            - `ERROR OUTPUT`
        - C 否:
            说明此任务还没有完成, 但想要记录信息故需要:
            - 将此次传入的参数插入到子任务状态记录表中
            - `SUCCESS OUTPUT`

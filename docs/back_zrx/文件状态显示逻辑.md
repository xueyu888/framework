1. 检查三个模块中文件 id 的存在情况:

- A: 此文件 id 是否在已归档模块中?
- B: 此文件 id 是否在父子任务状态记录模块中?
- C: 此文件 id 是否在文件块存储模块块中?

共有 8 种情况:
- A 是:
    - B 是:
        - C 是:
            异常情况, 已归档的文件既不会出现在文件块存储模块, 也不会出现在父子任务状态记录模块中, 故需要:
            - `ERROR OUTPUT`
        - C 否:
            异常情况, 已归档的文件不会出现在文件块存储模块中, 故需要:
            - `ERROR OUTPUT`
    - B 否:
        - C 是:
            异常情况, 已归档的文件不会出现在文件块存储模块中, 故需要:
            - `ERROR OUTPUT`
        - C 否:
            说明此文件已归档了, 故返回已归档状态.
            - `已归档`
- A 否:
    - B 是:
        - C 是:
            说明此文件正在审核或刚刚审核完成, 还没有进行归档, 这两个状态在父子任务状态记录模块中是存在记录的, 故返回父子任务状态记录模块中的文件状态:
            - `正在审核/已审核完毕`
        - C 否:
            异常情况, 归档一定需要同时清除父子任务状态记录模块和文件块存储模块中的记录, 故需要:
            - `ERROR OUTPUT`
    - B 否:
        - C 是:
            说明此文件刚清洗完, 还没有开始审核, 故返回:
            - `已清洗完待审核`
        - C 否:
            异常情况,三个地方都不存在, 说明此文件不存在, 故需要:
            - `ERROR OUTPUT`

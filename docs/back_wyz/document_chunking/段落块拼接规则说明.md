# 段落块拼接规则说明

本文件不是 `framework` 模块，而是对 `framework/document_chunking/L1-M0-文档分块.md` 中 `B4` 的补充说明。

## 说明目的

- 解释段落块如何从“已判定角色的有序段落块序列”组合成文本块。
- 为人工讨论和后续实现提供更展开的规则描述。
- 不承担框架层的 `Lx-Mn` 编号、组合闭环和验证编号责任。

## 核心规则

- 输入对象必须是已经完成 title/body 角色判型、且顺序稳定的段落块序列。
- 每个文本块必须由且仅由一个标题块作为起点。
- 一个标题块之后可以吸收零个或多个连续正文块。
- 当遇到下一个标题块或序列结束时，当前文本块闭合。
- 任一段落块不得同时归属于多个文本块。
- 不允许逆序吸收、跳序吸收、无标题起点成组或多个标题并入同一文本块。

## 推荐输出信息

- 文本块 ID
- 标题块 ID
- 正文块 ID 集合
- 起止顺序范围
- 闭合原因

## 与框架文件的关系

- 结构性定义以 `L1-M0` 中的 `B4`、`R3`、`V4` 为准。
- 本文件只负责展开说明，不单独参与框架 lint、层级引用图和生成闭环。

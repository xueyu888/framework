# 置物架框架标准（L2）

## 0. 规范关系
本文件为置物架领域标准，必须遵循：
- `standards/L0/规范总纲与树形结构.md`
- `standards/L1/框架设计核心标准.md`
- `standards/L1/可追溯性标准.md`
- `standards/L1/可删减性标准.md`
本文件位于 `L2`，承接 `L1` 方法标准并向 `L3` 实现层传导。

## 1. 目标（Goal）
定义可拆卸组装的开放式立体堆叠装置，以提升单位占地面积下的存取效率。

## 2. 边界定义（Boundary）
- `N`：层数上限（Boundary.max_layers）
- `W`：单层承重上限（Boundary.max_payload_weight_per_layer）
- `S`：单层可容纳三维空间上限（Boundary.max_volume_space_per_layer: [L, W, H]）
- `A`：总占地面积投影上限（Boundary.max_footprint_area）

边界收敛约定（用于固定占地模式）：
- 当目标明确为“固定占地构型”时，`A` 从“上限”收敛为“目标固定值”。
- 在该模式下，底层占地必须满足 `base_footprint == A_target`。
- 网格离散采用“近方形包络网格”：`grid_cells >= A_target`，并在该网格内枚举连通的 `A_target` 底层占地图形（而非强制 `A` 被整除成 `1xA` 线形网格）。
- 允许边界掩码（Boundary Mask）对网格单元做启用/取消，用于观察“去掉部分边界后”可形成的组合。

## 3. 模块（最小可行基，Module）
- `M1` 支撑柱（Module.PILLAR）：定义高度边界，作为对抗重力的主承力通道。
- `M2` 隔板（Module.PANEL）：切分 Z 轴空间，提供水平承载面，并将载荷分布传递给连接点。
- `M3` 标准接口（Module.CONNECTOR）：定义组装契约，负责 M1 与 M2 之间的力学传导与动态锚定。

## 4. 组合原则（Combination Principles）
- `C1` 模块契约（Contract）：当组合包含承托面 `M2` 时，必须同时包含 `M1` 与 `M3`，并且 `len(combo) >= 2`。
- `C2` 支撑连续性（Support Continuity）：承托单元通过立柱/接口形成垂向承载路径，且各层平面占用需满足“接口可桥接连通”（含边接触与角接触的连接拓扑）；不强制“上层单元必须与下一层同位重叠”。
- `C3` 投影稳定性（Projection Stability）：承托面保持水平，系统整体重心垂线投影必须落于占地 `A` 内部区域。
- `C4` 层完整性（Layer Completeness）：在多层模式（`N > 1`）下，每一层都必须至少激活一个承托单元；禁止出现声明多层但某层为空的分型。


## 5. 验证（Verification）
通过条件：
- 边界有效
- 组合属于有效组合集
- 结构原则满足 `C1/C2/C3/C4`
- `target_efficiency > baseline_efficiency`

效率口径约定：
- 当 `B2`（固定占地）启用时，效率按目标占地 `A_target` 计算。
- 当 `B2` 关闭时，效率按组合实际底层占地 `A_actual` 计算（用于观察放宽固定占地后的真实效率变化）。

规则实验室（Rule Lab）语义：
- 关闭某条组合原则后，`有效组合集` 必须按“当前启用规则集合”重新生成，并据此重算分组、组合列表与 3D 浏览范围。
- 关闭规则不等于“仅在验算阶段忽略失败项”；必须改变候选通过集合（在同一边界下通常会出现更多可浏览分型）。

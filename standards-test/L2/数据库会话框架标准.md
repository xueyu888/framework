


# 置物架框架标准（L2）

## 0. 规范关系
本文件为置物架领域标准，必须遵循：
- `standards/L0/规范总纲与树形结构.md`
- `standards/L1/框架设计核心标准.md`
- `standards/L1/可追溯性标准.md`
- `standards/L1/可删减性标准.md`
本文件位于 `L2`，承接 `L1` 方法标准并向 `L3` 实现层传导。

## 1. 目标（Goal）
实现同步/异步数据库会话的获取与释放

## 2. 边界定义（Boundary）
- `B1`: 操作类型（Operation Type）：定义数据库会话的操作类型，包括同步操作与异步操作
- `B2`: 连接池策略（Connection Pool Strategy）：pooled | nullpool
- `B3`: 会话行为策略（Session Behavior Strategy）：定义会话在获取与释放时的行为，包括阻塞与非阻塞模式

## 3. 模块（最小可行基，Module）

- `M1` SessionSpec：输入模型、默认值、合法性校验、错误类型定义。
- `M2` EngineSessionCore：按输入创建/缓存/销毁 engine 与 session factory，管理资源所有权。
- `M3` SessionGateway：统一对外 API，调用方只通过网关拿会话，不直接接触 engine。

## 4. 组合原则（Combination Principles）

- `C1` 结构顺序固定：M1 -> M2 -> M3。
- `C2` 资源所有权固定：engine 生命周期只归 M2 管。
- `C3` 接口隔离：M3 不得自行创建/销毁 engine。
- `C4` 输入前置：任何会话获取前必须先通过 M1 校验。
- `C5` 防爆炸约束：mode/pool_strategy 是输入参数，不新增组合模块维度。
- `C6` 任一规则不满足即判定组合 invalid。

## 5. 验证（Verification）

目标达成验证：逐条验证 G1~G4，确认对外能力真实可用。
边界符合验证：逐条验证 B1~B5，确认合法输入可执行、非法输入被拦截。
组合规则验证：验证 M1 -> M2 -> M3 的结构顺序与职责边界未被破坏。
生命周期验证：验证会话与引擎资源在正常和异常路径下都能正确释放。

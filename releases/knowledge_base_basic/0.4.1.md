## 中文说明

本次发布对应 `knowledge_base_basic` 样例完成最终 runtime/export-driven 收口后的版本。

当前主链是：

`framework markdown -> package registry -> project.toml -> config slicing -> package compile -> runtime exports -> runtime assembly -> canonical evidence`

本次变更摘要：

- runtime 主链改成由 package compile results 和 child export graph 自然收敛
- 删除旧的知识库专属 builder / projection / app 主路径
- `knowledge_base_workbench` 不再作为 runtime 主脑，只保留为普通元数据语义
- `runtime_snapshot.py`、`derived_governance_manifest.json`、`derived_governance_tree.json` 成为当前命名

对使用者可见的影响：

- 默认服务入口仍是知识库工作台，但装配来源已经改为统一 package exports
- 生成物、治理视图与 discovery audit 现在都与 canonical-first 主链一致

安装或获取方式：

1. 获取本目录下的 `knowledge-base-basic-0.4.1.tar.gz`
2. 解压后进入目录
3. 执行 `uv sync`
4. 执行 `uv run python src/main.py serve`
5. 浏览器访问 `http://127.0.0.1:8000/knowledge-base`

注意事项：

- 若你修改了 framework markdown 或 `projects/<project_id>/project.toml`，必须重新执行 `uv run python scripts/materialize_project.py`

## English Notes

This release corresponds to the finalized `knowledge_base_basic` sample after the runtime/export-driven rewrite was fully closed.

The active chain is:

`framework markdown -> package registry -> project.toml -> config slicing -> package compile -> runtime exports -> runtime assembly -> canonical evidence`

Summary:

- The runtime mainline now converges from package compile results and child export graphs
- The old knowledge-base-specific builder / projection / app main path was removed
- `knowledge_base_workbench` no longer acts as the runtime control center and remains only as ordinary metadata
- `runtime_snapshot.py`, `derived_governance_manifest.json`, and `derived_governance_tree.json` are the current artifact names

User-visible impact:

- The default served experience is still the knowledge-base workbench, but assembly now comes from unified package exports
- Generated artifacts, governance views, and the discovery audit now match the canonical-first mainline

Install / access:

1. Download `knowledge-base-basic-0.4.1.tar.gz` from this directory
2. Extract the archive and enter the folder
3. Run `uv sync`
4. Run `uv run python src/main.py serve`
5. Open `http://127.0.0.1:8000/knowledge-base`

Notes:

- If you change framework markdown or `projects/<project_id>/project.toml`, rerun `uv run python scripts/materialize_project.py`

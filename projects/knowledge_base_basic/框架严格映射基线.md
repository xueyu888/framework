# knowledge_base_basic 框架严格映射基线

## 1. 目的

本文件只回答一个问题：

> `knowledge_base_basic` 现在是如何沿着 `Framework -> Product -> Implementation -> Code -> Evidence` 严格收敛，并且把关键代码对象反查回上游来源的。

这里不再把多份生成文件并列描述为平行真相源。当前知识库主链只有一个机器真相源：

- `generated/canonical_graph.json`

其它生成文件都是派生视图。

## 2. 当前主链

```text
framework/*.md
  -> FrameworkModule / FrameworkBase / FrameworkRule
product_spec.toml + product_spec/*.toml
  -> Product Module
implementation_config.toml
  -> Implementation Module
      -> Code Module
          -> runtime symbols
          -> canonical_graph.json
              -> framework_ir.json
              -> product_spec.json
              -> implementation_bundle.py
              -> generation_manifest.json
              -> governance_manifest.json
              -> governance_tree.json
              -> strict_zone_report.json
              -> object_coverage_report.json
```

关键入口：

- Framework 解析：[src/framework_ir/](../../src/framework_ir/)
- 项目编译：[knowledge_base.py](../../src/project_runtime/knowledge_base.py)
- 前端 contract：[contracts.py](../../src/frontend_kernel/contracts.py)
- 工作台 contract：[workbench.py](../../src/knowledge_base_framework/workbench.py)
- 运行时代码：[app.py](../../src/knowledge_base_runtime/app.py)、[frontend.py](../../src/knowledge_base_runtime/frontend.py)、[backend.py](../../src/knowledge_base_runtime/backend.py)
- 治理与反查：[governance.py](../../src/project_runtime/governance.py)

## 3. Framework -> Product

知识库项目当前直接依赖三份 framework markdown：

- [前端框架标准模块](../../framework/frontend/L2-M0-前端框架标准模块.md)
- [知识库工作台场景模块](../../framework/knowledge_base/L2-M0-知识库工作台场景模块.md)
- [知识库接口框架标准模块](../../framework/backend/L2-M0-知识库接口框架标准模块.md)

这三份文档会被解析成 `FrameworkModule`，模块内部显式包含：

- `capabilities`
- `boundaries`
- `bases`
- `rules`
- `verifications`

随后 Product 层只消费这些模块导出面，把它们具体化到：

- [product_spec.toml](./product_spec.toml)
- [product_spec/](./product_spec)

当前已稳定对齐的产品 section：

- `surface / visual / route / a11y`
- `library / preview / chat / context / return`
- `[[documents]]`

## 4. Product -> Implementation

Implementation 层只消费 Product 层导出，不改写产品真相。

来源：

- [implementation_config.toml](./implementation_config.toml)

当前实现细化只收敛四类决策：

- `frontend`
- `backend`
- `evidence`
- `artifacts`

其中 `[artifacts]` 负责声明：

- 哪个文件名是 canonical source
- 哪些文件名是派生视图

## 5. Implementation -> Code

`src/project_runtime/knowledge_base.py` 会把上游三层编译成分层模块：

- `KnowledgeBaseFrameworkLayer`
- `KnowledgeBaseProductModule`
- `KnowledgeBaseImplementationModule`
- `KnowledgeBaseCodeModule`
- `KnowledgeBaseEvidenceModule`

Code 层当前稳定导出的关键对象包括：

- `frontend_contract`
- `workbench_contract`
- `ui_spec`
- `backend_spec`
- `product_view`
- `public_summary`
- `implementation_effects`

运行时代码只消费这些 Code 层导出，不再自行推导 Product / Implementation 真相。

## 6. Code -> Evidence

Evidence 层当前唯一机器真相源：

- `generated/canonical_graph.json`

它显式分成五层：

- `layers.framework`
- `layers.product`
- `layers.implementation`
- `layers.code`
- `layers.evidence`

其它生成文件的定位如下：

| 文件 | 定位 |
| --- | --- |
| `framework_ir.json` | Framework 层派生视图 |
| `product_spec.json` | Product 层派生视图 |
| `implementation_bundle.py` | Code 层导入友好视图 |
| `generation_manifest.json` | 生成纪律与 effect 视图 |
| `governance_manifest.json` | 高风险结构对象治理视图 |
| `governance_tree.json` | GUI/树状追溯视图 |
| `strict_zone_report.json` | strict zone 派生视图 |
| `object_coverage_report.json` | 结构覆盖派生视图 |

## 7. 关键代码对象与上游来源

当前知识库主链的高风险对象，已经通过治理链绑定回上游：

| Symbol | 代码落点 | 上游来源 |
| --- | --- | --- |
| `kb.runtime.page_routes` | `src/knowledge_base_runtime/app.py` | framework + product.route + implementation.evidence |
| `kb.frontend.surface_contract` | `src/frontend_kernel/contracts.py` | frontend framework + product.surface/route/a11y |
| `kb.workbench.surface_contract` | `src/knowledge_base_framework/workbench.py` | knowledge_base framework + product.library/preview/chat/context/return |
| `kb.ui.surface_spec` | `src/project_runtime/knowledge_base.py` | framework + product + implementation.frontend |
| `kb.backend.surface_spec` | `src/project_runtime/knowledge_base.py` | framework + product + implementation.backend |
| `kb.api.library_contracts` | `src/knowledge_base_runtime/backend.py` | backend framework + product.library/preview/route |
| `kb.api.chat_contract` | `src/knowledge_base_runtime/backend.py` | backend framework + product.chat/context/return/route |
| `kb.answer.behavior` | `src/knowledge_base_runtime/backend.py` | knowledge_base framework + product.chat/context/return + implementation.backend |

## 8. 当前结论

`knowledge_base_basic` 当前已经形成了严格的一条链：

- Framework 是 Markdown-first 作者源
- Product 只具体化 framework 能力空间
- Implementation 只细化实现路径
- Code 只消费 Implementation 导出
- Evidence 只消费 Code 导出

机器侧只有一个 canonical source：

- `generated/canonical_graph.json`

这就是当前 knowledge_base_basic 的严格映射基线。

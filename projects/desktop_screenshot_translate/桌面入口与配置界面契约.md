# AiTrans 桌面入口与配置界面契约

## 1. 文档职责

本文件回答 `AiTrans` 在当前项目实例中如何交付托盘入口、快捷键降级提示与首次配置界面。

它不替代：

- `framework/aitrans/L2-M0-桌面宿主层.md` 的领域共性宿主语义
- `framework/frontend/*.md` 的通用前端结构语言
- `implementation_config.toml` 中关于宿主、桥接、配置持久化与发布的技术选型

它的作用是把“托盘入口”“设置界面”“权限与降级引导”这些能力，编译成当前项目可直接实现的前端和宿主契约。

## 2. 托盘入口目标

- 托盘是系统级入口，不是应用主页。
- 托盘首要职责是承接“开始截图”和“进入设置”，而不是展示调试能力或派生结果页。
- 托盘菜单必须短、稳、低认知负担。
- 托盘文案必须偏产品语义，而不是偏实现或排障语义。

## 3. 托盘菜单契约

### 3.1 顶级动作

- 开始截图翻译
- 设置与连接
- 检查更新
- 安装已下载更新（仅在有已下载更新时启用）
- 退出

### 3.2 状态提示

- 可展示快捷键状态和自动更新状态
- 状态提示必须是只读信息，不应膨胀成大段说明

### 3.3 明确禁止

- 托盘直接打开一个无会话结果面板
- 托盘顶级菜单暴露“打开配置目录”“复制配置路径”这类排障动作
- 托盘承载过多内部状态或诊断项

## 4. 首次配置界面目标

- 首次配置界面必须把用户带到“能开始截图翻译”的状态
- 普通用户主路径是：填写端点 -> 保存 -> 立即开始使用
- 普通用户不需要理解 `%APPDATA%`、`runtime-overrides.json`、Provider 诊断结构
- 能力缺失或降级提示必须是可行动的，而不是调试日志的翻译版

## 5. 首次配置界面结构

### 5.1 首屏必须包含

- 当前是否可开始截图的总状态
- `translation.base_url`
- `translation.api_key`
- `capture_shortcut`
- 保存并立即生效

### 5.2 首屏可轻量包含

- 快捷键是否可用
- 截图能力是否可用
- 当前入口模式（托盘 / 托盘+快捷键）

### 5.3 默认折叠的高级区

- 配置文件路径
- 打开配置文件
- 打开配置目录
- 复制配置路径

## 6. 配置界面动作契约

- `保存并立即生效`：保存当前配置并更新运行时状态
- `保存并开始截图`：保存成功后直接进入截图
- `重新检查`：重新读取当前运行时状态
- `完成`：关闭配置界面

## 7. 降级提示契约

- 快捷键冲突：提示“当前已降级为托盘入口”，并继续允许用户保存新快捷键
- 截图能力不可用：提示“当前无法截图”，并告诉用户先解决能力问题
- 端点未配置：提示“尚未完成翻译端点配置”，但不要提前展示文件路径和 JSON

## 8. 文案与视觉限制

- 不允许把“配置路径”做成首屏主视觉块
- 不允许把“高级操作”放在主表单之前
- 不允许首屏出现大段面向开发者的实现说明
- 不允许把排障动作伪装成普通用户主路径

# desktop_screenshot_translate 框架严格映射基线

## 1. 目的

本文件把截图翻译工具当前实例化链路写清楚：

1. `framework -> product spec`
2. `framework -> implementation config`
3. `framework -> generated evidence`

当前项目不是直接手写桌面业务代码，而是先把产品真相和实现细化固定下来，再由生成器物化出可追溯产物。

## 4. 当前结论

当前实例已经把以下真相固定下来：

- 用户通过托盘或全局快捷键触发截图
- 截图后进入 OCR 与翻译闭环
- 结果以悬浮面板交付，并至少支持复制、关闭和重新截图
- 平台范围、隐私姿态、更新通道和时延预算在产品层已固定
- 实现层只细化桌面宿主、采集链、Provider 链和发布路径
- 当前实现配置已明确采用 Windows MVP 优先路线，但未改写最终产品层平台真相

生成产物位于 `generated/`，只能通过物化脚本重建，不应手改。

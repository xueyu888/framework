# 数据清洗:DataWasher

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文档分块：支持对已完成归一化并已入库的文档执行资格识别；对于具备清洗资格的文档，再按输入分块标准将文档分为多个块，并按稳定输出格式生成新的分块结果文件；每个块都需要保留与原文的对应关系并预留属性扩展接口。
- `C2` Tag标注能力：支持将分出来的块按照内容添加Tag。
- `C3` 数据提取能力：支持通过大模型或文档模板的方式，按照添加的Tag提取对应的数据或内容。
- `C4` 结果对比能力：支持将通过大模型和文档模板的方式提取的数据进行对比。
- `C5` 故障排除能力：若`C4`中结果对比不一致，支持需要调整数据提取能力，最终达成一致。
- `C6` 人工审核能力：当`C4`中结果达成统一后，与人工提取的数据进行比对审核。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文档（normalized_document）：`data_washer` 仅处理已完成归一化并已入库的文档。来源：`C1`。
- `QUALIFIED` 资格结论（qualification_result）：每个归一文档在分块前必须被识别为具备清洗资格或不具备清洗资格。来源：`C1`。
- `CHUNK_SPEC` 分块标准（chunk_specification）：分块必须依据输入分块标准执行。来源：`C1`。
- `B` 分块数量（block）：`B > 1`。来源：`C1`。
- `OUTPUT_FORMAT` 输出格式（output_format）：分块结果必须按稳定输出格式写入新文件。来源：`C1`。
- `ARTIFACT` 结果文件（chunk_artifact）：分块后必须生成独立于原文档的新文件作为结果载体。来源：`C1`。
- `SOURCE_MAP` 原文映射（source_mapping）：每个分块结果都必须可对应到原文锚点、原文位置或原文范围。来源：`C1`。
- `ATTR_EXT` 属性扩展接口（attribute_extension）：每个分块结果都必须预留可扩展属性接口，但不在本层定义具体属性字段。来源：`C1`。
- `TN` 标签数量（tag_number）：`TN >= 1`。来源：`C2`。
- `TT` 标签种类（tag_type）：`TT >= 1`。来源：`C2`。
- `EG` 提取依据（extract_gist）：`EG`。来源：`C1 + C2 + C3`。
- `ET` 提取目标（extract_target）：`ET`。来源：`C1 + C2 + C3`。
- `CS` 对比内容（compare_substance）：`CS`。来源：`C3 + C4`。
- `CC` 对比结论（compare_conclusion）：`CC = TRUE | FALSE`。来源：`C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文档分块：`L1.M0[B1,B2,B3,B4]`；先将已完成归一化的文档接入为稳定文档变量，再执行资格识别并判断其是否具备进入清洗流程的资格；对于具备清洗资格的文档，按照输入分块标准将文档分为多个内容块，再按稳定输出格式生成新的分块结果文件；每个块记录都必须保留原文映射并预留属性扩展接口。来源：`C1 + NORM_DOC + QUALIFIED + CHUNK_SPEC + B + OUTPUT_FORMAT + ARTIFACT + SOURCE_MAP + ATTR_EXT`。
- `B2` 块标注：`L1.M1[B1,B2,B3]`；消费稳定块记录集合，为每个块绑定可追踪 `Tag`，并输出稳定标注关系集合。来源：`C1 + C2 + B + TN + TT`。
- `B3` 数据提取：按照提取依据与提取目标，从已标注内容块中提取结构化结果。来源：`C2 + C3 + EG + ET`。
- `B4` 数据对比：将多次提取的数据进行对比。来源：`C3 + C4 + ET + CS + CC`。
- `B5` 参数调整：调整数据提取的参数。来源：`C5 + EG`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 分块标注组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：`B2` 只能消费 `B1` 生成的分块结果文件中的块记录；每个 Tag 必须绑定唯一块标识，且不得脱离块记录集合单独存在。
  - `R1.3` 输出结构：`BT_set`（元素：`BT`，每个 `BT` 必须同时包含 `block_id` 与 `tag_id`）。
  - `R1.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/B/OUTPUT_FORMAT/ARTIFACT/SOURCE_MAP/ATTR_EXT/TN/TT`。
- `R2` 标注提取组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：`B3` 只能基于已标注块执行提取；每条提取结果必须绑定来源块、触发 Tag、提取依据与提取目标，未绑定任一项则组合无效。
  - `R2.3` 输出结构：`ER_set`（元素：`ER`，每个 `ER` 必须包含 `block_id`、`tag_id`、`extract_gist`、`extract_target`）。
  - `R2.4` 边界绑定：`TN/TT/EG/ET`。
- `R3` 提取对比组合
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：`B4` 只能对相同提取目标、相同对比内容范围内的两组或多组提取结果执行对比；来源块范围、字段语义或结果结构不一致时不得组合。
  - `R3.3` 输出结构：`CR_set`（元素：`CR`，每个 `CR` 必须包含参与对比的 `extract_result_id_set`、`compare_substance`、`compare_conclusion`）。
  - `R3.4` 边界绑定：`ET/CS/CC`。
- `R4` 提取调参组合
  - `R4.1` 参与基：`B3 + B5`。
  - `R4.2` 组合方式：`B5` 只能调整 `B3` 的提取参数、提取依据或规则版本；调整动作不得直接修改原始块内容与 Tag 绑定，只能生成新的提取版本。
  - `R4.3` 输出结构：`AP_set`（元素：`AP`，每个 `AP` 必须包含 `extract_target`、`parameter_version`、`extract_gist`）。
  - `R4.4` 边界绑定：`EG/ET`。
- `R5` 对比调参闭环组合
  - `R5.1` 参与基：`B3 + B4 + B5`。
  - `R5.2` 组合方式：只有进入 `B4` 的可比提取结果才允许进入 `B5` 调整；`B5` 每次调整后必须回到 `B3` 重新生成新版本结果，旧版本结果保留用于复比。
  - `R5.3` 输出结构：`Loop_set`（元素：`Loop`，每个 `Loop` 必须包含 `compare_result_id`、`parameter_version`、`new_extract_result_id`）。
  - `R5.4` 边界绑定：`EG/ET/CS/CC`。
- `R6` 全链路合法组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：合法依赖顺序固定为 `B1 -> B2 -> B3`，`B4` 与 `B5` 仅能挂接在 `B3` 之后；下游基不得逆向改写上游基的对象标识与边界定义。
  - `R6.3` 输出结构：`Chain_set`（元素：`Chain`，每个 `Chain` 必须保持归一文档、资格识别结果、分块结果文件、块记录、原文映射、Tag、提取结果、对比结果、参数版本的可追踪关联）。
  - `R6.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/B/OUTPUT_FORMAT/ARTIFACT/SOURCE_MAP/ATTR_EXT/TN/TT/EG/ET/CS/CC`。
- `R7` 禁止组合
  - `R7.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R7.2` 组合方式：跳过 `B1` 直接进入 `B2/B3/B4/B5`、跳过 `B2` 直接提取、跳过 `B3` 直接对比或调参、`B5` 直接覆盖旧提取结果、下游基逆向改写上游块记录或 Tag 绑定、以及绕过稳定分块产物直接接入下游链路的组合均无效。
  - `R7.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效基、失效关系与失效原因）。
  - `R7.4` 边界绑定：`NORM_DOC/QUALIFIED/CHUNK_SPEC/B/OUTPUT_FORMAT/ARTIFACT/SOURCE_MAP/ATTR_EXT/TN/TT/EG/ET/CS/CC`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则完整性：每个 `R*` 必须明确参与基、组合方式、输出结构与边界绑定。
- `V3` 依赖顺序合法性：所有有效组合必须满足 `B1 -> B2 -> B3` 的固定依赖顺序，`B4/B5` 不得前置。
- `V4` 绑定一致性：归一文档、资格识别结果、分块结果文件、块记录、原文映射、Tag、提取结果、对比结果、参数版本之间必须存在可校验的绑定关系。
- `V5` 可比与可回放性：进入对比的结果必须可比，进入调参闭环的结果必须可回放到具体参数版本。
- `V6` 最小必要性：移除任一参与合法组合的 `B*` 后，该组合至少一项合法性条件失败。
- `V7` 结论表达：逐条输出 `R* -> 有效/无效 / 参数边界 / 失效原因` 的结论。
- `V8` 层级引用方向：若 `L0` 基需要展开到 `L1`，必须由 `L0` 显式声明其引用的 `L1` 基集合，`L1` 不得反向声明“上承”作为主表达。

# 文档分块:DocumentChunking

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本分段能力：将文本根据段落切分为有序段落块序列的能力。
- `C2` 角色判型能力：提供依据显式模板判定段落标题角色的能力。
- `C3` 段落块组合能力：将数个段落块组合为一个文本块的能力。
- `C4` 分块结果输出能力：将组合后的所有文本块打包输出为一个对象的能力。
- `C5` 归属关系输出能力：将段落块和文本块之间的归属关系输出的能力。
- `C6` 结果回溯能力：能根据结果回溯中间状态的能力。
- `C7` 非能力项：不负责跨文档聚类、摘要改写、向量化索引、版面渲染或下游检索排序。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文本边界：输入必须为单文档纯文本，且已完成换行、空白与编码归一。来源：`C1`。
- `CUT_RELY` 分段依据边界：分段依据必须明确，来源：`C1`。
- `BLOCK_NUM` 分段数量边界：段落块具有一定数量范围限制，来源：`C1`。
- `ROLE_JUDGE` 角色判型依据边界：判型依据必须明确，来源：`C2`。
- `ROLE_TYPE` 段落块角色边界：段落块只能被判定为标题或正文，来源：`C1 + C2`。
- `BLOCK_COMBINE` 段落块组合边界：段落块的组合依据必须明确，来源：`C1 + C2 + C3`。
- `OUTPUT` 输出状态边界：模块输出的结果状态必须明确，来源：`C4 + C5 + C6`。


## 3. 最小可行基（Minimum Viable Bases）

- `B1` 段落分界模板：分段依据，标明分段位置。来源：`C1 + CUT_RELY + BLOCK_NUM`。
- `B2` 段落块：段落文本容器，存放通过 `B1` 拆分出来的一段文本，并保留顺序与定位信息。来源：`C1 + C6 + BLOCK_NUM`。
- `B3` 文本角色识别模块：依据 `ROLE_JUDGE` 对已装载文本的 `B2` 进行角色判型并赋予 `title/body` 标签。来源：`C2 + ROLE_JUDGE + ROLE_TYPE`。
- `B4` 段落块匹配关系：将多个 `B2` 归并为单个文本块的匹配依据。来源：`C3 + BLOCK_COMBINE`。
- `B5` 归属关系表：段落块和文本块之间的归属关系表。来源：`C5 + C6 + OUTPUT`。
- `B6` 输出结构：文档分块模块输出结果的稳定承载结构。来源：`C4 + OUTPUT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 段落切块组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：`B1` 只能作用于满足 `NORM_DOC` 的单文档纯文本；每次合法分界命中必须生成一个按原文顺序排列的 `B2`，且各 `B2` 不得重叠、跳序或跨文档混合。
  - `R1.3` 输出结构：`ParagraphBlock_set`（元素：`PB`，每个 `PB` 必须包含 `block_id`、`order_index`、`text`、`start_offset`、`end_offset`）。
  - `R1.4` 边界绑定：`NORM_DOC/CUT_RELY/BLOCK_NUM`。
- `R2` 段落块角色识别组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：每个 `B2` 在进入后续组合前都必须经 `B3` 完成角色判型，并获得唯一的 `title/body` 角色标签；未完成角色识别的段落块不得进入后续匹配。
  - `R2.3` 输出结构：`LabeledParagraphBlock_set`（元素：`LPB`，每个 `LPB` 必须包含 `block_id`、`order_index`、`text`、`start_offset`、`end_offset`、`block_role`）。
  - `R2.4` 边界绑定：`ROLE_JUDGE/ROLE_TYPE`。
- `R3` 段落块匹配组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：`B4` 只能消费已完成角色判型且保持稳定顺序的 `B2` 序列；每个文本块必须由且仅由一个标题块作为锚点，并吸收其后直到下一个标题块出现前的连续正文块；任一 `B2` 不得同时归属多个文本块，也不得跳序归组。
  - `R3.3` 输出结构：`ChunkMatch_set`（元素：`CM`，每个 `CM` 必须包含 `text_chunk_id`、`title_block_id`、`body_block_id_set`、`start_order`、`end_order`）。
  - `R3.4` 边界绑定：`ROLE_JUDGE/ROLE_TYPE/BLOCK_COMBINE`。
- `R4` 归属登记组合
  - `R4.1` 参与基：`B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：每个合法 `CM` 必须为其包含的全部 `B2` 生成归属记录；每条记录只能把一个段落块登记到一个文本块，且必须显式标明该段落块在文本块中的角色与顺序位置。
  - `R4.3` 输出结构：`OwnershipRecord_set`（元素：`OR`，每个 `OR` 必须包含 `block_id`、`text_chunk_id`、`block_role`、`position_in_chunk`）。
  - `R4.4` 边界绑定：`ROLE_TYPE/BLOCK_COMBINE/OUTPUT`。
- `R5` 输出封装组合
  - `R5.1` 参与基：`B2 + B3 + B4 + B5 + B6`。
  - `R5.2` 组合方式：全部合法文本块必须按 `start_order` 输出为稳定序列；`B6` 必须同时承载文本块集合、归属关系表与必要的回溯字段，使结果能够回连到原始 `B2` 序列。
  - `R5.3` 输出结构：`DocumentChunkingOutput`（必须包含 `text_chunk_set`、`ownership_record_set`、`paragraph_block_set`、`trace_meta`）。
  - `R5.4` 边界绑定：`OUTPUT/BLOCK_NUM/BLOCK_COMBINE`。
- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R6.2` 组合方式：未完成归一化即切块、无显式分段依据、段落块角色不在 `title/body` 中、正文块在无标题锚点时被归组、同一段落块被重复消费、跨文档归组、跳序归组、缺失归属记录、或输出结果无法回溯到原始段落块的组合均无效。
  - `R6.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效基、失效关系与失效原因）。
  - `R6.4` 边界绑定：`NORM_DOC/CUT_RELY/BLOCK_NUM/ROLE_JUDGE/ROLE_TYPE/BLOCK_COMBINE/OUTPUT`。

## 5. 验证（Verification）

- `V1` 输入与分界有效性：进入 `R1` 的输入必须满足 `NORM_DOC/CUT_RELY/BLOCK_NUM` 约束，且 `B1` 必须能给出可执行的稳定切块依据。
- `V2` 段落块合法性：`R1` 产出的 `ParagraphBlock_set` 必须保持顺序唯一、范围不重叠、块内容非空，且每个 `PB` 都可定位到原文范围。
- `V3` 角色完备性：`R2` 产出的 `LabeledParagraphBlock_set` 必须保证每个 `B2` 都依据 `ROLE_JUDGE` 获得确定角色，且角色值只能属于 `title/body`。
- `V4` 匹配合法性：`R3` 产出的每个 `CM` 都必须满足“一个标题锚点 + 零个或多个连续正文块”的结构，不得存在无标题归组、跨组重叠或跳序组合。
- `V5` 归属一致性：`R4` 产出的 `OwnershipRecord_set` 必须覆盖所有已进入文本块的 `B2`，并保证每个 `block_id` 只对应一个 `text_chunk_id`。
- `V6` 输出稳定性与可回溯性：`R5` 产出的 `DocumentChunkingOutput` 必须同时保留文本块序列、归属关系与段落块序列，且必须能从结果回溯到中间 `B2/CM/OR` 状态。
- `V7` 禁止组合可检出性：`R6` 列举的无效组合必须能够被判定并输出明确的失效原因。
- `V8` 最小必要性：移除任一 `B*` 后，`V2~V6` 至少一项失败。
- `V9` 结论表达：逐条输出 `R* -> 有效/无效 / 边界 / 失效原因` 的结论。

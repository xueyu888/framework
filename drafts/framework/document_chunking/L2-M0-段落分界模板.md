# 段落分界模板:ParagraphDecompositionTemplate

@framework

## 1. 能力声明（Capability Statement）

- `C1` 字符串输入能力：支持输入不同的需要进行查找匹配的字符串。
- `C2` 匹配能力：根据输入的字符串和匹配规则进行匹配。
- `C3` 匹配结果输出能力：返回匹配结果，包括成功和失败。

## 2. 边界定义（Boundary / 参数）

- `STR_LENGTH` 字符串长度需要进行限制。来源：`C1`。
- `MATCH_RULE` 匹配规则需要进行明确。来源：`C2`。
- `MATCH_RESULT` 匹配结果需要采用统一的规则，错误码或匹配到的位置。来源：`C3`。 

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 字符串。匹配的目标。来源：`C1 + STR_LENGTH`。
- `B2` 匹配规则。规定匹配方式。来源：`C2 + MATCH_RULE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 字符串匹配组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：每个 `B2` 必须绑定一个满足长度约束的 `B1`，形成一次可执行的匹配任务；脱离目标字符串单独存在的匹配规则不构成有效组合。
  - `R1.3` 输出结构：`MatchTask_set`（元素：`MT`，每个 `MT` 必须包含 `target_string`、`rule_id`、`rule_value`）。
  - `R1.4` 边界绑定：`STR_LENGTH/MATCH_RULE`。
- `R2` 禁止组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：空字符串、超出长度限制的字符串、未声明或不可执行的匹配规则、规则与字符串脱离绑定、以及输出结果不符合统一返回格式的组合均无效。
  - `R2.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效基、失效关系与失效原因）。
  - `R2.4` 边界绑定：`STR_LENGTH/MATCH_RULE/MATCH_RESULT`。


## 5. 验证（Verification）

- `V1` 字符串有效性：`B1` 必须满足 `STR_LENGTH` 约束，且必须能够作为可匹配的目标字符串被稳定输入。
- `V2` 规则有效性：`B2` 必须满足 `MATCH_RULE` 约束，且规则内容必须明确、可执行、可区分。
- `V3` 匹配任务合法性：`R1` 产出的 `MatchTask_set` 必须保证每个匹配任务都同时绑定一个合法 `B1` 与一个合法 `B2`。
- `V4` 结果结构合法性：`R2` 产出的 `MatchResult_set` 必须满足 `MATCH_RESULT` 约束，命中结果与失败结果都必须能被统一解析。
- `V5` 禁止组合可检出性：`R3` 列举的无效组合必须能够被判定，并输出明确失效原因。
- `V6` 最小必要性：移除任一 `B*` 后，`V3~V4` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> 有效/无效 / 边界 / 失效原因` 的结论。

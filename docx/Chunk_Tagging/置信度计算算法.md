# 置信度计算算法

## 1. 目标

本文档定义文本块针对单个标签进行匹配置信度计算的最小可用算法。

算法目标是：

- 输入一个文本块和一个标签定义。
- 基于标签定义中的关键词规则或正则规则计算命中情况。
- 将计算结果归一化到 `0~1` 范围内，作为该文本块对该标签的匹配置信度。

## 2. 输入结构

单次计算输入包括：

- 一个文本块
- 一个标签定义

其中标签定义至少包含：

- 标签名
- 匹配规则列表

单条匹配规则至少包含：

- 匹配方式：精准匹配或正则匹配
- 匹配内容：关键词或正则表达式
- 作用位置：标题或正文
- 权重：用于计算该规则命中后的贡献值

这里保留 `作用位置` 的原因是：

- 标题命中直接判定 `1.0`
- 正文命中进入加权统计

因此规则配置中必须显式区分“标题规则”和“正文规则”。

## 3. 文本预处理

文本块在计算前划分为：

- 标题
- 正文

正文按句子切分。最小版本中，句子切分符可先按以下字符处理：

- `。`
- `！`
- `？`
- `.`
- `!`
- `?`

若正文切分后句子数量为 `0`，则按 `1` 处理，以避免除零。

## 4. 计算流程

### 4.1 标题命中优先

先对标题执行匹配，只检查 `scope = title` 的规则。

若标题命中任一标题规则，无论命中方式为精准匹配还是正则匹配，直接判定：

`confidence = 1.0`

此时不再继续正文评分。

### 4.2 正文命中统计

若标题未命中，则对正文执行匹配，只检查 `scope = body` 的规则。

对每条正文规则统计命中次数：

- 精准匹配：统计关键词在正文中的命中次数
- 正则匹配：统计正则表达式在正文中的命中次数

对每条规则计算加权命中值：

`rule_score_i = hit_count_i * weight_i`

将所有正文规则的得分求和：

`weighted_hits = sum(rule_score_i)`

### 4.3 正文置信度计算

设正文句子数为：

`sentence_count`

则正文置信度计算公式为：

`body_score = weighted_hits / sentence_count`

为保证结果落在 `0~1` 范围内，进行截断归一化：

`confidence = min(body_score, 1.0)`

## 5. 最终结果

最终置信度输出规则如下：

- 若标题命中：`confidence = 1.0`
- 若标题未命中：`confidence = min(weighted_hits / sentence_count, 1.0)`

最终输出为单个 `0~1` 范围内的数值。

## 6. 解释性

该最小版本算法具有以下特点：

- 简单，易于实现
- 可解释，容易回看是哪条规则命中
- 适合第一版验证标签定义和规则配置是否有效

其核心含义是：

- 标题命中表示高置信度命中，直接给满分
- 标题未命中时，正文中的加权命中越密集，得分越高
- 正文越长但命中越稀疏，得分越低

## 7. 后续可扩展方向

后续若需要提升算法稳定性，可继续扩展：

- 为不同规则类型设置不同默认权重
- 为单条规则设置命中次数上限，避免重复词刷高分数
- 将标题命中分数从固定 `1.0` 改为可配置阈值
- 引入负向规则、示例文本或语义相似度计算
- 将正文句子切分从简单符号切分升级为更稳定的句法切分

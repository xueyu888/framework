# 置信度计算

本文档用于说明文本块与标签之间的置信度计算思路。该部分不再作为 `framework_drafts` 中的独立模块维护，而作为 `Chunk_Tagging` 方案中的实现说明保留在 `docx` 目录下。

## 1. 输入

- 单个待评分文本块。
- 单个标签定义。

## 2. 输出

- 一个 `0~1.0` 范围内的置信度结果。

## 3. 约束

- 单次计算只处理一个文本块与一个标签定义。
- 输入文本块必须满足文本块结构要求。
- 输入标签定义必须满足标签定义模块的完整组合要求。
- 输出结果必须落在 `0~1.0` 之间。

## 4. 计算方式

- 先将输入文本块划分为标题和正文。
- 若标题命中标签定义中的任一标题规则，则直接输出 `1.0` 作为置信度。
- 若标题未命中，则对正文按句子切分，并对标签定义中的规则分别统计命中次数。
- 按“命中次数 × 规则权重”求和得到 `weighted_hits`。
- 再以 `weighted_hits / sentence_count` 计算正文得分。
- 最后将结果截断映射到 `0~1` 范围内，作为该文本块对该标签的最终置信度。

# 规范总纲与树形结构（L0）

## 1. 目的
本文件是仓库规范总纲，负责定义标准树、层级关系和变更传导规则。

## 2. 标准树（强制）

`standards/` 目录必须与标准层级一一对应，不允许平铺混放：

```text
standards/
├── L0/
│   └── 规范总纲与树形结构.md
├── L1/
│   ├── 框架设计核心标准.md
│   ├── 可追溯性标准.md
│   └── 可删减性标准.md
├── L2/
│   ├── 置物架框架标准.md
│   └── 窗帘框架标准.md
└── L3/
    └── mapping_registry.json
```

仓库级标准树（含实现与证据）如下：

```text
L0 规范总纲
└── standards/L0/规范总纲与树形结构.md
    └── L1 核心标准层
        ├── standards/L1/框架设计核心标准.md
        ├── standards/L1/可追溯性标准.md
        ├── standards/L1/可删减性标准.md
        ├── AGENTS.md
        └── L2 领域标准层
            ├── standards/L2/置物架框架标准.md
            └── standards/L2/窗帘框架标准.md
                └── L3 领域实现与证据
                    ├── src/shelf_framework.py
                    ├── src/main.py
                    ├── scripts/validate_strict_mapping.py
                    ├── standards/L3/mapping_registry.json
                    └── docs/logic_record.json
```

## 3. 树化映射规则（强制）
- 所有“标准 -> 实现”的一一对应关系，必须挂载在 `standards/L3/mapping_registry.json` 的 `tree` 字段。
- `tree` 必须具备稳定 `id`，并使用 `children` 维护父子关系，形成可遍历的树结构（类似 AST）。
- `tree` 中每个 `kind=file` 节点必须指向唯一文件，禁止重复挂载同一文件。
- 位于 `standards/` 内的文件，必须放在与其 `level` 一致的目录（例如 `L2` 文件必须在 `standards/L2/`）。

## 4. 变更传导规则（强制）
- 当 L0 变更时，L1/L2/L3 必须同步评估并按需更新。
- 当 L1 变更时，L2/L3 必须同步评估并按需更新。
- 当 L2 变更时，L3 必须同步更新。
- 当 L3 变更时，必须执行反向验证，确认不违反 L0/L1/L2。
- 以上规则必须通过树化校验自动检测，而不是人工口头确认。

## 5. 验证机制
机器可读映射：`standards/L3/mapping_registry.json`

验证命令：
```bash
uv run python scripts/validate_strict_mapping.py
uv run python scripts/validate_strict_mapping.py --check-changes
```

## 6. 评审准入门槛
合并前至少满足：
- 标准树关系未被破坏
- 映射验证通过
- 变更方向满足传导规则

# 置物架框架标准（L2）

## 0. 规范关系
本文件为置物架领域标准，必须遵循：
- `standards/L0/规范总纲与树形结构.md`
- `standards/L1/框架设计核心标准.md`
- `standards/L1/可追溯性标准.md`
- `standards/L1/可删减性标准.md`
本文件位于 `L2`，承接 `L1` 方法标准并向 `L3` 实现层传导。


## 1. 目标（Goal）
定义可拆卸组装的开放式立体堆叠装置，在给定占地面积条件下提升物品存取效率。

## 2. 边界定义（Boundary）
- `N`：总层数参数（`BoundaryDefinition.layers_n`，`N >= 1`）
- `P`：目标承重需求（`BoundaryDefinition.payload_p_per_layer`，用于合格判定，不作为组合枚举约束）
- `x`：单层净宽（`BoundaryDefinition.space_s_per_layer.width`）
- `y`：单层净深（`BoundaryDefinition.space_s_per_layer.depth`）
- `h`：单层高度（`BoundaryDefinition.space_s_per_layer.height`），同时作为侧面支撑板高度上限（当前验证场景可取 `h=30`）
- `nx`：水平方向 `x` 轴拼接数量（`nx >= 1`，如 `1x2` 中 `nx=2`）
- `ny`：水平方向 `y` 轴拼接数量（`ny >= 1`，如 `1x2` 中 `ny=1`）
- `O`：开口尺寸（`BoundaryDefinition.opening_o`）
- `A`：占地面积（`BoundaryDefinition.footprint_a`）
- `L_type`：层载荷模型（均布/集中）
- `γ`：安全系数（设计荷载放大系数）
- `δ_max`：层板允许挠度上限（可采用 `L/xxx` 形式）

说明：
- 当启用承重验证时，`P/L_type/γ/δ_max` 必须同时给定；否则仅可判定几何与连接合法性，不能判定承重通过。
- 承重相关要求只影响“是否合格”判定，不影响组合合法性枚举结果。
- 外部属性集 `ExternalPropertySet`（解耦属性，不改变三基定义）用于承重计算输入，至少包含：`rod_capacity`、`connector_capacity`、`board_capacity`、`support_panel_capacity`、`board_deflection_capacity`。


## 3. 模块（最小可行基，Module）
- `M1` 杆：结构骨架与承力通道；承重验证时必须具备可计算承载属性（如强度、截面、有效长度）。
- `M2` 连接接口：定义连接语义、对齐与载荷传递；承重验证时必须具备可校验连接容量（抗剪/抗拉/抗弯等）。
- `M3` 板：提供承载面，并可在规则允许时作为支撑件参与传力；承重验证时必须区分“层板/侧支撑板”并具备强度与挠度可校验属性。

说明：
- 三基不变；承重能力参数通过外部属性集注入，实现结构组合与力学参数解耦。


## 4. 组合原则（Combination Principles）
组合对象：
- `ShelfStructure`：置物架总装结构，可由 `nx * ny` 个子置物架在水平方向拼接构成；每个子置物架由支撑件（杆或板）、层板、连接接口构成。

按功能分类（可扩展，不限制条目数量）：

### A. 支撑与空间
- `R1`：支撑件类型仅允许 `M1`（杆）或 `M3`（作为支撑的板），且每个支撑件必须在实例级标注支撑角色。
- `R2`：每层等效支撑必须满足 `E_total = 4`，并采用固定计数：`E(杆)=1`、`E(支撑板)=2`。
- `R3`：支撑板约束统一为：覆盖同层两个相邻角点、`0 < h_panel <= h`、同层数量不超过 2。
- `R4`：若同层两支撑板形成 `90°`，共享角点只计一次等效支撑；每层四角仍必须形成有效支撑（缺角无效）。
- `R5`：每个子置物架层数 `n_i` 必须满足 `1 <= n_i <= N`。
- `R6`：整体结构必须满足 `span_x > 0`、`span_y > 0`、`span_z > 0`，并形成按高度单调分层的承载体系；杆件允许跨层连续布置。

### B. 连接与拼接
- `R7`：所有结构连接（支撑件-层板、支撑件-支撑件、板-板 `90°`）必须通过 `M2` 接口实现。
- `R8`：禁止非法连接（非法穿透、非法相交、悬空连接）。
- `R9`：横向拼接规模由 `nx, ny` 唯一定义，并必须按二维网格对齐实现（如 `1x2`）。
- `R10`：拼接后结构必须单连通，且不允许孤立构件与错层悬空拼接。

### C. 边界与传力
- `R11`：每层层板必须与四角支撑形成闭合框架。
- `R12`：总装结构外包范围不得超过 `A`。
- `R13`：开口约束 `O` 在每个有效子置物架层上必须可判定。
- `R14`：每层至少一个侧面必须保持开放。
- `R15`：开放性约束：所有平面板（层板与侧支撑板）必须至少具备一个与外界连通的接口；该接口可为直接外露边，或通过连续开放路径与外界连通。
- `R16`：所有承载层必须存在连续传力路径。
- `R17`：承重不足不判为“组合无效”，仅在验证阶段判为“不合格”。

组合结果必须可判定为“有效/无效”，并可枚举有效组合集合（参数空间由边界定义给定）。

## 5. 验证（Verification）
通过条件：
- 边界有效
- 组合属于有效组合集
- 承重合格：在给定外部属性集与载荷工况下，层板强度/挠度、支撑件承载、连接接口承载均满足 `P/L_type/γ/δ_max` 判定标准
- `target_efficiency > baseline_efficiency`

失败原因至少包含：
- 几何规则失败（`R1-Rn`）
- 承重能力不足（层板/支撑件/连接接口中的具体失败点）

# 路径门控模块框架标准（L2）

## 0. 规范关系
本文件为路径门控模块标准，参考：
- `archive/data_station/standards/L0/数据站框架标准.md`
- `archive/data_station/standards/L1/上传路径模块框架标准.md`

本文件位于 `L2`，承接 `L1` 中 `M1_1` 的定义，对路径门控模块进行实现前框架化约束，并向实现层传导。

## 1. 目标（Goal）
- 构建一个确定性的路径门控模块，在上传链路入口对 UI 路径/API 路由进行放行或拒绝判定。
- 确保仅命中 `path_scope` 的请求进入 `M1_2`，越界请求在入口被拦截并返回可观察结果。

## 2. 边界定义（Boundary）
说明：本节将路径门控模块视为一个整体黑盒，不拆分内部实现细节。

- `PathGateBoundary.event_scope`：仅覆盖“请求进入上传入口”到“输出 allow/deny 决策”的门控事件。
- `PathGateBoundary.path_scope`：仅定义允许放行的 UI/API 路径规则（精确/前缀/模板匹配）。
- `PathGateBoundary.input_contract`：输入仅包含 `path/method/request_id` 等门控必需字段。
- `PathGateBoundary.output_contract`：输出仅包含 `allow/deny`、`reason_code` 与下一步路由结果。
- `PathGateBoundary.state_contract`：状态仅允许 `gate_pending -> matching -> allowed/denied` 合法流转。
- `PathGateBoundary.governance_contract`：规则维护受 `M5` 控制，门控记录接入 `M3` 追溯接口。

去重约束：
- 门控模块只解决“能否进入上传链路”，不承担上传鉴权、文件校验和入库处理。

## 3. 模块（最小可行基，Module）
- `M1_1_1` 规则加载子模块：加载并缓存 `path_scope` 规则集与版本信息。
- `M1_1_2` 路径标准化子模块：对路径与方法进行规范化处理（大小写、斜杠、模板参数展开）。
- `M1_1_3` 路径匹配子模块：基于规则集执行匹配，生成 `allow/deny` 初始判定。
- `M1_1_4` 决策出站子模块：输出门控结果（决策、原因码、下一步路由指令），并触发 `M3` 追溯记录接口。

说明：
- `M1_1_1-M1_1_4` 共同构成 `M1_1` 路径门控模块的最小可行基，不再按功能继续拆分。
- `authority/traceability` 为横切约束，作用于 `M1_1_1-M1_1_4` 全链路。

## 4. 组合原则（Combination Principles）
- 子模块组合按 `R1 -> R2 -> R3` 编排。
- `R1` 规则准备：`M1_1_1` 完成规则加载，`M1_1_2` 完成输入标准化。
- `R2` 匹配判定：`M1_1_3` 对标准化后的输入执行路径匹配并生成决策。
- `R3` 决策出站：`M1_1_4` 输出决策结果并触发追溯记录；`allow` 才可进入 `M1_2`，`deny` 必须在门控层终止。

## 5. 验证（Verification）
判定公式：
- `PASS = BoundaryAssertions(all) AND CombinedOutcomeChecks(all) AND GoalChecks(all)`

组合验证说明：
- 组合过程遵循第 4 节组合原则，验证阶段仅对组合形成的结果进行判定。

通过条件：
- `V-B0` 路径边界通过：仅定义内路径可放行进入上传链路，越界路径全部拒绝。
- `V-B1` 输入边界通过：门控输入字段完整且可解析（`request_id/path/method`）。
- `V-B2` 输出边界通过：每次门控都有明确 `allow/deny + reason_code` 输出。
- `V-B3` 状态边界通过：门控状态仅在定义状态机内流转。
- `V-B4` 治理边界通过：规则维护受 `M5` 控制，门控决策经 `M3` 追溯接口记录。
- `V-C1` 组合结果完整：规则加载、标准化、匹配、决策出站全链路可执行。
- `V-C2` 组合结果一致：`allow` 请求进入 `M1_2`，`deny` 请求被门控终止且返回明确原因。
- `V-C3` 组合结果稳态：重复请求在同规则版本下输出一致，规则版本切换可追溯。
- `V-G1` 目标闭环通过：覆盖 UI/API 白名单、黑名单、边界路径三类样例均得到预期结果。

失败原因至少包含：
- 任一边界断言失败（`V-B0` 至 `V-B4`）。
- 任一组合结果约束失败（`V-C1` 至 `V-C3`）。
- 目标闭环失败（`V-G1`）。

证据记录要求：
- 采用验收矩阵记录 `case_id`、输入路径、方法、规则版本、期望决策、实际决策、原因码、证据位置。

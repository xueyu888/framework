# 文件上传最小示例

这个示例用于演示一个最基础的“网页上传文件”流程，只覆盖两件事：

- 选择文件并发起上传
- 接收上传结果并反馈给用户

示例文件：

- `index.html`：前端页面，负责选择文件、发起请求、展示结果
- `upload_server.py`：后端服务，负责接收文件、保存文件、返回 JSON 结果

## 运行方式

在仓库根目录执行：

```bash
uv run python archive/data_station/example/upload_server.py
```

浏览器打开：

```text
http://127.0.0.1:8000
```

## 这个示例演示了什么

前端做的事：

- 选择本地文件
- 进行最基础的大小检查
- 把文件作为请求体发送给后端
- 显示成功或失败结果

后端做的事：

- 接收上传请求
- 检查文件名和文件大小
- 把文件保存到 `archive/data_station/example/uploads/`
- 返回上传结果 JSON

## 注意

这是教学示例，不是生产方案。

它故意保持最小化，没有实现这些内容：

- 登录鉴权
- 数据库记录
- 病毒扫描
- 分片上传
- 断点续传
- 复杂文件类型校验
